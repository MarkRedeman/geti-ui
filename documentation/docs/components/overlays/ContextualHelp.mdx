# ContextualHelp

ContextualHelp places a small icon button next to a UI element that opens a popover with additional guidance. Use it to clarify form fields, feature labels, or settings without cluttering the layout with persistent text.

```tsx
import { ContextualHelp } from '@geti-ai/ui';
import { Content, Heading, Text } from '@adobe/react-spectrum';
```

- [React Spectrum ContextualHelp ↗](https://react-spectrum.adobe.com/react-spectrum/ContextualHelp.html)
- [Geti UI Theming](/components/theming)

## Examples

### Help variant

```tsx preview
import { ContextualHelp } from '@geti-ai/ui';
import { Content, Heading, Text } from '@adobe/react-spectrum';

export default function App() {
  return (
    <ContextualHelp variant="help">
      <Heading>Need help?</Heading>
      <Content>
        <Text>
          This field accepts a maximum of 64 characters.
          Use a descriptive name to easily identify the item later.
        </Text>
      </Content>
    </ContextualHelp>
  );
}
```

### Info variant

```tsx preview
import { ContextualHelp } from '@geti-ai/ui';
import { Content, Heading, Text } from '@adobe/react-spectrum';

export default function App() {
  return (
    <ContextualHelp variant="info">
      <Heading>About this feature</Heading>
      <Content>
        <Text>This feature is currently in beta. Some functionality may change before the final release.</Text>
      </Content>
    </ContextualHelp>
  );
}
```

### Next to a label

```tsx preview
import { ContextualHelp } from '@geti-ai/ui';
import { Content, Flex, Heading, Text } from '@adobe/react-spectrum';

export default function App() {
  return (
    <Flex gap="size-100" alignItems="center">
      <Text>Dataset name</Text>
      <ContextualHelp variant="help">
        <Heading>Dataset name</Heading>
        <Content>
          <Text>
            Enter a unique name for your dataset.
            This name will be displayed throughout the application.
          </Text>
        </Content>
      </ContextualHelp>
    </Flex>
  );
}
```

### Top placement

```tsx preview
import { ContextualHelp } from '@geti-ai/ui';
import { Content, Heading, Text } from '@adobe/react-spectrum';

export default function App() {
  return (
    <div style={{ marginTop: '6rem' }}>
      <ContextualHelp variant="help" placement="top">
        <Heading>Top placement</Heading>
        <Content>
          <Text>This help popover appears above the icon button.</Text>
        </Content>
      </ContextualHelp>
    </div>
  );
}
```

## Responsibility

`ContextualHelp` wraps React Spectrum's `ContextualHelp`. It renders a small icon button (question mark for `help`, info circle for `info`) that opens a read-only popover with a `Heading` and `Content`. It is not suited for interactive content — use `Popover` or `CustomPopover` when the popover needs form elements or actions.

## Variants and states

| Variant | Icon | Typical use |
|---|---|---|
| `help` | Question mark (?) | Explaining how to fill in a form field |
| `info` | Info circle (ⓘ) | Providing background context about a feature |

## Props (deep dive)

### Content

- `children`: must include a `Heading` and `Content` from `@adobe/react-spectrum`. The content is read-only; no action buttons are rendered inside the popover.
- `variant`: `"help"` or `"info"` — controls the icon and tone.

### Placement

- `placement`: positions the popover relative to the icon button. Accepts `"top"`, `"bottom"`, `"start"`, `"end"`, and corner variants. Defaults to `"bottom start"`.

### Accessibility

- The icon button has a built-in accessible label ("Help" or "Info") provided by React Spectrum.
- Focus returns to the icon button when the popover closes.
- Content inside the popover is not reachable until the popover opens.

## Related components

- [`Popover`](/components/overlays/Popover) — use for popovers that need arbitrary `Dialog` content.
- [`Tooltip`](/components/overlays/Tooltip) — use for brief one-line hints; ContextualHelp is better for multi-sentence explanations.
