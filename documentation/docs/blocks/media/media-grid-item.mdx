# Media grid item

`MediaGridItem` is the composable item shell used inside `MediaGrid`.

It provides:

- selectable pressable container
- four overlay slots (`topLeft`, `topRight`, `bottomLeft`, `bottomRight`)
- hover/selected behavior for top corners
- persistent top-right actions while menu is open

Use it when you need domain-specific overlays and interactions (checkboxes, action menus, quality/status badges, IDs) but still want consistent grid item behavior.

```tsx
import {
  MediaGridItem,
  MediaGridThumbnailItem,
  MediaGridItemCheckbox,
  MediaGridItemMenu,
  MediaGridItemInfo,
  MediaGridItemStatus,
  type MediaGridRenderContext,
} from '@geti-ai/blocks';
```

## Reference-style custom item

```tsx preview
import { useState } from 'react';
import { ThemeProvider } from '@geti-ai/ui';
import {
  MediaGridItem,
  MediaGridItemCheckbox,
  MediaGridItemInfo,
  MediaGridItemMenu,
} from '@geti-ai/blocks';

const media = {
  id: 'media-1',
  name: 'Asset 1',
  thumbnailUrl: 'https://picsum.photos/seed/grid-item-single/400/400',
  score: 0.92,
};

export default function App() {
  const [isSelected, setIsSelected] = useState(true);

  return (
    <ThemeProvider>
      <div style={{ width: 260, height: 260 }}>
        <MediaGridItem
          isSelected={isSelected}
          onPress={() => setIsSelected((current) => !current)}
          onDoublePress={() => console.log('open preview', media.id)}
          topLeft={
            <MediaGridItemCheckbox
              ariaLabel={`Select ${media.name}`}
              isSelected={isSelected}
              onChange={() => setIsSelected((current) => !current)}
            />
          }
          topRight={
            <MediaGridItemMenu
              ariaLabel="Media actions"
              actions={[
                { key: 'annotate', label: 'Annotate' },
                { key: 'download', label: 'Download' },
                { key: 'delete', label: 'Delete' },
              ]}
              onAction={(key) => console.log(key, media.id)}
            />
          }
          bottomLeft={<MediaGridItemInfo>{media.score}</MediaGridItemInfo>}
          bottomRight={<MediaGridItemInfo>{media.id}</MediaGridItemInfo>}
        >
          <img
            src={media.thumbnailUrl}
            alt={media.name}
            style={{ width: '100%', height: '100%', objectFit: 'cover', display: 'block' }}
          />
        </MediaGridItem>
      </div>
    </ThemeProvider>
  );
}
```

## `MediaGridItem` props

| Prop | Type | Required | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | Yes | Main item content (typically image/video thumbnail). |
| `isSelected` | `boolean` | No | Controls selected visuals. Default: `false`. |
| `isPlaceholder` | `boolean` | No | Applies placeholder visual treatment. Default: `false`. |
| `onPress` | `() => void` | No | Press interaction handler. |
| `onDoublePress` | `() => void` | No | Double-press interaction handler. |
| `topLeft` | `ReactNode` | No | Optional top-left overlay slot. |
| `topRight` | `ReactNode` | No | Optional top-right overlay slot. |
| `bottomLeft` | `ReactNode` | No | Optional bottom-left overlay slot. |
| `bottomRight` | `ReactNode` | No | Optional bottom-right overlay slot. |

## Integration notes

- Keep `MediaGridItem` inside a fixed-size grid/row/column cell (it is designed to fill parent width/height).
- Use overlay parts (`MediaGridItemCheckbox`, `MediaGridItemMenu`, etc.) for consistent interaction layering.
- Keep selection state in your container (`selectedKeys`) and derive `isSelected` from grid context.
- Use `onDoublePress` for “open/preview” actions, and `topRight` menu for secondary actions.
