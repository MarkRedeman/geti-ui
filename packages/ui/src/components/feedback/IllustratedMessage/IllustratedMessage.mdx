## Responsibility

`IllustratedMessage` is a specialized layout component designed to fill a large, prominent empty area. It typically consists of a decorative illustration, a heading (title), and body content (description).

### When to use IllustratedMessage
- **Empty States**: Filling a large empty table, list, or canvas when no results are found.
- **Section-level Errors**: Displaying full-page or section-level errors like a 404 or 500 page.
- **First-use Onboarding**: Prompting a user to take an initial action, such as "Create your first dataset."
- **Drag-and-Drop Targets**: Providing a visual target for file uploads.

### Comparison with other components
| Scenario | Use instead |
| :--- | :--- |
| Brief, dismissible status message | `Toast` or `InlineAlert` |
| Warning or error inline within a form | `InlineAlert` |
| destructive confirmation with action | `AlertDialog` |

### Usage Rules
- **Always provide a Heading**: The heading acts as the accessible name of the message for screen readers.
- **Spectrum Illustrations**: Use standard Adobe Spectrum illustrations for visual consistency.
- **Not for inline validation**: Use field-level error messages for form validation instead of `IllustratedMessage`.

## Variants

### Standard Layout
Combines an illustration, a heading, and content to provide a clear explanation for the current state.

## Accessibility

### Roles and Labelling
- `IllustratedMessage` renders as a generic container with no specific landmark role by default.
- **Accessible Name**: The label is derived from the `Heading` child. If a `Heading` is omitted, the illustration itself must have an `aria-label`.
- **Decorative Illustrations**: When a `Heading` is present, the illustration should be marked as decorative (`role="presentation"` and `aria-hidden="true"`) to avoid duplicate announcements.

### Dynamic States
If the message represents a live change in state (e.g., a search results list that just became empty), wrap the component in an `aria-live="polite"` region.
