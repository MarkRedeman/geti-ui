## Responsibility

Provides a mutually exclusive selection input composed of two co-located components: `RadioGroup` (the container managing selection state and accessibility) and `Radio` (an individual option). Both are thin wrappers over Adobe React Spectrum so Geti consumers import from one place.

## Design

Both components are pure pass-throughs:

```tsx
// RadioGroup.tsx
export interface RadioGroupProps extends SpectrumRadioGroupProps {}
export const RadioGroup = (props: RadioGroupProps) => <SpectrumRadioGroup {...props} />;

// Radio.tsx
export interface RadioProps extends SpectrumRadioProps {}
export const Radio = (props: RadioProps) => <SpectrumRadio {...props} />;
```

No CSS module, no `UNSAFE_className`, no internal state. Full Spectrum API preserved: `label`, `value`, `defaultValue`, `onChange`, `isDisabled`, `orientation`, `validationState` on the group; `value`, `isDisabled`, `children` on each `Radio`.

## Flow

`RadioGroup` wraps children in an accessible `role="radiogroup"` with `aria-labelledby`. Each `Radio` child renders an `<input type="radio">` tied to the group's selection state via Spectrum's context. Selecting a `Radio` fires `RadioGroup`'s `onChange` with the new value.

## Integration

- **Depends on**: `@adobe/react-spectrum` (`RadioGroup`, `Radio`, `SpectrumRadioGroupProps`, `SpectrumRadioProps`)
- **Usage pattern**: `<RadioGroup><Radio value="a">A</Radio><Radio value="b">B</Radio></RadioGroup>`
- **Used by**: settings forms, mode selectors, any single-choice input
