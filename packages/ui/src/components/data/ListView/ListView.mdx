# ListView

An interactive collection of items with support for selection, actions, and drag-and-drop reordering.

```tsx
import { ListView, Item } from '@geti-ai/ui';
```

- [React Spectrum V3 ListView](https://react-spectrum.adobe.com/react-spectrum/ListView.html)
- [React Aria GridList (Headless Equivalent)](https://react-spectrum.adobe.com/react-aria/GridList.html)

<Stories />

## Responsibility

`ListView` is the right choice when users need to **interact with a collection of items** as individual entities — navigating into them, selecting them, or performing actions on them. It is distinct from simpler list primitives in that it separates **row selection** from **row actions**: a row can be opened or actioned (via `onAction`) independently of whether it is selected.

Key responsibilities:

- **Rich Item Rendering**: Displays items with thumbnails, descriptions, metadata, or secondary actions (e.g., a file browser or dataset list).
- **Batch Operations**: Lets users select one or many items and act on the entire selection.
- **Independent Actions**: Rows support an independent activation action (e.g., double-click to open, Enter to navigate) that is decoupled from selection.
- **Drag and Drop**: Supports reordering items within the list or transferring items between lists.

### When to use

- When items have thumbnails, descriptions, metadata, or secondary actions (e.g., a model list, a dataset browser, an annotation queue).
- When users need to **select multiple items** and apply a batch operation (delete, export, assign label).
- When rows need both a **selection state** and a separate **activation action** — for example, a checkbox to select and a double-click to open.
- When the list supports **drag-and-drop reordering** or cross-list transfers.

### When to avoid

- **vs. `ListBox`**: Use `ListBox` for simple selection inside a form or dropdown where no nested interactive elements or row actions are needed. `ListBox` renders as `role="listbox"` and is best for option-picking, not record navigation.
- **vs. `TableView`**: Use `TableView` for **structured, multi-column data** that requires sorting, column resizing, or visual comparison across fields (e.g., a training job table). `ListView` is better for heterogeneous or media-rich items where a grid of columns would feel excessive.
- **vs. `CardView`**: Use `CardView` for **visual, card-style asset browsing** with grid or waterfall layouts where the visual thumbnail is the primary affordance. Switch to `ListView` when users prefer a compact list layout or when metadata density matters more than visual presentation.
- For **simple read-only enumeration** with no interaction — use a plain `<ul>` or a `Content` block.

## Design

`ListView` is a **thin wrapper** around Spectrum's `SpectrumListView`. It extends `SpectrumListViewProps` directly without adding or modifying props, keeping the Geti wrapper as a stable integration point for future customisation (e.g., analytics, theming overrides, or migration to `react-aria-components`).

```tsx
// packages/ui/src/components/data/ListView/ListView.tsx

export interface ListViewProps<T extends object> extends SpectrumListViewProps<T> {}

export const ListView = <T extends object>(props: ListViewProps<T>) => <SpectrumListView {...props} />;
```

**`Item` co-location**

`Item` is re-exported directly from `@adobe/react-spectrum` alongside `ListView` so consumers never need to import from two packages:

```tsx
// ListView.tsx
export const Item = SpectrumItem; // re-exported from @adobe/react-spectrum
```

This means:

```tsx
// One import covers both the container and its children
import { ListView, Item } from '@geti-ai/ui';
```

**Generic type `<T extends object>`**

`ListView` is generic over `T extends object`. This type parameter is used when supplying items via the `items` prop (collection API) instead of static `Item` children — for example, when your list is driven by server data:

```tsx
const files = [
    { id: 'f1', name: 'dataset-v1.zip', size: '1.2 GB' },
    { id: 'f2', name: 'dataset-v2.zip', size: '1.8 GB' },
];

<ListView<(typeof files)[number]> items={files} aria-label="Dataset files" selectionMode="multiple">
    {(file) => <Item key={file.id}>{file.name}</Item>}
</ListView>;
```

**Focus strategy: Roving Tabindex**

`ListView` uses a **roving tabindex** strategy for keyboard navigation. The entire list is a single tab stop in the page's tab order — `tabIndex=0` is assigned to the currently focused row, and all other rows are set to `tabIndex=-1`. Once a user tabs into the list, they navigate between rows using the `Arrow` keys rather than pressing `Tab`. This prevents users from having to tab through every row to reach content below the list.

**DOM role: `role="grid"`**

`ListView` renders as `role="grid"` (not `role="list"`). This composite role signals to assistive technology that the collection contains interactive cells — supporting two-dimensional keyboard navigation and complex selection semantics. Each row is `role="row"` and each cell is `role="gridcell"`, matching the ARIA grid pattern.

**Theme integration**

`ListView` inherits all Spectrum design tokens (row height, selection highlight colour, border, focus ring) from the `ThemeProvider` context. No per-component CSS overrides are needed in standard usage.

## Variants

### Density

`ListView` supports three density variants to balance scannability and information depth:

- `compact`: Minimises vertical space — ideal for data-dense dashboards or toolbars.
- `regular` (default): The balanced default for most content.
- `spacious`: Adds extra padding for more relaxed, touch-friendly layouts.

### Selection Modes

- `none`: No selection allowed. Use when the list is purely navigational (every row fires `onAction`).
- `single`: One item can be selected at a time (replace mode by default — arrow keys move selection).
- `multiple`: Multiple items can be selected using checkboxes or highlight selection (toggle mode by default).

### Loading States

The `loadingState` prop communicates async status to both the UI and assistive technology:

- `loading`: Initial load — shows a spinner.
- `loadingMore`: Pagination in progress — shows a spinner at the bottom of the list.
- `sorting`: A sort operation is in progress.
- `error`: Data fetch failed.
- `idle`: Data loaded, list is interactive.

## Accessibility

`ListView` implements the [WAI-ARIA Grid pattern](https://www.w3.org/WAI/ARIA/apg/patterns/grid/).

### ARIA role structure

| Element                 | Role       | Notes                                                                                              |
| ----------------------- | ---------- | -------------------------------------------------------------------------------------------------- |
| `ListView` root         | `grid`     | Composite widget role. Announces to screen readers that the collection contains interactive cells. |
| Each item row           | `row`      | One `row` per `Item`. Receives `aria-selected` when `selectionMode` is active.                     |
| Each item cell          | `gridcell` | Contains the item's content and any inline actions.                                                |
| Checkbox (multi-select) | `checkbox` | Rendered automatically inside the `gridcell` when `selectionMode="multiple"`.                      |

The `role="grid"` composite role is intentional: it communicates to AT that rows contain interactive content (selection, actions, drag handles) rather than being purely informational list items. This is why `ListView` uses `grid` instead of the simpler `list` / `listitem` roles.

### Mandatory `aria-label`

Always provide `aria-label` (or `aria-labelledby`) on `ListView`. Without it, the `grid` landmark is announced without context, making it impossible for screen-reader users to understand what the list contains — especially if multiple lists appear on the same page.

```tsx
// Good — screen reader announces "Project datasets, grid"
<ListView aria-label="Project datasets" selectionMode="multiple">…</ListView>

// Also acceptable — label sourced from a visible heading
<h2 id="datasets-heading">Project datasets</h2>
<ListView aria-labelledby="datasets-heading" selectionMode="multiple">…</ListView>
```

### Focus management

`ListView` uses a **roving tabindex** strategy. The entire grid is a single tab stop — `tabIndex=0` is placed on the currently focused row, all others are `tabIndex=-1`. Once focus enters the list, users navigate with `Arrow` keys rather than `Tab`, which prevents `Tab` from landing on every row in the page's focus order.

### Keyboard interactions

| Key                          | Behaviour                                                                                             |
| ---------------------------- | ----------------------------------------------------------------------------------------------------- |
| `Tab`                        | Moves focus into the `ListView`, landing on the most recently focused row (or the first row if none). |
| `Arrow Down` / `Arrow Up`    | Moves focus to the next / previous row.                                                               |
| `Arrow Right` / `Arrow Left` | Moves focus between cells within a row (e.g., row label → inline action button).                      |
| `Home`                       | Moves focus to the first row.                                                                         |
| `End`                        | Moves focus to the last row.                                                                          |
| `Space`                      | **Toggle mode (multiple):** Toggles selection of the focused row without activating it.               |
| `Enter`                      | Activates the focused row — fires `onAction`.                                                         |
| `Escape`                     | Clears the selection (if `selectionMode` is active).                                                  |
| `Ctrl/Cmd + A`               | Selects all rows (when `selectionMode="multiple"`).                                                   |
| **Drag & drop**              |                                                                                                       |
| `Enter` (on drag handle)     | Enters drag mode for the focused row(s).                                                              |
| `Arrow Up` / `Arrow Down`    | Moves the dragged item(s) while in drag mode.                                                         |
| `Enter` (during drag)        | Drops the item(s) at the current position.                                                            |
| `Escape` (during drag)       | Cancels the drag and returns items to their original positions.                                       |

## Integration

- **Dependencies**: `@adobe/react-spectrum` — `ListView`, `Item`, `SpectrumListViewProps`, and `useDragAndDrop` are all sourced from this package. `Item` is re-exported from `@geti-ai/ui` as a convenience so consumers only need one import.
- **ThemeProvider**: Must be present in the React tree. Row height, selection colour, focus ring, and drag indicator all derive from Spectrum tokens resolved by `ThemeProvider`.

### Selection vs. Action

`selectionMode` and `onAction` serve different purposes and can be used together or independently:

| Intent                           | Mechanism                             | Typical use                             |
| -------------------------------- | ------------------------------------- | --------------------------------------- |
| Mark items for a batch operation | `selectionMode` + `onSelectionChange` | "Select 3 files → Delete"               |
| Open / navigate to a single item | `onAction`                            | "Click row → open detail panel"         |
| Both at the same time            | `selectionMode` + `onAction`          | "Checkbox to select, click row to open" |

```tsx
import { useState } from 'react';
import type { Key, Selection } from '@adobe/react-spectrum';
import { ListView, Item } from '@geti-ai/ui';

const files = [
    { id: 'f1', name: 'dataset-v1.zip' },
    { id: 'f2', name: 'dataset-v2.zip' },
    { id: 'f3', name: 'annotations.json' },
];

export function FileManager() {
    const [selectedKeys, setSelectedKeys] = useState<Selection>(new Set());

    const handleAction = (key: Key) => {
        // Fired when a row is "activated" (Enter key or single-click in action mode).
        // Independent of selection — this navigates, not selects.
        console.log('Open file:', key);
    };

    return (
        <ListView<(typeof files)[number]>
            items={files}
            aria-label="Project files"
            selectionMode="multiple"
            selectedKeys={selectedKeys}
            onSelectionChange={setSelectedKeys}
            onAction={handleAction}
            width="size-6000"
        >
            {(file) => <Item key={file.id}>{file.name}</Item>}
        </ListView>
    );
}
```

> **Tip:** When `selectionMode="multiple"`, clicking a row selects it. `onAction` is triggered by pressing `Enter` (or double-clicking, depending on the `selectionBehavior` prop). Set `selectionBehavior="replace"` to make single-click activate rows and use checkboxes for selection.

### Controlled vs. uncontrolled selection

```tsx
// Uncontrolled — ListView manages selected keys internally
<ListView aria-label="Files" selectionMode="multiple" defaultSelectedKeys={['f1']}>
    <Item key="f1">dataset-v1.zip</Item>
    <Item key="f2">dataset-v2.zip</Item>
</ListView>;

// Controlled — you own the state
const [selected, setSelected] = useState<Selection>(new Set(['f1']));

<ListView aria-label="Files" selectionMode="multiple" selectedKeys={selected} onSelectionChange={setSelected}>
    <Item key="f1">dataset-v1.zip</Item>
    <Item key="f2">dataset-v2.zip</Item>
</ListView>;
```

Use `defaultSelectedKeys` (uncontrolled) when no sibling component needs to react to the selection. Use `selectedKeys` + `onSelectionChange` (controlled) when:

- A toolbar or action bar needs to read the current selection to enable/disable batch actions.
- The selection is derived from or persisted in application state (e.g., a Redux store).
- You need to constrain or validate which items can be selected.

### Drag and Drop

`ListView` supports drag-and-drop reordering via the `dragAndDropHooks` prop. The hooks are provided by the `useDragAndDrop` utility from `@adobe/react-spectrum`, which accepts handlers for the full drag lifecycle.

```tsx
import { useDragAndDrop } from '@adobe/react-spectrum';
import type { Key } from '@adobe/react-spectrum';
import { useState } from 'react';
import { ListView, Item } from '@geti-ai/ui';

interface FileItem {
    id: string;
    name: string;
}

export function ReorderableList() {
    const [items, setItems] = useState<FileItem[]>([
        { id: 'f1', name: 'dataset-v1.zip' },
        { id: 'f2', name: 'dataset-v2.zip' },
        { id: 'f3', name: 'annotations.json' },
        { id: 'f4', name: 'model-weights.pt' },
    ]);

    const { dragAndDropHooks } = useDragAndDrop({
        getItems: (keys) =>
            [...keys].map((key) => ({
                'text/plain': items.find((item) => item.id === key)?.name ?? String(key),
            })),

        onReorder(event) {
            const { keys, target } = event;
            const movedIds = [...keys] as string[];
            const targetId = target.key as string;
            const targetPos = target.dropPosition; // 'before' | 'after'

            setItems((prev) => {
                const moved = prev.filter((item) => movedIds.includes(item.id));
                const remaining = prev.filter((item) => !movedIds.includes(item.id));
                const insertIndex = remaining.findIndex((item) => item.id === targetId);
                const offset = targetPos === 'before' ? 0 : 1;
                const next = [...remaining];
                next.splice(insertIndex + offset, 0, ...moved);
                return next;
            });
        },
    });

    return (
        <ListView<FileItem>
            items={items}
            aria-label="Reorderable file list"
            dragAndDropHooks={dragAndDropHooks}
            selectionMode="multiple"
            width="size-6000"
        >
            {(item) => <Item key={item.id}>{item.name}</Item>}
        </ListView>
    );
}
```

> **Key points:**
>
> - `getItems` serialises the dragged rows into a transfer format (here `text/plain`). Use a structured MIME type (e.g., `application/vnd.geti.file`) for cross-list transfers.
> - `onReorder` fires only when items are dropped back into the **same** list. Use `onInsert` for drops from an external source.
> - Pass `selectionMode="multiple"` alongside `dragAndDropHooks` to allow dragging multiple selected rows at once.

### Pairing with a search/filter bar

A common pattern is to pair `ListView` with a `SearchField` or filter controls above it. The list is driven by filtered data derived from the search state:

```tsx
import { useState } from 'react';
import { SearchField, Flex } from '@adobe/react-spectrum';
import { ListView, Item } from '@geti-ai/ui';

const allFiles = [
    { id: 'f1', name: 'dataset-v1.zip' },
    { id: 'f2', name: 'dataset-v2.zip' },
    { id: 'f3', name: 'annotations.json' },
];

export function FilterableList() {
    const [query, setQuery] = useState('');
    const filtered = allFiles.filter((f) => f.name.toLowerCase().includes(query.toLowerCase()));

    return (
        <Flex direction="column" gap="size-100" width="size-6000">
            <SearchField label="Filter files" value={query} onChange={setQuery} aria-controls="file-list" />
            <ListView<(typeof allFiles)[number]>
                id="file-list"
                items={filtered}
                aria-label="Filtered file list"
                selectionMode="multiple"
            >
                {(file) => <Item key={file.id}>{file.name}</Item>}
            </ListView>
        </Flex>
    );
}
```

### Pairing with an action toolbar

Connect `onSelectionChange` to a toolbar to enable batch actions when rows are selected:

```tsx
import { useState } from 'react';
import type { Selection } from '@adobe/react-spectrum';
import { ActionButton, Flex } from '@adobe/react-spectrum';
import { ListView, Item } from '@geti-ai/ui';

export function ListWithToolbar() {
    const [selectedKeys, setSelectedKeys] = useState<Selection>(new Set());
    const hasSelection = selectedKeys === 'all' || selectedKeys.size > 0;

    return (
        <Flex direction="column" gap="size-100" width="size-6000">
            <Flex gap="size-100">
                <ActionButton isDisabled={!hasSelection} onPress={() => console.log('Delete', selectedKeys)}>
                    Delete selected
                </ActionButton>
                <ActionButton isDisabled={!hasSelection} onPress={() => console.log('Export', selectedKeys)}>
                    Export
                </ActionButton>
            </Flex>
            <ListView
                aria-label="Project files"
                selectionMode="multiple"
                selectedKeys={selectedKeys}
                onSelectionChange={setSelectedKeys}
                width="size-6000"
            >
                <Item key="f1">dataset-v1.zip</Item>
                <Item key="f2">dataset-v2.zip</Item>
                <Item key="f3">annotations.json</Item>
            </ListView>
        </Flex>
    );
}
```

## Related components

- **`TableView`** — use instead when data is multi-column and users need to sort, compare, or resize columns; `TableView` virtualises structured tabular data.
- **`CardView`** — use when visual thumbnails are the primary affordance and a 2D grid layout is preferred over a linear list.
- **`ListBox`** — use for simple option selection inside a form or dropdown where no nested interactive elements, row actions, or drag-and-drop are needed.
- **`ActionBar`** — pair with `ListView` inside `ActionBarContainer` to present bulk-action buttons when rows are selected.
