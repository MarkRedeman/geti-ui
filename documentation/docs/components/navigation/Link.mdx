# Link

Link renders a styled anchor for navigating to internal pages or external URLs. Use it wherever plain text navigation is needed ŌĆö documentation, breadcrumb helpers, or inline contextual references.

```tsx
import { Link } from '@geti-ai/ui';
```

- [React Spectrum Link ŌåŚ](https://react-spectrum.adobe.com/react-spectrum/Link.html)
- [Geti UI Theming](/components/theming)

## Examples

### Default (primary)

```tsx preview
import { Link } from '@geti-ai/ui';

export default function App() {
  return <Link href="https://example.com">Learn more</Link>;
}
```

### Primary variant

```tsx preview
import { Link } from '@geti-ai/ui';

export default function App() {
  return <Link variant="primary" href="https://example.com">Primary link</Link>;
}
```

### Secondary variant

```tsx preview
import { Link } from '@geti-ai/ui';

export default function App() {
  return <Link variant="secondary" href="https://example.com">Secondary link</Link>;
}
```

### Quiet (no underline)

```tsx preview
import { Link } from '@geti-ai/ui';

export default function App() {
  return <Link isQuiet href="https://example.com">Quiet link</Link>;
}
```

## Responsibility

`Link` is a thin wrapper around React Spectrum `Link` with no behavioral changes. It inherits Geti theme tokens through `ThemeProvider` and participates in Spectrum's focus/keyboard navigation ring.

## Variants and states

| Variant | Typical use |
|---|---|
| `primary` (default) | Standard inline navigation link |
| `secondary` | De-emphasised secondary references |
| `overBackground` | Links on non-default coloured backgrounds |

---

| Prop | Notes |
|---|---|
| `isQuiet` | Removes underline; use sparingly for dense layouts |

## Props (deep dive)

### Content

- `children`: visible link text.
- `href`: navigation destination.
- `target`: e.g. `_blank` for new tab ŌĆö add `rel="noopener noreferrer"` when opening external links.

### Visual

- `variant`: `primary | secondary | overBackground`.
- `isQuiet`: suppress underline decoration.

### Events

- `onPress`: preferred handler for press / click interaction.

### Accessibility

- always use descriptive link text (`View project report`, not `click here`),
- links must have visible focus rings ŌĆö do not override with custom CSS.

## Related components

- `Breadcrumbs` ŌĆö hierarchical path navigation using links.
- `Button` ŌĆö use when the target triggers an action rather than navigation.
