# Menu

Menu presents a list of actions or options in a floating panel triggered by a button press. Use it for contextual actions on a selected item, editor commands, or any operation set that would be too wide to display inline.

```tsx
import { MenuItem as Item, Menu, MenuTrigger, MenuSection as Section } from '@geti/ui';
```

- [React Spectrum Menu ↗](https://react-spectrum.adobe.com/react-spectrum/Menu.html)
- [Geti UI Theming](/components/theming)

## Responsibility

`Menu` is a thin wrapper around React Spectrum `Menu`. Compose it inside a `MenuTrigger` with a pressable trigger element. Spectrum handles focus management, keyboard navigation, and ARIA role assignments automatically.

## Variants and states

| Prop | Options | Notes |
|---|---|---|
| `selectionMode` | `none` (default) / `single` / `multiple` | Controls whether items can be checked |
| `disabledKeys` | string[] | Renders specific items non-interactive |

## Props (deep dive)

### Content

- `children`: `Item` or `Section` elements. Each `Item` requires a unique `key`.
- `Section`: groups items with an optional visible `title`.

### Value / state

- `selectionMode`: `none | single | multiple`.
- `defaultSelectedKeys` / `selectedKeys` + `onSelectionChange`: manage checked states.
- `disabledKeys`: items that cannot be selected.

### Events

- `onAction`: called with the key of the pressed item (use instead of `onSelectionChange` for action-only menus).

### Accessibility

- `Menu` renders a `role="menu"` with `role="menuitem"` children automatically.
- Use `textValue` on an `Item` if its label is not plain text (e.g. icon-only items).

## Related components

- `ActionMenu` — a pre-composed `MenuTrigger` + `ActionButton` + `Menu` with a ⋯ trigger icon.
- `Picker` — use when selecting a value to display in a form field, not for triggering actions.
