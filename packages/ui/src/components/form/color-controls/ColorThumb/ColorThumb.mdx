## Responsibility

`ColorThumb` is the only color component in this library that does **not** wrap `@adobe/react-spectrum`. It is a small, static, purely presentational element: a square block of solid color used to show the currently selected color at a glance â€” for example, inside a trigger button before the color picker opens, or in a table cell alongside a color label.

Key responsibilities:
- **Color Preview**: Renders a solid square with `background-color` driven by a CSS custom property (`--color-thumb-bg`), keeping dynamic color values in CSS rather than inline `style` attributes.
- **Flexible Sizing**: The `size` prop controls both width and height uniformly, defaulting to `10` px. Sizes are integer pixel values â€” not Spectrum dimension tokens.
- **CSS Custom Property Pattern**: Uses `UNSAFE_style` to inject `--color-thumb-bg` into the underlying `View`. The CSS module then references `var(--color-thumb-bg)` for `background-color`. This is the correct Spectrum-compatible pattern for dynamic color values in a CSS module.
- **Composable**: Extends `ViewProps` (excluding `width`, `height`, and `UNSAFE_style`), so all Spectrum layout and spacing props (`margin`, `flex`, `alignSelf`, etc.) are available.

### When to use
- To show the current color selection inside a trigger button (e.g., the `ActionButton` trigger in `ColorPickerDialog`).
- As an inline color indicator in table cells, list items, or annotation label chips.
- Anywhere a compact, non-interactive color swatch is needed without the overhead of `ColorSwatch` from the swatch picker family.

### When to avoid
- When the swatch needs to be selectable â€” use `ColorSwatch` inside `ColorSwatchPicker` instead.
- When you need to display a gradient or pattern â€” `ColorThumb` only renders solid colors.
- When the color string might be `undefined` â€” always guard against `undefined` before passing to `color`, as an empty custom property will render a transparent background with no visible indicator.

## Design

Built on the library's own `View` component with a CSS module and a CSS custom property:

```tsx
export const ColorThumb = ({
  color,
  size = 10,
  UNSAFE_className,
  UNSAFE_style,
  ...rest
}: ColorThumbProps) => (
  <View
    {...rest}
    width={size}
    height={size}
    UNSAFE_className={clsx(styles.colorThumb, UNSAFE_className)}
    UNSAFE_style={{
      '--color-thumb-bg': color,
      ...UNSAFE_style,
    } as CSSPropertiesWithVars}
  />
);
```

`ColorThumb.module.css` owns the `background-color: var(--color-thumb-bg)` declaration. This keeps the dynamic color value out of inline styles while remaining fully compatible with TypeScript's `CSSProperties` type via the `CSSPropertiesWithVars` alias (`CSSProperties & Record<\`--${string}\`, string>`).

## Integration

- **Depends on**: library's own `View` (`../../../ui/View/View`), `clsx`, `ColorThumb.module.css`
- **Does not depend on**: `@adobe/react-spectrum` directly â€” `View` is the only dependency, and `View` wraps Spectrum's `View`
- **Used by**: `ColorPickerDialog` uses the related `ColorSwatch` (a Spectrum component) for its trigger; `ColorThumb` is available for all other inline color indicator use cases
- **Not interactive**: `ColorThumb` renders no button, no press handler, and no focusable element. If you need a clickable swatch, wrap it in a `PressableElement` or use `ColorSwatch` within `ColorSwatchPicker`.

### Usage examples

```tsx

  <Flex gap="size-100" alignItems="center">
    <ColorThumb color={currentColor} size={16} />

  </Flex>

// In a table cell
```

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `color` | `string` | **Required** | Any valid CSS color string (`#rrggbb`, `rgb(...)`, `hsl(...)`, named colors). |
| `size` | `number` | `10` | Width and height in pixels. |
| `UNSAFE_className` | `string` | â€” | Additional CSS class names. Merged with the module class via `clsx`. |
| `UNSAFE_style` | `CSSProperties` | â€” | Additional inline styles. Merged after the `--color-thumb-bg` custom property. |

All `ViewProps` layout and spacing props (`margin`, `flex`, `alignSelf`, `position`, etc.) are also accepted via `...rest`.

## Accessibility

- **Presentational**: `ColorThumb` is a purely visual element. It renders as a `<div>` (via `View`) with no semantic role, no `aria-label`, and no keyboard interaction.
- **Color-only information**: If `ColorThumb` is the sole indicator of a value, ensure that the surrounding context provides a text alternative. For example, always pair it with a visible color name or hex value in the adjacent text, so users who cannot perceive color still receive the information.
- **Screen reader treatment**: Because it carries no accessible label, screen readers will typically skip it. If the thumb is meaningful in isolation (e.g., inside a button), add `aria-label` to the **parent button**, not to the thumb itself.
- **Do not focus**: `ColorThumb` is not in the tab order. Never add a `tabIndex` or `onPress` handler directly to it.

## Related components
- `ColorSwatch` â€” use instead when the color display needs a standardized size, shape, and optional interactivity in a `ColorSwatchPicker`.
- `ColorPickerDialog` â€” use `ColorThumb` as the visual swatch inside the trigger `ActionButton` of a color picker.
- `ActionButton` â€” the typical parent element; wrap an `ActionButton` around `ColorThumb` to create a compact color picker trigger.
