# DateField

<Subtitle>

## Responsibility

`DateField` is a thin wrapper around Adobe React Spectrum's `DateField`. It presents a date value as a row of individually editable segments (e.g., `MM / DD / YYYY`) directly in the form layout — no popover, no calendar grid. Users tab between segments and type digits or use arrow keys to adjust values.

Key responsibilities:

- **Segment-based editing**: Each part of the date (month, day, year — and optionally hour, minute, second for `CalendarDateTime`) is a separate, focusable, editable field.
- **Type safety**: Generic over `T extends DateValue`, accepting `CalendarDate`, `CalendarDateTime`, or `ZonedDateTime` from `@internationalized/date`.
- **Form integration**: Exposes `label`, `description`, `errorMessage`, `isRequired`, `validationState`, and `isDisabled` to compose cleanly inside a `Form`.
- **Inline placement**: Renders in the normal document flow — useful in table cells, compact sidebars, or anywhere a full calendar popup would be disruptive.

### When to use

- In dense forms where a calendar popup would be spatially disruptive — inline table cell editors, filter bars, settings panels.
- When the exact date is already known and the user will type it directly rather than browse a calendar.
- When you need both date and time in a single compact field (pass a `CalendarDateTime` value).
- When accessibility requires that the date input be reachable and operable with keyboard only, without a mouse-driven popup.

### When to avoid

- When users need to browse for a date (e.g., "find a free slot next month") — use `DatePicker` or an inline `Calendar`.
- When users are selecting a date **range** — use `DateRangePicker` or a pair of `DateField` components with explicit start/end semantics.
- When the field is part of a mobile-first form — native date pickers are better on touch devices; `DatePicker` passes through to the platform picker more gracefully.

## Variants

### Default

Renders `MM / DD / YYYY` segments with a visible label. The user tabs or clicks into each segment and types.

### With description

Add `description` to provide helper text beneath the field (e.g., format hints, business rules about valid dates).

### Invalid

Set `validationState="invalid"` and `errorMessage` to surface a form validation error. The field border turns red and the message appears below.

### Disabled

`isDisabled` grays out all segments and removes them from the tab order. Use when the date is locked.

## Design

`DateField` is a pure pass-through wrapper, generic over Spectrum's `DateValue` union.

```tsx
export const DateField = <T extends DateValue>(props: SpectrumDateFieldProps<T>) => <SpectrumDateField {...props} />;

export type { SpectrumDateFieldProps as DateFieldProps };
```

Typical controlled usage:

```tsx
import { DateField } from '@geti/ui';
import { CalendarDate } from '@internationalized/date';

const [date, setDate] = useState<CalendarDate | null>(null);

<DateField label="Event date" value={date} onChange={setDate} isRequired minValue={new CalendarDate(2020, 1, 1)} />;
```

For date + time combined, pass a `CalendarDateTime`:

```tsx
import { CalendarDateTime } from '@internationalized/date';

<DateField<CalendarDateTime> label="Scheduled at" value={scheduledAt} onChange={setScheduledAt} />;
```

## Integration

- **Depends on**: `@adobe/react-spectrum` (`DateField`, `SpectrumDateFieldProps`), `@react-types/datepicker` (`DateValue`)
- **Date types**: `CalendarDate` for date-only, `CalendarDateTime` for date+time without timezone, `ZonedDateTime` for date+time with timezone. All from `@internationalized/date`.
- **Contrast with DatePicker**: `DatePicker` adds a calendar popover trigger to `DateField`. If you need only keyboard entry, use `DateField` — it is lighter and avoids overlay management.
- **Contrast with TimeField**: `TimeField` is scoped to time-only values (`Time`, `ZonedDateTime`). Use `DateField` when you need date components (or date + time combined).
- **Used in**: compact form rows, inline table cell editors, filter sidebars.

## Accessibility

`DateField` implements the [ARIA spinbutton pattern](https://www.w3.org/WAI/ARIA/apg/patterns/spinbutton/) for each editable segment.

- **Group role**: The field's container has `role="group"` with the field label as its accessible name via `aria-labelledby`.
- **Segment roles**: Each date segment (`MM`, `DD`, `YYYY`) has `role="spinbutton"` with `aria-valuemin`, `aria-valuemax`, and `aria-valuenow` reflecting the current digit.
- **Required/invalid**: `aria-required` and `aria-invalid` are applied to the group when the corresponding props are set. The `errorMessage` is linked via `aria-describedby`.
- **Keyboard navigation**:

| Key                   | Behaviour                                                         |
| --------------------- | ----------------------------------------------------------------- |
| `Tab`                 | Move focus into the field (first segment)                         |
| `Tab` / `Shift + Tab` | Move focus between segments                                       |
| `0`–`9`               | Type digits into the focused segment; auto-advances when complete |
| `↑` / `↓`             | Increment or decrement the focused segment's value by 1           |
| `Backspace`           | Clear the focused segment                                         |

- **Auto-advance**: Spectrum auto-advances focus to the next segment as soon as a complete, valid unit is entered (e.g., typing `1` then `2` in the month segment completes it and moves to the day segment). This removes the need to press `Tab` between every segment.
- **Placeholder communication**: Unfilled segments announce their placeholder (e.g., "month") so screen reader users know which segment they are on and what to type.

## Related components

- `DatePicker` — use instead when a calendar popover is needed alongside the field for visual date selection.
- `TimeField` — use alongside `DateField` when both a date and a time must be entered independently.
- `Calendar` — use instead when a standalone always-visible calendar grid is preferred over a segmented input.
