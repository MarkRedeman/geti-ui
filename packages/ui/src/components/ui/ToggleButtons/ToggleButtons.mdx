## Responsibility

`ToggleButtons` is a custom implementation of a selection group where exactly one option is active at a time. It is built using a collection of Geti `Button` components styled to appear as a single, cohesive "pill" or "segmented" control.

### What this component does
- Renders an array of options as a horizontal button group.
- Automatically handles visual selection states: the selected button uses `variant="accent"` (blue fill), while others use `variant="secondary"` (outlined).
- Applies `aria-pressed` attributes to correctly communicate state to assistive technologies.
- Provides a `getLabel` mapper for rendering icons or custom React nodes instead of raw text.

### Key Constraints
- **Single Selection**: This component is strictly mutually exclusive. It does not support multiple simultaneous selections.
- **Controlled State**: The component is fully controlled. The caller must provide the `selectedOption` and handle the `onOptionChange` callback.

---

## Variants

### Text Options
By default, the component renders the string or number values from the `options` array directly as button labels.

### Icon Options
Use the `getLabel` prop to map option values to icons or custom elements. This is commonly used for view-switching (e.g., Grid vs. List).

### Disabled Group
Set `isDisabled` to `true` to disable all buttons in the group simultaneously.

## Implementation Details

### Styling
The component uses a custom CSS module (`ToggleButtons.module.css`) to join the buttons into a single visual unit by removing inner border radii and adjacent borders.

### Accessibility
- **Group Role**: The container uses `role="group"` and `aria-label="Selection options"` to identify the collection.
- **Pressed State**: Individual buttons use `aria-pressed` to signal their selection status.
- **Note**: This custom implementation does not currently support arrow-key navigation within the group (roving tab index); each button remains an independent tab stop.

## Spectrum 2 Migration

`ToggleButtons` predates the official `ToggleButtonGroup` and `SegmentedControl` primitives in the Spectrum ecosystem. For new implementations or during migration, the following official replacements are recommended:

### Recommended Replacements
- **`SegmentedControl`** from `@react-spectrum/s2`: The closest semantic match for mutually exclusive view switching.
- **`ToggleButtonGroup`** from `react-aria-components` or `@react-spectrum/s2`: Best for groups that may require multi-selection or more advanced ARIA behaviors.

### Why Migrate?
Official Spectrum components provide superior accessibility:
- **Correct ARIA Roles**: Container uses `role="radiogroup"` with `role="radio"` items for single selection.
- **Keyboard Navigation**: Implements roving tab index (arrow keys move focus within the group; Tab moves focus in/out as a single unit).
- **Selection Models**: Supports both `single` and `multiple` selection modes natively.
