## Responsibility

The `TextField` component provides a single-line input field that allows users to enter and edit text. It is built on top of Adobe's React Spectrum and ensures a consistent look and feel across the application while maintaining high accessibility standards.

### When to use
- Use for short, bounded text inputs like names, email addresses, URLs, or passwords.
- Use when the input is expected to be a single line.

### When to use something else
- Use [TextArea](/docs/components-form-textarea--docs) for multi-line text input.
- Use [SearchField](/docs/components-form-searchfield--docs) for search-specific inputs which include a clear button and search icon.
- Use [NumberField](/docs/components-form-numberfield--docs) for numeric inputs that require formatting or increment/decrement controls.

## Variants

### States
- **Disabled**: The field is not focusable or interactive. Use `isDisabled`.
- **Read-only**: The field is focusable and the text can be copied, but the value cannot be changed. Use `isReadOnly`.
- **Invalid**: Indicates a validation error. In V3, use `validationState="invalid"`.
- **Required**: Marks the field as mandatory for form submission. Use `isRequired`.

### Constraints
- **Character Limit**: Use `maxLength` to limit the number of characters.
- **Pattern**: Use `pattern` for regex-based validation.

## Props

### Content
| Prop | Type | Description |
|---|---|---|
| `label` | `ReactNode` | The content to display as the label. |
| `placeholder` | `string` | Temporary text that appears in the field when it is empty. |
| `description` | `ReactNode` | Help text displayed below the field. |
| `errorMessage` | `ReactNode` | Error message displayed below the field when it is invalid. |
| `contextualHelp` | `ReactNode` | A `<ContextualHelp>` element placed next to the label. |

### Value / State
| Prop | Type | Default | Description |
|---|---|---|---|
| `value` | `string` | — | The current value (controlled). |
| `defaultValue` | `string` | — | The initial value (uncontrolled). |
| `type` | `string` | `'text'` | The type of input (e.g., `'text'`, `'email'`, `'password'`). |
| `validationState` | `'valid' \| 'invalid'` | — | Whether the input is valid or invalid. |
| `isRequired` | `boolean` | `false` | Whether user input is required on the field. |
| `isReadOnly` | `boolean` | `false` | Whether the field can be edited. |
| `isDisabled` | `boolean` | `false` | Whether the field is disabled. |

### Events
| Prop | Type | Description |
|---|---|---|
| `onChange` | `(value: string) => void` | Handler that is called when the value changes. |
| `onFocus` | `(e: FocusEvent) => void` | Handler that is called when the field receives focus. |
| `onBlur` | `(e: FocusEvent) => void` | Handler that is called when the field loses focus. |

### Styles
| Prop | Type | Description |
|---|---|---|
| `labelPosition` | `'top' \| 'side'` | The position of the label relative to the input. |
| `labelAlign` | `'start' \| 'end'` | The horizontal alignment of the label. |
| `width` | `DimensionValue` | The width of the component. |
| `isQuiet` | `boolean` | Whether the input should be displayed with a quiet style. |

## Accessibility

- The underlying `<input>` element is automatically labeled by the `label` prop via `aria-labelledby`.
- Support for `aria-describedby` for descriptions and error messages.
- Full keyboard support: **Tab** to focus, and standard text editing shortcuts.
- High contrast mode support and screen reader compatibility via Adobe Spectrum.

## Spectrum 2 Migration

When migrating to Spectrum 2 (S2), keep the following changes in mind:

| V3 Prop | S2 Equivalent | Notes |
|---|---|---|
| `validationState="invalid"` | `isInvalid` | Boolean flag replaces the string enum. |
| `validationState="valid"` | — | No visual "valid" state in S2. |
| `isQuiet` | — | The quiet variant has been removed in S2. |
| `width="size-2000"` | `styles={style({ width: 160 })}` | Dimensional props are replaced by the `style()` macro. |
| `size` | `size` | S2 adds a `size` prop (`'S'`, `'M'`, `'L'`, `'XL'`). |
