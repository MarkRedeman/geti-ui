## Responsibility

The `PasswordField` component is a specialized wrapper around Spectrum's `TextField`. It is specifically designed for sensitive data entry, providing an integrated "Show/Hide" toggle to let users verify their input while maintaining masking by default.

### When to use
- For login forms, registration, and account settings.
- When collecting any sensitive information that should be masked (PINs, API keys).
- When a reveal toggle is necessary for user confidence and error reduction.

### When to use something else
- Use [TextField](/docs/components-form-textfield--docs) with `type="text"` for non-sensitive text.
- Use [NumberField](/docs/components-form-numberfield--docs) for purely numeric values that are not sensitive.

## Variants

### States
- **Revealed**: The toggle button allows users to switch between masked (dots) and plain text.
- **New Password**: Use the `isNewPassword` prop to display a password-rule hint (e.g., "minimum 8 characters").
- **Error**: Displays a custom error message and sets the field to an invalid state. Use the `error` prop.

### Standard States
- **Disabled**: The field and toggle are non-interactive. Use `isDisabled`.
- **Read-only**: The text can be revealed and copied but not edited. Use `isReadOnly`.

## Accessibility

- The underlying input uses `type="password"` for native browser masking and secure entry announcement.
- The reveal toggle is a native button with appropriate labeling.
- **ARIA association**: The `error` message and `isNewPassword` hint are automatically linked to the input via `aria-describedby`.
- **Autocomplete**: Supports standard `autoComplete` values like `"current-password"` or `"new-password"` to assist password managers.
- Full keyboard support for focusing the field and interacting with the reveal toggle.
