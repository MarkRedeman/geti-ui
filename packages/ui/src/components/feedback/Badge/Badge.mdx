# Badge

A compact, colour-coded label used to communicate status, category, or count at a glance.

```tsx
import { Badge } from '@geti/ui';
```

- [React Spectrum Badge](https://react-spectrum.adobe.com/react-spectrum/Badge.html)

<Stories />

## Responsibility

`Badge` registers Adobe React Spectrum's `Badge` in Geti's public API surface, making it available as a first-class design system primitive without modifying its behaviour or adding new props.

Its purpose is to draw immediate, scannable attention to a small piece of metadata â€” a state (`"Active"`, `"Failed"`), a classification (`"Beta"`, `"New"`), or a count â€” directly adjacent to the element it annotates.

Key characteristics:
- **Categorical signalling**: The `variant` prop encodes semantic meaning through colour, using Spectrum's full palette (`positive`, `negative`, `info`, `neutral`, and eight accent colours).
- **Icon support**: Accepts an icon + label pair or an icon alone (with `aria-label` required in the icon-only case).
- **Passive display**: `Badge` is a non-interactive display primitive. It does not receive focus, handle press events, or accept `onPress`.
- **Theme aware**: Visual tokens are resolved by `ThemeProvider`; no component-level CSS overrides are needed.

### When to use
- To communicate the lifecycle state of an entity (e.g. `"Training"`, `"Deployed"`, `"Archived"`).
- To label a feature or item as `"New"` or `"Beta"` when that metadata is genuinely relevant to the user's decision.
- To show small counts or totals when space is constrained and a full number label would be too heavy.
- Inside list rows, card headers, or navigation items where a quick visual scan matters.

### When to avoid
- When the state is operational and changes in real time â€” use `StatusLight` instead, which provides a pulsing dot and a continuous presence affordance.
- When the message is a contextual alert (info, warning, error) that needs a title and body â€” use `InlineAlert`.
- When the content is interactive (e.g. removable tags) â€” use `Tag` from the `data/` category, which supports `onRemove`.
- Do not use multiple `Badge` variants on the same element if each colour carries a different semantic meaning; it creates ambiguity.

## Design

`Badge` is a direct Spectrum pass-through. `BadgeProps extends SpectrumBadgeProps` with zero additions or overrides:

```tsx
import { Badge as SpectrumBadge, SpectrumBadgeProps } from '@adobe/react-spectrum';

export interface BadgeProps extends SpectrumBadgeProps {}

export const Badge = (props: BadgeProps) => <SpectrumBadge {...props} />;
```

No state, no effects, no refs. All layout, colour, and sizing is delegated to Spectrum's internal implementation.

## Integration

```tsx
// Typical usage: status badge on a list item
<Badge variant="positive">Active</Badge>

// With icon and label
<Badge variant="positive">
  <Checkmark />
  <span>Approved</span>
</Badge>

// Icon-only â€” aria-label is mandatory
<Badge variant="info" aria-label="Information">
  <Info />
</Badge>

// Count badge
<Badge variant="neutral">12</Badge>
```

- **Depends on**: `@adobe/react-spectrum` (`Badge`, `SpectrumBadgeProps`)
- **Theme**: Resolved automatically by `ThemeProvider`. No CSS imports required.
- **Related**: `StatusLight` (operational state), `Tag` (interactive/removable), `InlineAlert` (rich contextual message)
- **Placement**: Typically used inside list rows, `Card` headers, navigation labels, and data table cells

All `SpectrumBadgeProps` are accepted as-is. The key props are:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'neutral' \| 'info' \| 'positive' \| 'negative' \| 'indigo' \| 'yellow' \| 'magenta' \| 'fuchsia' \| 'purple' \| 'seafoam'` | `'neutral'` | Determines the badge colour. Semantic variants (`positive`, `negative`, `info`) carry meaning; accent variants are for categorical differentiation only. |
| `children` | `ReactNode` | â€” | The badge content. Can be a string, a number, or an `<icon> + <span>` pair. |
| `aria-label` | `string` | â€” | **Required** when `children` contains only an icon and no visible text. |

## Accessibility

- **Role**: Spectrum renders `Badge` as a `<span>` with no interactive role. Screen readers announce the text content inline with surrounding content.
- **Icon-only badges**: When `children` contains only an icon, provide `aria-label` to give the badge a meaningful name. Without it, screen readers will announce nothing or read the SVG title, which is unreliable.
- **Colour alone**: `Badge` uses both colour and text to convey meaning. Never rely on colour as the sole differentiator â€” ensure the text label is also descriptive (e.g. `"Error"` not just a red dot).
- **No keyboard interaction**: `Badge` is not focusable and does not respond to keyboard events. If you need a clickable label, use a `Button` or `Tag`.
- **Context**: Because `Badge` is an inline `<span>`, it is announced as part of the surrounding sentence. Position it close to the element it describes so the association is clear to screen reader users.

## Related components
- **`StatusLight`** â€” use for live, changeable operational states (e.g. "Active", "Offline") that persist over time; `Badge` is for static categorical metadata.
- **`Tag`** â€” use when the label needs to be removable or interactive; `Badge` is a non-interactive display primitive.
- **`InlineAlert`** â€” use when the annotation requires a title, body text, and/or an action button rather than a compact inline label.
- **`StatusLight`** (accent variants) â€” use for colour-coded category legends in data visualisation rather than `Badge` when a dot indicator is more appropriate than a pill.
