# Tag

`Tag` displays a small categorisation label with an optional accent dot, prefix element, or suffix element. Use it to show metadata inline тАФ annotation labels, model types, status categories тАФ without needing a full Spectrum `TagGroup`.

```tsx
import { Tag } from '@geti-ai/ui';
```

- [Geti UI Theming](/components/theming)

## Examples

### Default (with accent dot)

```tsx preview
import { Tag } from '@geti-ai/ui';

export default function App() {
  return <Tag text="Detection" />;
}
```

### No dot

```tsx preview
import { Tag } from '@geti-ai/ui';

export default function App() {
  return <Tag text="Classification" withDot={false} />;
}
```

### Dark mode

```tsx preview
import { Tag } from '@geti-ai/ui';

export default function App() {
  return <Tag text="Segmentation" darkMode />;
}
```

### With tooltip

```tsx preview

```

## Responsibility

`Tag` is a Geti-authored component with no direct Spectrum equivalent. It renders a small pill element using Spectrum design tokens for sizing, colour, and spacing. The default accent dot uses the Geti `--energy-blue` brand token.

For a collection of removable or selectable tags rendered from data, use `TagGroup` instead.

## Variants and states

| Feature | Default | Notes |
|---|---|---|
| Accent dot | `true` | Disabled by setting `withDot={false}` |
| Background | Light grey | `darkMode` switches to a darker grey |
| Prefix | none | Overrides the dot with a custom element |
| Suffix | none | Appended after the text |

## Props (deep dive)

### Content

- `text`: the label string displayed inside the tag.
- `prefix`: custom element rendered before the text; replaces the accent dot when provided.
- `suffix`: custom element rendered after the text.
- `withDot`: whether to show the accent dot. Defaults to `true`; set `false` when using a custom `prefix`.

### Appearance

- `darkMode`: uses a darker background token, useful inside already-dark panels.
- `className` / `style`: additional class or inline CSS for the container.

### Interaction / accessibility

- `tooltip`: native `title` attribute тАФ shown on hover.
- `aria-label`: defaults to the `text` value; override for non-obvious labels.
- `id`: also used as `data-testid` for automated tests.

## Related components

- `TagGroup` тАФ use for a managed, removable collection of tags rendered from a data array.
- `Badge` тАФ use for short colour-coded status labels that do not need a dot prefix.
- `StatusLight` тАФ use when the coloured indicator is the primary visual signal, not the text.
