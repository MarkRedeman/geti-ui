# Button

Buttons trigger user actions such as save, submit, or delete. Use Button for clear, high-signal actions in forms, dialogs, and page-level workflows.

```tsx
import { Button } from '@geti-ui/ui';
```

- [React Spectrum Button ↗](https://react-spectrum.adobe.com/react-spectrum/Button.html)
- [Geti UI Theming](/components/theming)

## Examples

### Accent (default)

```tsx preview
import { Button } from '@geti-ui/ui';

export default function App() {
  return <Button variant="accent">Save</Button>;
}
```

### Primary

```tsx preview
import { Button } from '@geti-ui/ui';

export default function App() {
  return <Button variant="primary">Edit</Button>;
}
```

### Secondary (outline)

```tsx preview
import { Button } from '@geti-ui/ui';

export default function App() {
  return <Button variant="secondary" style="outline">Cancel</Button>;
}
```

### Negative

```tsx preview
import { Button } from '@geti-ui/ui';

export default function App() {
  return <Button variant="negative">Delete</Button>;
}
```

### Pending

```tsx preview
import { Button } from '@geti-ui/ui';

export default function App() {
  return <Button isPending>Saving...</Button>;
}
```

### Disabled

```tsx preview
import { Button } from '@geti-ui/ui';

export default function App() {
  return <Button isDisabled>Disabled</Button>;
}
```


## Responsibility

`Button` is a Geti wrapper around React Spectrum `Button` with Geti-safe defaults and variant constraints.

What this wrapper does for consumers:

- defaults to `variant="accent"` for primary actions,
- excludes legacy variants that are not part of Geti design language,
- keeps press behavior accessible via Spectrum's press interactions,
- supports Geti theme tokens through `ThemeProvider`.

## Variants and states

| Variant | Typical use |
|---|---|
| `accent` (default) | Primary CTA such as save/submit/create |
| `primary` | High-importance secondary action |
| `secondary` | Neutral actions such as cancel/back |
| `negative` | Destructive actions such as delete/remove |

---

| State | Prop | Notes |
|---|---|---|
| Loading | `isPending` | Shows spinner and blocks interaction |
| Disabled | `isDisabled` | Non-interactive and visually muted |

## Props (deep dive)

### Content

- `children`: button label/content.
- `elementType`: render as custom element (e.g. anchor).
- `href`: navigation target (renders as an anchor by default).

### Value / visual

- `variant`: `accent | primary | secondary | negative`.
- `style`: `fill | outline`.
- `isDisabled`: disable interaction.
- `isPending`: loading state with spinner.

### Events

- `onPress`: preferred action handler (mouse/touch/keyboard).
- `onPressStart` / `onPressEnd`: advanced interaction timing.

### Accessibility

- use clear, specific button text (`Save changes`, not `OK`),
- use `isDisabled` (not custom CSS) to correctly expose disabled semantics.

## Related components

- [`ActionButton`](/components/ui/ActionButton): use for compact icon-centric actions (toolbars, inline actions).
- [`ToggleButton`](/components/ui/ToggleButton): use when action has persistent selected/unselected state.
- [`ToggleButtons`](/components/ui/ToggleButtons): use for segmented mutually-exclusive selections.
- [`PressableElement`](/components/ui/PressableElement): use press semantics on non-button container UIs.
