# PasswordField

A secure text input field with a built-in reveal toggle for passwords and sensitive secrets.

```tsx
import { PasswordField } from '@geti/ui';
```

- [React Spectrum V3 TextField (type="password")](https://react-spectrum.adobe.com/react-spectrum/TextField.html)

<Stories />

## Responsibility

`PasswordField` is a Geti-specific wrapper around Spectrum's `TextField` with `type="password"` and an integrated show/hide toggle button. It masks the value by default and lets users reveal their input to catch typos. An `isNewPassword` prop displays a password-rule hint below the field, and an `error` prop drives the invalid state with a custom message.

### When to use
- Login forms, registration flows, and account settings.
- When collecting any sensitive information that should be masked: passwords, PINs, API keys, secret tokens.
- When a reveal toggle reduces user errors on password entry.

### When to avoid
- Use [`TextField`](/docs/components-form-textfield--docs) for non-sensitive text input.
- Use [`NumberField`](/docs/components-form-numberfield--docs) for numeric values that are not sensitive.

## States

- **Revealed**: The toggle button switches between masked (dots) and visible plain text.
- **New Password** (`isNewPassword`): Displays a password-rule hint below the field.
- **Error** (`error`): Renders a custom error message and marks the field as invalid.
- **Disabled** (`isDisabled`): The field and toggle are non-interactive.
- **Read-only** (`isReadOnly`): The value can be revealed and copied but not edited.

## Props

### Custom Props
| Prop | Type | Default | Description |
|---|---|---|---|
| `isNewPassword` | `boolean` | `false` | Shows a password-rule hint below the field. |
| `error` | `string` | — | Displays a specific error message and marks the field as invalid. |

### Standard TextField Props
| Prop | Type | Default | Description |
|---|---|---|---|
| `label` | `ReactNode` | — | The label for the password field. |
| `value` | `string` | — | The current value (controlled). |
| `defaultValue` | `string` | — | The initial value (uncontrolled). |
| `placeholder` | `string` | — | Hint text when the field is empty. |
| `isDisabled` | `boolean` | `false` | Whether the field is non-interactive. |
| `isReadOnly` | `boolean` | `false` | Whether the field is read-only. |
| `onChange` | `(value: string) => void` | — | Called when the password value changes. |

## Related components
- `TextField` — use instead for general non-sensitive single-line text inputs.
- `Form` — wrap `PasswordField` in `Form` for aligned label layout and submit handling.
