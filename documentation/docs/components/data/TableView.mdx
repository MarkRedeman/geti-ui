# TableView

`TableView` renders a data table with sortable columns, row selection, and virtualized scrolling. Use it when data is naturally tabular â€” rows and named columns â€” and you need sorting or multi-select.

```tsx
import { TableView, TableHeader, TableBody, Column, Row, Cell } from '@geti-ai/ui';
```

## Examples

### Basic sortable table

```tsx preview
import { TableView, TableHeader, TableBody, Column, Row, Cell } from '@geti-ai/ui';

const columns = [
  { name: 'Name', uid: 'name' },
  { name: 'Type', uid: 'type' },
  { name: 'Date', uid: 'date' },
];
const rows = [
  { id: 1, name: 'project-alpha', type: 'Detection', date: '2024-01-15' },
  { id: 2, name: 'project-beta', type: 'Classification', date: '2024-02-20' },
  { id: 3, name: 'project-gamma', type: 'Segmentation', date: '2024-03-05' },
];

export default function App() {
  return (
    <TableView aria-label="Projects" height="size-3000" width="size-6000">
      <TableHeader columns={columns}>
        {(col) => <Column key={col.uid} allowsSorting>{col.name}</Column>}
      </TableHeader>
      <TableBody items={rows}>
        {(item) => <Row>{(key) => <Cell>{item[key]}</Cell>}</Row>}
      </TableBody>
    </TableView>
  );
}
```

## Responsibility

`TableView` and its companion pieces (`TableHeader`, `TableBody`, `Column`, `Row`, `Cell`) are direct re-exports of React Spectrum's table components. The wrapper keeps imports on a single package path. All sorting, selection, and virtualization come from Spectrum.

## Variants and states

| `density` | Row height |
|---|---|
| `compact` | Tight row height for data-heavy views |
| `regular` (default) | Standard height |
| `spacious` | More breathing room per row |

| `selectionMode` | Behaviour |
|---|---|
| `none` | No row selection |
| `single` | One row at a time |
| `multiple` | Checkbox column with multi-select |

## Props (deep dive)

### Content

- `children`: `TableHeader` + `TableBody` pair.
- `Column`: accepts `allowsSorting`, `isRowHeader`, `width`, `minWidth`, `maxWidth`.
- `Row`: renders each data row; the child function receives the column key.
- `Cell`: leaf content of each table cell.

### Value / state

- `selectionMode`: row selection behaviour.
- `selectedKeys` / `defaultSelectedKeys`: controlled or uncontrolled selection.
- `sortDescriptor` / `onSortChange`: controlled sort state and change handler.
- `density`: row height variant.

### Events

- `onSelectionChange(keys)`: fires when row selection changes.
- `onSortChange(descriptor)`: fires when a sortable column header is clicked.
- `onAction(key)`: fires when a row is activated (Enter or double-click).

### Accessibility

- Provide `aria-label` on `TableView`.
- Mark at least one `Column` with `isRowHeader` so screen readers announce it as the row identifier.

## Related components

- `ActionBar` â€” combine with `selectionMode="multiple"` for bulk row operations.
- `ListView` â€” use for non-tabular lists that don't need column headers.
- `VirtualizedListLayout` â€” use when you need a fully custom virtualized list outside Spectrum's table constraints.
