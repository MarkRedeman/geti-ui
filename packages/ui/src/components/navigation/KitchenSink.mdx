The `navigation/` category owns **components that move the user through the application** — switching views, traversing page hierarchies, opening contextual action sets, and changing display modes. Every component here either changes what the user is looking at or presents a set of choices about where to go next.

Components in this folder:

| Component | Purpose |
|---|---|
| `Tabs` / `TabList` / `TabPanels` / `Item` | Sectioned view switcher with accessible tab controls |
| `Menu` / `MenuTrigger` / `Item` / `Section` | Floating action/option menu triggered by a button |
| `ActionMenu` | Pre-composed `ActionButton` + `Menu` in one component |
| `Breadcrumbs` / `Item` | Hierarchical location trail with ancestor links |
| `Link` | Accessible inline hyperlink |
| `MediaViewModes` | Icon button + menu to switch between list/grid view modes |

---

## Design

### Thin Spectrum wrappers + child-primitive re-exports

All navigation components follow the same thin-wrapper pattern seen in the rest of the library:

```tsx
export const Menu = <T extends object>(props: MenuProps<T>) => <SpectrumMenu {...props} />;
```

Compound components (those that require multiple cooperating pieces) re-export their children from the same module rather than requiring consumers to import from `@adobe/react-spectrum`:

```tsx
// Menu/Item.tsx
export { Item } from '@adobe/react-spectrum';

// Menu/Section.tsx
export { Section } from '@adobe/react-spectrum';
```

This pattern is repeated for `Tabs` (`TabList`, `TabPanels`, `Item`), `Breadcrumbs` (`Item`), and `Menu` (`MenuTrigger`, `Item`, `Section`). The consumer only needs to import from `@geti/ui`.

### Tabs decomposition

`Tabs` is decomposed into four co-located files:
- `Tabs.tsx` — the outer wrapper
- `TabList.tsx` — the row of tab triggers
- `TabPanels.tsx` — the panel container
- `Item.tsx` — re-export of Spectrum `Item` for individual tab definitions

### ActionMenu — convenience composite

`ActionMenu` wraps Spectrum's `SpectrumActionMenu` which itself combines `ActionButton` + `Menu` trigger logic. It requires no configuration beyond providing `Item` children.

### MediaViewModes — Geti-native composite

`MediaViewModes` is a bespoke Geti component with no direct Spectrum equivalent. It:
1. Renders the icon for the **current** view mode as the `ActionButton` label.
2. Opens a `Menu` (from `@adobe/react-spectrum` directly) with all available modes.
3. Handles `onAction` by matching the selection key to a `ViewModes` enum value and calling `setViewMode`.
4. Wraps the `ActionButton` in a `TooltipTrigger`/`Tooltip` for discoverability.

Icons come from `@spectrum-icons/workflow` (List, ViewGrid, ModernGridView, ClassicGridView).

---

## Flow

- **Tabs**: controlled by Spectrum's `selectedKey` prop; `onSelectionChange` propagates the selected tab key to the parent.
- **Menu / ActionMenu / MediaViewModes**: open state is managed internally by `MenuTrigger`; `onAction` or `onSelectionChange` propagates user choices up.
- **Breadcrumbs**: stateless — renders the provided `Item` hierarchy; navigation is delegated to anchor `href` or `onPress` on each item.
- **Link**: purely presentational; navigation is the anchor's `href`.

---

## Integration

- **Depends on**: `@adobe/react-spectrum` (all components), `@spectrum-icons/workflow` (icons in `MediaViewModes`).
- **Does NOT depend on** any other Geti component category (navigation is a foundational category).
- **Consumed by**: application shell (tabs for workspace switching), toolbar areas (ActionMenu, MediaViewModes), page headers (Breadcrumbs), and inline content (Link).
- **Theme**: inherits Geti dark tokens via `ThemeProvider`; `--spectrum-tabs-*` overrides in `geti-dark.module.css` ensure the tab selection indicator uses the Geti blue.
