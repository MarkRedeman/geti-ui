# DatePicker

`DatePicker` combines a segmented `DateField` with a Calendar popover, letting users enter or pick a date (and optionally time). Use it in forms that require a precise date value.

```tsx
import { DatePicker } from '@geti/ui';
```

## Responsibility

`DatePicker` wraps React Spectrum's `DatePicker` and applies `width="size-2400"` explicitly to fix a Safari mobile layout-flicker bug. All other props pass through unchanged.

## Variants and states

| State | Prop | Notes |
|---|---|---|
| Date only | `granularity="day"` (default) | Shows day/month/year segments |
| Date + time | `granularity="minute"` or `"second"` | Adds time segments |
| Disabled | `isDisabled` | Non-interactive |
| Read-only | `isReadOnly` | Shows value, no editing |
| Required | `isRequired` | Adds required indicator |
| Invalid | `validationState="invalid"` + `errorMessage` | Error styling |

## Props (deep dive)

### Content

- `label`: visible field label.
- `description`: helper text below the field.
- `errorMessage`: shown when `validationState="invalid"`.

### Value / state

- `value` / `defaultValue`: controlled/uncontrolled date value (accepts `CalendarDate`, `CalendarDateTime`, `ZonedDateTime`).
- `minValue` / `maxValue`: constrains the selectable range.
- `granularity`: controls the smallest time unit shown (`"day"`, `"hour"`, `"minute"`, `"second"`).
- `isDisabled`, `isReadOnly`, `isRequired`.

### Events

- `onChange(value)`: fires when the user commits a new date.

### Accessibility

- Always provide a `label` ‚Äî the field cannot be identified without it.
- The calendar popover is keyboard-navigable (arrow keys move days, Page Up/Down move months).

## Related components

- `DateField` ‚Äî use when you want the segmented input only, without the calendar popover.
- `DateRangePicker` ‚Äî use when a start and end date are both required.
- `Calendar` ‚Äî use when you need the calendar grid standalone (without the input).
