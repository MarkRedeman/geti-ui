# Picker

<Subtitle>A select-only dropdown that lets users choose a single value from a predefined, fixed list of options.</Subtitle>

```tsx
import { Picker, Item } from '@geti/ui';
```

- [React Spectrum Picker](https://react-spectrum.adobe.com/react-spectrum/Picker.html)

## Responsibility

`Picker` is a pure pass-through wrapper around Adobe React Spectrum's `Picker`, co-exporting `Item` from the same import. It is the correct choice when the set of valid options is known at design time and the user may not type arbitrary input ŌĆö the dropdown enforces a closed selection.

Key responsibilities:
- **Closed Selection**: Users can only choose from the options provided as `<Item>` children or via the `items` prop. There is no free-text fallback.
- **Typed Generic**: The `<T extends object>` generic allows the item type to be inferred from `items`, giving full TypeScript type safety on `selectedKey`, `defaultSelectedKey`, and `onSelectionChange`.
- **Full Spectrum API**: No props are removed or defaulted. Every Spectrum `Picker` prop ŌĆö `label`, `placeholder`, `isRequired`, `isDisabled`, `validationState`, `width`, `items`, `disabledKeys`, `sections`, and more ŌĆö is available.
- **Static and Dynamic Options**: Supports both JSX children (`<Item key="x">Label</Item>`) and dynamic `items` prop with a render function for data-driven lists.

### When to use
- When the list of options is fixed and exhaustive ŌĆö model types, export formats, locale settings, sort orders.
- When the user should not be able to enter a value outside the defined set.
- When there are more than ~4 options (use `RadioGroup` for 4 or fewer options in a form).
- When the options list is too long to display inline but does not require search/filter ŌĆö up to ~20 items is a comfortable Picker range.

### When to avoid
- When users need to type a value or filter a long list ŌĆö use `ComboBox` instead.
- When the list has fewer than 3 options and all should be visible at once ŌĆö use `RadioGroup`.
- When multiple items can be selected simultaneously ŌĆö Spectrum does not support multi-select in `Picker`; use a `ListBox` with `selectionMode="multiple"`.
- When the options are user-generated or unbounded ŌĆö `ComboBox` is the appropriate control.

## Design

Generic thin wrapper with a co-exported `Item`:

```tsx
export interface PickerProps<T extends object> extends SpectrumPickerProps<T> {}

export const Picker = <T extends object>(props: PickerProps<T>) => (
  <SpectrumPicker {...props} />
);

export { Item } from '@adobe/react-spectrum';
```

No CSS overrides, no local state. Always import `Item` from `@geti/ui` alongside `Picker` ŌĆö do not import it directly from `@adobe/react-spectrum` in application code, to keep the dependency abstraction consistent.

## Flow

```tsx
// Static children (most common)
<Picker label="Export format" onSelectionChange={(key) => setFormat(key)}>
  <Item key="png">PNG</Item>
  <Item key="jpeg">JPEG</Item>
  <Item key="webp">WebP</Item>
</Picker>

// Controlled selection
const [sortKey, setSortKey] = useState<Key>('date-desc');
<Picker
  label="Sort by"
  selectedKey={sortKey}
  onSelectionChange={(key) => setSortKey(key as string)}
>
  <Item key="date-desc">Newest first</Item>
  <Item key="date-asc">Oldest first</Item>
  <Item key="name-asc">Name AŌĆōZ</Item>
</Picker>

// Dynamic items from data
<Picker
  label="Model type"
  items={modelTypes}
  onSelectionChange={handleModelChange}
>
  {(model) => <Item key={model.id}>{model.name}</Item>}
</Picker>

// With placeholder and validation
<Picker
  label="Region"
  placeholder="Select a regionŌĆ”"
  isRequired
  validationState={region ? 'valid' : 'invalid'}
  errorMessage="Please select a region."
>
  <Item key="us-east">US East</Item>
  <Item key="eu-west">EU West</Item>
  <Item key="ap-south">AP South</Item>
</Picker>
```

## Integration

- **Depends on**: `@adobe/react-spectrum` ŌĆö `Picker`, `SpectrumPickerProps`, `Item`
- **Always pair with**: `Item` ŌĆö co-exported from `@geti/ui` to define individual options
- **Contrast**: `ComboBox` (free-form text + filtered suggestions) vs `Picker` (strictly closed option set)
- **Sections**: For grouped options, use `Section` from `@adobe/react-spectrum` as a child of `Picker` to add visual groupings with headings

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `label` | `string` | ŌĆö | The visible label for the picker. |
| `placeholder` | `string` | ŌĆö | Placeholder text shown when no option is selected. |
| `selectedKey` | `Key` | ŌĆö | The key of the currently selected item (controlled). |
| `defaultSelectedKey` | `Key` | ŌĆö | The initially selected key (uncontrolled). |
| `onSelectionChange` | `(key: Key) => void` | ŌĆö | Called when the selection changes. |
| `items` | `Iterable<T>` | ŌĆö | Data source for dynamic item lists. Use with a render function as `children`. |
| `disabledKeys` | `Iterable<Key>` | ŌĆö | Keys of items that should be disabled. |
| `isDisabled` | `boolean` | `false` | Disables the entire picker. |
| `isRequired` | `boolean` | `false` | Marks the field as required in a form. |
| `validationState` | `'valid' \| 'invalid'` | ŌĆö | Controls inline validation styling. |
| `errorMessage` | `ReactNode` | ŌĆö | Error message shown when `validationState="invalid"`. |
| `width` | `DimensionValue` | ŌĆö | The width of the trigger button. |

## Accessibility

- **ARIA pattern**: Spectrum implements `Picker` using the **ARIA Listbox** pattern. The trigger button has `role="button"` with `aria-haspopup="listbox"` and `aria-expanded` reflecting open state. The dropdown renders as `role="listbox"` with each option as `role="option"`.
- **Labels**: The `label` prop renders a visible `<label>` element associated with the trigger via `for`/`id`. If `label` is omitted, provide `aria-label` on the `Picker` to maintain an accessible name.
- **Keyboard ŌĆö trigger**: **Space**, **Enter**, or **Arrow Down** opens the listbox. **Alt + Arrow Down** opens without moving selection.
- **Keyboard ŌĆö listbox**: **Arrow Up/Down** moves focus between options. **Enter** or **Space** selects the focused option and closes the listbox. **Escape** or **Tab** closes the listbox without changing selection.
- **Type-ahead**: Pressing alphanumeric keys while the listbox is open moves focus to the first option starting with that character.
- **Disabled options**: Items listed in `disabledKeys` have `aria-disabled="true"` and are skipped during keyboard navigation.
- **Required and validation**: `isRequired` sets `aria-required="true"`. `validationState="invalid"` sets `aria-invalid="true"` and `errorMessage` is linked via `aria-describedby`.
