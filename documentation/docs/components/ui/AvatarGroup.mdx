# AvatarGroup

AvatarGroup renders a horizontal row of Avatar components. When the list exceeds the configured maximum, it shows a `+N` overflow badge instead of clipping. Use it for compact team member lists, annotation author stacks, and collaboration indicators.

```tsx
import { AvatarGroup } from '@geti-ai/ui';
```

## Examples

### Default (max 3, showing overflow)

```tsx preview
import { AvatarGroup } from '@geti-ai/ui';

export default function App() {
  return (
    <AvatarGroup
      avatars={[
        { src: 'https://i.pravatar.cc/150?img=1', alt: 'User 1' },
        { src: 'https://i.pravatar.cc/150?img=2', alt: 'User 2' },
        { src: 'https://i.pravatar.cc/150?img=3', alt: 'User 3' },
        { src: 'https://i.pravatar.cc/150?img=4', alt: 'User 4' },
        { src: 'https://i.pravatar.cc/150?img=5', alt: 'User 5' },
      ]}
      max={3}
    />
  );
}
```

## Responsibility

`AvatarGroup` is a Geti-native component (not a Spectrum wrapper). It:

- renders up to `max` avatars with a negative left margin to produce overlapping stacks,
- shows a `+N` badge when the total count exceeds `max`,
- applies a uniform `size` to all avatars in the group when provided,
- exposes a `role="group"` container with an `aria-label` of `"N avatars"` for screen readers.

## Variants and states

`AvatarGroup` has no visual variants. Control visual weight through `size` and `max`.

| Prop | Default | Effect |
|---|---|---|
| `max` | `3` | Maximum visible avatars before overflow badge |
| `size` | inherited per avatar | Uniform size token applied to all avatars |

## Props (deep dive)

### Content

- `avatars`: array of `AvatarProps` objects. Each item is passed directly to an `Avatar`. Ensure each entry has a unique `alt` for accessible rendering.

### Visual

- `max`: number â€” avatars beyond this count are hidden and replaced by `+N`.
- `size`: size token string â€” overrides individual avatar sizes for uniformity.

### Accessibility

- the wrapper element has `role="group"` and `aria-label` reflecting total count,
- the overflow badge has `aria-label` of `"N more"`,
- each individual `Avatar` requires a meaningful `alt` in the `avatars` array.
