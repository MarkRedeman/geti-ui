## Responsibility

`TreeView` is used to present **nested hierarchical data** where parent-child relationships are meaningful and users need to expand and collapse branches to explore depth—for example, file directories, organizational hierarchies, or nested project structures.

### When to use
- **Hierarchical Data**: When data has 2+ levels of true parent/child hierarchy.
- **Expand/Collapse**: When users need to explore different depths of a tree at their own pace.
- **Distinct Roles**: When parent nodes and leaf nodes represent different types of entities (e.g., folders vs. files).

### Comparison with other components
| Situation | Use instead |
| :--- | :--- |
| Flat list of items | `ListView` |
| Application-level navigation | `VerticalNav` |
| Nested navigation that acts like a menu | `Menu` with submenus |

### Usage Rules
- **Depth**: Limit nesting to 3–4 levels; deeper structures can be cognitively overwhelming.
- **Accessible Labels**: Always provide `aria-label` or `aria-labelledby` on the `<TreeView>` root.
- **Text Values**: Set `textValue` on every `<Item>` (or `<TreeViewItem>`) to a plain-text string for screen readers and type-ahead.
- **Selection vs. Action**: Use `onAction` for opening/navigating items, and `selectionMode` for batch operations (e.g., selecting items for deletion).

## Variants

### Selection Modes
Supports `none` (default), `single`, and `multiple` selection modes.

### Density
`TreeView` supports density variants to control the vertical space between items, similar to `ListView`.

## Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `children` | `ItemElement \| ItemRenderer` | — | The items or branches in the tree. |
| `selectionMode` | `'none' \| 'single' \| 'multiple'` | `'none'` | The selection mode for the tree. |
| `onAction` | `(key: Key) => void` | — | Handler called when an item is activated. |
| `expandedKeys` | `Iterable<Key>` | — | The set of keys for expanded branches. |
| `onExpandedChange` | `(keys: Set<Key>) => void` | — | Handler called when the expansion state changes. |

## Accessibility

### Roles and Navigation
- `TreeView` follows the [W3C ARIA Tree View pattern](https://www.w3.org/WAI/ARIA/apg/patterns/treeview/).
- **Keyboard Navigation**:
    - `Up/Down Arrows`: Move focus between visible items.
    - `Right Arrow`: Expand a parent node or move focus to its first child.
    - `Left Arrow`: Collapse an open parent node or move focus to its parent.
    - `Home/End`: Jump to the first or last visible item.
- **Required Attributes**: Parent nodes are marked with `aria-expanded` while leaf nodes are not.

### Labelling Requirement
An `aria-label` or `aria-labelledby` is **mandatory** on the `<TreeView>` root, as it is a landmark that provides context for the entire collection.
