# Loading

The canonical, multi-mode loading indicator for Geti — combines a Spectrum progress spinner and the Intel-branded animation into a single, placement-aware component.

```tsx
import { Loading } from '@geti/ui';
```

- [React Spectrum ProgressCircle](https://react-spectrum.adobe.com/react-spectrum/ProgressCircle.html)

<Stories />

## Responsibility

`Loading` is the **single source of truth** for "something is loading" communication across the Geti application. It unifies two visual styles (`spinner`, `intel`) and three layout modes (`inline`, `overlay`, `fullscreen`) into one composable API, eliminating the need for ad-hoc CSS positioning on every loading state.

Key responsibilities:
- **Mode-based positioning**: Computes a `CSSProperties` object for each mode, removing the need for consumers to add wrapper divs with `position: fixed` or `position: absolute`.
- **Variant switching**: Renders either a Spectrum `ProgressCircle` or an Intel-branded WebP animation, keeping the outer wrapper structure identical for both.
- **Size normalization**: The `size` prop maps to Spectrum's `S | M | L` tokens for the spinner, and to explicit pixel sizes (`24 | 48 | 192 px`) for the Intel animation via `INTEL_SIZE_MAP`.
- **Scrim support**: For `overlay` and `fullscreen` modes, a `backgroundColor` can be passed via the `style` prop to create a dimming scrim without a separate overlay element.

### When to use
- As the primary loading indicator during data fetching, model inference, or navigation transitions.
- Use `mode="overlay"` when loading content within a bounded container (e.g., a card, panel, or modal) that should be blocked while loading.
- Use `mode="fullscreen"` for application-level loading states (e.g., initial app bootstrap, full-page transitions).
- Use `mode="inline"` when the spinner should participate in document flow (e.g., inside a button group, toolbar, or list item).
- Use `variant="intel"` for prominent, brand-forward splash screens on the Intel platform.

### When to avoid
- For skeleton-style content placeholders — use `Skeleton`, which preserves layout and communicates content shape.
- For progress with a known completion percentage — use `ProgressBar` or `ProgressCircle` directly.
- For tiny inline status icons — use `StatusLight`.
- Do not use `IntelBrandedLoading` in new code; use `<Loading variant="intel" mode="inline" size="L" />` instead.

## Variants

### Mode

| Mode | Positioning | Typical use |
|------|-------------|-------------|
| `inline` | `display: flex`, no positioning | Participates in document flow; inside a container |
| `overlay` | `position: absolute; inset: 0` | Blocks a `position: relative` parent container |
| `fullscreen` | `position: fixed; inset: 0` | Full-application blocking overlay |

### Variant

| Variant | Inner content | Size mapping |
|---------|---------------|--------------|
| `spinner` | Spectrum `ProgressCircle isIndeterminate` | Spectrum S / M / L tokens |
| `intel` | `<img>` from Intel loading WebP asset | `S=24px`, `M=48px`, `L=192px` |

## Design

Two orthogonal axes — `mode` and `variant` — are resolved independently at render time. The outer wrapper always receives the mode style; the inner content switches on variant. This means all combinations are structurally equivalent, simplifying testing and inspection.

```tsx
// Inline spinner — participates in document flow
<Loading mode="inline" size="M" />

// Absolute overlay with a semi-transparent scrim
<Loading
  mode="overlay"
  style={{ backgroundColor: 'rgba(0, 0, 0, 0.4)' }}
/>

// Fixed fullscreen spinner (default mode)
<Loading />

// Intel-branded large animation
<Loading variant="intel" mode="inline" size="L" />
```

## Integration

- **`VirtualizedListLayout`** uses `Loading` as the default `renderLoading` sentinel when `isLoading` is true.
- **`IntelBrandedLoading`** is a deprecated convenience wrapper that delegates to `<Loading variant="intel" mode="inline" size="L">`. New code should use `Loading` directly.
- **`ProgressCircle`** is a peer component in the `feedback/` category. `Loading` depends on it for the spinner variant — do not replace `ProgressCircle` with a raw Spectrum import inside `Loading`.
- The `IntelLoadingWebp` asset is a static import handled by the build pipeline. No special loader configuration is required.
- `LoadingProps` extends `Omit<SpectrumProgressCircleProps, 'variant'>`. All Spectrum `ProgressCircle` props (e.g., `aria-label`) are forwarded in `spinner` mode; they are silently ignored in `intel` mode.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `mode` | `'inline' \| 'overlay' \| 'fullscreen'` | `'fullscreen'` | Controls the CSS positioning strategy of the outer wrapper. |
| `variant` | `'spinner' \| 'intel'` | `'spinner'` | Selects the visual loading indicator. |
| `size` | `'S' \| 'M' \| 'L'` | `'L'` | Size of the indicator. Pixel values differ between variants (see table above). |
| `style` | `CSSProperties` | `{}` | Additional inline styles merged into the outer wrapper (e.g., `backgroundColor` for a scrim). |
| `className` | `string` | — | Additional CSS class names on the outer wrapper. |

All other props from `SpectrumProgressCircleProps` are forwarded to `ProgressCircle` in `spinner` mode.

## Accessibility

- **Spinner variant**: `ProgressCircle` receives `aria-label="Loading..."` and `isIndeterminate`, which maps to `role="progressbar"` with `aria-valuenow` omitted (correct for indeterminate progress per WAI-ARIA).
- **Intel variant**: The `<img>` element carries `role="progressbar"` and `alt="Loading"`, providing equivalent semantics without a Spectrum component.
- **Overlay/fullscreen modes**: When using these modes as blocking overlays, consider setting `aria-busy="true"` on the region being loaded and using `aria-live="polite"` on a status region to announce when loading completes.
- **Focus trapping**: `Loading` does not trap focus. For modal loading states where user interaction must be blocked, wrap the content in a `Dialog` or use the application-level state management to prevent tab navigation.
- **Keyboard**: The loading indicator itself is not interactive and receives no focus.

## Related components
- **`Skeleton`** — use when the content shape is known and layout preservation reduces perceived load time; `Skeleton` complements `Loading` for structured placeholder UIs.
- **`ProgressBar`** — use when the loading operation has a measurable completion percentage that should be communicated to the user.
- **`ProgressCircle`** — use as the raw spinner primitive in tight layout slots (e.g. inside a button, table cell) without `Loading`'s positioning wrapper.
- **`IntelBrandedLoading`** — deprecated; use `<Loading variant="intel" mode="inline" size="L" />` for the Intel-branded animation in new code.

## Example use cases
<details>
  <summary>Typical use cases</summary>

- Blocking a project settings panel while configuration data is being fetched — `<Loading mode="overlay" style={{ backgroundColor: 'rgba(0,0,0,0.4)' }} />`.
- Displaying a fullscreen spinner during the initial application bootstrap sequence — `<Loading mode="fullscreen" />`.
- Showing a small inline spinner next to a "Save" button label while the save request is in flight — `<Loading mode="inline" size="S" />`.
- Rendering the Intel-branded animation on a prominent splash screen — `<Loading variant="intel" mode="inline" size="L" />`.
- Overlaying a loading indicator on a `VirtualizedListLayout` canvas while annotation data is paged in.
