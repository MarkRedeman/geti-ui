# Breadcrumbs

Breadcrumbs show the user's current location in a hierarchy and let them navigate back to any ancestor page. Use them at the top of deep content hierarchies such as project → dataset → media views.

```tsx
import { Breadcrumbs, BreadcrumbItem as Item } from '@geti/ui';
```

- [React Spectrum Breadcrumbs ↗](https://react-spectrum.adobe.com/react-spectrum/Breadcrumbs.html)
- [Geti UI Theming](/components/theming)

## Responsibility

`Breadcrumbs` is a thin wrapper around React Spectrum `Breadcrumbs`. The last `Item` (current page) is rendered as non-interactive text; all preceding items are rendered as links. Spectrum handles overflow collapsing automatically when space is constrained.

## Variants and states

| Size | Typical use |
|---|---|
| `S` | Compact headers or dense toolbar areas |
| `M` (default) | Standard page-level navigation |
| `L` | Featured or hero-area navigation |

---

| Prop | Notes |
|---|---|
| `isDisabled` | Disables all link interactions |
| `showRoot` | Always show the first item even when collapsed |

## Props (deep dive)

### Content

- `children`: one or more `Item` elements. Each `Item` takes a `key` and optional `href`.
- The last `Item` (no `href`) represents the current page and is not interactive.

### Visual

- `size`: `S | M | L`.
- `showRoot`: keep the root item visible during overflow collapse.

### Accessibility

- Spectrum renders this as a `<nav>` with `aria-label="Breadcrumbs"` automatically.
- The current page item receives `aria-current="page"`.

## Related components

- `Link` — standalone inline links without hierarchy semantics.
- `Tabs` — use for peer-level section switching rather than hierarchical navigation.
