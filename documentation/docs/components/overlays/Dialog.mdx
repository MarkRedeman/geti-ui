# Dialog

Dialog presents content in a modal overlay that requires the user's attention before returning to the page. Use it for confirmation prompts, input forms, detail views, or any workflow that should visually interrupt the current context.

```tsx
import { Dialog, DialogTrigger } from '@geti/ui';
```

- [React Spectrum Dialog ↗](https://react-spectrum.adobe.com/react-spectrum/Dialog.html)
- [Geti UI Theming](/components/theming)

## Responsibility

`Dialog` and `DialogTrigger` are thin wrappers around React Spectrum equivalents. `DialogTrigger` links open/close state to the trigger's press event. `Dialog` renders the modal shell with Spectrum's focus trap, backdrop, and ARIA role management.

## Variants and states

| Prop | Options | Notes |
|---|---|---|
| `size` on `Dialog` | `S \| M \| L` | Controls the maximum width of the dialog |
| `isDismissable` on `DialogTrigger` | boolean | Allows closing via backdrop click or Escape |
| `type` on `DialogTrigger` | `modal \| fullscreen \| fullscreenTakeover \| tray \| popover` | Defaults to `modal` |

## Props (deep dive)

### Dialog

- `children`: use Spectrum's `Heading`, `Divider`, `Content`, `Footer`, and `ButtonGroup` to compose the dialog body.
- `size`: `S | M | L` — preferred width.

### DialogTrigger

- `children`: trigger element (first child) and `Dialog` or render-prop returning one (second child).
- `isDismissable`: allow backdrop/Escape dismiss.
- `type`: determines the overlay style.
- `onOpenChange`: called when the dialog opens or closes.

### Accessibility

- `Dialog` renders with `role="dialog"` and `aria-modal="true"`.
- Focus is trapped inside the dialog while open and restored to the trigger when closed.
- Always include a visible `Heading` — Spectrum uses it as the dialog's accessible name.

## Related components

- `AlertDialog` — use for short confirmation or destructive-action prompts with structured button labels.
- `DialogContainer` — use when there is no persistent trigger element (programmatic open).
- `Popover` — use for smaller contextual overlays that do not require full focus-trap semantics.
