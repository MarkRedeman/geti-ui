# Skills

Geti UI publishes a [skills discovery endpoint](https://docs.geti-ui.markredeman.nl/.well-known/skills/index.json) that lets AI agents install curated instruction sets for the Geti design system libraries.

Skills follow the [`.well-known/skills/`](https://skills.md) convention — a lightweight standard for distributing AI agent skills over HTTP.

## Install

Install all available Geti UI skills at once:

```bash
npx skills add https://docs.geti-ui.markredeman.nl
```

This discovers and installs all skills from the endpoint. You can also install a specific skill by its URL:

```bash
npx skills add https://docs.geti-ui.markredeman.nl/.well-known/skills/geti-ui
```

## Available skills

The documentation site publishes **4 skills**, each aligned with a library package:

### Geti UI

**ID:** `geti-ui`

The core component library (`@geti-ai/ui`). Covers 80+ React components built on Adobe React Spectrum, organized into categories: UI, Form Controls, Date Controls, Color Controls, Pickers, Data Display, Status & Feedback, Navigation, Overlays, and Layout & Structure.

Includes documentation for all components plus asset pages (icons, images, domains, primary tools).

```bash
npx skills add https://docs.geti-ui.markredeman.nl/.well-known/skills/geti-ui
```

### Geti UI Charts

**ID:** `geti-ui-charts`

Data visualization with `@geti-ai/charts`. Covers chart primitives (Line, Area, Bar, Scatter, Pie, Donut, Radar, RadialBar, Meter, Treemap, Sparkline), compositions, theming, and 18 machine learning-specific chart types.

```bash
npx skills add https://docs.geti-ui.markredeman.nl/.well-known/skills/geti-ui-charts
```

### Geti UI Smart Tools

**ID:** `geti-ui-smart-tools`

Browser-based computer vision annotation tools from `@geti-ai/smart-tools`. Covers OpenCV.js tools (GrabCut, Intelligent Scissors, Watershed, SSIM) and ONNX Runtime tools (RITM, Segment Anything).

```bash
npx skills add https://docs.geti-ui.markredeman.nl/.well-known/skills/geti-ui-smart-tools
```

### Geti UI Examples

**ID:** `geti-ui-examples`

Composition patterns and kitchensink examples showing how to combine Geti UI components into real application interfaces. Includes category kitchensinks and 9 composed examples (job management, model list, annotation, media filter, toolbar, etc.).

```bash
npx skills add https://docs.geti-ui.markredeman.nl/.well-known/skills/geti-ui-examples
```

## What's in a skill?

Each skill descriptor contains:

| Field | Description |
|-------|-------------|
| `name` | Skill identifier (e.g. `geti-ui`) |
| `description` | One-line summary of what the skill covers |
| `instructions` | Detailed setup and usage instructions the agent follows |
| `resources` | URLs to individual documentation pages the agent can reference |

The instructions teach the agent how to install the library, set up theming, use import patterns, and follow conventions. The resources link to the full documentation pages generated by the docs site.

## How it works

The skills endpoint is generated automatically at build time by an [Rspress plugin](/ai/skills#architecture). When the documentation site is built:

1. The `@rspress/plugin-llms` plugin generates `llms.txt` — an index of all documentation pages
2. The `skills-endpoint` plugin reads `llms.txt` and categorizes pages by path prefix
3. It generates `.well-known/skills/index.json` (the registry) and individual skill descriptor files
4. These are served as static files from the documentation site

This means skills are always in sync with the latest published documentation.

## Architecture

```
  docs.geti-ui.markredeman.nl
              │
              ├── .well-known/skills/
              │   ├── index.json          ← skill registry
              │   ├── geti-ui             ← component skill
              │   ├── geti-ui-charts      ← charts skill
              │   ├── geti-ui-smart-tools ← smart tools skill
              │   └── geti-ui-examples    ← examples skill
              │
              ├── llms.txt                ← page index (from @rspress/plugin-llms)
              ├── llms-full.txt           ← full docs concatenated
              └── *.md                    ← individual doc pages
```

## Updating skills

To update installed skills to the latest version:

```bash
npx skills update
```

This re-fetches the skill descriptors and resources from the documentation site.
