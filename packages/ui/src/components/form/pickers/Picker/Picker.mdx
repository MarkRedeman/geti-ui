# Picker

<Subtitle>A select-only dropdown that enforces a closed choice from a fixed, predefined list of options.</Subtitle>

```tsx
import { Picker, PickerItem as Item } from '@geti-ai/ui';
```

- [React Spectrum Picker](https://react-spectrum.adobe.com/react-spectrum/Picker.html)

<Stories />

## Responsibility

`Picker` is a pure pass-through wrapper around Adobe React Spectrum's `Picker`. It is the correct choice when the set of valid options is known at design time and the user may **not** type arbitrary input — the dropdown enforces a closed selection with no free-text fallback.

Key responsibilities:

- **Closed selection**: Users can only choose from the options provided as `<Item>` children or via the `items` prop.
- **Typed generic**: `<T extends object>` allows the item type to be inferred from `items`, giving full TypeScript type safety on `selectedKey`, `defaultSelectedKey`, and `onSelectionChange`.
- **Full Spectrum API**: No props are removed or defaulted. Every Spectrum `Picker` prop — `label`, `placeholder`, `isRequired`, `isDisabled`, `validationState`, `width`, `items`, `disabledKeys`, `sections` — is available.
- **Static and dynamic options**: Supports both JSX children (`<Item key="x">Label</Item>`) and the dynamic `items` prop with a render function for data-driven lists.

### When to use

- The list of options is **fixed and exhaustive** — model types, export formats, locale settings, sort orders.
- The user should **not** be able to enter a value outside the defined set.
- There are **more than ~4 options** (use `RadioGroup` for 4 or fewer inline options).
- The options list is up to ~20 items and does not require filtering — above that, reach for `ComboBox`.

### When to avoid

- Users need to filter a long list — use `ComboBox` instead.
- Fewer than 3 options that should all be visible at once — use `RadioGroup`.
- Multiple items can be selected simultaneously — use a `ListBox` with `selectionMode="multiple"`.
- Options are user-generated or unbounded — `ComboBox` is the appropriate control.

## Variants and states

### Default (no selection)

Renders the `placeholder` text (or an empty trigger) until the user makes a choice.

### Default selected

Pass `defaultSelectedKey` to pre-select an item in uncontrolled usage. The key must match one of the item keys in the collection.

### Disabled

`isDisabled` grays out the trigger and removes it from the tab order. Use when the value is locked or context-dependent (e.g., a field that is only editable after another choice is made).

### Required with validation

Combine `isRequired`, `validationState`, and `errorMessage` to surface form-level errors. The trigger border turns red and the message appears below.

### Grouped options (Sections)

Use `Section` from `@adobe/react-spectrum` as a child of `Picker` to add visual groupings with headings. Sections are purely presentational — keys must still be unique across all sections.

## Props

### Content

| Prop           | Type            | Default | Description                                                                      |
| -------------- | --------------- | ------- | -------------------------------------------------------------------------------- |
| `label`        | `ReactNode`     | —       | Visible label rendered above the trigger. Omit only when providing `aria-label`. |
| `placeholder`  | `string`        | —       | Text shown in the trigger when no item is selected.                              |
| `items`        | `Iterable<T>`   | —       | Data source for dynamic collections. Pair with a render-function `children`.     |
| `disabledKeys` | `Iterable<Key>` | —       | Keys of items that are shown but cannot be selected.                             |

### Value

| Prop                 | Type                 | Default | Description                                   |
| -------------------- | -------------------- | ------- | --------------------------------------------- |
| `selectedKey`        | `Key \| null`        | —       | The currently selected item key (controlled). |
| `defaultSelectedKey` | `Key`                | —       | The initially selected key (uncontrolled).    |
| `onSelectionChange`  | `(key: Key) => void` | —       | Called when the user selects an item.         |

### Validation

| Prop              | Type                   | Default | Description                                        |
| ----------------- | ---------------------- | ------- | -------------------------------------------------- |
| `isRequired`      | `boolean`              | `false` | Marks the field required; sets `aria-required`.    |
| `isDisabled`      | `boolean`              | `false` | Disables the entire picker.                        |
| `validationState` | `'valid' \| 'invalid'` | —       | Controls inline validation styling.                |
| `errorMessage`    | `ReactNode`            | —       | Error text shown when `validationState="invalid"`. |

### Layout

| Prop    | Type             | Default | Description                  |
| ------- | ---------------- | ------- | ---------------------------- |
| `width` | `DimensionValue` | —       | Width of the trigger button. |

## Usage examples

```tsx
// Static children (most common)
<Picker label="Export format" onSelectionChange={(key) => setFormat(key)}>
  <Item key="png">PNG</Item>
  <Item key="jpeg">JPEG</Item>
  <Item key="webp">WebP</Item>
</Picker>

// Controlled selection
const [sortKey, setSortKey] = useState<Key>('date-desc');
<Picker
  label="Sort by"
  selectedKey={sortKey}
  onSelectionChange={(key) => setSortKey(key as string)}
>
  <Item key="date-desc">Newest first</Item>
  <Item key="date-asc">Oldest first</Item>
  <Item key="name-asc">Name A–Z</Item>
</Picker>

// Dynamic items from data
<Picker label="Model type" items={modelTypes} onSelectionChange={handleModelChange}>
  {(model) => <Item key={model.id}>{model.name}</Item>}
</Picker>

// Required with validation
<Picker
  label="Region"
  placeholder="Select a region…"
  isRequired
  validationState={region ? 'valid' : 'invalid'}
  errorMessage="Please select a region."
>
  <Item key="us-east">US East</Item>
  <Item key="eu-west">EU West</Item>
  <Item key="ap-south">AP South</Item>
</Picker>
```

## Accessibility

- **ARIA pattern**: Spectrum implements `Picker` using the ARIA Listbox pattern. The trigger has `role="button"` with `aria-haspopup="listbox"` and `aria-expanded` reflecting open state. The dropdown renders as `role="listbox"` with each option as `role="option"`.
- **Labels**: The `label` prop renders a visible `<label>` element associated with the trigger via `for`/`id`. If `label` is omitted, provide `aria-label` on the `Picker` to maintain an accessible name.
- **Keyboard — trigger**: `Space`, `Enter`, or `Arrow Down` opens the listbox. `Alt + Arrow Down` opens without moving selection.
- **Keyboard — listbox**: `Arrow Up/Down` moves focus between options. `Enter` or `Space` selects the focused option and closes. `Escape` or `Tab` closes without changing selection.
- **Type-ahead**: Alphanumeric keys while the listbox is open move focus to the first matching option.
- **Disabled options**: Items in `disabledKeys` have `aria-disabled="true"` and are skipped during keyboard navigation.
- **Validation**: `isRequired` sets `aria-required="true"`. `validationState="invalid"` sets `aria-invalid="true"` and `errorMessage` is linked via `aria-describedby`.

## Related components

- **`ComboBox`** — use instead when users need to type or filter a long or dynamic list of options.
- **`RadioGroup`** — use instead when there are 3 or fewer options and displaying them all inline is preferred.
- **`CheckboxGroup`** — use instead when multiple options may be selected simultaneously.
