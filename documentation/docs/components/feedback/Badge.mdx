# Badge

`Badge` displays a short colour-coded label for categorisation and status indication. Use it to call attention to a category or condition on a data item — model state, annotation status, severity level.

```tsx
import { Badge } from '@geti/ui';
```

- [React Spectrum Badge ↗](https://react-spectrum.adobe.com/react-spectrum/Badge.html)
- [Geti UI Theming](/components/theming)

## Responsibility

`Badge` is a thin wrapper around React Spectrum's `Badge`. It passes all props through and adds no Geti-specific defaults. The wrapper keeps the import path consistent with the rest of `@geti/ui`.

## Variants and states

| Variant | Semantic meaning |
|---|---|
| `neutral` | No semantic meaning; general category label |
| `info` | Informational, ongoing, or pending |
| `positive` | Success, healthy, active |
| `negative` | Error, failure, inactive |
| `notice` | Warning, requires attention |
| `indigo` `yellow` `magenta` `fuchsia` `purple` `seafoam` | Non-semantic categorical colours |

## Props (deep dive)

### Content

- `children`: the badge label text, or an icon, or both together in a fragment.
- `variant`: one of the semantic or categorical colour names listed above.

### Accessibility

- When `children` contains only an icon, provide `aria-label` with a text description.
- Avoid using `Badge` as the only indicator of critical state — pair it with text or other signals.

## Related components

- `StatusLight` — use when a coloured dot is the primary signal, not a filled chip.
- `Tag` — use for Geti-styled category chips with an accent dot inside data rows.
- `InlineAlert` — use when a status message needs a heading and body text, not just a label.
