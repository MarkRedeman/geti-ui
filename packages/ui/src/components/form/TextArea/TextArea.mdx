# TextArea

A multi-line text input field for long-form content like comments, descriptions, notes, and addresses.

```tsx
import { TextArea } from '@geti-ai/ui';
```

- [React Spectrum V3 TextArea](https://react-spectrum.adobe.com/react-spectrum/TextArea.html)

<Stories />

## Responsibility

`TextArea` is a transparent wrapper around Adobe React Spectrum's `TextArea`. It renders a `<textarea>` with automatic label association, validation state styling, `description` and `errorMessage` rendering, and keyboard handling — all without extra configuration.

`onChange` receives the **string value** directly — no `.target.value` unwrapping needed. Unlike `TextField`, pressing `Enter` inserts a new line rather than submitting a form.

### When to use

- For long-form text: comments, notes, descriptions, bios, addresses, free-form feedback.
- When the input is expected to span multiple lines or line breaks in the value need to be preserved.
- When `TextField` feels too constraining for the expected content length.

### When to avoid

- **Single-line input** — use `TextField`.
- **Search inputs** — use `SearchField`.
- **Numeric inputs** — use `NumberField`.
- **Rich text or formatting** — `TextArea` is plain text; use a dedicated rich-text editor.
- **When no label is feasible** — always provide `label` or `aria-label`.

## States

| State     | Prop                        | Description                                             |
| --------- | --------------------------- | ------------------------------------------------------- |
| Default   | —                           | Standard interactive field.                             |
| Invalid   | `validationState="invalid"` | Red border, error icon, `errorMessage` below the field. |
| Valid     | `validationState="valid"`   | Green border, checkmark. Use sparingly.                 |
| Disabled  | `isDisabled={true}`         | Non-focusable, visually dimmed.                         |
| Read-only | `isReadOnly={true}`         | Focusable and copy-able, but not editable.              |
| Required  | `isRequired={true}`         | Visual asterisk and `aria-required="true"`.             |
| Quiet     | `isQuiet={true}`            | Underline-only border for dense layouts.                |

Use `labelPosition="side"` to place the label to the left for aligned form layouts.

## Props

<Controls of={DocStories.Default} />

### Content

| Prop             | Type        | Description                                                                                      |
| ---------------- | ----------- | ------------------------------------------------------------------------------------------------ |
| `label`          | `ReactNode` | The label above or beside the field. Required for accessibility unless `aria-label` is provided. |
| `placeholder`    | `string`    | Hint text when empty. Supplement labels — do not substitute for them.                            |
| `description`    | `ReactNode` | Persistent helper text below the field.                                                          |
| `errorMessage`   | `ReactNode` | Error message shown when `validationState="invalid"`.                                            |
| `contextualHelp` | `ReactNode` | A `<ContextualHelp>` popover anchored to the label.                                              |

### Value / State

| Prop              | Type                   | Default | Description                                 |
| ----------------- | ---------------------- | ------- | ------------------------------------------- |
| `value`           | `string`               | —       | The current value (controlled).             |
| `defaultValue`    | `string`               | —       | The initial value (uncontrolled).           |
| `validationState` | `'valid' \| 'invalid'` | —       | Visual and ARIA validation state.           |
| `isRequired`      | `boolean`              | `false` | Adds a visual asterisk and `aria-required`. |
| `isReadOnly`      | `boolean`              | `false` | Value visible but not editable.             |
| `isDisabled`      | `boolean`              | `false` | Non-interactive and visually dimmed.        |
| `minLength`       | `number`               | —       | Minimum number of characters required.      |
| `maxLength`       | `number`               | —       | Maximum number of characters allowed.       |
| `isQuiet`         | `boolean`              | `false` | Underline-only border style.                |

### Events

| Prop       | Type                      | Description                                                                                 |
| ---------- | ------------------------- | ------------------------------------------------------------------------------------------- |
| `onChange` | `(value: string) => void` | Called on every value change. Receives the new string directly — no `.target.value` needed. |
| `onBlur`   | `(e: FocusEvent) => void` | Called when the field loses focus. Commonly used to trigger validation.                     |

### Layout

| Prop            | Type              | Description                           |
| --------------- | ----------------- | ------------------------------------- |
| `labelPosition` | `'top' \| 'side'` | Label position relative to the input. |
| `width`         | `DimensionValue`  | The width of the entire component.    |

## Related components

- `TextField` — use instead for single-line text inputs.
- `SearchField` — use instead when the input is for filtering or querying content.
- `NumberField` — use instead for numeric-only inputs with stepper buttons.

## Example use cases

<details>
  <summary>Typical use cases</summary>

- Writing a project description or dataset annotation notes in a creation or edit dialog.
- Providing free-form feedback or comments on a model's performance in a review form.
- Entering a multi-line address or postal description in a user settings form.
- Composing an issue report or bug description to be submitted to a support workflow.
