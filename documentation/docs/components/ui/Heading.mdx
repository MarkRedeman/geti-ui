# Heading

`Heading` renders section and page titles with Spectrum heading semantics.

```tsx
import { Heading } from '@geti-ui/ui';
```

- [React Spectrum Heading ↗](https://react-spectrum.adobe.com/react-spectrum/Heading.html)
- [Geti UI Theming](/components/theming)

## Examples

### Basic heading

```tsx preview
import { Heading } from '@geti-ui/ui';

export default function App() {
  return <Heading>Dataset overview</Heading>;
}
```

## Responsibility

`Heading` is a thin wrapper around React Spectrum `Heading`. It keeps heading primitives available from the `@geti-ui/ui` entry point for consistent imports across components and docs.

## Variants and states

`Heading` has no wrapper-specific variants. Visual scale and weight come from Spectrum and parent context.

## Props (deep dive)

### Content

- `children`: heading text or inline markup.

### Semantics

- `level`: heading level (`1`-`6`) used for semantic hierarchy and accessibility.
- Pick levels based on document structure, not visual style preference.

### Accessibility

- Use headings in logical order to preserve document structure.
- Keep heading copy short and descriptive.
