# View

View is a general-purpose layout container that provides direct access to Spectrum's design tokens as JSX props.

```tsx
import { View } from '@geti-ai/ui';
```

- [React Spectrum V3 View](https://react-spectrum.adobe.com/react-spectrum/View.html)

<Stories />

## Responsibility

The `View` component is the Spectrum-aware equivalent of a plain `<div>`. It acts as a semantically-neutral container that allows you to apply Spectrum design tokens (padding, border, background color, etc.) directly as props, without writing custom CSS.

Key responsibilities:

- **Theme Awareness**: Props like `backgroundColor="gray-100"` automatically adapt to the active color scheme (dark/light mode).
- **Scale Awareness**: Dimension tokens like `padding="size-200"` automatically scale based on the user's platform (e.g., larger for touch, smaller for mouse).
- **Responsive Layouts**: Supports the `Responsive<T>` prop shape (e.g., `width={{ base: 'size-1000', L: 'size-2000' }}`) for breakpoint-aware sizing.
- **Logical Properties**: Supports RTL-aware spacing through logical props like `paddingStart`, `paddingEnd`, `marginStart`, and `marginEnd`.

### When to use

- As a generic container that needs Spectrum-aware styling (borders, padding, backgrounds).
- To establish a layout context for nested components.
- When you need a container that responds to different breakpoints.
- To wrap content that requires a specific semantic HTML element (via `elementType`).

### When to avoid

- For purely structural layouts (use `Flex` or `Grid` instead).
- When a more specialized component exists (e.g., use `Well` for a recessed container with specific styling).

## Variants

### Bordered Container

Use `borderWidth`, `borderColor`, and `borderRadius` to create framed sections or cards.

### Semantic Elements

Use the `elementType` prop to render the `View` as a specific HTML element, such as `main`, `section`, `article`, or `aside`.

## Props

### Content & Structure

| Prop          | Type                    | Default | Description                                      |
| ------------- | ----------------------- | ------- | ------------------------------------------------ |
| `children`    | `ReactNode`             | -       | The content to be rendered inside the container. |
| `elementType` | `string \| ElementType` | `'div'` | The HTML element to render.                      |

### Style Tokens

| Category       | Key Props                                    | Examples                                                |
| -------------- | -------------------------------------------- | ------------------------------------------------------- |
| **Background** | `backgroundColor`                            | `'gray-100'`, `'accent'`, `'negative'`, `'transparent'` |
| **Spacing**    | `padding`, `margin`                          | `'size-50'`, `'size-200'`, `'size-400'`                 |
| **Border**     | `borderWidth`, `borderColor`, `borderRadius` | `'thin'`, `'dark'`, `'medium'`, `'large'`               |
| **Sizing**     | `width`, `height`, `minWidth`, `maxWidth`    | `'size-2000'`, `'100%'`, `400`                          |
| **Layout**     | `flex`, `gridArea`, `overflow`, `isHidden`   | Flex/Grid child and visibility controls.                |

### Positioning

| Prop                                | Type                            | Default    | Description                               |
| ----------------------------------- | ------------------------------- | ---------- | ----------------------------------------- |
| `position`                          | `'static' \| 'relative' \| ...` | `'static'` | CSS positioning mode.                     |
| `top` / `bottom` / `left` / `right` | `DimensionValue`                | -          | Positional offsets using Spectrum tokens. |

By default, `View` renders as a `div` with no ARIA role. Use the `elementType` prop to render more descriptive HTML5 landmark elements (e.g., `elementType="section"`) when appropriate for document structure. Standard ARIA attributes (e.g., `role`, `aria-label`) can be passed through via DOM props.

## Related components

- `Flex` — use for one-dimensional (row or column) flex layouts with Spectrum tokens.
- `Grid` — use for two-dimensional grid layouts with Spectrum tokens.
- `PressableElement` — use when the container itself must be interactive (pressable/clickable).

## Example use cases

<details>
  <summary>Typical use cases</summary>

- Wrapping a sidebar panel section with a bordered `View` to visually separate it from adjacent panels.
- Creating a padded card container with a `backgroundColor` token for a subtle raised appearance.
- Using `elementType="section"` to give a major content block the correct HTML5 landmark semantics.
- Applying responsive `width` or `height` tokens to a container that adjusts between breakpoints.
