# DateField

`DateField` provides a segmented keyboard input for entering a date value, with each date segment (day, month, year) editable independently. `TimeField` is a co-located companion for time-only values.

```tsx
import { DateField, TimeField } from '@geti-ai/ui';
```

- [React Spectrum DateField ↗](https://react-spectrum.adobe.com/react-spectrum/DateField.html)
- [React Spectrum TimeField ↗](https://react-spectrum.adobe.com/react-spectrum/TimeField.html)
- [Geti UI Theming](/components/theming)

## Examples

### DateField — default

```tsx preview
import { DateField } from '@geti-ai/ui';

export default function App() {
  return <DateField label="Event date" />;
}
```

```tsx preview

  return <DateField label="Birth date" description="Please enter your date of birth." />;
```

## Responsibility

Both `DateField` and `TimeField` are direct pass-through wrappers of the equivalent React Spectrum components. No defaults are changed. Use `DateField` for date-only values and `TimeField` for time-only values; combine both in a layout when you need a full date-time entry without the calendar popover.

## Variants and states

| State | Prop | Notes |
|---|---|---|
| Valid | `validationState="valid"` | Green indicator |
| Invalid | `validationState="invalid"` + `errorMessage` | Red outline + error |
| Disabled | `isDisabled` | Non-interactive |
| Read-only | `isReadOnly` | Non-editable display |
| Required | `isRequired` | Adds required indicator |

## Props (deep dive)

### Content

- `label`: visible field label.
- `description`: helper text.
- `errorMessage`: validation error text.

### Value / state (DateField)

- `value` / `defaultValue`: `CalendarDate | CalendarDateTime | ZonedDateTime`.
- `minValue` / `maxValue`: constrains the accepted range.
- `granularity`: smallest unit to display (`"day"`, `"hour"`, `"minute"`, `"second"`).
- `isDisabled`, `isReadOnly`, `isRequired`.

### Value / state (TimeField)

- `value` / `defaultValue`: `Time` or `ZonedDateTime`.
- `minValue` / `maxValue`: time range constraint.
- `granularity`: `"hour"`, `"minute"`, or `"second"`.
- `hourCycle`: `12` or `24`.

### Events

- `onChange(value)`: fires when the user finishes editing a segment.

### Accessibility

- Always supply `label` — the segmented input cannot be labelled by context alone.
- Each segment receives its own `aria-label` from Spectrum automatically.

## Related components

- `DatePicker` — use when the user benefits from a calendar popover for picking dates.
- `DateRangePicker` — use for a start + end date range with calendar.
- `Calendar` — use for a standalone calendar without text input.
