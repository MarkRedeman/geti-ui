## Responsibility

`PressableElement` is an architectural bridge component. it allows any arbitrary layout or container to accept **Spectrum style props** (padding, margin, background color, etc.) while remaining fully keyboard and pointer accessible via `react-aria-components` (RAC).

It is the primary choice for creating custom interactive surfaces that do not fit the semantic model of a `Button` or `Link`, such as clickable cards, list items, or complex thumbnail wrappers.

### Key Features
- **Hybrid Styling**: Uses the internal Spectrum `useStyleProps` hook to translate tokens like `padding="size-200"` into CSS variables.
- **Accessible Press**: Leverages the RAC `Pressable` component to handle focus, keyboard (Enter/Space), and touch interactions uniformly.
- **Ref Forwarding**: Ensures that interactions and focus are correctly managed on the underlying DOM element.
- **Truncation Support**: Includes a built-in `isTruncated` utility for handling text overflow in interactive containers.

---

## Variants

### Truncation
When `isTruncated` is `true`, the wrapper applies CSS ellipsis styles to ensure content fits within the specified `width`.

### Double Click
In addition to standard press events, `PressableElement` supports a native `onDoubleClick` handler, which is useful for specialized workflows like "double-click to rename" or "double-click to open."

### Styles

`PressableElement` accepts all standard [Spectrum layout and spacing props](https://react-spectrum.adobe.com/react-spectrum/styling.html).

## Implementation Details

The component manually splits its props between the RAC `Pressable` wrapper and an inner `div` that receives the visual styles:

```tsx
<Pressable {...pressableProps}>
  <div {...styleProps} onDoubleClick={onDoubleClick}>
    {children}
  </div>
</Pressable>
```

- **Outer Wrapper**: `Pressable` (from RAC) manages the interaction state and ARIA attributes.
- **Inner Element**: A `div` that uses Spectrum's `useStyleProps` to apply layout, spacing, and custom `UNSAFE_style` / `UNSAFE_className` overrides.

---

## Accessibility

- **Keyboard Support**: Automatically handles `tabIndex` and keyboard activation (Enter/Space).
- **Focus Indicator**: Inherits Spectrum-compliant focus ring behavior when focused via keyboard.
- **Roles**: When wrapping non-semantic elements, ensure the child has an appropriate `role` (e.g., `role="button"`, `role="link"`, or `role="gridcell"`) if it is not inherently interactive.
