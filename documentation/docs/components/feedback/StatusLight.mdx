# StatusLight

`StatusLight` pairs a coloured dot with a text label to communicate the state of an entity. Use it in data rows, dashboards, and detail panels to show at a glance whether something is active, failed, in progress, or in another named state.

```tsx
import { StatusLight } from '@geti-ai/ui';
```

- [React Spectrum StatusLight ↗](https://react-spectrum.adobe.com/react-spectrum/StatusLight.html)
- [Geti UI Theming](/components/theming)

## Examples

### Neutral

```tsx preview
import { StatusLight } from '@geti-ai/ui';

export default function App() {
  return <StatusLight variant="neutral">Neutral</StatusLight>;
}
```

### Positive — active or healthy

```tsx preview
import { StatusLight } from '@geti-ai/ui';

export default function App() {
  return <StatusLight variant="positive">Active</StatusLight>;
}
```

### Negative — error or failure

```tsx preview
import { StatusLight } from '@geti-ai/ui';

export default function App() {
  return <StatusLight variant="negative">Error</StatusLight>;
}
```

### Notice — warning

```tsx preview
import { StatusLight } from '@geti-ai/ui';

export default function App() {
  return <StatusLight variant="notice">Warning</StatusLight>;
}
```

### Info

```tsx preview
import { StatusLight } from '@geti-ai/ui';

export default function App() {
  return <StatusLight variant="info">Processing</StatusLight>;
}
```

### Disabled

```tsx preview
import { StatusLight } from '@geti-ai/ui';

export default function App() {
  return <StatusLight variant="positive" isDisabled>Disabled</StatusLight>;
}
```

## Responsibility

`StatusLight` is a thin wrapper around React Spectrum's `StatusLight`. All styling, dot colours, and accessibility behaviour come from Spectrum. The wrapper keeps the import path consistent with the rest of `@geti-ai/ui`.

## Variants and states

**Semantic variants** (use these when the status carries meaning):

| Variant | Meaning |
|---|---|
| `neutral` | No specific meaning |
| `positive` | Active, healthy, success |
| `negative` | Failed, error, inactive |
| `notice` | Warning, attention needed |
| `info` | In progress, informational |

**Categorical variants** (use for data-visualisation colour coding without semantic meaning):

`celery` · `chartreuse` · `yellow` · `magenta` · `fuchsia` · `purple` · `indigo` · `seafoam`

## Props (deep dive)

### Content

- `children`: the text label shown next to the dot.
- `variant`: semantic or categorical colour name.

### State

- `isDisabled`: dims the light and text to communicate inactivity.

### Accessibility

- The `children` text is the accessible label — always provide meaningful text.
- `StatusLight` does not have an interactive role; if clicking it triggers navigation, use a different component.
