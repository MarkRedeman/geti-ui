# Custom Charts

Use primitives directly to build charts that are not covered by the [composition components](./compositions).

## When to reach for primitives

- You need a chart type not in the composition library (e.g. funnel, candlestick, sankey).
- You need to compose multiple Recharts layers in a single chart.
- You need per-series control beyond what `SeriesConfig` exposes.

Note: custom charts use **raw Recharts chart roots** (`AreaChart`, `ComposedChart`, etc.) together with Geti primitives (`GetiXAxis`, `GetiYAxis`, `GetiTooltip`, `GetiCartesianGrid`, ...).

## Adding a new composition chart (contributor workflow)

When a pattern is reused often and should be easy for product teams to consume, add a composition chart in `packages/charts/src/components`.

Checklist:

1. Create `packages/charts/src/components/<Name>Chart.tsx`
   - Functional component
   - `animate?: boolean` with default `false`
   - `aria-label` passthrough
   - expose extension props (`...AxisProps`, tooltip/legend/grid props) where relevant
   - **Use `getAxisTickStyle`, `getAxisLineStyle`, `getAxisTickLineStyle` from `../utils/axisStyles`** for all axis styling — do not inline theme token reads
2. Export from `packages/charts/src/index.ts`
3. Add docs pages:
   - `documentation/docs/charts/compositions/<name>-chart.mdx`
   - `documentation/docs/charts/primitives/<name>-chart.mdx`
   - **Update this file (`custom-charts.mdx`):** if the new chart makes an existing "when to reach for primitives" example stale, update that section too
4. Add links/navigation:
   - `documentation/docs/charts/compositions.mdx`
   - `documentation/docs/charts/primitives.mdx`
   - `documentation/rspress.config.ts`
5. Add dataset subset theming example to:
   - `documentation/docs/charts/theming/dataset-subsets.mdx`
6. Verify:
   - `npm run type-check --workspace=@geti-ai/charts`
   - `npm run build --workspace=@geti-ai/charts`

## Building a custom area chart

```tsx preview
import { Area, AreaChart, ComposedChart } from 'recharts';
import {
  GetiCartesianGrid,
  GetiXAxis,
  GetiYAxis,
  GetiTooltip,
  GetiLegend,
  GetiResponsiveContainer,
} from '@geti-ai/charts/primitives';

const data = [
  { day: 'Mon', images: 120, annotations: 98 },
  { day: 'Tue', images: 145, annotations: 130 },
  { day: 'Wed', images: 132, annotations: 120 },
  { day: 'Thu', images: 160, annotations: 155 },
  { day: 'Fri', images: 178, annotations: 170 },
];

export default function App() {
  return (
    <GetiResponsiveContainer height={260}>
      <AreaChart data={data}>
        <defs>
          <linearGradient id="imgGrad" x1="0" y1="0" x2="0" y2="1">
            <stop offset="5%" stopColor="var(--spectrum-global-color-blue-500)" stopOpacity={0.3} />
            <stop offset="95%" stopColor="var(--spectrum-global-color-blue-500)" stopOpacity={0} />
          </linearGradient>
          <linearGradient id="annGrad" x1="0" y1="0" x2="0" y2="1">
            <stop offset="5%" stopColor="var(--spectrum-global-color-green-500)" stopOpacity={0.3} />
            <stop offset="95%" stopColor="var(--spectrum-global-color-green-500)" stopOpacity={0} />
          </linearGradient>
        </defs>
        <GetiCartesianGrid />
        <GetiXAxis dataKey="day" />
        <GetiYAxis />
        <GetiTooltip />
        <GetiLegend />
        <Area
          type="monotone"
          dataKey="images"
          name="Images uploaded"
          stroke="var(--spectrum-global-color-blue-500)"
          fill="url(#imgGrad)"
          isAnimationActive={false}
        />
        <Area
          type="monotone"
          dataKey="annotations"
          name="Annotations created"
          stroke="var(--spectrum-global-color-green-500)"
          fill="url(#annGrad)"
          isAnimationActive={false}
        />
      </AreaChart>
    </GetiResponsiveContainer>
  );
}
```

## Mixing chart types

Recharts `ComposedChart` lets you layer bars and lines in a single chart. Use it with Geti primitives for consistent styling:

```tsx preview
import { Bar, Line, ComposedChart } from 'recharts';
import {
  GetiCartesianGrid,
  GetiXAxis,
  GetiYAxis,
  GetiTooltip,
  GetiLegend,
  GetiResponsiveContainer,
} from '@geti-ai/charts/primitives';

const data = [
  { week: 'W1', images: 200, accuracy: 71 },
  { week: 'W2', images: 350, accuracy: 76 },
  { week: 'W3', images: 280, accuracy: 80 },
  { week: 'W4', images: 410, accuracy: 84 },
  { week: 'W5', images: 390, accuracy: 87 },
];

export default function App() {
  return (
    <GetiResponsiveContainer height={280}>
      <ComposedChart data={data}>
        <GetiCartesianGrid />
        <GetiXAxis dataKey="week" />
        <GetiYAxis yAxisId="left" orientation="left" />
        <GetiYAxis yAxisId="right" orientation="right" domain={[60, 100]} />
        <GetiTooltip />
        <GetiLegend />
        <Bar
          yAxisId="left"
          dataKey="images"
          name="Images uploaded"
          fill="var(--spectrum-global-color-blue-700)"
          isAnimationActive={false}
        />
        <Line
          yAxisId="right"
          type="monotone"
          dataKey="accuracy"
          name="Accuracy (%)"
          stroke="var(--spectrum-global-color-green-400)"
          dot={false}
          isAnimationActive={false}
        />
      </ComposedChart>
    </GetiResponsiveContainer>
  );
}
```

## Log scale with primitives

Pass `scale="log"` directly on `GetiYAxis` (or `GetiXAxis`) when building custom charts. Always set an explicit `domain` — Recharts log scale requires strictly positive values.

```tsx preview
import { ScatterChart, Scatter } from 'recharts';
import {
  GetiCartesianGrid,
  GetiXAxis,
  GetiYAxis,
  GetiTooltip,
  GetiResponsiveContainer,
} from '@geti-ai/charts/primitives';

// Model size (MB) vs mAP — both axes span orders of magnitude
const models = [
  { size: 2,   map: 28.1, name: 'MobileNet-v3-S' },
  { size: 5,   map: 34.6, name: 'MobileNet-v3-L' },
  { size: 14,  map: 41.2, name: 'EfficientDet-D0' },
  { size: 52,  map: 49.8, name: 'EfficientDet-D3' },
  { size: 98,  map: 55.3, name: 'ResNet-50 FPN' },
  { size: 232, map: 60.7, name: 'ResNet-101 FPN' },
  { size: 440, map: 63.1, name: 'Swin-L' },
];

export default function App() {
  return (
    <GetiResponsiveContainer height={300}>
      <ScatterChart margin={{ top: 10, right: 20, bottom: 20, left: 20 }}>
        <GetiCartesianGrid />
        <GetiXAxis
          dataKey="size"
          name="Model size (MB)"
          scale="log"
          domain={[1, 'auto']}
          type="number"
          label={{ value: 'Model size (MB) — log scale', position: 'insideBottom', offset: -10, fill: 'var(--spectrum-global-color-gray-400)', fontSize: 12 }}
        />
        <GetiYAxis
          dataKey="map"
          name="mAP"
          scale="log"
          domain={[20, 70]}
          label={{ value: 'mAP — log scale', angle: -90, position: 'insideLeft', fill: 'var(--spectrum-global-color-gray-400)', fontSize: 12 }}
        />
        <GetiTooltip cursor={{ strokeDasharray: '3 3' }} />
        <Scatter
          data={models}
          fill="var(--spectrum-global-color-blue-400)"
          isAnimationActive={false}
        />
      </ScatterChart>
    </GetiResponsiveContainer>
  );
}
```

## Animation in custom charts

Custom charts use Recharts props directly. Pass `isAnimationActive={false}` (the library default) on every series element to keep animations off, or pass `isAnimationActive={true}` to opt in:

```tsx
// Opt-in — animated line
<Line dataKey="value" isAnimationActive />

// Explicit off (matches library default)
<Line dataKey="value" isAnimationActive={false} />
```

## Machine Learning custom chart examples

These examples intentionally use primitives/custom composition patterns rather than dedicated "ML primitive" pages.

### Training metrics (custom)

```tsx preview
import { Line, LineChart } from 'recharts';
import { GetiCartesianGrid, GetiLegend, GetiResponsiveContainer, GetiTooltip, GetiXAxis, GetiYAxis } from '@geti-ai/charts/primitives';

const data = [
  { epoch: 1, trainLoss: 1.42, valLoss: 1.53 },
  { epoch: 2, trainLoss: 1.08, valLoss: 1.20 },
  { epoch: 3, trainLoss: 0.86, valLoss: 0.98 },
  { epoch: 4, trainLoss: 0.70, valLoss: 0.83 },
];

export default function App() {
  return (
    <GetiResponsiveContainer height={300}>
      <LineChart data={data}>
        <GetiCartesianGrid />
        <GetiXAxis dataKey="epoch" />
        <GetiYAxis />
        <GetiTooltip />
        <GetiLegend />
        <Line dataKey="trainLoss" name="Train" stroke="var(--spectrum-global-color-green-500)" dot={false} isAnimationActive={false} />
        <Line dataKey="valLoss" name="Validation" stroke="var(--spectrum-global-color-red-500)" dot={false} strokeDasharray="5 3" isAnimationActive={false} />
      </LineChart>
    </GetiResponsiveContainer>
  );
}
```

### Confusion matrix (custom grid)

```tsx preview
import { GetiChartContainer } from '@geti-ai/charts/primitives';

const labels = ['Car', 'Person', 'Bike'];
const matrix = [
  [124, 7, 3],
  [8, 116, 5],
  [4, 9, 98],
];
const max = Math.max(...matrix.flat());

export default function App() {
  return (
    <GetiChartContainer aria-label="Confusion matrix custom chart" height={340}>
      <div style={{ display: 'grid', gridTemplateColumns: `96px repeat(${labels.length}, 1fr)`, gridTemplateRows: `32px repeat(${labels.length}, 1fr)`, gap: 4, height: '100%', padding: 8 }}>
        <div />
        {labels.map((label) => <div key={`c-${label}`} style={{ textAlign: 'center', fontWeight: 600 }}>{label}</div>)}
        {matrix.map((row, r) => (
          <div key={`row-${r}`} style={{ display: 'contents' }}>
            <div style={{ fontWeight: 600 }}>{labels[r]}</div>
            {row.map((v, c) => <div key={`${r}-${c}`} style={{ display: 'grid', placeItems: 'center', border: '1px solid var(--spectrum-global-color-gray-700)', borderRadius: 4, backgroundColor: `rgba(76,155,232,${0.15 + 0.75 * (v / max)})`, fontWeight: 600 }}>{v}</div>)}
          </div>
        ))}
      </div>
    </GetiChartContainer>
  );
}
```

### PR / ROC (custom)

Use raw `LineChart` + `ReferenceLine` with `GetiXAxis`/`GetiYAxis` domain `[0,1]` for evaluation curves.
