## Responsibility

`DatePicker` combines a `DateField` (keyboard segment input) with a `Calendar` popover so users can either type a date or pick one visually. Wraps Spectrum's `DatePicker` with a Safari mobile layout fix.

## Design

Thin wrapper with one hardcoded default prop applied before the spread:

```tsx
export const DatePicker = <T extends DateValue>(props: SpectrumDatePickerProps<T>) =>
    <SpectrumDatePicker width="size-2400" {...props} />;
export type { SpectrumDatePickerProps as DatePickerProps };
```

`width="size-2400"` is set before `...props` as a Safari mobile fix — prevents layout flickering when the calendar popover opens. Callers can override by passing their own `width` (spread wins). No CSS module or custom styling.

## Flow

Props in → `width="size-2400"` applied first → caller's `...props` spread (can override width) → `SpectrumDatePicker` renders field + calendar button → user selects date → `onChange` fires with `DateValue`.

## Integration

- **Depends on**: `@adobe/react-spectrum`, `@react-types/datepicker` (`DateValue`)
- **Contrast**: `DateField` (keyboard only) vs `DatePicker` (keyboard + calendar popover)
- **Companion**: `DateRangePicker` (same Safari fix, selects a start + end date range)
