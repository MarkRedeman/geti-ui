## Responsibility

The `Avatar` component is a lightweight wrapper around Adobe's React Spectrum `Avatar`. It ensures consistent styling and a centralized import point for the Geti UI design system.

The `AvatarGroup` is a custom layout component that manages a collection of avatars, providing an overlapping stack effect and an automatic overflow badge.

---

## Avatar

### Variants

#### Sizes
Avatars support a range of named Spectrum size tokens. The default size is `avatar-size-100` (24px).

| Size Token | Pixel Value (Approx) |
|------------|----------------------|
| `avatar-size-50` | 16px |
| `avatar-size-75` | 20px |
| `avatar-size-100` | 24px (Default) |
| `avatar-size-200` | 32px |
| `avatar-size-300` | 40px |
| `avatar-size-400` | 48px |
| `avatar-size-500` | 64px |
| `avatar-size-600` | 80px |
| `avatar-size-700` | 96px |

Custom numeric pixel values are also supported (e.g., `size={50}`).

#### Disabled
Use the `isDisabled` prop to render the avatar in a visually muted state.

#### Fallback
If the `src` URL is invalid or the image fails to load, a generic silhouette placeholder is rendered automatically.

### Props

<Controls of={DocStories.Default} />

#### Content

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `src` | `string` | — | The image URL for the avatar. |
| `alt` | `string` | `null` | Accessible text description. Required for standalone avatars. |

#### Value

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `string \| number` | `'avatar-size-100'` | Size token or custom pixel number. |
| `isDisabled` | `boolean` | `false` | Whether the avatar is visually disabled. |

#### Styles

The `Avatar` component accepts all standard [Spectrum layout and spacing props](https://react-spectrum.adobe.com/react-spectrum/styling.html), including `margin`, `position`, and `flex` properties.

## AvatarGroup

`AvatarGroup` is a Geti-specific layout component used to display multiple users in a compact, overlapping stack.

### Variants

#### Max Count
The `max` prop controls how many avatars are shown before the list is truncated with a `+N` badge.

#### Uniform Sizing
Providing a `size` prop to the `AvatarGroup` will override the sizes of all individual avatars within the group.

### Props

<Controls of={DocStories.Group} />

#### Content

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `avatars` | `AvatarProps[]` | **Required** | Array of prop objects for each avatar in the group. |

#### Value

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `max` | `number` | `3` | Maximum number of avatars to display before showing an overflow badge. |
| `size` | `string \| number` | — | Uniform size override for all avatars in the group. |

## Accessibility

- **Alt Text**: Standalone avatars must provide a meaningful `alt` description for screen readers.
- **Group Labeling**: `AvatarGroup` uses `role="group"` and `aria-label` to identify the collection of users.
- **Overflow Badge**: The overflow badge includes an `aria-label` (e.g., "2 more") to ensure the total count is accessible.
