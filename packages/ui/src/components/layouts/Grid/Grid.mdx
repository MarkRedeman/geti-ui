# Grid

<Subtitle>Grid is a layout container that uses CSS Grid to create complex two-dimensional layouts with precisely controlled rows and columns.</Subtitle>

```tsx

```

## Responsibility

`Grid` is the layout primitive for two-dimensional composition. Where `Flex` handles a single axis, `Grid` controls both rows and columns simultaneously, making it the right choice for page scaffolding, dashboard panels, card galleries, and any layout where alignment across two axes matters.

Key responsibilities:
- **Two-Dimensional Layout**: Define rows and columns independently with `rows` and `columns` props.
- **Named Areas**: Use the `areas` prop to build readable, intent-expressive layout templates (e.g., `'header header'`, `'sidebar content'`).
- **Flexible Track Sizing**: Supports `fr` units, `auto`, percentage widths, and Spectrum dimension tokens — all in the same declaration.
- **Responsive Control**: Define entirely different grid structures for different viewport sizes using responsive prop objects.

### When to use
- To build the overall skeleton of a page or complex UI panel (header, sidebar, content, footer).
- For card galleries where items should align on both row and column axes.
- When a layout has a fixed-width sidebar alongside a fluid content region.
- For form layouts where labels and inputs must be aligned across multiple rows.
- When items need to span multiple rows or columns.

### When to avoid
- For simple horizontal or vertical stacks — use `Flex` instead.
- When you only need to align items along a single axis.
- For wrapping tag or chip lists — `Flex wrap` is simpler and more semantically appropriate.

## Design

`Grid` is a **thin wrapper** around Adobe React Spectrum's `Grid`. The full `GridProps` interface is re-exported and all props are forwarded without modification.

```tsx
// Grid.tsx — the complete implementation
import { Grid as SpectrumGrid } from '@adobe/react-spectrum';
import type { GridProps } from '@adobe/react-spectrum';

export interface GridComponentProps extends GridProps {}

export const Grid = (props: GridComponentProps) => <SpectrumGrid {...props} />;
```

The thin-wrapper pattern provides namespace isolation — consuming code imports from `@geti/ui` — while keeping maintenance cost at zero: every Spectrum Grid improvement flows through automatically.

## Flow

```
<Grid columns={['200px', '1fr']} gap="size-200">
        │
        ▼
  GridComponentProps (geti-ui re-export of GridProps)
        │
        ▼
  <SpectrumGrid> (Adobe React Spectrum)
        │
        ▼
  Resolves "size-200" → var(--spectrum-global-dimension-size-200) → 16px
  Resolves DimensionValue array → "200px 1fr"
        │
        ▼
  Renders <div style="display:grid; grid-template-columns:200px 1fr; gap:16px;">
```

- **Dimension tokens** in track arrays (e.g., `['size-2000', '1fr']`) are resolved by Spectrum against the active theme variables before being passed to CSS.
- **Named areas** are written as an array of strings — each string is one row — and Spectrum translates them into the `grid-template-areas` CSS property.
- **Responsive objects** (e.g., `columns={{ base: '1fr', L: 'repeat(3, 1fr)' }}`) use Spectrum's CSS media-query system; no JavaScript breakpoint listeners.

## Variants

### Fixed Columns

```tsx
<Grid columns="repeat(3, 1fr)" gap="size-200">
    <Card>...</Card>
    <Card>...</Card>
    <Card>...</Card>
</Grid>
```

### Mixed Track Sizes

```tsx
// Fixed sidebar, fluid content, fixed aside
<Grid columns={['200px', '1fr', '240px']} gap="size-300">
    <Sidebar />
    <Main />
    <Details />
</Grid>
```

### Named Areas

```tsx
<Grid
    areas={[
        'header  header',
        'sidebar content',
        'footer  footer',
    ]}
    columns={['240px', '1fr']}
    rows={['size-800', '1fr', 'size-600']}
    gap="size-200"
>
    <View gridArea="header">...</View>
    <View gridArea="sidebar">...</View>
    <View gridArea="content">...</View>
    <View gridArea="footer">...</View>
</Grid>
```

### Card Gallery

```tsx
// Auto-fill card gallery: as many columns as fit, min 260px each
<Grid columns="repeat(auto-fill, minmax(260px, 1fr))" gap="size-300">
    {projects.map(p => <Card key={p.id}>{p.name}</Card>)}
</Grid>
```

### Responsive Columns

Use a responsive object to collapse columns on smaller viewports.

```tsx
<Grid
    columns={{ base: '1fr', M: 'repeat(2, 1fr)', L: 'repeat(3, 1fr)' }}
    gap={{ base: 'size-200', L: 'size-300' }}
>
    {items.map(item => <Card key={item.id}>{item.name}</Card>)}
</Grid>
```

Spectrum's named breakpoints are:
| Key | Approximate viewport width |
| --- | --- |
| `base` | All sizes (mobile-first default) |
| `S` | ≥ 480 px |
| `M` | ≥ 640 px |
| `L` | ≥ 1024 px |
| `XL` | ≥ 1280 px |

### Alignment

```tsx
// Centre all items within their grid cells
<Grid columns="repeat(3, 1fr)" justifyItems="center" alignItems="center">
    ...
</Grid>
```

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `areas` | `Responsive<string[]>` | - | Named grid area template. Each string represents one row. |
| `columns` | `Responsive<DimensionValue[] \| string>` | - | Track sizes for columns. Accepts Spectrum tokens, CSS values, or `fr` units. |
| `rows` | `Responsive<DimensionValue[] \| string>` | - | Track sizes for rows. |
| `gap` | `Responsive<DimensionValue>` | - | The space between grid items (both axes). |
| `columnGap` | `Responsive<DimensionValue>` | - | Horizontal gap between columns. |
| `rowGap` | `Responsive<DimensionValue>` | - | Vertical gap between rows. |
| `justifyContent` | `Responsive<JustifyContent>` | - | How the whole grid is aligned horizontally within the container. |
| `alignContent` | `Responsive<AlignContent>` | - | How the whole grid is aligned vertically within the container. |
| `justifyItems` | `Responsive<JustifyItems>` | - | Default horizontal alignment of items within their cells. |
| `alignItems` | `Responsive<AlignItems>` | - | Default vertical alignment of items within their cells. |
| `autoColumns` | `Responsive<DimensionValue>` | - | Size for implicitly created columns. |
| `autoRows` | `Responsive<DimensionValue>` | - | Size for implicitly created rows. |
| `elementType` | `string` | `'div'` | The HTML element or custom component to render. |
| `children` | `ReactNode` | - | The grid items. |

All remaining `GridProps` from `@adobe/react-spectrum` are forwarded transparently.

## Integration

`Grid` depends solely on `@adobe/react-spectrum`. It carries no Geti-specific theme overrides. Grid containers are structurally invisible — apply colours, borders, or padding to child `View` components or to individual named areas.

```tsx
import { Grid, View } from '@geti/ui';

// Typical page scaffold using Grid + View
<Grid
    areas={['header', 'content', 'footer']}
    rows={['size-800', '1fr', 'size-600']}
    height="100vh"
>
    <View gridArea="header" backgroundColor="gray-100" padding="size-200">
        <Header />
    </View>
    <View gridArea="content" overflow="auto" padding="size-400">
        <Outlet />
    </View>
    <View gridArea="footer" backgroundColor="gray-100" padding="size-200">
        <Footer />
    </View>
</Grid>
```

## Accessibility

- **Semantic HTML**: `Grid` renders as a `<div>` by default. Use the `elementType` prop to promote it to a meaningful element for the content it contains:
  - `elementType="main"` for the primary page content area.
  - `elementType="section"` for a self-contained section of the page (add `aria-label`).
  - `elementType="ul"` (with `<li>` children) when the grid items form a list.
- **Logical Reading Order**: Ensure the DOM order of grid children matches the intended reading and tab order. CSS Grid's `grid-column` and `grid-row` placement can reorder items visually without changing DOM order, which can confuse screen reader users if used carelessly.
- **Named Areas and ARIA**: Named areas defined in the `areas` prop are purely a CSS concept and have no automatic accessibility meaning. Assign appropriate roles and labels to the child elements placed inside each area.
- **Landmark Regions**: When the grid represents a significant page structure, label each region explicitly:

```tsx
<Grid areas={['header', 'main', 'aside']} columns={['1fr', '280px']} rows={['size-800', '1fr']}>
    <View elementType="header" gridArea="header" aria-label="Page header">
        <Header />
    </View>
    <View elementType="main" gridArea="main" aria-label="Main content">
        <Content />
    </View>
    <View elementType="aside" gridArea="aside" aria-label="Supporting details">
        <Aside />
    </View>
</Grid>
```

- **Focus Management**: Grid layout does not affect focus order. Items are focused in DOM order regardless of their visual position. Plan your DOM order to match the intended Tab sequence.
