## Responsibility

The `Button` component is a thin, intentional wrapper around Adobe React Spectrum's `Button`. Rather than reimplementing behaviour, it enforces Geti-specific constraints and defaults at the boundary, keeping downstream consumers insulated from upstream API changes.

Key responsibilities:
- **Legacy Variant Removal**: The Spectrum `cta` and `overBackground` variants are excluded from the public API. These variants do not exist in the Geti design language and would produce visually inconsistent results.
- **Accent Default**: The component defaults to the `accent` variant (blue), which is the primary Geti call-to-action colour. This means `<Button>Save</Button>` does the right thing without requiring an explicit `variant` prop.
- **Class Merging**: `UNSAFE_className` is passed through `clsx` before reaching Spectrum, so multiple class strings, conditional objects, and arrays are all handled correctly without creating invalid attribute values.
- **Ref Forwarding**: Accepts `FocusableRef<HTMLElement>` â€” Spectrum's ref type â€” enabling programmatic focus management (e.g., returning focus to a trigger after a dialog closes).

### When to use
- For the primary action on a page or within a section (e.g., "Create Project", "Save", "Submit"). Use `variant="accent"` (the default).
- For secondary actions that are important but should not compete visually with the primary CTA (e.g., "Edit", "Export"). Use `variant="primary"`.
- For neutral or cancel actions that carry low visual weight (e.g., "Cancel", "Back"). Use `variant="secondary"`, optionally combined with `style="outline"`.
- For destructive or irreversible actions (e.g., "Delete", "Remove"). Use `variant="negative"` to communicate risk clearly.
- When a pending async operation is triggered by the button. Use `isPending` to disable interaction and show a loading spinner in-place.

### When to avoid
- For navigation-only actions where the destination is a URL â€” prefer a link (`<a>`) or a `Button` with `elementType="a"` and `href`.
- As a container for non-interactive content â€” buttons must have a meaningful press action.
- For icon-only actions inside toolbars â€” prefer `ActionButton` from `@adobe/react-spectrum`, which has a more appropriate compact affordance.
- When you need to trigger menu items or toggles â€” use `MenuTrigger`/`ToggleButton` instead.

## Design

`Button` follows the *thin wrapper* pattern used throughout Geti UI: the component adds constraints and defaults at the seam between Geti's design language and the upstream library, then passes all remaining props through unchanged.

```tsx
// packages/ui/src/components/ui/Button/Button.tsx

type VariantWithoutLegacy = Exclude<SpectrumButtonProps['variant'], 'cta' | 'overBackground'>;

export interface ButtonProps extends Omit<SpectrumButtonProps, 'variant'> {
    variant?: VariantWithoutLegacy;   // narrows the type, removes legacy values
    UNSAFE_className?: string;
    ref?: FocusableRef<HTMLElement>;
}

export const Button = ({ variant = 'accent', UNSAFE_className, ...rest }: ButtonProps) => (
    <SpectrumButton 
        {...rest} 
        variant={variant} 
        UNSAFE_className={clsx(UNSAFE_className) || undefined} 
    />
);
```

### Custom Styling with `UNSAFE_className`

While we generally prefer using Spectrum's built-in props for layout and spacing, `UNSAFE_className` is used when Geti's visual designs deviate significantly from Spectrum's defaults.

- **Class Merging**: We use `clsx` to merge consumer-provided classes with any internal overrides.
- **Normalization**: The `|| undefined` pattern ensures that if no classes are present, the `class` attribute is omitted from the DOM entirely.
- **Token Overrides**: Custom CSS should target the element via a CSS Module and override Spectrum's internal tokens (e.g., `--spectrum-button-m-background-color-accent`) to ensure the design system's theme remains consistent.

For more details on our styling philosophy and available tokens, see the [Theming Guide](?path=/docs/theme-theming--docs).

### Prop mapping

| Geti `ButtonProps`     | Spectrum `SpectrumButtonProps` | Notes |
|------------------------|-------------------------------|-------|
| `variant` (narrowed)   | `variant`                      | Excludes `'cta'` and `'overBackground'`; defaults to `'accent'` |
| `UNSAFE_className`     | `UNSAFE_className`             | Normalised via `clsx` before forwarding |
| `ref`                  | `ref`                          | Typed as `FocusableRef<HTMLElement>` |
| `...rest`              | all other props                | Passed through without transformation |

## Flow

`Button` is a **purely uncontrolled, event-driven** component â€” it holds no internal state of its own. The entire event flow is managed by Spectrum's press system and delegated upward via callbacks.


**Key points:**
- `onPress` is the canonical handler. It fires for mouse clicks, touch taps, and `Space`/`Enter` key presses alike â€” you do not need separate `onClick` and `onKeyDown` handlers.
- `PressEvent` carries a `pointerType` field (`'mouse'`, `'touch'`, `'keyboard'`, `'virtual'`) so you can distinguish gesture origins if needed.
- When `isPending` or `isDisabled` is `true`, Spectrum suppresses all press events before they reach your handler.
- There is no controlled "pressed" state to manage â€” `Button` is always stateless from the consumer's perspective.

### Layout and spacing

The `Button` component accepts all standard [Spectrum layout and spacing props](https://react-spectrum.adobe.com/react-spectrum/styling.html), including `margin`, `marginTop`, `marginEnd`, `position`, `flex`, `width`, and `height`. These are forwarded directly to Spectrum.

## Accessibility

- **ARIA role**: Renders as `role="button"` by default. When `elementType="a"` is set, renders as a semantic `<a>` with appropriate keyboard handling preserved by Spectrum.
- **Keyboard interaction**: `Space` and `Enter` both trigger `onPress`. This is standard button behaviour and is handled automatically â€” no `onKeyDown` override is needed.
- **Labeling**: The button's accessible name is derived from its `children`. For icon-only buttons, always add an `aria-label` to provide a text alternative for screen readers.
- **Disabled state**: `isDisabled` sets `aria-disabled` and suppresses all interaction events. Avoid removing a button from the DOM as a substitute for disabling it â€” screen reader users benefit from knowing a button exists even when it is inactive.
- **Pending state**: `isPending` adds `aria-busy` to communicate that an operation is in progress, in addition to visually showing a spinner.
- **Focus management**: The forwarded `ref` (typed as `FocusableRef<HTMLElement>`) enables programmatic focus via Spectrum's `focusSafely` utilities. Use this to return focus to the trigger button after a dialog or popover closes.

## Integration

- **Dependency**: `@adobe/react-spectrum` â€” `Button` and `SpectrumButtonProps` are imported from this package.
- **Peer dependency**: `@react-types/shared` â€” used for `FocusableRef<HTMLElement>`.
- **ThemeProvider**: Must be present in the React tree for Spectrum tokens (colour, spacing, typography) to resolve correctly. Stories and application roots should wrap content in `<ThemeProvider>`.
- **DialogTrigger**: When a `Button` opens a `Dialog`, use the `Button` as the first child of `<DialogTrigger>`. Spectrum automatically links the button to the dialog via `aria-haspopup` and `aria-expanded`, and restores focus to the button when the dialog closes.
- **Form submission**: `Button` does not submit HTML forms by default. To submit a form, either handle `onPress` explicitly, or set `type="submit"` via `elementType` and a native `<button>` element.
