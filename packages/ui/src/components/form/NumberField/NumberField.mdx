## Responsibility

The `NumberField` component is specifically designed for numeric input. It provides built-in validation for minimum and maximum values, supports localized formatting (e.g., currency, percentages), and includes stepper buttons for quick adjustments. It ensures that the input is always a valid number according to the specified constraints.

### Key Features
- **Stepper Buttons**: Provides easy increment and decrement actions, which are touch-friendly and keyboard-accessible.
- **Localized Formatting**: Uses `Intl.NumberFormat` for locale-aware display of currencies, percentages, and units.
- **Strict Validation**: Prevents users from entering non-numeric characters (except for locale-specific decimal and grouping separators) and enforces `minValue`/`maxValue` limits.
- **Form Integration**: Submits the raw numeric value to forms, ensuring consistent data handling regardless of the displayed format.

### NumberField vs. TextField
- Use **NumberField** for any input that must be a number. It provides better accessibility and UX for numeric data than a generic `TextField`.
- Use **TextField** only for general text entry where numeric constraints are not applicable.

## Variants

### Value Constraints
- **Min/Max**: Use `minValue` and `maxValue` to define the allowed range.
- **Step**: Use `step` to define the increment/decrement interval (defaults to 1).

### Formatting
The `formatOptions` prop allows for advanced formatting:
- **Currency**: `{ style: 'currency', currency: 'USD' }`
- **Percentage**: `{ style: 'percent' }`
- **Decimals**: `{ minimumFractionDigits: 2, maximumFractionDigits: 2 }`

### Visual Options
- **Hide Stepper**: Set `hideStepper` to `true` to remove the increment/decrement buttons while keeping the numeric validation logic.
- **Quiet**: Use `isQuiet` for a more minimal look.

## Props

### Content
| Prop | Type | Description |
| --- | --- | --- |
| `label` | `ReactNode` | The label for the number field. |
| `description` | `ReactNode` | A description for the field. |
| `errorMessage` | `ReactNode` | An error message for the field. |
| `placeholder` | `string` | Temporary text displayed when the field is empty. |

### Value/State
| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `value` | `number` | — | The current value (controlled). |
| `defaultValue` | `number` | — | The default value (uncontrolled). |
| `minValue` | `number` | — | The minimum allowed value. |
| `maxValue` | `number` | — | The maximum allowed value. |
| `step` | `number` | 1 | The amount to increment or decrement. |

### Events
| Prop | Type | Description |
| --- | --- | --- |
| `onChange` | `(value: number) => void` | Handler called when the value changes. |
| `onFocus/onBlur` | `(e: FocusEvent) => void` | Handlers for focus and blur events. |

### Styles
| Prop | Type | Description |
| --- | --- | --- |
| `formatOptions` | `Intl.NumberFormatOptions` | Localized formatting options. |
| `hideStepper` | `boolean` | Whether to hide the stepper buttons. |
| `isQuiet` | `boolean` | Whether the field is quiet. |
| `isDisabled` | `boolean` | Whether the input is disabled. |
| `validationState` | `'valid' \| 'invalid'` | Whether the input is valid or invalid. |

## Accessibility

- **ARIA Role**: Implements `role="spinbutton"`.
- **Keyboard Support**:
    - `ArrowUp/Down`: Increments or decrements the value by the `step` amount.
    - `PageUp/Down`: Increments or decrements the value by a larger amount (if configured).
    - `Home/End`: Sets the value to the `minValue` or `maxValue`.
- **Labeling**: Always provide a `label`. Stepper buttons are automatically labeled for screen readers (e.g., "Increment", "Decrement").
