## Responsibility

The `SearchField` component provides a standard search input interface. It automatically handles the search icon, the "clear" button functionality, and standard search-related keyboard interactions like pressing <kbd>Enter</kbd> to submit.

### When to use
- For search bars, filter inputs, and global search fields.
- When you want an integrated "clear" button to quickly reset the input.

### When to use something else
- Use [TextField](/docs/components-form-textfield--docs) for general data entry (names, emails, descriptions).
- Use [ComboBox](/docs/components-form-pickers-combobox--docs) if you need a search input with a dropdown of suggestions.

## Variants

### States
- **Disabled**: The field is not interactive. Use `isDisabled`.
- **Read-only**: The text is selectable but cannot be changed. Use `isReadOnly`.
- **Invalid**: Indicates a validation error. In V3, use `validationState="invalid"`.
- **Required**: Marks the search as mandatory for a form. Use `isRequired`.

### Visual Styles
- **Quiet**: Use `isQuiet` for a minimal, underline-only style (V3).
- **Clear Button**: Automatically appears once text is entered.

## Props

### Content
| Prop | Type | Description |
|---|---|---|
| `label` | `ReactNode` | The label for the search field. |
| `placeholder` | `string` | Temporary text displayed when the field is empty. |
| `description` | `ReactNode` | Help text displayed below the field. |
| `errorMessage` | `ReactNode` | Error message displayed when the field is invalid. |

### Value / State
| Prop | Type | Default | Description |
|---|---|---|---|
| `value` | `string` | — | The current value (controlled). |
| `defaultValue` | `string` | — | The initial value (uncontrolled). |
| `validationState` | `'valid' \| 'invalid'` | — | Whether the input is valid or invalid. |
| `isDisabled` | `boolean` | `false` | Whether the field is disabled. |

### Events
| Prop | Type | Description |
|---|---|---|
| `onChange` | `(value: string) => void` | Called when the value changes. |
| `onSubmit` | `(value: string) => void` | Called when the user presses <kbd>Enter</kbd>. |
| `onClear` | `() => void` | Called when the clear button is pressed. |

## Accessibility

- **ARIA Role**: Uses `role="searchbox"` for specialized search semantics.
- **Keyboard Support**:
    - <kbd>Enter</kbd>: Triggers the `onSubmit` event.
    - <kbd>Escape</kbd>: Clears the input and triggers `onClear`.
- **Labeling**: Always provide a `label`. Use `aria-label` or `aria-labelledby` if the label is visually hidden.
- **Ref Forwarding**: Forwards its ref to the underlying input, allowing imperative actions like `focus()` or `clear()`.
