## Responsibility

`AlertDialog` is a specialized variant of a Dialog designed for critical confirmations, such as deleting data or overwriting a file. It enforces a standardized layout with a title, a body (content), and a set of action buttons (Primary, Secondary, and Cancel).

Key responsibilities:
- **Critical Feedback**: Provides clear visual and semantic cues for the importance of the action (e.g., using a "destructive" variant).
- **Standardized Actions**: Ensures a consistent button placement and labeling across the application.
- **Accessibility**: Automatically assigns `role="alertdialog"` and manages focus transitions.

### When to use
- For destructive actions (e.g., "Delete Project").
- For critical confirmations that require the user's immediate attention.
- When you need a simple "Yes/No" or "Confirm/Cancel" pattern with minimal content.

### When to avoid
- For complex forms or rich content (use `Dialog` instead).
- For non-critical information or simple alerts (use `Toast` or `Dialog` with an "information" variant).

## Variants

### Confirmation
The default variant, used for standard confirmations.

### Destructive
Used for actions that result in data loss or are irreversible. The primary action button is styled with a "negative" or "danger" color (red).

### Error / Warning / Information
Specialized variants that include a status icon in the heading to convey the nature of the alert.

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `title` | `string` | - | The title of the alert dialog. |
| `children` | `ReactNode` | - | The message or content of the alert dialog. |
| `variant` | `'confirmation' \| 'information' \| 'destructive' \| 'error' \| 'warning'` | `'confirmation'` | The visual style of the alert dialog. |
| `primaryActionLabel` | `string` | - | The label for the primary action button. |
| `secondaryActionLabel` | `string` | - | The label for the optional secondary action button. |
| `cancelLabel` | `string` | - | The label for the cancel button. |
| `onPrimaryAction` | `() => void` | - | Handler called when the primary action button is pressed. |
| `onSecondaryAction` | `() => void` | - | Handler called when the secondary action button is pressed. |
| `onCancel` | `() => void` | - | Handler called when the cancel button is pressed. |
| `autoFocusButton` | `'primary' \| 'secondary' \| 'cancel'` | - | Which button should be automatically focused when the dialog opens. |

## Accessibility

- **Keyboard Support**: `Tab` cycles focus between the action buttons. `Esc` dismisses the dialog and triggers the `onCancel` handler.
- **Aria Roles**: The dialog is assigned `role="alertdialog"`.
- **Labeling**: The `title` prop is automatically linked as the accessible label for the dialog.
