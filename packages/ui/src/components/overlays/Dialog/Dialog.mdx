## Responsibility

The `Dialog` component provides the content and structure for a modal window. It is typically used in conjunction with a `DialogTrigger`, which manages the overlay and associates the dialog with a trigger element (like a Button).

Together, they handle:
- **Focus Management**: Automatically traps focus within the dialog and restores it to the trigger upon closing.
- **Overlay Management**: Handles positioning, backdrop/underlay rendering, and dismissal logic.
- **Accessibility**: Implements the ARIA dialog pattern, ensuring screen readers correctly identify the dialog and its purpose.

### When to use
- To prompt the user for a decision or confirmation.
- To gather input via a form without leaving the current context.
- To display additional information or details related to an item.

### When to avoid
- For transient feedback or notifications (use `Toast` instead).
- For simple tooltips or non-interactive hints (use `Tooltip` or `ContextualHelp`).

## Variants

### Modal
The default dialog type. It renders an underlay that blocks interaction with the rest of the application until the dialog is dismissed.

### Popover
A smaller dialog that appears relative to its trigger. It does not block interaction with the rest of the application and is dismissed when clicking outside. Use `type="popover"` on the `DialogTrigger`.

### Fullscreen
Occupies the entire viewport. Useful for complex tasks or mobile-first experiences. Use `type="fullscreen"` on the `DialogTrigger`.

### Dismissible
Allows the user to close the dialog by clicking an "X" button in the corner. Enabled by the `isDismissable` prop on the `Dialog` component.

## Props

### Dialog Props
| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode \| (close) => ReactNode` | - | The contents of the dialog. Supports slot-based children (`Heading`, `Divider`, `Content`, `ButtonGroup`, `Footer`). |
| `size` | `'S' \| 'M' \| 'L'` | `'M'` | The width of the dialog. |
| `isDismissable` | `boolean` | `false` | Whether the dialog can be dismissed via a close button. |

### DialogTrigger Props
| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `type` | `'modal' \| 'popover' \| 'tray' \| 'fullscreen' \| 'fullscreenTakeover'` | `'modal'` | The type of overlay to render. |
| `mobileType` | `'modal' \| 'tray' \| 'fullscreen'` | `'tray'` | The type of overlay to render on mobile devices. |
| `isOpen` | `boolean` | - | Whether the dialog is open (controlled). |
| `defaultOpen` | `boolean` | `false` | Whether the dialog is open by default (uncontrolled). |
| `onOpenChange` | `(isOpen: boolean) => void` | - | Handler called when the open state changes. |

## Accessibility

- **Keyboard Support**: `Tab` cycles focus within the dialog. `Esc` dismisses the dialog (unless `isKeyboardDismissDisabled` is set).
- **Aria Roles**: The dialog is assigned `role="dialog"` or `role="alertdialog"`.
- **Labeling**: The `Heading` component inside the dialog is automatically linked as the accessible label for the dialog via `aria-labelledby`.
