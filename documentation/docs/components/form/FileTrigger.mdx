# FileTrigger

FileTrigger attaches native file picker behaviour to any child element. When the child is pressed, the system file dialog opens. Use it to build upload buttons, image pickers, and document import flows.

```tsx
import { Button, FileTrigger } from '@geti-ai/ui';
```

- [React Aria FileTrigger ↗](https://react-spectrum.adobe.com/react-aria/FileTrigger.html)
- [Geti UI Theming](/components/theming)

## Examples

### Basic upload button

```tsx preview
import { Button, FileTrigger } from '@geti-ai/ui';

export default function App() {
  return (
    <FileTrigger>
      <Button variant="accent">Upload file</Button>
    </FileTrigger>
  );
}
```

### Images only

```tsx preview
import { Button, FileTrigger } from '@geti-ai/ui';

export default function App() {
  return (
    <FileTrigger acceptedFileTypes={['image/*']}>
      <Button variant="primary">Upload image</Button>
    </FileTrigger>
  );
}
```

### Multiple files

```tsx preview
import { Button, FileTrigger } from '@geti-ai/ui';

export default function App() {
  return (
    <FileTrigger allowsMultiple>
      <Button variant="accent">Upload files</Button>
    </FileTrigger>
  );
}
```

## Responsibility

`FileTrigger` is a thin wrapper around `react-aria-components`' `FileTrigger`. All props pass through unchanged. The wrapper provides a `@geti-ai/ui` import path.

FileTrigger handles no visual rendering of its own — it delegates entirely to its `children` for visual presentation.

## Variants and states

`FileTrigger` is stateless. State management (selected files, upload progress) is the responsibility of the parent component.

## Props (deep dive)

### Content

- `children`: the trigger element — typically a `Button` or `ActionButton`.

### Behaviour

- `acceptedFileTypes`: array of MIME type strings or file extensions (e.g. `['image/*']`, `['.json', '.csv']`).
- `allowsMultiple`: when `true`, the file dialog allows selecting multiple files.
- `allowsDirectory`: when `true`, allows selecting a directory instead of individual files.

### Events

- `onSelect`: `(files: FileList | null) => void` — called with selected files after the dialog closes.

### Accessibility

- the trigger element (`children`) must be an accessible interactive element,
- the accessible name of the button should describe the action (e.g. "Upload dataset images").

## Example use cases

<details>
  <summary>Typical use cases</summary>

- Dataset upload button: open file dialog to select a ZIP archive.
- Profile photo upload: accept `image/*` types only.
- Annotation import: accept a single `.json` or `.xml` file.
- Bulk media upload: allow multiple files via a single click.
