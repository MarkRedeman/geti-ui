# IntelBrandedLoading

A deprecated convenience wrapper that centres the Intel-branded loading animation in a full-height container — use `Loading` directly in new code.

```tsx
// New code — preferred
import { Loading } from '@geti/ui';
<Loading variant="intel" mode="inline" size="L" />

// Legacy code only
import { IntelBrandedLoading } from '@geti/ui';
```

> ⚠️ **Deprecated.** `IntelBrandedLoading` exists solely for backward compatibility. It adds no capability beyond `<Loading variant="intel" mode="inline" size="L" />` combined with a centering `Flex`. All new implementations should use `Loading` directly and handle their own centering.

<Stories />

## Responsibility

`IntelBrandedLoading` is a **thin backward-compatibility shim** that displays the Intel-branded animated logo centred in a full-height container. It was created before `Loading` gained `variant` and `mode` props; it is now superseded entirely by `Loading`.

Its single configurable surface is `height?: string` — the height of the outer `Flex` container, defaulting to `'100vh'`. No other styling, layout, or animation customization is exposed.

### When to use
- **Only** when migrating or maintaining existing code that already uses `IntelBrandedLoading`.

### When to avoid
- **All new code.** Use `<Loading variant="intel" mode="inline" size="L" />` instead.
- When you need sizing other than `L` (192 px) — `Loading` exposes `size` directly.
- When you need overlay or fullscreen positioning — `Loading` supports `mode="overlay"` and `mode="fullscreen"`.

## Design

A thin composition:

```tsx
// What IntelBrandedLoading renders internally:
<Flex justifyContent="center" alignItems="center" height={height} direction="column">
  <Loading variant="intel" mode="inline" size="L" style={{ height }} />
</Flex>
```

The `height` prop is passed to both the `Flex` container and the inner `Loading` component's `style`. This duplication is a legacy artefact — migrating callers should not replicate it.

### Migration path

```tsx
// Before
<IntelBrandedLoading height="400px" />

// After
<div style={{ display: 'flex', justifyContent: 'center', alignItems: 'center', height: '400px' }}>
  <Loading variant="intel" mode="inline" size="L" />
</div>
```

## Integration

- Delegates all rendering to `Loading` in the sibling `Loading/` folder.
- Depends on `Flex` from `@adobe/react-spectrum` for centering layout.
- `Loading` in turn renders the Intel WebP animation from `intel-loading.webp` (co-located in the `IntelBrandedLoading/` folder — this is the only file in the folder that is not deprecated).

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `height` | `string` | `'100vh'` | The height of the outer centering container. Passed to both `Flex` and `Loading`. |

## Accessibility

- Inherits all accessibility semantics from `Loading`: the inner `<img>` carries `role="progressbar"` and `alt="Loading"`.
- The outer `Flex` container is a generic `<div>` and carries no ARIA role.
- See the [Loading accessibility section](/docs/feedback-loading--docs) for complete guidance on announcing loading states to screen readers.

## Related components
- **`Loading`** — the replacement for `IntelBrandedLoading`; use `<Loading variant="intel" mode="inline" size="L" />` in all new code.
- **`ProgressCircle`** — the underlying spinner primitive used by `Loading`'s default (`spinner`) variant for compact inline or overlay states.
- **`Skeleton`** — use for layout-preserving content placeholders during data fetching rather than a full-area branded animation.
