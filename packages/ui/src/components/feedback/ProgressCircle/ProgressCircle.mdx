# ProgressCircle

<Subtitle>A circular spinner that communicates the progress of an operation in a compact form, supporting both determinate and indeterminate states.</Subtitle>

```tsx
import { ProgressCircle } from '@geti/ui';
```

- [React Spectrum ProgressCircle](https://react-spectrum.adobe.com/react-spectrum/ProgressCircle.html)

<Stories />

## Responsibility

`ProgressCircle` registers Adobe React Spectrum's `ProgressCircle` in Geti's public API surface, providing the standard circular loading indicator for contexts where a linear bar is too wide or visually heavy.

Its primary use case in Geti is the indeterminate spinner: signalling that work is in progress without implying a known completion percentage. It is the rendering primitive used by the `Loading` component, which wraps it in positioning logic (fullscreen, overlay, inline). Use `ProgressCircle` directly when you need the spinner itself without any positioning wrapper — for example, inside a button, an icon slot, or a table cell.

Key characteristics:
- **Three sizes**: `S` (24 px), `M` (48 px, default), `L` — tuned for inline, standard, and hero loading contexts respectively.
- **Determinate and indeterminate**: A `value` prop drives a precise arc fill; `isIndeterminate` animates a continuous sweep.
- **No visible label**: Unlike `ProgressBar` and `Meter`, `ProgressCircle` renders no visible text. An `aria-label` is the sole accessible name and is always required.
- **Theme aware**: Visual tokens are resolved by `ThemeProvider`; no component-level CSS overrides are needed.

### When to use
- Inside `Loading` implicitly — `Loading` renders `ProgressCircle` for its `spinner` variant.
- Directly when you need a spinner in a specific slot without the container logic of `Loading` (e.g. inside a `Button` while `isPending`, in a list row during an inline fetch, or in a `Tooltip` trigger area).
- For compact determinate arcs when a `ProgressBar` would be too wide for the available space.
- Choose size `S` for tight spaces (icon rows, table cells), `M` for card-level indicators, `L` for full-panel loading states.

### When to avoid
- When you need both a spinner and a progress percentage — use `ProgressBar` instead, which provides a label and value display.
- When the operation is better described by a static level (e.g. disk usage) — use `Meter`.
- Do not use multiple `ProgressCircle` components simultaneously on the same screen without careful consideration; multiple spinning indicators create anxiety and distract from content.
- For page-level and overlay loading patterns, prefer `Loading` which handles positioning, sizing defaults, and the Intel-branded animation variant.

## Design

`ProgressCircle` is a direct Spectrum pass-through. `ProgressCircleProps extends SpectrumProgressCircleProps` with zero additions or overrides:

```tsx
import { ProgressCircle as SpectrumProgressCircle, SpectrumProgressCircleProps } from '@adobe/react-spectrum';

export interface ProgressCircleProps extends SpectrumProgressCircleProps {}

export const ProgressCircle = (props: ProgressCircleProps) => <SpectrumProgressCircle {...props} />;
```

No state, no effects, no refs. Spectrum manages SVG arc rendering, animation, and ARIA attributes.

## Flow

```tsx
// Standard indeterminate spinner — always requires aria-label
<ProgressCircle aria-label="Loading" isIndeterminate />

// Determinate arc at 60%
<ProgressCircle aria-label="Saving" value={60} />

// Small size — for inline/tight contexts
<ProgressCircle aria-label="Loading" isIndeterminate size="S" />

// Large size — for panel-level loading states
<ProgressCircle aria-label="Loading dataset" isIndeterminate size="L" />

// Embedded in a button during a pending action
// (prefer Button's isPending prop when available; this shows the underlying primitive)
<Flex alignItems="center" gap="size-100">
  <ProgressCircle aria-label="Saving" isIndeterminate size="S" />
  <Text>Saving changes…</Text>
</Flex>

// Used by Loading internally
// <Loading mode="inline" size="M" /> renders:
<ProgressCircle aria-label="Loading..." isIndeterminate size="M" />
```

## Integration

| Concern | Detail |
|---------|--------|
| **Package** | `@adobe/react-spectrum` — `ProgressCircle` |
| **Theme** | Resolved automatically by `ThemeProvider`. No CSS imports required. |
| **Used by** | `Loading` (spinner variant) — wraps `ProgressCircle` with `isIndeterminate` in a positioned container |
| **Related** | `Loading` (positioned spinner with modes), `ProgressBar` (linear progress with label), `Meter` (static quantity level) |

All `SpectrumProgressCircleProps` are accepted as-is. The primary props are:

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `aria-label` | `string` | — | **Required.** The accessible name. There is no visible label; this is the sole name for screen readers. |
| `isIndeterminate` | `boolean` | `false` | When `true`, animates a continuous sweep. `value` is ignored. |
| `value` | `number` | `0` | Current progress value for determinate mode. Interpreted within `[minValue, maxValue]`. |
| `minValue` | `number` | `0` | Minimum value of the range. |
| `maxValue` | `number` | `100` | Maximum value of the range. |
| `size` | `'S' \| 'M' \| 'L'` | `'M'` | Controls the rendered diameter of the circle. |

## Accessibility

- **Role**: Spectrum renders `ProgressCircle` as a `<div role="progressbar">` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax` set from the numeric props. In indeterminate mode, `aria-valuenow` is omitted.
- **`aria-label` is mandatory**: `ProgressCircle` renders no visible text. Without `aria-label`, screen readers announce an unnamed progress bar — always provide a concise, task-describing label (e.g. `"Loading models"`, `"Saving changes"`, `"Uploading file"`).
- **`aria-labelledby`**: Accepted as an alternative when a nearby visible element (e.g. a heading) serves as the label. Provide the `id` of that element.
- **Indeterminate state**: When `isIndeterminate` is `true`, the element is announced as an active, in-progress indicator. Screen readers do not repeat the announcement as the animation runs.
- **Animation and motion**: The spinning arc is a CSS animation. Users who have `prefers-reduced-motion` enabled will see Spectrum's reduced-motion variant automatically (a static arc or subtler animation depending on the Spectrum version).
- **No keyboard interaction**: `ProgressCircle` is a read-only display element. It is not focusable and does not respond to keyboard or pointer events.
