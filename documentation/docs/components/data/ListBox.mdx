# ListBox

`ListBox` displays a list of options from which users can select. It is used as a standalone selection surface â€” typically inside popovers, panels, or custom pickers â€” rather than inside forms (use `Picker` or `ComboBox` for form contexts).

```tsx
import { ListBox, ListBoxItem } from '@geti/ui';
```

## Responsibility

`ListBox` and `ListBoxItem` are direct pass-throughs of React Spectrum's `ListBox` and `Item` components. The wrapper exists to keep imports consistent with the rest of Geti UI and to allow future Geti-specific defaults.

## Variants and states

| `selectionMode` | Behaviour |
|---|---|
| `none` (default) | No selection, display only |
| `single` | One item can be selected at a time |
| `multiple` | Multiple items can be selected |

## Props (deep dive)

### Content

- `children`: `ListBoxItem` elements. Each item requires a unique `key`.
- `items` + render function: for dynamic collections, pass `items` array and a render child function.

### Value / state

- `selectionMode`: `none | single | multiple`.
- `selectedKeys` / `defaultSelectedKeys`: controlled or uncontrolled selection.
- `disabledKeys`: set of keys to disable.

### Events

- `onSelectionChange(keys)`: called with the new `Selection` when the user selects an item.

### Accessibility

- Always provide `aria-label` â€” `ListBox` has no visible label by default.
- Each `ListBoxItem` announces its text content to screen readers automatically.

## Related components

- `Picker` â€” use when a single option needs to be chosen inside a form field.
- `ComboBox` â€” use when you want a searchable dropdown.
- `ListView` â€” use when each option needs richer interaction (actions, drag-and-drop).
- `TagGroup` â€” use when selected items should be displayed as removable tags.
