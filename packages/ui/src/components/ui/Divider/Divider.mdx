## Responsibility

The `Divider` component is a simple but essential visual aid for organizing content. It helps users distinguish between different sections of a page, group related items in a menu, or provide a visual break in a dense layout.

Key responsibilities:
- **Visual Separation**: Clearly marks the boundaries between content sections.
- **Contextual Grouping**: Helps group related elements in menus, toolbars, or lists.
- **Layout Consistency**: Maintains a standard visual style for separations across the application.

### When to use
- To separate logical sections of a page (e.g., between a header and body).
- Between items in a list or menu to group them into sub-categories.
- In toolbars to separate different functional groups of actions.

### When to avoid
- When natural white space or background color changes are sufficient to distinguish sections.
- For purely decorative lines that don't represent a logical separation.
- Inside simple components that already have built-in borders or spacing.

## Variants

### Orientation
- **Horizontal (Default)**: Used to separate content stacked vertically (e.g., in a column layout).
- **Vertical**: Used to separate content arranged horizontally (e.g., in a row layout or toolbar).

### Size (Thickness)
The `size` prop controls the thickness of the divider line:
- `S`: 1px (Thinnest)
- `M`: 2px
- `L`: 4px (Thickest, default in Geti UI)

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `orientation` | `'horizontal' \| 'vertical'` | `'horizontal'` | The axis the divider aligns to. |
| `size` | `'S' \| 'M' \| 'L'` | `'L'` | Controls the thickness of the divider line. |

## Accessibility

- **Roles**: Renders as an `<hr>` element with `role="separator"`.
- **Aria Hidden**: If the divider is purely decorative and does not represent a meaningful separation in the content structure, it should be marked with `aria-hidden="true"`.
- **Labeling**: For complex layouts, an `aria-label` can be added to the divider to describe the nature of the separation for screen reader users.

## Spectrum 2 Migration

The migration to Spectrum 2 (S2) involves minor prop changes and a shift in styling methodology.

### Key Changes
- **Default Size**: In S2, the default size changes from `L` to `M`.
- **Static Color**: A new `staticColor` prop (`'white' \| 'black' \| 'auto'`) is introduced for use over backgrounds where the theme color might not have enough contrast.
- **Styles Macro**: Layout props (like `margin` or `flex`) are replaced by the S2 `styles` prop and `style()` macro.
- **Dialog Redesign**: S2 Dialogs no longer require an explicit `Divider` component; the layout is handled automatically by the `Dialog` itself.

### Migration Example

**V3 Style (Current):**
```tsx
<Flex direction="column" gap="size-200">
  <Header>Settings</Header>
  <Divider size="L" />
  <Content>General settings content...</Content>
</Flex>
```

**S2 Style (Future):**
```tsx
import { style } from '@react-spectrum/s2/style' with { type: 'macro' };

<div className={style({ display: 'flex', flexDirection: 'column', gap: 16 })}>
  <Header>Settings</Header>
  <Divider size="M" />
  <Content>General settings content...</Content>
</div>
```
