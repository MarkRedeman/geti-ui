# Loading

`Loading` renders a progress spinner (or the Intel branded animation) in one of three layout modes: inline within a container, as a full-screen overlay, or as an absolutely-positioned overlay over a parent. Use it consistently for all loading states across Geti.

```tsx
import { Loading } from '@geti-ai/ui';
```

- [Geti UI Theming](/components/theming)

## Examples

### Inline spinner (medium)

```tsx preview
import { Loading } from '@geti-ai/ui';

export default function App() {
  return <Loading mode="inline" size="M" />;
}
```

### Inline spinner (small)

```tsx preview
import { Loading } from '@geti-ai/ui';

export default function App() {
  return <Loading mode="inline" size="S" />;
}
```

### Inline spinner (large)

```tsx preview
import { Loading } from '@geti-ai/ui';

export default function App() {
  return <Loading mode="inline" size="L" />;
}
```

### Intel branded animation

```tsx preview
import { Loading } from '@geti-ai/ui';

export default function App() {
  return <Loading variant="intel" mode="inline" size="M" />;
}
```

### Overlay over a container

```tsx preview
import { Loading } from '@geti-ai/ui';

export default function App() {
  return (
    <div style={{ position: 'relative', width: 300, height: 150, border: '1px dashed #555' }}>
      <p style={{ padding: 16 }}>Content underneath</p>
      <Loading mode="overlay" style={{ backgroundColor: 'rgba(0,0,0,0.5)' }} />
    </div>
  );
}
```

## Responsibility

`Loading` is a Geti-authored component with no direct Spectrum equivalent. It provides:

- a unified entry point for all loading states so the codebase uses one component everywhere,
- three layout modes (inline, overlay, fullscreen) without requiring consumers to write positioning CSS,
- the Intel branded animation as an alternative to the default spinner.

The spinner itself is Geti's `ProgressCircle` (wrapped Spectrum `ProgressCircle`).

## Variants and states

| `mode` | Layout |
|---|---|
| `inline` | Centered flex inside its natural document flow position |
| `overlay` | `position: absolute; inset: 0` — covers nearest positioned ancestor |
| `fullscreen` (default) | `position: fixed; inset: 0` — covers the entire viewport |

| `variant` | Visual |
|---|---|
| `spinner` (default) | Adobe Spectrum progress circle |
| `intel` | Intel branded animated WebP |

| `size` | Diameter |
|---|---|
| `S` | 24 px (Intel: 24 px) |
| `M` | Standard medium |
| `L` (default) | Large — 192 px for Intel variant |

## Props (deep dive)

### Display

- `mode`: layout mode — `inline | overlay | fullscreen`.
- `variant`: spinner style — `spinner | intel`.
- `size`: `S | M | L`.

### Customisation

- `style`: additional inline CSS applied to the wrapper `<div>`.
- `className`: additional class names on the wrapper.

### Accessibility

- The spinner renders `<ProgressCircle aria-label="Loading..." isIndeterminate />`.
- The Intel variant renders `<img role="progressbar" alt="Loading" />`.
- For `overlay` / `fullscreen` modes, consider also setting `aria-busy="true"` on the blocked region.
