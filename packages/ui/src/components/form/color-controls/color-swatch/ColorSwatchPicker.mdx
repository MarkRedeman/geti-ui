# ColorSwatchPicker

<Subtitle>An interactive selection group that lets users choose a color from a predefined palette of swatches.</Subtitle>

```tsx
import { ColorSwatchPicker, ColorSwatchPickerItem } from '@geti/ui';
```

## Responsibility

`ColorSwatchPicker` is a thin wrapper around Adobe React Spectrum's `ColorSwatchPicker`. It manages the selection state for a group of color swatches and calls `onChange` with the newly chosen `Color` when the user selects one. Each swatch in the group is represented by a `ColorSwatchPickerItem`.

Key responsibilities:
- **Selection management**: Tracks which swatch is currently selected, applies a selection ring to it, and fires `onChange(Color)` on selection.
- **Palette composition**: Accepts `ColorSwatchPickerItem` children, each representing one color option in the set.
- **Layout control**: `density` (`compact`, `regular`, `spacious`) controls the gap between swatches; `size` and `rounding` propagate to all child swatches.
- **Controlled or uncontrolled**: Supports both a `value` prop (controlled) and `defaultValue` (uncontrolled) to integrate with either React state or HTML form submission.

### When to use
- When presenting a predefined, finite set of color options (brand palette, annotation colors, label colors).
- When the color options are curated and known ahead of time — the user picks from the set rather than constructing a color.
- As the "Presets" section of a larger color picker UI (e.g., inside `ColorPickerDialog`) for quick access to common colors.
- In any form where a color choice should be presented as a constrained selection rather than a free-form input.

### When to avoid
- When users need arbitrary color freedom — use `ColorPickerDialog` (which includes `ColorArea`, `ColorSlider`, and `ColorField`).
- When there is only one color to display with no selection intent — use `ColorSwatch` (the static display element) instead.
- When the palette is very large (> ~20 colors) — a `Picker` dropdown with color names may be more navigable than a wide swatch grid.

---

## Variants

### Default
A horizontal row of `ColorSwatchPickerItem` children. The selected item receives a visible focus ring and selection indicator.

### Density
Control the gap between swatches:
- `compact` — tight spacing, suitable for toolbars or embedded panels.
- `regular` — default spacing.
- `spacious` — wider gaps, suited to standalone palette panels.

### Size
`S`, `M`, `L` — propagated to all children. Use `S` in dense UI contexts; `L` for touch-primary surfaces.

### Rounding
`none`, `default`, `full` — propagated to all children. `full` gives a classic palette-circle aesthetic.

## Design

`ColorSwatchPicker` is a pure pass-through wrapper with an explicit interface extension.

`ColorSwatchPickerItem` wraps Spectrum's `ColorSwatch` (not `SpectrumColorSwatchPickerItem`), because in React Spectrum v3 the items inside a picker are just styled `ColorSwatch` elements — the picker container manages the selection role.

```tsx
export interface ColorSwatchPickerProps extends SpectrumColorSwatchPickerProps {}

export const ColorSwatchPicker = (props: ColorSwatchPickerProps) => (
  <SpectrumColorSwatchPicker {...props} />
);

// Items are ColorSwatch elements — the picker handles selection semantics
export interface ColorSwatchPickerItemProps extends SpectrumColorSwatchProps {}

export const ColorSwatchPickerItem = (props: ColorSwatchPickerItemProps) => (
  <SpectrumColorSwatch {...props} />
);
```

Typical controlled usage:

```tsx
import { ColorSwatchPicker, ColorSwatchPickerItem } from '@geti/ui';
import { parseColor, Color } from '@adobe/react-spectrum';

const [selectedColor, setSelectedColor] = useState<Color>(parseColor('#ff0000'));

<ColorSwatchPicker
  aria-label="Annotation color"
  value={selectedColor}
  onChange={setSelectedColor}
  size="M"
  rounding="full"
  density="regular"
>
  <ColorSwatchPickerItem color="#ff0000" aria-label="Red" />
  <ColorSwatchPickerItem color="#00c851" aria-label="Green" />
  <ColorSwatchPickerItem color="#007bff" aria-label="Blue" />
  <ColorSwatchPickerItem color="#ffbb33" aria-label="Yellow" />
  <ColorSwatchPickerItem color="#ff4444" aria-label="Orange" />
</ColorSwatchPicker>
```

Use inside `ColorPickerDialog` as a presets row:

```tsx
<ColorSwatchPicker value={color} onChange={setColor}>
  <ColorSwatch color="#ff0000" aria-label="Red" />
  <ColorSwatch color="#00ff00" aria-label="Green" />
  <ColorSwatch color="#0000ff" aria-label="Blue" />
</ColorSwatchPicker>
```

> **Note**: Inside `ColorPickerDialog`, raw `ColorSwatch` elements (rather than `ColorSwatchPickerItem`) are used as children — both work because `ColorSwatchPickerItem` is itself a `ColorSwatch` wrapper.

## Flow

1. `ColorSwatchPicker` renders a list of `ColorSwatchPickerItem` (or `ColorSwatch`) children.
2. The currently selected swatch (matched by color value against `value`) receives a visible selection ring.
3. User clicks on a swatch — or navigates with `←` / `→` arrow keys and presses `Enter` / `Space`.
4. `onChange(Color)` fires with the `Color` object of the chosen swatch.
5. The parent updates its state; the selection ring moves to the newly selected swatch.

## Integration

- **Depends on**: `@adobe/react-spectrum` (`ColorSwatchPicker`, `ColorSwatch`, `SpectrumColorSwatchPickerProps`, `SpectrumColorSwatchProps`)
- **Color type**: `value` and `onChange` use Adobe's `Color` object. Use `parseColor(hexString)` from `@adobe/react-spectrum` to convert a hex or CSS string to `Color`. Call `.toString('hex')` on the result to convert back.
- **Children**: Pass `ColorSwatchPickerItem` (from `@geti/ui`) or raw `ColorSwatch` elements. Both work — `ColorSwatchPickerItem` is the idiomatic choice; `ColorSwatch` can be used directly as seen in `ColorPickerDialog`.
- **Composed into**: `ColorPickerDialog` uses `ColorSwatchPicker` as its "Presets" section. Pair it with `ColorArea`, `ColorSlider`, and `ColorWheel` for a full picker.
- **Contrast with ColorSwatch**: `ColorSwatch` alone is a static display element. `ColorSwatchPicker` wraps a group of them in a managed selection container.

## Accessibility

`ColorSwatchPicker` implements the [ARIA radiogroup pattern](https://www.w3.org/WAI/ARIA/apg/patterns/radio/), treating each swatch as a radio button within the group.

- **Group role**: The picker container has `role="radiogroup"` with its accessible name provided by `aria-label` (or `aria-labelledby`). **Always provide `aria-label`** — without it, the group is announced with no context.
- **Item roles**: Each `ColorSwatchPickerItem` has `role="radio"` with `aria-checked="true"` on the selected swatch and `aria-checked="false"` on all others.
- **Color labels**: Each `ColorSwatchPickerItem` must have `aria-label` with the color name (e.g., `"Red"`, `"Blue"`). Without it, screen readers cannot communicate which color option the user is focused on — color alone is never sufficient.
- **Selection announcement**: When the user moves to a swatch, the screen reader announces its `aria-label` and whether it is selected (e.g., "Red, radio button, checked").
- **Keyboard navigation**:

| Key | Behaviour |
|-----|-----------|
| `Tab` | Move focus into the picker group (lands on the selected item, or the first item if none selected) |
| `←` / `↑` | Move focus to the previous swatch in the group |
| `→` / `↓` | Move focus to the next swatch in the group |
| `Enter` / `Space` | Select the currently focused swatch |

- **Single tab stop**: The picker group is a single tab stop — internal navigation uses arrow keys, not `Tab`. This follows standard `radiogroup` convention and prevents users from having to tab through every color option.
- **Focus management**: Focus stays within the group until `Tab` is pressed to move to the next focusable element in the page.
