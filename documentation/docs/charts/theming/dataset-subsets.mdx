# Dataset subsets palette

Use the built-in dataset subset palette utilities to keep **training / validation / testing** colors consistent across charts.

Default mapping:

- `train` → green
- `validation` / `val` → red
- `test` → yellow

## Basic usage

```tsx
import { withDatasetSubsetPalette } from '@geti-ai/charts';

const series = withDatasetSubsetPalette([
  { dataKey: 'train', name: 'Training set' },
  { dataKey: 'val', name: 'Validation set' },
  { dataKey: 'test', name: 'Test set' },
]);
```

## Override palette and aliases

```tsx
import { withDatasetSubsetPalette } from '@geti-ai/charts';

const series = withDatasetSubsetPalette(
  [
    { dataKey: 'training' },
    { dataKey: 'validation' },
    { dataKey: 'testing' },
  ],
  {
    palette: {
      train: '#22c55e',
      validation: '#ef4444',
      test: '#f59e0b',
    },
    aliases: {
      train: ['training'],
      validation: ['validation'],
      test: ['testing'],
    },
  }
);
```

## Pie chart example

For pie-like charts (single series), use `getDatasetSubsetColor` directly per slice:

```tsx preview
import { PieChart, getDatasetSubsetColor } from '@geti-ai/charts';

const data = [
  { name: 'Train', value: 70 },
  { name: 'Validation', value: 20 },
  { name: 'Test', value: 10 },
];

export default function App() {
  return (
    <PieChart
      data={data}
      colors={[
        getDatasetSubsetColor('train')!,
        getDatasetSubsetColor('validation')!,
        getDatasetSubsetColor('test')!,
      ]}
      showLabels
      aria-label="Dataset subsets as pie slices"
      height={320}
    />
  );
}
```


## Scatter / bubble chart example

```tsx preview
import { ScatterChart, withDatasetSubsetPalette } from '@geti-ai/charts';

const train = [
  { x: 2.8e6, y: 0.47, z: 9.8 },
  { x: 1.1e7, y: 0.54, z: 27.0 },
];
const val = [
  { x: 2.8e6, y: 0.44, z: 9.8 },
  { x: 1.1e7, y: 0.50, z: 27.0 },
];
const test = [
  { x: 2.8e6, y: 0.42, z: 9.8 },
  { x: 1.1e7, y: 0.48, z: 27.0 },
];

export default function App() {
  const series = withDatasetSubsetPalette(
    [
      { name: 'Train', data: train, zKey: 'z' },
      { name: 'Validation', data: val, zKey: 'z' },
      { name: 'Test', data: test, zKey: 'z' },
    ],
    { matchBy: 'name' }
  );

  return (
    <ScatterChart
      series={series}
      zAxisProps={{ dataKey: 'z', range: [80, 320], name: 'GFLOPs' }}
      showLegend
      xScale={{ scale: 'log', domain: [1e6, 'auto'] }}
      xAxisLabel="Parameters"
      yAxisLabel="mAP @ 50:95"
      aria-label="Train/validation/test model comparison"
      height={320}
    />
  );
}
```

```tsx preview
      height={320}

```

## Radar / radial / meter / treemap

For the upcoming charts, use the same `withDatasetSubsetPalette` or `getDatasetSubsetColor` patterns based on whether the chart API is multi-series (`withDatasetSubsetPalette`) or single-series slices (`getDatasetSubsetColor`).

## Radar chart example

```tsx preview
import { RadarChart, withDatasetSubsetPalette } from '@geti-ai/charts';

const data = [
  { metric: 'mAP', train: 0.88, val: 0.82, test: 0.80 },
  { metric: 'Recall', train: 0.91, val: 0.86, test: 0.84 },
  { metric: 'Precision', train: 0.89, val: 0.84, test: 0.83 },
  { metric: 'Stability', train: 0.76, val: 0.71, test: 0.70 },
  { metric: 'Latency score', train: 0.66, val: 0.69, test: 0.68 },
];

export default function App() {
  const series = withDatasetSubsetPalette([
    { dataKey: 'train', name: 'Training', fillOpacity: 0.2 },
    { dataKey: 'val', name: 'Validation', fillOpacity: 0.2 },
    { dataKey: 'test', name: 'Test', fillOpacity: 0.2 },
  ]);

  return (
    <RadarChart
      data={data}
      categoryKey="metric"
      series={series}
      aria-label="Dataset subset radar comparison"
      height={360}
    />
  );
}
```
