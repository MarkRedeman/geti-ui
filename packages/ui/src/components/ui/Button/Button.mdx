## Responsibility

The `Button` component is a lightweight wrapper around Adobe's React Spectrum `Button`. It enforces Geti-specific defaults, handles ref forwarding, and ensures a clean integration with the design system.

Key refinements include:
- **Simplified Variants**: Legacy Spectrum variants like `cta` and `overBackground` are removed.
- **Accent Default**: The component defaults to the `accent` variant (blue), which is the primary Geti call-to-action color.
- **Ref Forwarding**: Supports `FocusableRef<HTMLElement>` for precise focus management.

---

## Variants

### Styles
Buttons can be rendered with a solid `fill` or an `outline` style. The default is `fill`.

### Semantic Variants

| Variant | Use Case | Visual Weight |
|---------|----------|---------------|
| `accent` (Default) | **Primary CTA**. Use for the most important action on a page (e.g., "Save", "Create"). | High |
| `primary` | Secondary important actions that shouldn't compete with the main CTA. | Medium |
| `secondary` | Tertiary actions or cancel buttons. Often paired with `style="outline"`. | Low |
| `negative` | **Destructive actions**. Use for delete or remove operations. | High (Warning) |

## Props

<Controls of={DocStories.Default} />

### Content

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | The label text or content to display inside the button. |
| `elementType` | `string \| ElementType` | `'button'` | The HTML element to render (e.g., `'a'`, `'button'`). |
| `href` | `string` | — | A URL to link to if `elementType="a"`. |

### Value / State

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `'accent' \| 'primary' \| 'secondary' \| 'negative'` | `'accent'` | The visual style of the button. |
| `style` | `'fill' \| 'outline'` | `'fill'` | The background/border style of the button. |
| `isDisabled` | `boolean` | `false` | Whether the button is disabled. |
| `isPending` | `boolean` | `false` | Shows a loading spinner and disables interaction. |

### Events

| Prop | Type | Description |
|------|------|-------------|
| `onPress` | `(e: PressEvent) => void` | Handler called when the button is pressed (click/touch/keyboard). |
| `onPressStart` | `(e: PressEvent) => void` | Handler called when a press interaction starts. |
| `onPressEnd` | `(e: PressEvent) => void` | Handler called when a press interaction ends. |

### Styles

The `Button` component accepts all standard [Spectrum layout and spacing props](https://react-spectrum.adobe.com/react-spectrum/styling.html), including `margin`, `position`, `flex`, `width`, and `height`.

## Accessibility

- **Labels**: Ensure buttons have a clear, descriptive label. If using an icon-only button, provide an `aria-label`.
- **Keyboard Navigation**: Buttons are fully accessible via the keyboard (Space and Enter keys).
- **Focus Management**: Use the forwarded `ref` to manage focus programmatically when needed (e.g., returning focus after a dialog closes).

---

## Spectrum 2 Migration

When migrating to Spectrum 2 (`@react-spectrum/s2`), be aware of the following changes:

- **`style` → `fillStyle`**: The `style` prop is renamed to `fillStyle` in S2.
- **`size` Prop**: S2 introduces a `size` prop (`'S' \| 'M' \| 'L' \| 'XL'`).
- **Styles Macro**: Layout and spacing props are consolidated into a new `styles` macro system.
- **New Variants**: S2 adds `premium` (gold) and `genai` (gradient) variants.
