## Responsibility

`PhotoPlaceholder` fills a functional gap in the Adobe Spectrum design system: providing a high-quality, initials-based fallback for missing images. While Spectrum's `Avatar` component is strictly image-only, `PhotoPlaceholder` ensures every entity has a distinct visual identity even without a profile photo.

### Key Capabilities
- **Deterministic Color**: Background colors are hashed from an `indicator` string (e.g., email or ID), ensuring the same user always has the same color.
- **Initials Rendering**: Automatically extracts and displays the first initial from the `name` prop.
- **Smart Contrast**: Dynamically calculates whether to use light or dark text based on the background color's perceived luminance, ensuring WCAG-compliant contrast.
- **Flexible Shape**: Defaults to a circular "avatar" style but supports custom `borderRadius` for square or rounded-rect variants.

---

## Variants

### Sizing
The component supports standard Spectrum dimension tokens for `width` and `height`. The default size is `size-1600` (64px).

### Custom Shape
Use the `borderRadius` prop to change the shape from the default circle (`50%`) to a rounded square or other geometric form.

### Content

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `name` | `string` | **Required** | The name of the person or entity. Extracts the first letter for display. |
| `indicator` | `string` | **Required** | A unique string (email, ID) used to deterministically pick a background color. |

### Value / Style overrides

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `width` | `string \| number` | `'size-1600'` | Spectrum dimension token or numeric pixel value for width. |

| `borderRadius` | `string` | `'50%'` | CSS border-radius value (e.g., `'8px'`, `'100%'`). |

## Implementation Details

### Color Pipeline
The component uses a perceptional luminance formula (ITU-R 601 weights) to ensure text readability:
1. **Hash**: The `indicator` is hashed to a hex color.
2. **Luminance**: `(r*299 + g*587 + b*114) / 1000`.
3. **Threshold**: If luminance > 128, use dark text; otherwise, use light text.

### Layout
Static centering logic (Flexbox) is handled in `PhotoPlaceholder.module.css`, while dynamic colors and radius are applied via `UNSAFE_style` on a Spectrum `View`.

---

## Accessibility

- **Role**: The component uses `role="img"` on the container to identify as a single visual entity.
- **Label**: The `aria-label` is set to the full `name` (or `indicator`), ensuring screen readers announce the person's name rather than just the initial.
- **Hiding Text**: The visible initial character is marked with `aria-hidden="true"` to prevent redundant announcements.

---

## Spectrum 2 Migration

`PhotoPlaceholder` is a standalone Geti component and does not wrap any Spectrum logic that is changing in Spectrum 2. It is expected to remain a custom component throughout all migration phases.

### Future Path
In Phase 3 (Tailwind CSS), the component can be simplified to a pure HTML/Tailwind implementation:
