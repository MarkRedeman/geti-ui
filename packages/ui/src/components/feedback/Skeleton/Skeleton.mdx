## Responsibility

Renders an animated placeholder block during content load, communicating to users that layout is reserved and content is incoming. Supports rectangular and circular shapes, and can be constrained to a 1:1 aspect ratio for image placeholders.

## Design

Zero Spectrum dependency — this is a fully custom component using CSS-in-JS only.

Key patterns:
- **Shimmer animation** — `background: linear-gradient(...)` with `animation: 'shimmer 2s ease infinite'`. The `shimmer` keyframe is defined externally in `geti-global.module.css` (a global stylesheet), not co-located here. The component references the animation name as a plain string.
- **Shape control** — `isCircle?: boolean` toggles `borderRadius: '50%'` in the inline style object. Defaults to rectangular.
- **Aspect ratio** — `isAspectRatioOne?: boolean` sets `aspectRatio: '1'`, useful for square image skeletons without needing to specify both `width` and `height`.
- **ARIA** — renders `<span role="img" aria-label="Loading…" aria-busy="true">`, satisfying accessibility requirements without a visible text label.
- **Sizing** — consumers control size entirely through `width`, `height`, and optional `style` props; the component does not impose dimensions.

## Flow

```
props { width, height, isCircle, isAspectRatioOne, style, ...rest }
  → compute spanStyle { width, height, borderRadius?, aspectRatio?, shimmer animation, ... }
  → <span role="img" aria-label="Loading…" aria-busy="true" style={spanStyle} {...rest} />
```

No state, no effects, no refs.

## Integration

- `shimmer` keyframe must be present globally (defined in `geti-global.module.css`) for the animation to work; the Skeleton component itself does not import that stylesheet.
- Used in list/grid loading states alongside `Loading` (which handles the spinner/intel variants for indeterminate full-area loading).
- No Spectrum peer dependency — can be used outside of `ThemeProvider` if needed.
