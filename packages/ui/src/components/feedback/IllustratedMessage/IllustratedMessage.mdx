# IllustratedMessage

A layout component for empty states, error messages, or onboarding information.

```tsx
import { IllustratedMessage, Heading, Content } from '@geti/ui';
```

- [React Spectrum IllustratedMessage](https://react-spectrum.adobe.com/react-spectrum/IllustratedMessage.html)

<Stories />

## Responsibility

`IllustratedMessage` is a specialized layout component designed to fill a large, prominent empty area. It typically consists of a decorative illustration, a heading (title), and body content (description).

### When to use IllustratedMessage

- **Empty States**: Filling a large empty table, list, or canvas when no results are found.
- **Section-level Errors**: Displaying full-page or section-level errors like a 404 or 500 page.
- **First-use Onboarding**: Prompting a user to take an initial action, such as "Create your first dataset."
- **Drag-and-Drop Targets**: Providing a visual target for file uploads.

### Comparison with other components

| Scenario                              | Use instead              |
| :------------------------------------ | :----------------------- |
| Brief, dismissible status message     | `Toast` or `InlineAlert` |
| Warning or error inline within a form | `InlineAlert`            |
| Destructive confirmation with action  | `AlertDialog`            |

### Usage Rules

- **Always provide a Heading**: The heading acts as the accessible name of the message for screen readers.
- **Spectrum Illustrations**: Use standard Adobe Spectrum illustrations for visual consistency.
- **Not for inline validation**: Use field-level error messages for form validation instead of `IllustratedMessage`.

## Design

`IllustratedMessage` is a three-slot layout component that centers its children vertically and horizontally.

1. **Illustration Slot**: The top-most element, typically an SVG or image representing the state.
2. **Heading Slot**: A `Heading` component (renders as `<h3>` by default) that provides a clear title.
3. **Content Slot**: A `Content` component for detailed description and optional action buttons.

The layout is responsive and centers content within its parent container, making it ideal for full-page or full-section empty states.

## Integration

### Paired with DropZone

A common pattern for file uploads is to place an `IllustratedMessage` inside a `DropZone`.

```tsx
<DropZone onDrop={handleDrop}>
    <IllustratedMessage>
        <FileIcon />
        <Heading>Drop files here</Heading>
        <Content>Or click to browse from your computer</Content>
    </IllustratedMessage>
</DropZone>
```

### Empty List State

Use `IllustratedMessage` as the `renderEmptyState` prop in collection components.

```tsx
<ListView
    items={items}
    renderEmptyState={() => (
        <IllustratedMessage>
            <NoResultsIllustration />
            <Heading>No items found</Heading>
            <Content>Try adjusting your search or filters.</Content>
        </IllustratedMessage>
    )}
>
    {/* List items */}
</ListView>
```

| Prop                 | Type        | Default | Description                                                 |
| :------------------- | :---------- | :------ | :---------------------------------------------------------- |
| `children`           | `ReactNode` | —       | The illustration, `Heading`, and `Content` elements.        |
| `UNSAFE_className`   | `string`    | —       | CSS class name to apply to the container.                   |
| Spectrum style props | —           | —       | Supports standard layout props like `margin`, `width`, etc. |

## Accessibility

- `IllustratedMessage` renders as a generic container with no specific landmark role by default.
- **Accessible Name**: The label is derived from the `Heading` child. If a `Heading` is omitted, the illustration itself must have an `aria-label`.
- **Decorative Illustrations**: When a `Heading` is present, the illustration should be marked as decorative (`role="presentation"` and `aria-hidden="true"`) to avoid duplicate announcements.
- If the message represents a live change in state (e.g., a search results list that just became empty), wrap the component in an `aria-live="polite"` region.

## Related components

- **`InlineAlert`** — use for compact, in-flow status messages (info, warning, error) that don't require a large illustration; `IllustratedMessage` is for prominent empty-state areas.
- **`Toast`** — use for transient, auto-dismissing event notifications rather than persistent empty or error states.
- **`Loading`** — use alongside `IllustratedMessage` to replace the illustrated placeholder while data is being fetched, then reveal the message if the result is empty.
- **`ListView` / `CardView`** — these collection components accept an `IllustratedMessage` via `renderEmptyState` for their built-in empty state slot.

## Example use cases

<details>
  <summary>Typical use cases</summary>

- Filling an empty `ListView` when a project has no datasets yet — illustration + "No datasets" heading + "Create your first dataset" action button.
- Displaying a section-level 404 when a requested model is not found — illustration + "Model not found" heading + "Back to models" link.
- Providing a drag-and-drop upload target inside a `DropZone` — upload illustration + "Drop images here" heading + "Or click to browse" body.
- Showing an empty search results state in an image gallery when no annotations match the active filters.
- Presenting a first-use onboarding prompt when a workspace has no projects — illustration + "Welcome to Geti" heading + "Create a project" call-to-action.
