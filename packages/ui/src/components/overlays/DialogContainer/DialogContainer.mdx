# DialogContainer

A controlled dialog host that renders a modal independently of any trigger element ‚Äî the correct choice when the trigger may unmount while the dialog is still open, or when dialogs are spawned programmatically.

```tsx
import { DialogContainer } from '@geti-ai/ui';
```

- [React Spectrum DialogContainer](https://react-spectrum.adobe.com/react-spectrum/DialogContainer.html)

<Stories />

## Responsibility

`DialogContainer` solves the specific architectural problem of showing a Spectrum `Dialog` when **the trigger element cannot or should not remain in the DOM** while the dialog is open.

Key responsibilities:

- **Trigger-independent lifecycle**: Manages the modal overlay and focus trap without requiring a persistent trigger `<button>` in the component tree.
- **Controlled API**: Enforces an always-controlled pattern ‚Äî `isOpen` and `onDismiss` are required, making the dialog's open state explicit and testable.
- **Spectrum-native children**: Accepts any `Dialog` or `AlertDialog` as its child, inheriting the full Spectrum modal experience (title, divider, action buttons, animation, scrim).

### When to use

- When the trigger element is **transient** ‚Äî e.g., a row in a table or list that can be deleted, a canvas element that can be unmounted, or a context menu item that closes before the dialog opens.
- When a dialog is opened **programmatically** ‚Äî from a keyboard shortcut, a Redux/state action, a right-click context menu, or a canvas event ‚Äî where no button element exists as a natural trigger.
- When the trigger and dialog are in **different parts of the component tree** and wiring a `DialogTrigger` ref would be impractical.

### When to avoid

- When a **stable, persistent trigger button** exists ‚Äî use `DialogTrigger` instead, which pairs the trigger and dialog in a single unit and manages focus return automatically.
- For non-modal floating panels or popovers ‚Äî use `Popover` or `CustomPopover`.
- For tooltip or contextual-help overlays ‚Äî use `ContextualHelp`.

## Design

Thin wrapper ‚Äî `DialogContainerProps extends SpectrumDialogContainerProps`. The component body is `(props) => <SpectrumDialogContainer {...props} />`.

The critical design difference from `DialogTrigger`: `DialogContainer` is **always controlled**. `isOpen` and `onDismiss` govern the lifecycle entirely. Spectrum's `DialogContainer` handles mounting/unmounting the overlay, the scrim, focus trapping, and restore-focus-on-close without any trigger element involvement.

```tsx
const [isOpen, setIsOpen] = useState(false);

// Opened from a context menu item, keyboard shortcut, or programmatic action
<button onClick={() => setIsOpen(true)}>Open dialog</button>

<DialogContainer isOpen={isOpen} onDismiss={() => setIsOpen(false)}>
  <Dialog>
    <Heading>Confirm deletion</Heading>
    <Divider />
    <Content>Are you sure you want to delete this item?</Content>
    <ButtonGroup>
      <Button variant="secondary" onPress={() => setIsOpen(false)}>Cancel</Button>
      <Button variant="negative" onPress={handleDelete}>Delete</Button>
    </ButtonGroup>
  </Dialog>
</DialogContainer>
```

## Integration

- Used when the trigger is transient (disappears after click) or when dialogs are opened programmatically (e.g., from a context menu, keyboard shortcut, or Redux action).
- Use **`DialogTrigger`** instead when a stable trigger element (button) is present and persistent ‚Äî `DialogTrigger` provides tighter pairing between trigger and dialog, and a simpler API.
- Accepts the same `Dialog` and `AlertDialog` children as `DialogTrigger`.
- `onDismiss` is called for **all** dismissal paths: `Escape` key, scrim click, and any internal dialog close action. Ensure your state update (`setIsOpen(false)`) handles all these paths uniformly.
- When rendering an `AlertDialog`, set `variant` on `AlertDialog` (not on `DialogContainer`) to control the icon and colour scheme.

## Props

| Prop            | Type                                              | Default   | Description                                                                                              |
| --------------- | ------------------------------------------------- | --------- | -------------------------------------------------------------------------------------------------------- |
| `children`      | `ReactNode`                                       | ‚Äî         | **Required.** A single `Dialog` or `AlertDialog`.                                                        |
| `isOpen`        | `boolean`                                         | ‚Äî         | **Required.** Whether the dialog is currently open.                                                      |
| `onDismiss`     | `() => void`                                      | ‚Äî         | **Required.** Called when the user dismisses the dialog (Escape, scrim click, or internal close action). |
| `isDismissable` | `boolean`                                         | `false`   | Whether the dialog can be dismissed by clicking the scrim or pressing Escape.                            |
| `type`          | `'modal' \| 'fullscreen' \| 'fullscreenTakeover'` | `'modal'` | The visual presentation of the dialog overlay.                                                           |

## Accessibility

- **Focus trap**: Spectrum automatically traps keyboard focus within the open dialog. Users cannot Tab out of the dialog while it is open.
- **Focus restore**: When the dialog closes, Spectrum restores focus to the element that was focused before the dialog opened. Because `DialogContainer` has no built-in trigger element, Spectrum uses the **last focused element in the document** as the restore target. If your programmatic trigger (e.g., a button that triggered an action) is unmounted, ensure a sensible fallback element remains in the DOM to receive restored focus.
- **`Escape` key**: Dismisses the dialog and calls `onDismiss` (when `isDismissable` is `true` or the dialog type permits it). Spectrum handles this automatically.
- **Scrim click**: For `isDismissable` dialogs, clicking the backdrop calls `onDismiss`. Modal dialogs (`isDismissable={false}`) require an explicit in-dialog close action.
- **`role="dialog"`**: Spectrum renders the dialog with `role="dialog"` and `aria-modal="true"`. Ensure the `Dialog` child has a visible `Heading` element ‚Äî Spectrum wires `aria-labelledby` from the dialog to its `Heading` automatically, giving screen readers an accessible name for the modal.
- **Screen reader announcements**: When the dialog opens, screen readers announce the dialog name (from the `Heading`) and move their virtual cursor inside the modal. When it closes, they return to the trigger context.

## Related components

- **`DialogTrigger`** ‚Äî use when a stable trigger button exists in the component tree; `DialogContainer` is specifically for trigger-less or transient-trigger scenarios.
- **`AlertDialog`** ‚Äî pass as the child of `DialogContainer` for fixed-structure confirmation prompts; the container manages the overlay lifecycle while `AlertDialog` defines the content.
- **`Dialog`** ‚Äî pass as the child when a custom-slotted dialog layout (headings, forms, footers) is required.
- **`Popover`** ‚Äî use for non-blocking overlays that do not need to trap focus or block interaction with the rest of the UI.

## Example use cases

<details>
  <summary>Typical use cases</summary>

- **Context-menu action dialog**: A right-click context menu item triggers a delete confirmation; the context menu closes before the dialog opens, making `DialogContainer` the correct choice over `DialogTrigger`.
- **Keyboard shortcut dialog**: A global `Cmd+K` / `Ctrl+K` handler opens a search or command palette modal without any button trigger in the component tree.
- **Redux / state-machine driven dialog**: A saga or state action sets `isOpen: true` to show a progress or error dialog from outside any React subtree that contains a trigger element.
- **Row-action dialog in a table**: Clicking "Delete" in a table row opens a `DialogContainer` confirmation; the row may be removed from the DOM before the dialog closes.
- **Canvas event dialog**: A double-click on a canvas annotation opens an edit dialog; `triggerRef` is unavailable and state is managed at the canvas controller level.

</details>
