## Responsibility

The `Card` component is a fundamental building block of the Geti UI. It provides a bordered, themed container that can be either static (for informational content) or interactive (acting as a large button). It is the primary way to represent assets, projects, or any item collection in a grid or list.

Key responsibilities:
- **Visual Containment**: Groups related content (images, text, actions) into a distinct visual unit.
- **Selection State**: Provides built-in visual feedback for selection using theme-aware borders and background colors.
- **Semantic Branching**: Automatically switches between `<article>` (static) and `<button>` (interactive) based on the presence of an `onPress` handler.
- **Theme Integration**: Uses Spectrum's design tokens to ensure it looks correct in both dark and light modes.

### When to use
- To represent individual items in a collection (e.g., a project list, a dataset gallery).
- To group related information that requires a clear visual boundary and a "surface" metaphor.
- When an entire section of content needs to be clickable or selectable.

### When to avoid
- For simple grouping without the need for a border or background (use `Flex` or `Grid` instead).
- For secondary information that should be less prominent (use a `Well`).
- When the content is part of a sequential flow and doesn't represent a standalone entity.

## Variants

### Static vs. Interactive
- **Static**: If no `onPress` prop is provided, the card renders as an `<article>`. This is used for purely informational cards.
- **Interactive**: If an `onPress` handler is provided, the card renders as a `<button>`. This is used for cards that trigger an action or navigate to a new page.

### Selection
The `isSelected` prop applies a distinct "informative" background and a "focus" border color to the card, making it clear to the user which items are currently selected.

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The content to be rendered inside the card. |
| `onPress` | `(event: PressEvent) => void` | - | Handler called when the card is pressed. If present, the card renders as a button. |
| `isSelected` | `boolean` | `false` | Whether the card is in a selected state. |
| `isDisabled` | `boolean` | `false` | Whether the card (interactive mode) is disabled. |
| `aria-label` | `string` | - | An accessible label for the card. |

## Accessibility

- **Semantic HTML**: `Card` uses `<article>` for static content and `<button>` for interactive content, ensuring correct semantics for screen readers.
- **Selection State**: In interactive mode, `isSelected` is reflected via `aria-pressed` on the underlying button.
- **Focus Management**: Interactive cards are focusable and support standard keyboard interactions (`Enter`, `Space`).
- **Labeling**: Always provide an `aria-label` or ensure the card contains a heading that provides a clear name for the item.
