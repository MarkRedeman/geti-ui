## Responsibility

`Slider` is a single-handle range slider. It wraps Spectrum's `Slider` and adds Geti's energy-blue track/thumb override when `isFilled` is `true`. This is the primary slider for settings panels and filter controls where the filled-track visual is desired.

---

## Design

### The `isFilled` → `data-filled` pattern

Spectrum's `Slider` supports `isFilled` but uses its own accent colour for the fill track. To override with `--energy-blue`, Geti:

1. Passes `isFilled` through to Spectrum (unchanged — `props.isFilled` is in `...props`).
2. Mirrors the value as `data-filled={props.isFilled}` on the Spectrum component.
3. Uses a CSS module selector `[data-filled='true']` to override the Spectrum component-level tokens:

```css
/* Slider.module.css */
.slider[data-filled='true'] {
    --spectrum-slider-fill-track-color:              var(--energy-blue);
    --spectrum-slider-handle-border-color:           var(--energy-blue);
    --spectrum-slider-handle-border-color-down:      var(--energy-blue-lighter);
    --spectrum-slider-handle-border-color-hover:     var(--energy-blue-light);
}
```

The `.slider` class is applied unconditionally via `UNSAFE_className={clsx(styles.slider, UNSAFE_className)}`, so a caller's own class is always merged after.

### Props

```tsx
export interface SliderProps extends SpectrumSliderProps {}
```

No props are added or removed. All Spectrum props (`minValue`, `maxValue`, `step`, `value`, `onChange`, `label`, `isDisabled`, `isFilled`, etc.) pass through.

---

## Flow

Pure presentational. No internal state. `isFilled` controls the CSS override via `data-filled` attribute — no JavaScript logic is involved in the visual switch.

---

## Integration

- **Depends on**: `@adobe/react-spectrum` (`Slider`, `SpectrumSliderProps`), `clsx`, `./Slider.module.css`.
- **Tokens**: `--energy-blue`, `--energy-blue-light`, `--energy-blue-lighter` defined in `geti-global.module.css`.
- **Consumed by**: settings panels, annotation opacity controls, confidence threshold sliders.
- **Distinct from `RangeSlider`**: `Slider` has one handle (single value); `RangeSlider` has two handles (start + end range).
