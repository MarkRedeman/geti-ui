## Responsibility

The `Divider` component is a direct wrapper around Adobe's React Spectrum `Divider`. It is used to separate sections of content or group related elements visually.

### Key Usage Rules
1. **Layout Context**: A `Divider` has no intrinsic size. It stretches to fill its container's width (horizontal) or height (vertical). It **must** be placed within a layout container like `Flex` or `Grid` to be visible.
2. **Orientation Alignment**:
    - Use `orientation="horizontal"` (default) inside a column-direction layout.
    - Use `orientation="vertical"` inside a row-direction layout.
3. **Automatic Slotting**: When used inside a Spectrum `Dialog`, the `Divider` automatically slots itself between the header and the body content.

---

## Variants

### Orientation
Dividers can be horizontal or vertical. Vertical dividers are often used to separate items in a toolbar or navigation row.

### Thickness (Size)
The `size` prop controls the thickness of the divider line:
- `S`: 1px (Thinnest)
- `M`: 2px
- `L`: 4px (Thickest, default in v3)

## Props

<Controls of={DocStories.Horizontal} />

### Content / Value

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `orientation` | `'horizontal' \| 'vertical'` | `'horizontal'` | The axis the divider aligns to. |
| `size` | `'S' \| 'M' \| 'L'` | `'L'` | Controls the thickness of the divider. |

### Styles

The `Divider` component accepts all standard [Spectrum layout and spacing props](https://react-spectrum.adobe.com/react-spectrum/styling.html), including `margin`, `flex`, and `grid` properties.

---

## Accessibility

- **Role**: Renders as an `<hr>` element with `role="separator"`.
- **Labeling**: If the divider is used to separate distinct logical sections, consider providing an `aria-label` to describe the separation for screen reader users.

---

## Spectrum 2 Migration

When migrating to Spectrum 2 (`@react-spectrum/s2`), note the following changes:

- **Default Size Change**: In S2, the default `size` changes from `'L'` to `'M'`. This may require explicit prop updates to maintain visual parity.
- **Static Color**: S2 adds a `staticColor` prop (`'white' \| 'black' \| 'auto'`) for use over colored backgrounds.
- **Styles Macro**: Layout and spacing props are replaced by the S2 `styles` macro system.
