## Responsibility

The `AvatarGroup` component is a Geti-native layout component used to display multiple user avatars in a compact, overlapping row. It is commonly used to represent project members, annotators, commenters, or any scenario where a collection of people must be conveyed at a glance.

Key responsibilities:
- **Compact Layout**: Stacks avatars horizontally with a negative margin so they overlap, minimising the space used by a list of users.
- **Overflow Badge**: When the number of avatars exceeds the `max` threshold, a `+N` pill is shown so the total count is always visible without rendering every avatar.
- **Uniform Sizing**: An optional `size` prop propagates a single size token to every avatar in the group, overriding per-avatar sizes.
- **Accessible Group Semantics**: Wraps the stack in a `role="group"` element so screen readers understand the items belong together.

### When to use
- To show project members or team participants in a toolbar or card header.
- Alongside comment threads or annotation tasks where multiple users are involved.
- Anywhere a compact, at-a-glance representation of a user set is needed.

### When to avoid
- When you need to display a single avatar — use `Avatar` directly.
- When each avatar needs its own interactive control (tooltip, link, popover) — compose individual `Avatar` components manually.
- For large, unbound lists where a count chip alone would be more appropriate than a partial stack.

## Design

`AvatarGroup` is a Geti-native layout component that orchestrates multiple `Avatar` instances using inline CSS properties for the stacking effect.

- **Stacking Logic**: Each avatar is wrapped in a `<span>` with a negative `marginRight` (default `-8px`) to create the overlap. The last visible item also receives this margin, but the overflow badge uses a positive `marginLeft` (`12px`) to break out of the stack deliberately.
- **Size Propagation**: The group-level `size` prop takes precedence. If provided, it overrides any `size` set on the individual `avatars` prop objects.
- **Stable Keys**: The component prefers `avatarProps.alt` as a stable React `key`, falling back to the array `index` if `alt` is missing.

| Geti Prop | Type | Implementation Detail |
| :--- | :--- | :--- |
| `avatars` | `AvatarProps[]` | Array of Spectrum Avatar prop objects. |
| `max` | `number` | Threshold for overflow badge rendering. |
| `size` | `AvatarProps['size']` | Spectrum size token (e.g., `avatar-size-100`) or pixel number. |

## Flow

```
[avatars] array
  → .slice(0, max) 
  → map(Avatar) with negative overlap margin
  → if (length > max)
      → calculate (length - max)
      → render <span role="img">+N</span> with breakout margin
  → render wrapper with role="group"
```

## Integration

- **`ThemeProvider`**: Requires the Geti theme context to resolve Spectrum size tokens correctly.
- **`Avatar`**: Composes the `@geti/ui` `Avatar` component (which is a thin wrapper over Spectrum).
- **`PhotoPlaceholder`**: If an avatar in the group is missing a `src`, it will automatically trigger the fallback silhouette or initial-based placeholder logic.

## Variants

### Max Count
The `max` prop (default `3`) controls how many avatars are rendered before the remainder are collapsed into a `+N` overflow badge. Setting `max` equal to or greater than the total number of avatars disables the badge entirely.

### Uniform Size
Passing a `size` prop to the group applies that size token to every `Avatar` inside it, overriding any `size` values set on the individual `AvatarProps` items.

## Accessibility

- **Group Role**: The wrapping `<div>` carries `role="group"` and an `aria-label` of `"N avatars"` (where N is the total count) so assistive technologies announce the size of the collection.
- **Overflow Badge**: The `+N` pill includes its own `aria-label` (e.g. `"2 more"`) so screen readers can report the hidden count without relying on the visible text alone.
- **Individual Labels**: Each `Avatar` inside the group should have a meaningful `alt` prop so screen readers can identify individual users.
- **Decorative Images**: If an avatar is purely decorative and lacks an `alt` text, it should be marked as `aria-hidden`.
