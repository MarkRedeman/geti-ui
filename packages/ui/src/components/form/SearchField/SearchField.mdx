# SearchField

A specialized text input for search queries, with a built-in search icon, an automatic clear button, and search-specific keyboard interactions.

```tsx
import { SearchField } from '@geti-ai/ui';
```

- [React Spectrum V3 SearchField](https://react-spectrum.adobe.com/react-spectrum/SearchField.html)

<Stories />

## Responsibility

`SearchField` wraps Adobe React Spectrum's `SearchField` and adds `ref` forwarding via `TextFieldRef`. It renders with `role="searchbox"`, a persistent search icon, an automatic clear button (visible when the field has a value), and two distinct callbacks: `onChange` fires on every keystroke while `onSubmit` fires only on `Enter`.

The `forwardRef` wrapper allows parent components to call `ref.current.focus()` — useful for keyboard shortcuts that focus the search bar.

### When to use

- For search bars, filter inputs, and global search fields where the "search" affordance should be immediately visible.
- When you want a clear button so users can reset the query without manual text selection.
- When `onSubmit` (triggered by `Enter`) and `onChange` (live filtering) need to be handled separately.

### When to avoid

- **General data entry** — use `TextField`; `role="searchbox"` would be semantically wrong.
- **Search with suggestions** — use `ComboBox` when a dropdown of autocomplete options should appear.
- **Multi-line input** — use `TextArea`.
- **When no label is feasible** — always provide `label` or `aria-label`.

## States

| State               | Prop                        | Description                                             |
| ------------------- | --------------------------- | ------------------------------------------------------- |
| Default (empty)     | —                           | Search icon visible; clear button hidden.               |
| Active (with value) | —                           | Both search icon and clear button visible.              |
| Invalid             | `validationState="invalid"` | Red border, error icon, `errorMessage` below the field. |
| Disabled            | `isDisabled={true}`         | Non-focusable, visually dimmed. Clear button hidden.    |
| Read-only           | `isReadOnly={true}`         | Focusable and copy-able; clear button hidden.           |
| Required            | `isRequired={true}`         | Visual asterisk and `aria-required="true"`.             |
| Quiet               | `isQuiet={true}`            | Underline-only border for toolbars or dense layouts.    |

## Props

<Controls of={DocStories.Default} />

### Content

| Prop             | Type        | Description                                                                                      |
| ---------------- | ----------- | ------------------------------------------------------------------------------------------------ |
| `label`          | `ReactNode` | The label above or beside the field. Required for accessibility unless `aria-label` is provided. |
| `placeholder`    | `string`    | Hint text when empty. Supplement labels — do not substitute for them.                            |
| `description`    | `ReactNode` | Persistent helper text below the field.                                                          |
| `errorMessage`   | `ReactNode` | Error message shown when `validationState="invalid"`.                                            |
| `contextualHelp` | `ReactNode` | A `<ContextualHelp>` popover anchored to the label.                                              |

### Value / State

| Prop              | Type                   | Default | Description                                          |
| ----------------- | ---------------------- | ------- | ---------------------------------------------------- |
| `value`           | `string`               | —       | The current value (controlled).                      |
| `defaultValue`    | `string`               | —       | The initial value (uncontrolled).                    |
| `validationState` | `'valid' \| 'invalid'` | —       | Visual and ARIA validation state.                    |
| `isRequired`      | `boolean`              | `false` | Adds a visual asterisk and `aria-required`.          |
| `isReadOnly`      | `boolean`              | `false` | Value visible but not editable. Clear button hidden. |
| `isDisabled`      | `boolean`              | `false` | Non-interactive and visually dimmed.                 |
| `isQuiet`         | `boolean`              | `false` | Underline-only border style.                         |

### Events

| Prop       | Type                      | Description                                                                            |
| ---------- | ------------------------- | -------------------------------------------------------------------------------------- |
| `onChange` | `(value: string) => void` | Called on every keystroke. Use for live filtering; debounce for API calls.             |
| `onSubmit` | `(value: string) => void` | Called when the user presses `Enter`. Use for explicit search actions.                 |
| `onClear`  | `() => void`              | Called when the clear button is pressed or `Escape` is pressed with a non-empty value. |
| `onBlur`   | `(e: FocusEvent) => void` | Called when the field loses focus.                                                     |

### Layout

| Prop            | Type              | Description                           |
| --------------- | ----------------- | ------------------------------------- |
| `labelPosition` | `'top' \| 'side'` | Label position relative to the input. |
| `width`         | `DimensionValue`  | The width of the entire component.    |

## Related components

- `TextField` — use instead for general single-line data entry that is not a search query.
- `ComboBox` — use instead when a dropdown of suggestions should appear as the user types.
- `NumberField` — use instead for numeric-only inputs.
