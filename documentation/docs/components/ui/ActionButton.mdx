# ActionButton

ActionButton is a compact, icon-friendly button for toolbar actions, inline controls, and context menus. Use it when you need a low-footprint pressable action — not a primary page-level CTA.

```tsx
import { ActionButton } from '@geti-ai/ui';
```

- [React Spectrum ActionButton ↗](https://react-spectrum.adobe.com/react-spectrum/ActionButton.html)
- [Geti UI Theming](/components/theming)

## Examples

### Default

```tsx preview
import { ActionButton } from '@geti-ai/ui';

export default function App() {
  return <ActionButton>Action</ActionButton>;
}
```

### Dark variant

```tsx preview
import { ActionButton } from '@geti-ai/ui';

export default function App() {
  return <ActionButton colorVariant="dark">Dark</ActionButton>;
}
```

### Blue variant

```tsx preview
import { ActionButton } from '@geti-ai/ui';

export default function App() {
  return <ActionButton colorVariant="blue">Blue</ActionButton>;
}
```

### Quiet

```tsx preview
import { ActionButton } from '@geti-ai/ui';

export default function App() {
  return <ActionButton isQuiet>Quiet</ActionButton>;
}
```

### Disabled

```tsx preview
import { ActionButton } from '@geti-ai/ui';

export default function App() {
  return <ActionButton isDisabled>Disabled</ActionButton>;
}
```

## Responsibility

`ActionButton` wraps React Spectrum's `ActionButton` with an additional `colorVariant` prop for Geti-specific visual styles.

What this wrapper adds:

- `colorVariant="dark"` — subtle border on hover/active, suitable for dark panels.
- `colorVariant="light"` — blue-tinted hover/active, for light header areas.
- `colorVariant="blue"` — transparent background with blue text, for emphasis in sidebars.
- All standard Spectrum ActionButton features (quiet, disabled, icon support) pass through unchanged.

## Variants and states

| Variant | When to use |
|---|---|
| default | Neutral toolbar or panel actions |
| `dark` | Dark-background contexts (canvas toolbars) |
| `light` | Light-background header controls |
| `blue` | Accent-colored sidebar or secondary panel actions |

| State | Prop | Notes |
|---|---|---|
| Quiet | `isQuiet` | No border at rest; border on hover |
| Disabled | `isDisabled` | Non-interactive and visually muted |

## Props (deep dive)

### Content

- `children`: label text or icon node.
- `icon`: icon element shown before the label (standard Spectrum pattern).

### Visual

- `colorVariant`: `'dark' | 'light' | 'blue'` — Geti visual skin applied via CSS class.
- `isQuiet`: removes border at rest for a minimal look.

### Events

- `onPress`: primary action handler (keyboard and mouse safe).

### Accessibility

- always provide a text label or `aria-label` when using icon-only buttons,
- `isDisabled` correctly exposes disabled semantics to screen readers.

## Related components

- [`Button`](/components/ui/Button) — use for primary page-level CTAs (`Save`, `Delete`).
- [`ToggleButton`](/components/ui/ToggleButton) — use when the button has persistent selected/unselected state.
