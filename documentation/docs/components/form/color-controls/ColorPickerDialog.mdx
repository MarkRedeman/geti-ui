# ColorPickerDialog

`ColorPickerDialog` is a self-contained color picker that opens in a dialog. It combines `ColorArea`, `ColorWheel`, `ColorSlider` (hue + alpha), `ColorField` (hex), and a `ColorSwatchPicker` preset palette into a single trigger-and-confirm workflow.

```tsx
import { ColorPickerDialog } from '@geti-ui/ui';
```

- [Geti UI Theming](/components/theming)

## Examples

### Default

```tsx preview
import { ColorPickerDialog } from '@geti-ui/ui';

export default function App() {
  return (
    <ColorPickerDialog color="#ff0000" label="Pick Color" onColorChange={(hex) => console.log(hex)} />
  );
}
```

### Custom label

```tsx preview
import { ColorPickerDialog } from '@geti-ui/ui';

  return (
    <ColorPickerDialog color="#00ff00" label="Select Theme Color" onColorChange={(hex) => console.log(hex)} />

```

## Responsibility

`ColorPickerDialog` is a Geti-native composite component. It manages its own internal color state (HSBA format) and exposes a simplified API:

- Accepts a `color` string prop (any CSS color) and converts it to HSBA internally.
- On **Confirm**, calls `onColorChange` with the chosen color as a hex string.
- On **Cancel**, discards changes.
- Internally normalises to HSBA to avoid "Unknown color channel: alpha" errors when rendering the alpha `ColorSlider`.
- Falls back to red (`hsb(0, 100%, 100%)`) if an unparseable color string is provided.

## Variants and states

| Feature | Prop | Notes |
|---|---|---|
| Controlled open | `isOpen` + `onOpenChange` | Optional external control of dialog visibility |
| Label | `label` | Trigger button label and dialog aria-label (default: `"Pick Color"`) |
| Initial color | `color` | Any CSS color string; defaults to red |

## Props (deep dive)

### Content

- `label`: text shown on the trigger button and used as the dialog title hint. Defaults to `"Pick Color"`.

### Value / state

- `color`: any parseable CSS color string. Sets the starting color when the dialog opens.
- `isOpen` / `onOpenChange`: optional controlled dialog open state.

### Events

- `onColorChange(hex: string)`: called with the chosen hex color when the user presses **Confirm**.

### Accessibility

- The trigger `ActionButton` uses `aria-label={label}`.
- The `ColorSwatch` on the trigger is decorative (`aria-hidden`); the text label is visible to screen readers.
