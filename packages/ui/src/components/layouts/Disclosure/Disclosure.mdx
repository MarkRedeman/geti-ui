# Disclosure

<Subtitle>A Disclosure is a collapsible section that allows users to show or hide content as needed.</Subtitle>

## Responsibility

The `Disclosure` component provides a standard mechanism for progressive disclosure. It consists of a clickable header (trigger) and a collapsible panel (content). It is commonly used for individual collapsible sections or as a building block for an `Accordion`.

Key responsibilities:
- **Progressive Disclosure**: Keeps the UI clean by hiding non-essential content until the user requests it.
- **State Management**: Handles the expanded/collapsed state and the associated animation.
- **Accessibility**: Implements the ARIA disclosure pattern with proper button and region roles.

### When to use
- For a single, independent collapsible section (e.g., "Advanced Settings").
- Within an `Accordion` to group multiple related sections.
- For help text or additional details that should be hidden by default.

### When to avoid
- When the user needs to see the content at all times for quick scanning.
- To group interactive elements that shouldn't be hidden (use a `Well` or `Card`).
- For primary site-wide navigation (use a sidebar or `Tabs`).

## Variants

### Controlled vs. Uncontrolled
- **Uncontrolled (Default)**: The disclosure manages its own expansion state. Use `defaultExpanded` for the initial state.
- **Controlled**: The parent component manages the expansion state via the `isExpanded` and `onExpandedChange` props.

### Nested Header (S2 only)
Spectrum 2 introduces the `DisclosureHeader` component, which allows for a more complex header layout where action buttons or icons can be co-located with the trigger.

## Props

### Disclosure Props
| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The title and panel components. |
| `isExpanded` | `boolean` | - | Whether the section is expanded (controlled). |
| `defaultExpanded` | `boolean` | `false` | Whether the section is expanded by default (uncontrolled). |
| `onExpandedChange` | `(isExpanded: boolean) => void` | - | Handler called when the expansion state changes. |
| `isDisabled` | `boolean` | `false` | Whether the disclosure is disabled. |

### DisclosureTitle Props
| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The label or content of the clickable header. |

### DisclosurePanel Props
| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The content to be shown when expanded. |

## Accessibility

- **Roles**: The trigger is rendered as a `<button>` with `aria-expanded` and `aria-controls`. The panel has `role="group"` (or `role="region"` for significant content).
- **Keyboard Support**:
  - `Enter` or `Space` toggles the expansion of the section when the header is focused.
  - `Tab` moves focus between the header and any interactive elements inside the panel.

## Spectrum 2 Migration

The migration to Spectrum 2 (S2) introduces structural improvements and better visual scale controls.

### Structural Changes
- **Header Wrapper**: S2 adds a `DisclosureHeader` component to wrap the `DisclosureTitle` and any extra elements (like action buttons).
- **Sub-component Names**: Generic names like `DisclosureTitle` and `DisclosurePanel` are preserved, but the implementation is refined.

### New Features
- **Visual Scales**: New `size` and `density` props allow for better control over the disclosure's appearance.
- **Styles Macro**: Layout and spacing overrides use the `styles` prop and the `style()` macro.

### Migration Example

**V3 Style (Current):**
```tsx
<Disclosure defaultExpanded>
  <DisclosureTitle>Advanced Settings</DisclosureTitle>
  <DisclosurePanel>
    <Checkbox>Enable Beta Features</Checkbox>
  </DisclosurePanel>
</Disclosure>
```

**S2 Style (Future):**
```tsx
<Disclosure size="M" density="compact">
  <DisclosureHeader>
    <DisclosureTitle>Advanced Settings</DisclosureTitle>
    <ActionButton aria-label="Settings"><SettingsIcon /></ActionButton>
  </DisclosureHeader>
  <DisclosurePanel>
    <Checkbox>Enable Beta Features</Checkbox>
  </DisclosurePanel>
</Disclosure>
```
