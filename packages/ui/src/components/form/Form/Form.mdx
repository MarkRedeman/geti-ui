## Responsibility

The `Form` component is a semantic wrapper for groups of input fields. It renders a native `<form>` element and provides a React context that propagates layout settings (like label position) and shared states (like disabled or required) to all descendant Spectrum components.

### When to use
- When you have two or more related input fields that should be submitted together.
- To ensure consistent label alignment and layout across multiple fields in a single container.
- When you need to manage form-wide validation or disabled states.

### When to use something else
- Use a simple `<div>` for groupings that do not have form submission semantics.
- For single-field inputs that are not part of a larger form structure.

## Variants

### Label Layout
- **Top** (Default): Labels are placed above each field.
- **Side**: Labels are placed to the left of each field. Use `labelPosition="side"`.
- **Alignment**: When using side labels, use `labelAlign="start"` or `labelAlign="end"` to control text alignment.

### Necessity Indicator
- **Icon** (Default): Displays an asterisk `✱` for required fields.
- **Label**: Displays the text `"(required)"` or `"(optional)"` next to the label. Use `necessityIndicator="label"`.

### Validation Behavior
- **ARIA** (Default in V3): Validation is handled via ARIA attributes; it does not block native form submission.
- **Native**: Uses HTML5 constraint validation and blocks submission if fields are invalid. Use `validationBehavior="native"`.

## Props

### Layout & State
| Prop | Type | Default | Description |
|---|---|---|---|
| `labelPosition` | `'top' \| 'side'` | `'top'` | Position of labels for all child fields. |
| `labelAlign` | `'start' \| 'end'` | `'start'` | Alignment of labels (effective with `labelPosition="side"`). |
| `necessityIndicator` | `'icon' \| 'label'` | `'icon'` | How required/optional states are indicated. |
| `isDisabled` | `boolean` | `false` | Disables all fields within the form. |
| `isRequired` | `boolean` | `false` | Marks all fields within the form as required. |

### Validation
| Prop | Type | Default | Description |
|---|---|---|---|
| `validationBehavior` | `'aria' \| 'native'` | `'aria'` | Whether to use ARIA or native HTML5 validation. |
| `validationErrors` | `Record<string, string \| string[]>` | — | A map of field names to error messages (e.g., from a server). |

### Events
| Prop | Type | Description |
|---|---|---|
| `onSubmit` | `(e: FormEvent) => void` | Handler called when the form is submitted. |
| `onReset` | `(e: FormEvent) => void` | Handler called when the form is reset. |

## Accessibility

- Renders a native `<form>` element, creating an **ARIA `form` landmark**.
- **Important**: To be correctly identified as a landmark by screen readers, you should provide a label using `aria-label` or `aria-labelledby`.
- Automatically propagates accessibility-related props (like `isDisabled` or `isRequired`) to all child Spectrum components.
- Manages focus automatically by moving it to the first invalid field upon a failed submission (when using `validationBehavior="native"`).
