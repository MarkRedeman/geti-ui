# FileTrigger

A wrapper that allows any pressable component to open the system file picker.

```tsx
import { FileTrigger } from '@geti/ui';
```

- [React Aria Components FileTrigger](https://react-spectrum.adobe.com/react-aria/FileTrigger.html)

<Stories />

## Responsibility

`FileTrigger` is a transparent wrapper around `react-aria-components`' `FileTrigger` — the **only** component in `@geti/ui` sourced from `react-aria-components` rather than `@adobe/react-spectrum`. It is a headless primitive: it renders a visually hidden `<input type="file" aria-hidden="true">` and delegates the press event from any interactive child to open the OS file picker.

- **Hidden file input**: Manages the `<input type="file">` lifecycle, including resetting it after selection so the same file can be re-selected immediately.
- **Trigger delegation**: Programmatically clicks the hidden input when the child is activated (click, `Space`, or `Enter`), so the child retains full keyboard and screen reader semantics.
- **Type filtering**: `acceptedFileTypes` sets the `accept` attribute on the hidden input.
- **No visible output**: Renders only what `children` render plus the hidden input — no container, no border, no layout impact.

### When to use
- When you want a styled `Button`, `Link`, or other interactive element to trigger a file selection.
- As the click-to-browse companion inside a [`DropZone`](/docs/components-form-dropzone--docs).
- When you need control over accepted file types, multi-file selection, or directory selection.
- On mobile when you want to open a camera via `defaultCamera`.

### When to avoid
- **Drag-and-drop only** — use [`DropZone`](/docs/components-form-dropzone--docs) alone.
- **Native form submission** — use a `<input type="file" name="…">` if the browser needs to include the file in `FormData` without JavaScript.

## Variants

```tsx
// Single file
<FileTrigger onSelect={(files) => console.log(files)}>
    <Button>Select file</Button>
</FileTrigger>

// Multiple files
<FileTrigger onSelect={handleSelect} allowsMultiple>
    <Button>Select images</Button>
</FileTrigger>

// Restricted MIME types
<FileTrigger onSelect={handleSelect} acceptedFileTypes={['image/png', 'image/jpeg']}>
    <Button>Upload image</Button>
</FileTrigger>

// Disabled trigger — set isDisabled on the child, not FileTrigger
<FileTrigger onSelect={handleSelect}>
    <Button isDisabled>Upload (unavailable)</Button>
</FileTrigger>
```

Pair with `DropZone` to give users both drag-and-drop and click-to-browse paths in a single upload area.

## Props

<Controls of={DocStories.Default} />

### Events

| Prop | Type | Description |
|------|------|-------------|
| `onSelect` | `(files: FileList \| null) => void` | Called when the user selects files. If the user cancels the picker, this is not called. |

### File selection

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `acceptedFileTypes` | `readonly string[]` | — | MIME types or extensions to allow (e.g., `['image/*']`, `['.csv']`). Maps to the `accept` attribute. |
| `allowsMultiple` | `boolean` | `false` | Whether multiple files can be selected in one session. |
| `acceptDirectory` | `boolean` | `false` | Opens a directory/folder picker instead of a file picker. |
| `defaultCamera` | `'user' \| 'environment'` | — | Mobile only: `'user'` = front camera, `'environment'` = rear camera. |

### Content

| Prop | Type | Description |
|------|------|-------------|
| `children` | `ReactNode` | The interactive element that triggers the file picker. Must be a single pressable element. |

## Related components
- `DropZone` — pair with `FileTrigger` to provide both drag-and-drop and click-to-browse paths.
- `Button` — the most common child trigger; provides consistent Geti styling and press handling.
- `ActionButton` — use as the child when the trigger is in a toolbar with icon-only controls.

## Example use cases
<details>
  <summary>Typical use cases</summary>

- Triggering an image upload dialog from a "Browse files" button in a dataset creation wizard.
- Allowing users to upload a model weights file from a toolbar `ActionButton` in a model management panel.
- Replacing a project thumbnail by clicking an avatar or placeholder image.
- Uploading a CSV label file by pressing a `Button` inside a `DropZone` upload area.
