## Responsibility

`ListBox` displays a **standalone, permanently-visible list of options** that users can select from. Unlike a `Picker` or `ComboBox`, a `ListBox` does not hide behind a trigger—it is always visible in the layout.

### When to use ListBox
- **Permanent selection list**: Use when the list of options should be always visible, such as in a sidebar filter or a dedicated selection panel.
- **Simple Items**: Best for lists of text, icons, or avatars.
- **Single or Multiple Selection**: Supports both modes via `selectionMode`.

### Comparison with other components
- **vs. Picker / Select**: Use `Picker` for a compact single-select that opens a dropdown popover.
- **vs. ListView**: Use `ListView` when items need **interactive children** (like buttons or menus) or complex row actions. `ListBox` does not support nested interactive elements.

## Variants

### Selection Modes
- **Single Selection**: Users can select only one item at a time.
- **Multiple Selection**: Users can select multiple items using the keyboard or checkboxes.

### Sections
Items can be grouped into sections using the `Section` component (re-exported from Spectrum) to provide hierarchical organization with headers.

## Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `children` | `ItemElement \| ItemRenderer` | — | The items or sections to display. |
| `selectionMode` | `'none' \| 'single' \| 'multiple'` | `'none'` | The selection mode for the listbox. |
| `selectedKeys` | `'all' \| Iterable<Key>` | — | The currently selected keys in the collection. |
| `onSelectionChange` | `(keys: Selection) => void` | — | Handler that is called when the selection changes. |
| `items` | `Iterable<T>` | — | The list of items to display (for dynamic collections). |
| `disabledKeys` | `Iterable<Key>` | — | The item keys that should be disabled. |
| `autoFocus` | `boolean` | `false` | Whether the element should receive focus on render. |

## Accessibility

### Roles and Navigation
- The container uses `role="listbox"` and each item uses `role="option"`.
- **Keyboard Navigation**: 
    - `Up/Down Arrows`: Move focus between options.
    - `Home/End`: Jump to the first or last option.
    - `Space/Enter`: Select the focused option.
    - `Type-ahead`: Typing characters jumps to the matching option.
- **Required Labels**: Always provide an `aria-label` or `aria-labelledby` so screen readers can identify the list.

### Constraints
- **No nested interactivity**: Do not place buttons, links, or inputs inside a `ListBoxItem`. This breaks the ARIA model and keyboard navigation for screen readers.
