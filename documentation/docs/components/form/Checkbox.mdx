# Checkbox

Checkbox is a binary form control for opt-in/opt-out choices. Use it for individual boolean settings, multi-select item lists, and form agreement fields.

```tsx
import { Checkbox } from '@geti-ai/ui';
```

- [React Spectrum Checkbox ↗](https://react-spectrum.adobe.com/react-spectrum/Checkbox.html)
- [Geti UI Theming](/components/theming)

## Examples

### Default (uncontrolled)

```tsx preview
import { Checkbox } from '@geti-ai/ui';

export default function App() {
  return <Checkbox>Enable notifications</Checkbox>;
}
```

### Pre-checked

```tsx preview
import { Checkbox } from '@geti-ai/ui';

export default function App() {
  return <Checkbox defaultSelected>Auto-save enabled</Checkbox>;
}
```

### Indeterminate

```tsx preview
import { Checkbox } from '@geti-ai/ui';

export default function App() {
  return <Checkbox isIndeterminate>Select all</Checkbox>;
}
```

### Disabled

```tsx preview
import { Checkbox } from '@geti-ai/ui';

export default function App() {
  return <Checkbox isDisabled>Disabled option</Checkbox>;
}
```

### Invalid

```tsx preview
import { Checkbox } from '@geti-ai/ui';

export default function App() {
  return <Checkbox validationState="invalid">Required agreement</Checkbox>;
}
```

## Responsibility

`Checkbox` is a thin wrapper around React Spectrum's `Checkbox`. All props pass through unchanged. The wrapper provides a consistent `@geti-ai/ui` import path and Geti theming.

## Variants and states

| State | Prop | Notes |
|---|---|---|
| Checked | `isSelected` / `defaultSelected` | Controlled or uncontrolled |
| Indeterminate | `isIndeterminate` | Use for "select all" with partial selection |
| Disabled | `isDisabled` | Non-interactive |
| Invalid | `validationState="invalid"` | Shows error styling |
| Read-only | `isReadOnly` | Visible but not editable |

## Props (deep dive)

### Content

- `children`: label text.

### Value / state

- `isSelected`: controlled boolean value.
- `defaultSelected`: uncontrolled initial value.
- `isIndeterminate`: shows the dash/mixed state — use for "select all" master checkboxes.
- `value`: string value used when this checkbox is a child of `CheckboxGroup`.

### Events

- `onChange`: `(isSelected: boolean) => void`.

### Validation

- `validationState`: `'valid' | 'invalid'` — applies visual validation styling.
- `isRequired`: marks as required in native form validation.

### Accessibility

- always provide a visible text label as `children`,
- `isIndeterminate` sets `aria-checked="mixed"` automatically,
- when grouping, use `CheckboxGroup` instead of managing groups manually.

## Related components

- [`CheckboxGroup`](/components/form/CheckboxGroup) — use when multiple checkboxes share a label and value array.
- [`Switch`](/components/form/Switch) — use for live settings toggles that take effect immediately (not on form submit).
- [`RadioGroup`](/components/form/RadioGroup) — use when only one of several options can be selected.
