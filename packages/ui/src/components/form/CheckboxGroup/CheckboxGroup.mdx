## Responsibility

The `CheckboxGroup` component manages a collection of checkboxes, allowing users to select multiple options from a list. It ensures consistent labeling, orientation, and validation state across all items in the group.

### When to use
- When the user needs to select one or more options from a visible list.
- When the choices are not mutually exclusive.

### When to use something else
- Use [RadioGroup](/docs/components-form-radiogroup--docs) when the user must select exactly one option.
- Use a single [Checkbox](/docs/components-form-checkbox--docs) for a standalone binary choice (e.g., "I accept the terms").
- Use [Switch](/docs/components-form-switch--docs) for immediate on/off toggles.

## Variants

### States
- **Disabled**: Both the group (`isDisabled` on `CheckboxGroup`) and individual items can be disabled.
- **Invalid**: Indicates a validation error for the group. In V3, use `validationState="invalid"`.
- **Read-only**: The selection is visible but cannot be changed. Use `isReadOnly`.
- **Required**: Marks the group as requiring at least one selection. Use `isRequired`.
- **Emphasized**: Use `isEmphasized` for high-prominence (blue) styling across the group.

### Orientation
- **Vertical** (Default): Recommended for long lists of options.
- **Horizontal**: Best for small sets of options with short labels. Use `orientation="horizontal"`.

## Props

### CheckboxGroup
| Prop | Type | Default | Description |
|---|---|---|---|
| `label` | `ReactNode` | — | The label for the entire group. |
| `value` | `string[]` | — | Controlled array of selected values. |
| `defaultValue` | `string[]` | — | Initial selection state (uncontrolled). |
| `orientation` | `'vertical' \| 'horizontal'` | `'vertical'` | Layout axis for the checkboxes. |
| `validationState` | `'valid' \| 'invalid'` | — | Whether the input is valid or invalid. |
| `isRequired` | `boolean` | `false` | Whether at least one checkbox must be selected. |
| `isDisabled` | `boolean` | `false` | Disables all checkboxes within the group. |
| `isEmphasized` | `boolean` | `false` | Whether the checkboxes should use emphasized styling. |

### Checkbox
| Prop | Type | Description |
|---|---|---|
| `value` | `string` | The value to be submitted for this option. |
| `children` | `ReactNode` | The label text or content for this checkbox. |
| `isIndeterminate` | `boolean` | Displays a dash state (presentational only). |
| `isDisabled` | `boolean` | Whether this specific checkbox is disabled. |

## Accessibility

- The `CheckboxGroup` uses `role="group"` and is automatically labeled by the `label` prop via `aria-labelledby`.
- Individual checkboxes use native `<input type="checkbox">` elements (visually hidden) for standard behavior.
- **Keyboard support**: 
    - **Tab** moves focus through the checkboxes.
    - **Space** toggles the focused checkbox.
- Screen readers announce the group label, the individual checkbox label, and its checked state.

## Spectrum 2 Migration

When migrating to Spectrum 2 (S2), keep the following changes in mind:

| V3 Prop | S2 Equivalent | Notes |
|---|---|---|
| `validationState="invalid"` | `isInvalid` | Boolean flag replaces the string enum. |
| `validationState="valid"` | — | No visual "valid" state in S2. |
| `showErrorIcon` | — | Removed in S2 for accessibility reasons. |
| `isEmphasized` | `isEmphasized` | Remains the standard for accent-color highlights. |
| `size` | `size` | S2 adds a `size` prop (`'S'`, `'M'`, `'L'`, `'XL'`). |
| `validationBehavior` | `validationBehavior` | **Breaking**: Default flips from `'aria'` to `'native'`. |
