# ActionMenu

A self-contained "three dots" icon button that opens a contextual command list — the fastest way to attach per-item actions to a row, card, or toolbar slot.

```tsx
import { ActionMenu, Item, Section } from '@geti/ui';
```

- [React Spectrum V3 ActionMenu](https://react-spectrum.adobe.com/react-spectrum/ActionMenu.html)

<Stories />

## Responsibility

`ActionMenu` is a convenience composite that bundles an `ActionButton` trigger, a `MenuTrigger` overlay manager, and a `Menu` into a single component. The standard "three dots" (⋯) icon trigger and all open/close behaviour are provided out of the box — the caller only needs to supply menu items.

- **Contextual command hub**: Groups related actions that can be performed on a specific object (row, card, asset).
- **Space efficiency**: Hides multiple secondary actions behind a single compact trigger to reduce visual clutter.
- **Standardised trigger**: Enforces the system-standard icon and interaction pattern for consistency across all surfaces.

### When to use

- For actions attached to a **specific row in a table** or a **card in a grid** — e.g., Edit, Duplicate, Delete.
- When there are **3 or more secondary actions** that would otherwise clutter the UI if shown as individual buttons.
- For **"More actions"** in a toolbar or page header where the trigger must be icon-only to save horizontal space.
- In **compact list items** where a text-label button would be too wide.

### When to avoid

- When the trigger should display **text** (e.g., "Edit options") — compose `Menu` + `MenuTrigger` + `Button` manually instead.
- For **navigation between pages** — use `Link` or `Tabs`; the `menu` ARIA role implies commands, not navigation.
- For a **small number of primary actions** that should always be visible — use `ActionButton` or `Button` directly.
- When you need a **custom trigger icon** — `ActionMenu` uses the built-in Spectrum "three dots" icon and does not expose a prop to override it. Use `Menu` + `MenuTrigger` with your own trigger element in that case.

## Variants and states

### Size

Supports `S`, `M` (default), and `L` sizes for the trigger button via the `size` prop.

### Quiet

`isQuiet` removes the border and background from the trigger button, blending it into its container. Use in information-dense surfaces where the border adds unnecessary visual noise.

### Selection mode

Most `ActionMenu` items are commands, but you can also use `selectionMode="single"` or `selectionMode="multiple"` to allow users to select items from the menu (e.g., a "View Mode" setting on a card header).

## ActionMenu vs. manual composition

`ActionMenu` simplifies the three-component composition into a single-component API:

```tsx
// Manual composition (3 components)
<MenuTrigger>
    <ActionButton aria-label="Row actions">…</ActionButton>
    <Menu onAction={handleAction}>
        <Item key="edit">Edit</Item>
        <Item key="delete">Delete</Item>
    </Menu>
</MenuTrigger>

// ActionMenu (1 component — equivalent result)
<ActionMenu aria-label="Row actions" onAction={handleAction}>
    <Item key="edit">Edit</Item>
    <Item key="delete">Delete</Item>
</ActionMenu>
```

The "three dots" trigger icon is fixed — it is rendered by Spectrum's `ActionMenu` internals and cannot be replaced via a prop. If you need a different trigger icon or a text trigger, use `Menu` + `MenuTrigger` with a custom `ActionButton` trigger instead.

## Accessibility

`ActionMenu` implements the [WAI-ARIA Menu Button pattern](https://www.w3.org/WAI/ARIA/apg/patterns/menu-button/) via Spectrum.

**Always provide `aria-label`**: The trigger is icon-only (⋯). Without an `aria-label`, screen readers announce only "button" with no context. Use a description that reflects the **target object**:

```tsx
// Good — specific to the object
<ActionMenu aria-label={`Actions for ${project.name}`}>

// Avoid — too generic
<ActionMenu aria-label="More actions">
```

**Keyboard interactions**

| Key                                        | Behaviour                             |
| ------------------------------------------ | ------------------------------------- |
| `Enter` / `Space` / `Arrow Down` (trigger) | Opens menu, focuses first item        |
| `Arrow Up` (trigger)                       | Opens menu, focuses last item         |
| `Arrow Down` / `Arrow Up` (menu open)      | Moves focus between items             |
| `Enter` / `Space` (item focused)           | Activates item; closes menu           |
| `Escape`                                   | Closes menu, returns focus to trigger |
| Type-ahead                                 | Focus jumps to next matching item     |

## Related components

- **`Menu`** + **`MenuTrigger`** — use when you need a custom trigger element (text label, custom icon); `ActionMenu` is the shortcut for the standard "three dots" trigger.
- **`ActionButton`** — use when a single action is common enough to deserve a permanently visible button.
- **`ActionGroup`** — use when all actions should be permanently visible as a row of buttons.
- **`Tabs`** — use when options represent sibling content panels rather than commands.

## Example use cases

<details>
  <summary>Typical use cases</summary>

- Adding an "Edit / Duplicate / Delete" context menu to each row in a `TableView` projects list — `<ActionMenu aria-label={`Actions for ${project.name}`} onAction={handleAction}>`.
- Providing a quiet "three dots" menu in the top-right corner of a dataset `Card` for Rename, Export, and Delete actions.
- Collapsing secondary toolbar actions (Download report, Share, Archive) behind an `ActionMenu` trigger in a page header.
- Offering per-item actions in a compact `ListView` row — `isQuiet` keeps the trigger borderless to match the row style.
- Implementing a "More actions" menu on a model deployment card with `selectionMode="single"` to let users choose a default action.
