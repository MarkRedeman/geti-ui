# NumberField

A numeric input field with stepper buttons, range constraints, and locale-aware formatting for currencies, percentages, and units.

```tsx
import { NumberField } from '@geti-ai/ui';
```

- [React Spectrum V3 NumberField](https://react-spectrum.adobe.com/react-spectrum/NumberField.html)

<Stories />

## Responsibility

`NumberField` is a transparent wrapper around Adobe React Spectrum's `NumberField`. It provides stepper buttons, strict numeric validation (non-numeric characters are rejected), automatic value clamping to `minValue`/`maxValue` on blur, and locale-aware display formatting via `Intl.NumberFormat`. The raw `number` is always what `onChange` delivers — formatting is display-only.

### When to use

- For any input that must be a number: quantities, thresholds, confidence values, budgets, or counts.
- When you need to enforce a min/max range.
- When stepper buttons improve discoverability (e.g., adjusting a threshold by small increments).
- When locale-aware formatting (currency, percent, units) is required.

### When to avoid

- **General text entry** — use `TextField`.
- **Continuous range selection** — use `Slider` when a visual track is more intuitive.
- **Search inputs** — use `SearchField`.
- **When no label is feasible** — always provide `label` or `aria-label`.

## States

| State     | Prop                        | Description                                             |
| --------- | --------------------------- | ------------------------------------------------------- |
| Default   | —                           | Interactive field with stepper buttons.                 |
| Invalid   | `validationState="invalid"` | Red border, error icon, `errorMessage` below the field. |
| Valid     | `validationState="valid"`   | Green border, checkmark. Use sparingly.                 |
| Disabled  | `isDisabled={true}`         | Non-focusable, visually dimmed.                         |
| Read-only | `isReadOnly={true}`         | Focusable and copy-able; stepper buttons hidden.        |
| Required  | `isRequired={true}`         | Visual asterisk and `aria-required="true"`.             |
| Quiet     | `isQuiet={true}`            | Underline-only border for dense layouts.                |

## Props

<Controls of={DocStories.Default} />

### Content

| Prop             | Type        | Description                                                                                      |
| ---------------- | ----------- | ------------------------------------------------------------------------------------------------ |
| `label`          | `ReactNode` | The label above or beside the field. Required for accessibility unless `aria-label` is provided. |
| `placeholder`    | `string`    | Placeholder text when the field is empty. Supplement labels — do not use as a substitute.        |
| `description`    | `ReactNode` | Persistent helper text below the field.                                                          |
| `errorMessage`   | `ReactNode` | Error message shown when `validationState="invalid"`.                                            |
| `contextualHelp` | `ReactNode` | A `<ContextualHelp>` element next to the label.                                                  |

### Value / State

| Prop              | Type                       | Default | Description                                                                 |
| ----------------- | -------------------------- | ------- | --------------------------------------------------------------------------- |
| `value`           | `number`                   | —       | The current value (controlled). Use `NaN` to represent an empty field.      |
| `defaultValue`    | `number`                   | —       | The initial value (uncontrolled).                                           |
| `minValue`        | `number`                   | —       | The minimum allowed value. Clamped on blur.                                 |
| `maxValue`        | `number`                   | —       | The maximum allowed value. Clamped on blur.                                 |
| `step`            | `number`                   | `1`     | The increment/decrement amount for stepper buttons and arrow keys.          |
| `formatOptions`   | `Intl.NumberFormatOptions` | —       | Locale-aware display formatting (currency, percent, units, decimal places). |
| `hideStepper`     | `boolean`                  | `false` | Hides the increment/decrement stepper buttons.                              |
| `validationState` | `'valid' \| 'invalid'`     | —       | Visual and ARIA validation state.                                           |
| `isRequired`      | `boolean`                  | `false` | Adds a visual asterisk and `aria-required`.                                 |
| `isReadOnly`      | `boolean`                  | `false` | Value visible but not editable. Hides stepper buttons.                      |
| `isDisabled`      | `boolean`                  | `false` | Non-interactive and visually dimmed.                                        |
| `isQuiet`         | `boolean`                  | `false` | Underline-only border style.                                                |

### Events

| Prop       | Type                      | Description                                                                                 |
| ---------- | ------------------------- | ------------------------------------------------------------------------------------------- |
| `onChange` | `(value: number) => void` | Called when the value changes. Receives a `number` — not a string. `NaN` when empty.        |
| `onBlur`   | `(e: FocusEvent) => void` | Called when the field loses focus. Value is clamped to `minValue`/`maxValue` at this point. |

### Layout

| Prop            | Type              | Description                           |
| --------------- | ----------------- | ------------------------------------- |
| `labelPosition` | `'top' \| 'side'` | Label position relative to the input. |
| `width`         | `DimensionValue`  | The width of the entire component.    |

## Related components

- `Slider` — use instead when a visual range track is more intuitive than typing a value.
- `RangeSlider` — use instead when both a minimum and maximum numeric value must be selected.
- `TextField` — use instead for general text input.
