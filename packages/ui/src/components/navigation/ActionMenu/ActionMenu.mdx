## Responsibility

`ActionMenu` is a convenience component that combines a `MenuTrigger` and an icon-only `ActionButton` (typically with a "three dots" icon). It is the standard way to provide contextual actions for items in a list, table row, or card.

Key responsibilities:
- **Contextual Command Hub**: Groups related actions that can be performed on a specific object.
- **Space Efficiency**: Hides multiple actions behind a single trigger to reduce visual clutter.
- **Standardized Trigger**: Uses the system-standard icon and behavior for consistency.

### When to use
- For actions attached to a specific row in a table or card in a grid.
- When there are 3 or more secondary actions that would otherwise clutter the UI.
- To provide "More Actions" in a toolbar or header.

### When to avoid
- When the trigger should be a text button (use `Menu` + `MenuTrigger`).
- For navigation between different pages (use `Link` or `Tabs`).
- For a small number of primary actions that should always be visible (use `ActionButton` or `Button` directly).

## Variants

### Selection Mode
While most `ActionMenu` items are commands, you can also use `selectionMode="single"` or `selectionMode="multiple"` to allow users to select items from the menu (e.g., a "View Mode" setting).

### Size
Supports `S`, `M`, and `L` sizes for the trigger button.

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ItemElement \| ItemRenderer` | - | The menu items to display (using the `Item` or `Section` components). |
| `onAction` | `(key: Key) => void` | - | Handler called when a menu item is pressed. |
| `aria-label` | `string` | - | **Required.** An accessible label for the icon-only trigger button. |
| `size` | `'S' \| 'M' \| 'L'` | `'M'` | The size of the trigger button. |
| `isDisabled` | `boolean` | `false` | Whether the menu trigger is disabled. |
| `disabledKeys` | `Iterable<Key>` | `[]` | The keys of the items that should be disabled within the menu. |

## Accessibility

- **Icon Labeling**: Because the trigger is icon-only, an `aria-label` is mandatory. Without it, screen readers will only announce "button" with no context.
- **Keyboard Support**:
  - `Enter`, `Space`, or `Arrow Down` opens the menu.
  - `Arrow Down/Up` moves focus between menu items.
  - `Enter` or `Space` activates the focused item and closes the menu.
  - `Esc` closes the menu and returns focus to the trigger.
- **ARIA Roles**: The trigger has `role="button"` with `aria-haspopup="true"`. Menu items have `role="menuitem"`, `role="menuitemradio"`, or `role="menuitemcheckbox"` depending on the selection mode.

## Accessibility

### ARIA Roles

| Element | ARIA role | Notes |
| :--- | :--- | :--- |
| `ActionButton` trigger | `role="button"` | With `aria-haspopup="true"` and `aria-expanded` |
| Menu overlay container | `role="menu"` | Applied automatically by Spectrum |
| Each `<Item>` | `role="menuitem"` | Default when no `selectionMode` |
| Each `<Item>` (single select) | `role="menuitemradio"` | When `selectionMode="single"` |
| Each `<Item>` (multi select) | `role="menuitemcheckbox"` | When `selectionMode="multiple"` |
| `<Section>` group | `role="group"` | With `aria-label` from `title` prop |

### Keyboard Interactions

| Key | Behaviour |
| :--- | :--- |
| `Enter` / `Space` | Opens the menu when the trigger button is focused |
| `ArrowDown` | Opens the menu and moves focus to the first item |
| `ArrowUp` | Opens the menu and moves focus to the last item |
| `ArrowDown` / `ArrowUp` (open) | Moves focus between items |
| `Home` | Moves focus to the first item |
| `End` | Moves focus to the last item |
| `Enter` / `Space` (open) | Activates the focused item and closes the menu |
| `Escape` | Closes the menu and returns focus to the trigger |
| `Tab` | Closes the menu and moves focus to the next page element |
| Type-ahead | Typing a character jumps focus to the next matching item |

### Authoring Rules

- **Always provide `aria-label`**: The built-in trigger is icon-only (typically `⋯`). Without an `aria-label`, screen readers will announce only "button" with no context. Example: `<ActionMenu aria-label="Project actions">`.
- **Do not nest interactive elements inside `<Item>`**: The item itself is the interactive target. Any icons within an item should be decorative (`aria-hidden`).
- **Disabled items remain visible**: Prefer `disabledKeys` over omitting items to avoid confusing screen reader users who expect certain actions to be present.
- **Mobile tray**: On touch/mobile viewports, Spectrum automatically renders the menu in a bottom tray — no code change required.
