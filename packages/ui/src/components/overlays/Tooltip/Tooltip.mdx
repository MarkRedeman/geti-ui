# Tooltip

<Subtitle>Tooltips are brief, non-interactive labels that appear on hover or focus of a trigger element.</Subtitle>

## Responsibility

The `Tooltip` component provides contextual information about an element when the user hovers over it or focuses it. It is primarily used to provide a text label for icon-only buttons or to explain the purpose of a UI element.

Key responsibilities:
- **Contextual Labeling**: Provides a visible text label for elements that lack them (like icon buttons).
- **Progressive Disclosure**: Keeps the UI clean by hiding non-essential labels until needed.
- **Accessibility**: Automatically associates the tooltip text with the trigger element via `aria-describedby` or `aria-labelledby`.

### When to use
- For icon-only buttons (`ActionButton`) to provide their name/action.
- To provide a brief explanation of a UI element's function.
- When space is limited and a permanent label would clutter the interface.

### When to avoid
- For interactive content (tooltips must be non-interactive).
- For long descriptions or complex help text (use `ContextualHelp` instead).
- For essential information that must be visible at all times.
- On disabled elements, as tooltips are often inaccessible to keyboard and screen reader users in those cases.

## Variants

### Placement
Tooltips can be placed on any side of the trigger (`top`, `bottom`, `left`, `right`). They will automatically flip if there is not enough space.

### Semantic Variants (V3 only)
React Spectrum V3 supports semantic variants (`neutral`, `positive`, `negative`, `info`) which add a colored background and optional icon. Note that these are being deprecated in Spectrum 2.

## Props

### Tooltip Props
| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `children` | `ReactNode` | - | The text or content of the tooltip. |
| `variant` | `'neutral' \| 'positive' \| 'negative' \| 'info'` | `'neutral'` | The semantic variant of the tooltip (V3 only). |
| `showIcon` | `boolean` | `false` | Whether to show a status icon (requires a semantic variant). |

### TooltipTrigger Props
| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `delay` | `number` | `1500` | The delay in milliseconds before the tooltip appears. |
| `trigger` | `'hover' \| 'focus'` | - | Which event triggers the tooltip. |
| `isDisabled` | `boolean` | `false` | Whether the tooltip is disabled. |
| `placement` | `'top' \| 'bottom' \| 'left' \| 'right' \| ...` | `'top'` | The placement relative to the trigger. |

## Accessibility

- **Keyboard Support**: Tooltips appear when the trigger element receives keyboard focus.
- **ARIA**: `TooltipTrigger` manages the `aria-describedby` relationship automatically.
- **Timing**: Follows standard "warmup" and "cooldown" timers to prevent multiple tooltips from flashing as the user moves their mouse.

## Spectrum 2 Migration

The migration to Spectrum 2 (S2) involves simplifying the Tooltip component and moving some configuration.

### Key Changes
- **Simplified Tooltip**: S2 Tooltips no longer support semantic `variant` or `showIcon`. They are strictly for plain text labels.
- **Prop Relocation**: The `placement` prop moves exclusively to the `TooltipTrigger` in S2.
- **Disabled State Accessibility**: S2 emphasizes that tooltips on disabled elements are not accessible. For help text on disabled buttons, `ContextualHelp` is the recommended alternative.

### Migration Example

**V3 Style (Current):**
```tsx
<TooltipTrigger placement="bottom">
  <ActionButton aria-label="Delete"><Delete /></ActionButton>
  <Tooltip variant="negative" showIcon>Delete this item</Tooltip>
</TooltipTrigger>
```

**S2 Style (Future):**
```tsx
<TooltipTrigger placement="bottom">
  <ActionButton aria-label="Delete"><Delete /></ActionButton>
  <Tooltip>Delete this item</Tooltip>
</TooltipTrigger>
```
