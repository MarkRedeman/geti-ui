# Charts Overview

`@geti-ui/charts` provides Geti-themed chart components built on top of [Recharts](https://recharts.org). All components are dark-mode first and integrate with the Geti design token system.

## What's included

| Component | Description |
|---|---|
| `LineChart` | Time-series and continuous data trends |
| `AreaChart` | Continuous trends with optional filled areas |
| `BarChart` | Categorical comparisons, stacked or grouped |
| `ScatterChart` | Correlation between two continuous variables |
| `PieChart` | Proportional slices for single-series distribution |
| `DonutChart` | Pie variant with center hole for compact composition |
| `RadarChart` | Multi-metric profile comparison across categories |
| `RadialBarChart` | Circular progress-style category comparison |
| `MeterChart` | Half-donut KPI gauge with center value label |
| `TreemapChart` | Hierarchical proportional data as nested rectangles |
| `Sparkline` | Compact inline trend indicator with no axes |

## Primitives vs compositions

- **Primitives** are low-level, themed building blocks (axes, grid, tooltip, legend, responsive container, sparkline) that you compose manually inside Recharts charts.
- **Compositions** are higher-level ready-to-use chart components (`LineChart`, `AreaChart`, `BarChart`, `ScatterChart`, `PieChart`, `DonutChart`, `RadarChart`, `RadialBarChart`, `MeterChart`, `TreemapChart`, `Sparkline`) that already wire common chart parts for you.

### In practice

- Use **compositions** when your chart fits the standard APIs and you want fast implementation with sensible defaults.
- Use **primitives** when you need custom chart structures, mixed chart types, or low-level Recharts control.

```tsx
// Composition: batteries included
<LineChart data={data} xAxisKey="epoch" series={[{ dataKey: 'loss' }]} />

// Primitive approach: you wire parts manually
<GetiResponsiveContainer height={260}>
  <ComposedChart data={data}>
    <GetiCartesianGrid />
    <GetiXAxis dataKey="epoch" />
    <GetiYAxis />
    <GetiTooltip />
    <Line dataKey="loss" isAnimationActive={false} />
  </ComposedChart>
</GetiResponsiveContainer>
```

## Design principles

### Animation policy

**Animations are off by default.** Enabling them is opt-in per chart via the `animate` prop. This avoids unnecessary motion in dense dashboards and respects `prefers-reduced-motion`.

```tsx
// Animations off (default)
<LineChart data={data} series={[{ dataKey: 'value' }]} />

// Animations on (opt-in)
<LineChart data={data} series={[{ dataKey: 'value' }]} animate />
```

### Accessibility

- Provide an `aria-label` (or `aria-labelledby`) so assistive technologies can identify chart purpose.
- Recharts accessibility focus layer is disabled in composition charts to avoid distracting focus rectangles in dense dashboards.

### Axis scales

All composition charts (`LineChart`, `BarChart`, `AreaChart`, `ScatterChart`) accept `xScale` and `yScale` props to change the axis scale type. The default is linear/category — no changes needed for most charts.

```tsx
// Log scale on the Y axis (e.g. loss curves)
<LineChart
  data={data}
  xAxisKey="epoch"
  series={[{ dataKey: 'loss', name: 'Loss' }]}
  yScale={{ scale: 'log', domain: [0.001, 'auto'] }}
/>
```

Supported scale types: `'linear'`, `'log'`, `'sqrt'`, `'pow'`, `'time'`, `'band'`, `'point'`. See the [Compositions reference](./compositions) for full prop details and examples.

> **Log scale caveat:** all data values must be strictly positive (> 0). Set an explicit `domain` to avoid rendering artifacts near zero.

### Responsiveness

Charts fill their container width by default (`width="100%"`). Wrap in a sized container:

```tsx
<div style={{ width: '100%', height: 300 }}>
  <LineChart data={data} series={[{ dataKey: 'value' }]} />
</div>
```

## Quick example

```tsx preview
import { LineChart } from '@geti-ui/charts';

const data = [
  { month: 'Jan', accuracy: 72 },
  { month: 'Feb', accuracy: 78 },
  { month: 'Mar', accuracy: 81 },
  { month: 'Apr', accuracy: 85 },
  { month: 'May', accuracy: 88 },
  { month: 'Jun', accuracy: 91 },
];

export default function App() {
  return (
    <div style={{ width: '100%', height: 280 }}>
      <LineChart
        data={data}
        xAxisKey="month"
        series={[{ dataKey: 'accuracy', name: 'Accuracy (%)' }]}
        aria-label="Model accuracy over time"
      />
    </div>
  );
}
```

```tsx preview
const data = [
```

```tsx preview
}
```

## Primitives quick example

```tsx preview
import { ComposedChart, Line } from 'recharts';
import {
  GetiCartesianGrid,
  GetiXAxis,
  GetiYAxis,
  GetiTooltip,
  GetiLegend,
  GetiResponsiveContainer,
} from '@geti-ui/charts/primitives';

const data = [
  { t: '09:00', valA: 42, valB: 55 },
  { t: '10:00', valA: 58, valB: 63 },
  { t: '11:00', valA: 53, valB: 60 },
  { t: '12:00', valA: 67, valB: 72 },
  { t: '13:00', valA: 71, valB: 77 },
];

export default function App() {
  return (
    <GetiResponsiveContainer height={260}>
      <ComposedChart data={data}>
        <GetiCartesianGrid />
        <GetiXAxis dataKey="t" />
        <GetiYAxis />
        <GetiTooltip />
        <GetiLegend />
        <Line type="monotone" dataKey="valA" stroke="#4C9BE8" dot={false} isAnimationActive={false} />
        <Line type="monotone" dataKey="valB" stroke="#5CE6A2" dot={false} isAnimationActive={false} />
      </ComposedChart>
    </GetiResponsiveContainer>
  );
}
```
