## Responsibility

`ListView` is the right choice when users need to **interact with a collection of items** as individual entities—navigating into them, selecting them, or performing actions on them. It is distinct from simpler list primitives in that it separates **row selection** from **row actions**: a row can be opened or actioned (via `onAction`) independently of whether it is selected.

### When to use ListView
- **Rich Content**: When items have thumbnails, descriptions, metadata, or secondary actions (e.g., a file browser).
- **Batch Operations**: When users need to select one or many items and act on the selection as a batch.
- **Independent Actions**: When rows need an independent action (e.g., double-click to open) separate from selection.
- **Drag and Drop**: When the list supports reordering or cross-list transfers.

### Comparison with other components
- **vs. ListBox**: Use `ListBox` for simple selection inside a form where no nested interactive elements are needed.
- **vs. TableView**: Use `TableView` for structured, multi-column data that needs sorting or comparison across fields.
- **vs. CardView**: Use `CardView` for visual, card-style asset browsing with grid or waterfall layouts.

## Variants

### Density
`ListView` supports three density variants to balance scanability and information depth:
- `compact`: Minimizes vertical space.
- `regular`: The default spacing.
- `spacious`: Adds extra padding for more relaxed layouts.

### Selection Modes
- `none`: No selection allowed.
- `single`: One item can be selected.
- `multiple`: Multiple items can be selected using checkboxes or highlight selection.

## Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `children` | `ItemElement \| ItemRenderer` | — | The items to display in the list. |
| `selectionMode` | `'none' \| 'single' \| 'multiple'` | `'none'` | The selection mode for the list. |
| `onAction` | `(key: Key) => void` | — | Handler that is called when an item is "activated" (e.g., clicked or pressed Enter). |
| `density` | `'compact' \| 'regular' \| 'spacious'` | `'regular'` | The amount of vertical space each row takes up. |
| `loadingState` | `'loading' \| 'sorting' \| 'loadingMore' \| 'error' \| 'idle'` | — | The current loading state of the list. |
| `onLoadMore` | `() => void` | — | Handler called when the list is scrolled to the bottom. |
| `dragAndDropHooks` | `DragAndDropHooks` | — | Hooks to enable drag and drop functionality. |

## Accessibility

### Roles and Navigation
- `ListView` renders as `role="grid"`. Each row is `role="row"` with cells as `role="gridcell"`.
- **Keyboard Navigation**: Use `Up/Down Arrows` to move focus between rows.
- **Selection**: 
    - In `toggle` mode (default for multiple selection), `Space` toggles selection, and `Enter` triggers `onAction`.
    - In `replace` mode (default for single selection), `Up/Down Arrows` move both focus and selection together.
- **Required Labels**: Always provide an `aria-label` or `aria-labelledby` on the `ListView`.

### Focus Management
`ListView` uses a roving tabindex strategy, meaning the entire list is a single tab stop. Once focus enters the list, users navigate between items using arrow keys.
