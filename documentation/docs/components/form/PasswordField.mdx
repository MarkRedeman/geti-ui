# PasswordField

PasswordField is a masked text input with a show/hide toggle. Use it for login forms, account creation, and any field where the user enters a sensitive credential.

```tsx
import { PasswordField } from '@geti/ui';
```

## Responsibility

`PasswordField` is a Geti-native component built on top of Spectrum's `TextField`. It adds:

- a **show/hide toggle button** that switches between `type="password"` and `type="text"`,
- an optional **password rules hint** when `isNewPassword` is `true`,
- a custom `error` prop that renders an alert message with ARIA live region,
- proper ARIA linkage between the field and its error/hint messages via `aria-describedby`.

## Variants and states

| State/variant | Prop | Notes |
|---|---|---|
| New password hint | `isNewPassword` | Shows "Password must be at least 8 characters" hint |
| Custom error | `error` | Renders styled error message below field |
| Disabled | `isDisabled` | Non-interactive |

## Props (deep dive)

### Content

- `label`: visible field label.
- `placeholder`: input placeholder text.

### Behaviour

- `isNewPassword`: shows a hint that the password must be at least 8 characters.
- `error`: a custom error string. Rendered as a `role="alert"` below the input.

### Events

- `onChange`: inherited from Spectrum `TextField` — called with the current string value.

### Accessibility

- the show/hide toggle has `aria-label="Show password"` / `"Hide password"` for screen readers,
- `error` is linked via `aria-describedby` and rendered as `role="alert"`,
- the hint for `isNewPassword` is also linked via `aria-describedby`.

## Related components

- `TextField` — use for non-sensitive text inputs.
- `Form` — wraps password fields with other inputs in a submit flow.
