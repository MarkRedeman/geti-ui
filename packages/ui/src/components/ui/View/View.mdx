## Responsibility

The `View` component is the Spectrum-aware equivalent of a plain `<div>`. It acts as a semantically-neutral container that allows you to apply Spectrum design tokens (padding, border, background color, etc.) directly as props, without writing custom CSS.

### Key Capabilities
- **Theme Awareness**: Props like `backgroundColor="gray-100"` or `borderColor="dark"` automatically adapt to the active color scheme (dark/light mode).
- **Scale Awareness**: Dimension tokens like `padding="size-200"` automatically scale based on the user's platform (e.g., larger for touch, smaller for mouse).
- **Responsive Layouts**: Supports the `Responsive<T>` prop shape (e.g., `width={{ base: 'size-1000', L: 'size-2000' }}`) for breakpoint-aware sizing.
- **Logical Properties**: Supports RTL-aware spacing through logical props like `paddingStart`, `paddingEnd`, `marginStart`, and `marginEnd`.

---

## Variants

### Bordered Container
Use `borderWidth`, `borderColor`, and `borderRadius` to create framed sections or cards.

### Custom Layout
Combine `width`, `height`, and `position` props to establish layout contexts for nested components.

## Props

### Content

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | The content to be rendered inside the container. |
| `elementType` | `string \| ElementType` | `'div'` | The HTML element to render (e.g., `'section'`, `'article'`, `'aside'`). |

### Style Tokens

| Category | Key Props | Examples |
|----------|-----------|----------|
| **Background** | `backgroundColor` | `'gray-100'`, `'accent'`, `'negative'`, `'transparent'` |
| **Spacing** | `padding`, `margin` | `'size-50'`, `'size-200'`, `'size-400'` |
| **Border** | `borderWidth`, `borderColor`, `borderRadius` | `'thin'`, `'dark'`, `'medium'`, `'large'` |
| **Sizing** | `width`, `height`, `minWidth`, `maxWidth` | `'size-2000'`, `'100%'`, `400` |
| **Layout** | `flex`, `gridArea`, `overflow`, `isHidden` | Flex/Grid child and visibility controls. |

### Positioning

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `position` | `'static' \| 'relative' \| 'absolute' \| 'fixed' \| 'sticky'` | `'static'` | CSS positioning mode. |
| `top` / `bottom` / `left` / `right` / `start` / `end` | `DimensionValue` | — | Positional offsets using Spectrum tokens. |

## Accessibility

- **Generic Semantics**: By default, `View` renders as a `div` with no ARIA role.
- **Semantic Overrides**: Use the `elementType` prop to render more descriptive HTML5 landmark elements (e.g., `elementType="section"` or `elementType="main"`) when appropriate for document structure.
- **Roles**: Standard ARIA attributes (e.g., `role`, `aria-label`) can be passed through via DOM props.

---

## Spectrum 2 Migration

`View` **does not exist** in Spectrum 2 (`@react-spectrum/s2`). In the S2 ecosystem, the component is fully replaced by a plain `<div>` (or other HTML elements) styled with the **`style` macro**.

### Migration Example

<Source
  language="tsx"
  dark
  code={`
// v3 View
<View padding="size-200" borderRadius="medium" backgroundColor="gray-100">
  {children}
</View>

// S2 Replacement
import { style } from '@react-spectrum/s2/style' with { type: 'macro' };

<div className={style({ padding: 16, borderRadius: 'lg', backgroundColor: 'gray-100' })}>
  {children}
</div>
`}
/>

### Key Migration Deltas
- **Breakpoints**: `base` becomes `default`, `S` becomes `sm`, `M` becomes `md`, and `L` becomes `lg`.
- **Dimensions**: String tokens like `'size-200'` are replaced by numeric pixel values (e.g., `16`) in the style macro.
- **Borders**: Named width tokens like `'thin'` are replaced by numbers (e.g., `1`).
