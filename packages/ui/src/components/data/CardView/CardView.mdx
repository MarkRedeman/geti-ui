<Subtitle>A responsive grid collection for rendering visually rich cards.</Subtitle>

```tsx
import { CardView, Card } from '@geti/ui';
```

## Responsibility

`CardView` is used to render a collection of items as a responsive grid of cards. It is ideal for **visually rich content** such as images, media assets, user profiles, or product cards where 2D spatial browsing is more effective than a linear list.

### When to use CardView
- **Media-heavy objects**: When images or previews are the primary way users identify items.
- **2D Spatial Layout**: When users benefit from browsing a grid rather than scanning a vertical list.
- **Comparison at a glance**: When items have roughly equal conceptual weight and need to be compared visually.

### Comparison with other collections
- **vs. ListView**: Use `ListView` for text-heavy data where scannable density and linear order are more important.
- **vs. TableView**: Use `TableView` when items have multiple attributes that need to be compared across columns (sorting/filtering).

## Design

`CardView` is built as a flexible CSS Grid container that maps an array of `items` to a collection of `Card` components. 

- **Internal Anatomy**: The component provides a root `<div>` with `display: grid` and uses the `renderCard` prop to instantiate individual cards.
- **Responsive Behavior**: By default, it uses a fixed number of `columns`, but can be customized for different viewport sizes using standard layout props.
- **Layout Tokens**: It defaults to a `16px` gap, aligning with the Geti design system's spacing scale.

## Flow

1. **Data Ingestion**: The component receives an array of generic `items<T>`.
2. **Card Mapping**: For each item, the `renderCard` callback is executed, returning the props for a `Card`.
3. **Grid Rendering**: The grid container manages the positioning and spacing of the resulting card elements.
4. **Interaction**: User interactions (clicks, selection) are handled within the individual `Card` instances or by parent selection state.

## Integration

### Paired with ActionBar
To enable bulk actions in a `CardView`, it must be wrapped in an `ActionBarContainer`.

```tsx
<ActionBarContainer>
  <CardView
    items={assets}
    renderCard={(asset) => ({
      title: asset.name,
      isSelected: selectedKeys.has(asset.id),
      onSelectionChange: (selected) => handleSelection(asset.id, selected)
    })}
  />
  <ActionBar selectedItemCount={selectedKeys.size}>
    <Item key="download">Download</Item>
  </ActionBar>
</ActionBarContainer>
```

### Empty States
When the `items` array is empty, pair `CardView` with an `IllustratedMessage` to provide feedback to the user.

## Accessibility

`CardView` ensures that visually rich collections remain accessible to screen readers and keyboard users.

| Feature | Implementation |
| :--- | :--- |
| **Roles** | The container uses `role="list"` and each card is wrapped in a `role="listitem"`. |
| **Labels** | **Required:** Always provide an `aria-label` to the `CardView` for screen reader context. |
| **Navigation** | Supports standard `Tab` navigation between interactive cards. |
| **Interactive Content** | Ensure each card has a clear accessible name (via its title) and that any images have descriptive `alt` text. |

## Variants

### Custom Grid Layout
The current implementation uses a flexible CSS Grid layout. You can configure the density using the `columns` and `gap` props.

### Card Types
While `CardView` manages the collection, the individual cards are typically rendered using the `Card` component from `@geti/ui`. Depending on the use case, these can be:
- **Asset Cards**: For files, images, or media.
- **User Cards**: For people or profiles.
- **Interactive Cards**: Cards that act as buttons or links.

## Props

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `items` | `T[]` | — | **Required.** The collection of data items to render. |
| `renderCard` | `(item: T, index: number) => CardProps` | — | **Required.** A function that maps each data item to the props for a `Card` component. |
| `aria-label` | `string` | — | **Required.** An accessible label for the card collection. |
| `columns` | `number` | `3` | The number of columns in the grid layout. |
| `gap` | `string` | `'16px'` | The CSS gap between cards. |

## Accessibility

### Roles and Navigation
- The container uses `role="list"` and each card is wrapped in a `role="listitem"`. 
- **Required Labels**: Always provide an `aria-label` to the `CardView` so screen readers can identify the collection.
- **Card Content**: Ensure each card has a clear accessible name (via its title or a `textValue`) and that any images have descriptive `alt` text.

### Keyboard Support
In the current implementation, keyboard navigation follows the standard tab order of the cards.
