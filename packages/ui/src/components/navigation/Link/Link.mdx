# Link

An accessible hyperlink that integrates with Geti's theme and supports inline, quiet, and over-background display styles.

```tsx
import { Link } from '@geti-ai/ui';
```

- [React Spectrum Link](https://react-spectrum.adobe.com/react-spectrum/Link.html)

<Stories />

## Responsibility

`Link` registers Adobe React Spectrum's `Link` in Geti's public API surface, providing a consistently styled, accessible anchor that works in both standard navigation and client-side router contexts.

Its purpose is to give users a recognisable, underlined hyperlink affordance that is unambiguously distinct from a `Button`. Where a `Button` triggers an action within the current context, a `Link` navigates the user to a new location — whether that is an external URL, a different route, or an anchor within the same page.

Key characteristics:

- **Variants for context**: `primary` (accent-coloured), `secondary` (subdued), and `overBackground` (white, for use on coloured or image backgrounds) cover the three most common placement scenarios.
- **Quiet mode**: `isQuiet` removes the underline until hover or focus, reducing visual noise in dense layouts while preserving the affordance on interaction.
- **Router integration**: Spectrum's `Link` is intentionally router-agnostic. Pass `href` for standard anchor behaviour; supply `onPress` to intercept navigation and delegate to a client-side router (e.g. React Router's `navigate`).
- **Theme aware**: Visual tokens are resolved by `ThemeProvider`; no component-level CSS overrides are needed.

### When to use

- For inline text links within body copy, form helpers, or error messages (e.g. `"Forgot password?"`, `"Learn more"`).
- For navigation links in breadcrumbs, footers, and sidebars where the destination is a URL or route.
- When the action is navigational in nature and `<a>` semantics are correct.
- Use `overBackground` when the link appears on a coloured panel, image thumbnail, or dark banner where the default colour would have insufficient contrast.

### When to avoid

- When the element triggers an action rather than navigating — use `Button` or `ActionButton` instead.
- When the full content area should be clickable (e.g. a project card) — use `Card` with `onPress`.
- Do not use `Link` as a styling convenience for text that isn't actually navigable; this breaks the semantic contract and confuses screen reader users.
- Avoid placing links immediately adjacent to buttons that serve the same purpose; this creates redundant focus stops.

## Design

`Link` is a direct Spectrum pass-through. `LinkProps extends SpectrumLinkProps` with zero additions or overrides:

```tsx
import { Link as SpectrumLink, SpectrumLinkProps } from '@adobe/react-spectrum';

export interface LinkProps extends SpectrumLinkProps {}

export const Link = (props: LinkProps) => <SpectrumLink {...props} />;
```

No state, no effects, no refs. Spectrum handles focus ring rendering, press state styling, and anchor attribute forwarding.

### Usage examples

```tsx
// Standard navigation link
<Link href="https://docs.example.com">Read the documentation</Link>

// Quiet link — underline only on hover/focus
<Link href="/settings" isQuiet>Account settings</Link>

// Secondary variant — subdued colour for supporting text
<Link variant="secondary" href="/help">
  Need help?
</Link>

// Over a coloured background — renders in white
<View backgroundColor="informative" padding="size-200">
  <Link variant="overBackground" href="/learn-more">
    Learn more about this feature
  </Link>
</View>

// Client-side routing via onPress
<Link
  onPress={(e) => {
    e.continuePropagation();
    navigate('/projects/42');
  }}
>
  Open project
</Link>
```

All `SpectrumLinkProps` are accepted as-is. The primary props are:

| Prop         | Type                                           | Default     | Description                                                                                                                    |
| ------------ | ---------------------------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------------------------ |
| `children`   | `ReactNode`                                    | —           | The visible link text. Should be descriptive enough to make sense out of context (avoid `"click here"`).                       |
| `href`       | `string`                                       | —           | The destination URL. Omit when using `onPress` for client-side routing.                                                        |
| `variant`    | `'primary' \| 'secondary' \| 'overBackground'` | `'primary'` | Visual treatment of the link colour.                                                                                           |
| `isQuiet`    | `boolean`                                      | `false`     | Removes the underline until hover or focus.                                                                                    |
| `target`     | `string`                                       | —           | Standard anchor `target` attribute (e.g. `'_blank'`). When using `target="_blank"`, add `rel="noopener noreferrer"` via `rel`. |
| `rel`        | `string`                                       | —           | Standard anchor `rel` attribute.                                                                                               |
| `onPress`    | `(e: PressEvent) => void`                      | —           | Called on press; use for client-side navigation.                                                                               |
| `isDisabled` | `boolean`                                      | `false`     | Renders the link non-interactive and visually muted.                                                                           |

## Accessibility

- **Role**: Spectrum renders `Link` as an `<a>` element. Screen readers announce it with the `link` role and the link text. Avoid using generic text like `"here"` or `"click"` — the link text is the primary accessible name.
- **Keyboard navigation**: Links receive focus via `Tab` and are activated with `Enter`. Unlike buttons, `Space` does not activate a link; ensure your `onPress` integration does not break this convention.
- **Descriptive text**: Write link labels that are meaningful in isolation. Screen reader users often navigate by extracting a list of all links on a page; `"Learn more about annotation guidelines"` is far more useful than `"Learn more"`.
- **External links**: When `target="_blank"`, consider appending `" (opens in new tab)"` to the link text or providing an `aria-label` that conveys this. Some users are disoriented by unexpected context switches.
- **Disabled links**: `isDisabled` removes the link from the tab order and applies `aria-disabled="true"`. This is appropriate when the destination exists but is temporarily unavailable; if the link should not exist at all, remove it from the DOM instead.
- **`overBackground` contrast**: The white-on-colour rendering of `overBackground` must meet WCAG AA contrast requirements (4.5:1 for normal text). Test the specific background colour in your usage context.

## Related components

- **`Button`** / **`ActionButton`** — use when the element triggers an action in the current context rather than navigating to a new location; never use `Link` as a styled button.
- **`Breadcrumbs`** — use when the link is part of a structured navigation trail showing the user's position in the hierarchy.
- **`Menu`** / **`ActionMenu`** — use when multiple navigation targets should be grouped behind a single trigger rather than presented as individual inline links.
- **`Tabs`** — use when switching between sibling sections of the same page; prefer `Link` only when navigation changes the URL.

## Example use cases

<details>
  <summary>Typical use cases</summary>

- Providing a "Forgot password?" inline text link below a login form — `<Link href="/reset-password" isQuiet>Forgot password?</Link>`.
