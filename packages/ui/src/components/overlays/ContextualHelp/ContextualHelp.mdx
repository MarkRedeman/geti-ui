# ContextualHelp

A non-blocking overlay for supplemental guidance about a UI element or task.

```tsx
import { ContextualHelp, Heading, Content, Footer, Link } from '@geti/ui';
```

- [React Spectrum V3 ContextualHelp](https://react-spectrum.adobe.com/react-spectrum/ContextualHelp.html)

## Responsibility

`ContextualHelp` provides in-context, non-blocking guidance for users. It renders as a quiet icon button (either a `?` or an `â„ą`) that opens a popover-style dialog when clicked.

### When to use
- **Supplemental Guidance**: Use when a label, field, or UI element needs a deeper description than a simple tooltip can provide.
- **Task Concepts**: Use to explain a task, concept, or permission, optionally including a "Learn more" link.
- **Non-interactive Elements**: Ideal for providing context on plain text or disabled buttons that cannot receive focus for tooltips.

### Comparison with other components
| Situation | Use instead |
| :--- | :--- |
| Brief, transient label for an icon button | `Tooltip` |
| Persistent, inline feedback (error/warning) | `InlineAlert` |
| Modal workflow requiring a decision | `Dialog` or `AlertDialog` |

### Usage Rules
- **Heading and Content**: Both are required for a complete help dialog.
- **Footer Link**: Use the `<Footer>` slot with a standalone `<Link>` for "Learn more" links to external documentation.
- **Avoid Interaction**: Keep the popover content read-only. Do not put forms or complex interactive controls inside.

## Variants

### Help Variant (Default)
Renders a `?` icon. Best for in-depth guidance about a task or concept.

### Info Variant
Renders an `â„ą` icon. Best for brief, supplemental context about a specific label or feature.

## Accessibility

### Roles and Keyboard
- **Trigger**: The icon is a quiet `role="button"` that is fully keyboard-focusable.
- **Dialog**: The popover is a `role="dialog"` with `aria-modal="false"`, meaning it doesn't block interactions elsewhere but is clearly identified to screen readers.
- **Keyboard Navigation**:
    - `Enter / Space`: Opens or closes the popover.
    - `Escape`: Closes the popover and returns focus to the trigger icon.
    - `Tab`: Users can tab into the popover if it contains interactive elements like links.

### Touch Support
Unlike tooltips, `ContextualHelp` is triggered via press/click, making it accessible on touch screens where hover-only interactions are not available.
