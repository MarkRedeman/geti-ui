## Responsibility

The `Switch` component represents a binary choice (on/off) that typically takes effect immediately. It is commonly used in settings panels, preference toggles, and feature-flag controls.

### When to use
- For immediate activation/deactivation of a feature or setting.
- When the state change should happen without a "Submit" button.

### When to use something else
- Use [Checkbox](/docs/components-form-checkbox--docs) for selections within a list or form where changes take effect on submission.
- Use [ToggleButton](/docs/components-ui-primitives-togglebutton--docs) for a momentary toggle action, often within a toolbar.

## Variants

### States
- **Selected**: The "on" state. Use `isSelected` (controlled) or `defaultSelected` (uncontrolled).
- **Disabled**: The switch is non-interactive and visually dimmed. Use `isDisabled`.
- **Emphasized**: In Geti UI, the `isEmphasized` prop applies energy-blue design tokens to the track and handle, providing higher visual prominence.

### Validation
Note that `Switch` does **not** support an invalid or error state by design. It is used for binary activation, not for data entry that requires validation.

## Props

### Content
| Prop | Type | Description |
|---|---|---|
| `children` | `ReactNode` | The label text or content for the switch. |
| `aria-label` | `string` | An accessible label for the switch if no visible label is provided. |

### Value / State
| Prop | Type | Default | Description |
|---|---|---|---|
| `isSelected` | `boolean` | — | Whether the switch is in the "on" state (controlled). |
| `defaultSelected` | `boolean` | — | The initial "on" state (uncontrolled). |
| `isDisabled` | `boolean` | `false` | Whether the switch is disabled. |
| `isReadOnly` | `boolean` | `false` | Whether the switch value can be changed by the user. |

### Events
| Prop | Type | Description |
|---|---|---|
| `onChange` | `(isSelected: boolean) => void` | Handler called when the switch state is toggled. |

### Styles
| Prop | Type | Description |
|---|---|---|
| `isEmphasized` | `boolean` | Whether the switch should use a high-prominence (energy-blue) style. |
| `UNSAFE_className` | `string` | A custom CSS class name for the component. |

## Accessibility

- Renders as a visually-hidden `<input type="checkbox" role="switch">` inside a `<label>`.
- Full keyboard support: **Space** to toggle the switch, and **Tab** to move focus.
- Automatically handles state communication to screen readers via the `switch` role and `aria-checked` attribute.
- High contrast mode support via Adobe Spectrum.
