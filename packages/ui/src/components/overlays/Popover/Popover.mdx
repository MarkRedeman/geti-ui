# Popover

<Subtitle>Popovers are small, anchored overlays that appear relative to a trigger element and do not block the rest of the UI.</Subtitle>

```tsx
import { Popover } from '@geti/ui';
```

- [React Spectrum V3 Popover (via DialogTrigger)](https://react-spectrum.adobe.com/react-spectrum/DialogTrigger.html#popover)

<Stories />

## Responsibility

The `Popover` component provides a convenient shorthand for creating anchored overlays. In the current Geti UI implementation, it wraps `DialogTrigger` with a default `type="popover"`.

Key responsibilities:
- **Anchored Positioning**: Automatically calculates the best placement for the overlay relative to the trigger.
- **Non-blocking Interaction**: Allows the user to interact with the rest of the application while the popover is open.
- **Automatic Dismissal**: Closes when clicking outside the popover or pressing the `Esc` key.

### When to use
- For small menus or filters that should remain visible while the user works in the main area.
- For detail panels that provide extra context about a specific element.
- When you need an overlay that doesn't feel as heavy or disruptive as a modal dialog.

### When to avoid
- For critical tasks that require the user's full attention (use `Dialog` instead).
- For very simple, non-interactive text hints (use `Tooltip`).
- When the content is too large or complex for a small anchored window.

## Variants

### Placement
Popovers can be placed on any side of the trigger (`top`, `bottom`, `left`, `right`). The positioning engine will automatically flip the placement if there isn't enough space.

### Custom Popover
For cases requiring even more control over the overlay structure (e.g., adding an arrow or custom padding), Geti UI also provides a `CustomPopover` component based on `react-aria-components`.

## Props

`Popover` inherits all props from `DialogTrigger` but defaults the `type` to `'popover'`.

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `type` | `'modal' \| 'popover' \| 'tray' \| 'fullscreen'` | `'popover'` | The type of overlay to render. |
| `placement` | `'top' \| 'bottom' \| 'left' \| 'right' \| ...` | `'bottom'` | The preferred placement relative to the trigger. |
| `isOpen` | `boolean` | - | Whether the popover is open (controlled). |
| `onOpenChange` | `(isOpen: boolean) => void` | - | Handler called when the open state changes. |
| `hideArrow` | `boolean` | `true` | Whether to hide the arrow pointing to the trigger. |

## Accessibility

- **Keyboard Support**: `Esc` closes the popover. Focus is automatically managed within the popover content.
- **Aria Roles**: The trigger and overlay are linked via `aria-haspopup` and `aria-controls`.
- **Dismissal**: Clicking outside the popover follows the "light dismiss" pattern, which is standard for non-modal overlays.
