# PhotoPlaceholder

A deterministic initials-based placeholder used when a user or entity photo is unavailable.

```tsx
import { PhotoPlaceholder } from '@geti/ui';
```

<Stories />

## Responsibility

`PhotoPlaceholder` fills a functional gap in the Adobe Spectrum design system: providing a high-quality, initials-based fallback for missing images. While Spectrum's `Avatar` component is strictly image-only, `PhotoPlaceholder` ensures every entity has a distinct visual identity even without a profile photo.

Key capabilities:
- **Deterministic Color**: Background colors are hashed from an `indicator` string (e.g., email or ID), ensuring the same user always has the same color.
- **Initials Rendering**: Automatically extracts and displays the first initial from the `name` prop.
- **Smart Contrast**: Dynamically calculates whether to use light or dark text based on the background color's perceived luminance, ensuring WCAG-compliant contrast.
- **Flexible Shape**: Defaults to a circular "avatar" style but supports custom `borderRadius` for square or rounded-rect variants.

The color pipeline uses a perceptual luminance formula (ITU-R 601 weights): the `indicator` is hashed to a hex color, then `(r*299 + g*587 + b*114) / 1000` determines whether to render light or dark text. Centering is handled via CSS module Flexbox, while dynamic colors and radius are applied via `UNSAFE_style`. The component uses `role="img"` with `aria-label` set to the full `name` so screen readers announce the person's name; the visible initial is marked `aria-hidden="true"`.

## Variants

### Sizing
The component supports standard Spectrum dimension tokens for `width` and `height`. The default size is `size-1600` (64px).

### Custom Shape
Use the `borderRadius` prop to change the shape from the default circle (`50%`) to a rounded square or other geometric form.

## Props

<Controls of={DocStories.Default} />

### Content

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `name` | `string` | **Required** | The name of the person or entity. Extracts the first letter for display. |
| `indicator` | `string` | **Required** | A unique string (email, ID) used to deterministically pick a background color. |

### Value / Style overrides

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `width` | `string \| number` | `'size-1600'` | Spectrum dimension token or numeric pixel value for width. |
| `height` | `string \| number` | `'size-1600'` | Spectrum dimension token or numeric pixel value for height. |
| `borderRadius` | `string` | `'50%'` | CSS border-radius value (e.g., `'8px'`, `'100%'`). |

## Related components
- `Avatar` — use when an actual user photo is available; `PhotoPlaceholder` is its fallback partner.
- `AvatarGroup` — use when displaying a stack of multiple users; `PhotoPlaceholder` renders inside `Avatar` automatically on image error.
- `Image` — use for general-purpose images that are not person/entity representations.

## Example use cases
<details>
  <summary>Typical use cases</summary>

- Rendering an initials badge for a new user who has not uploaded a profile photo.
- Displaying a consistent color-coded placeholder for team members in a project member list.
- Showing a deterministic identity swatch in annotation tasks where user photos are not available.
- Using a square `borderRadius` variant to represent an organization or team entity rather than a person.
