# Menu

Menus display a list of actions or selections that users can choose from after interacting with a trigger.

```tsx
import { Menu, MenuTrigger, Item, Section } from '@geti/ui';
```

- [React Spectrum V3 Menu](https://react-spectrum.adobe.com/react-spectrum/Menu.html)

<Stories />

## Responsibility

`Menu` groups related commands or options into an overlay that appears on demand. It always works in tandem with a `MenuTrigger`, which owns the open/close state and wires the trigger element to the menu overlay.

Key responsibilities:

- **Command Dispatch**: Delivers a structured list of actions via `onAction` â€” fire-and-forget semantics.
- **Stateful Selection**: Supports single and multiple selection modes via `onSelectionChange` for settings and filters.
- **Visual Organisation**: Uses `Section` to group related items with optional titles and automatic dividers.
- **Focus Management**: Traps focus within the overlay and returns it to the trigger on close.

### When to use

- For a list of contextual actions associated with a specific UI element (e.g., an "Edit" button in a toolbar).
- For **overflow menus** â€” when a toolbar has more actions than can fit and the excess is collected behind a "More" trigger.
- For choosing from a small-to-medium list of options that should not permanently occupy screen space.
- For **filter or sort controls** where multiple settings are grouped under a single trigger.

### When to avoid

- When the selected value must remain visible after the menu closes â€” use `Picker` instead.
- When the list is long and requires search or filtering â€” use `ComboBox`.
- For primary site-wide navigation â€” use `VerticalNav` or `Tabs`; the `menu` ARIA role implies a command list, not a navigation landmark.
- When only two or three options are available and space permits â€” prefer `ActionGroup` or inline `Button`s to reduce interaction cost.

## Design

`Menu` is a **thin wrapper** around Spectrum's `SpectrumMenu`. It extends `SpectrumMenuProps` directly without adding props or modifying rendering, acting as a stable integration point for future migration.

```tsx
// packages/ui/src/components/navigation/Menu/Menu.tsx

export interface MenuProps<T extends object> extends SpectrumMenuProps<T> {}

export const Menu = <T extends object>(props: MenuProps<T>) => <SpectrumMenu {...props} />;
```

**`MenuTrigger` + `Menu` relationship**

`Menu` never manages its own visibility. Instead, `MenuTrigger` provides that context:

- The **first child** of `MenuTrigger` is the trigger element (any pressable component, typically `ActionButton` or `Button`).
- The **second child** is the `Menu` itself.

`MenuTrigger` renders no DOM node of its own â€” it is a pure behaviour provider that links the trigger's press event to the overlay lifecycle.

```tsx
<MenuTrigger>
    <ActionButton>Edit</ActionButton> {/* first child = trigger */}
    <Menu onAction={(key) => handle(key)}>
        {' '}
        {/* second child = menu */}
        <Item key="cut">Cut</Item>
        <Item key="copy">Copy</Item>
        <Item key="paste">Paste</Item>
    </Menu>
</MenuTrigger>
```

**Co-located re-exports**

The following primitives are re-exported from the `Menu` folder so consumers only need a single import source:

| Export        | Source                                  | Purpose                                                     |
| ------------- | --------------------------------------- | ----------------------------------------------------------- |
| `Menu`        | `Menu.tsx`                              | The menu overlay content.                                   |
| `MenuTrigger` | `MenuTrigger.tsx`                       | Open/close state manager and trigger linker.                |
| `Item`        | `Item.tsx` â†’ `@adobe/react-spectrum`    | Individual menu item (action or selectable option).         |
| `Section`     | `Section.tsx` â†’ `@adobe/react-spectrum` | Groups items with an optional title and a rendered divider. |

**Theme integration**

`Menu` inherits all Spectrum design tokens (background, border, typography, focus ring) from the `ThemeProvider` context. No per-component CSS overrides are needed in standard usage.

## Variants

### Selection Mode

- **None (default)**: Items are treated as commands. `onAction` fires with the item key on press; `onSelectionChange` is not used.
- **Single** (`selectionMode="single"`): One item can be active at a time, behaving like a radio group. `onSelectionChange` delivers a `Set` with the selected key.
- **Multiple** (`selectionMode="multiple"`): Any number of items can be selected simultaneously, behaving like a checkbox group.

### Sections and Dividers

Wrap groups of items in `Section` to produce a visible title and a separator line between groups. The `title` prop is optional; omitting it creates an untitled section with just a divider.

```tsx
<MenuTrigger>
    <ActionButton>Actions</ActionButton>
    <Menu onAction={(key) => handle(key)}>
        <Section title="Clipboard">
            <Item key="cut">Cut</Item>
            <Item key="copy">Copy</Item>
            <Item key="paste">Paste</Item>
        </Section>
        <Section title="Format">
            <Item key="bold">Bold</Item>
            <Item key="italic">Italic</Item>
            <Item key="underline">Underline</Item>
        </Section>
    </Menu>
</MenuTrigger>
```

Each `Section` renders as a `role="group"` with an `aria-label` derived from its `title` prop, satisfying WAI-ARIA grouping requirements automatically.

## Accessibility

`Menu` implements the full [WAI-ARIA Menu pattern](https://www.w3.org/WAI/ARIA/apg/patterns/menu/) via Spectrum's underlying implementation.

**ARIA roles**

| Element                     | Role                | Notes                                                                       |
| --------------------------- | ------------------- | --------------------------------------------------------------------------- |
| Menu overlay container      | `menu`              | Applied automatically by Spectrum to the overlay list.                      |
| Each `Item` (command mode)  | `menuitem`          | Default when `selectionMode="none"`.                                        |
| Each `Item` (single select) | `menuitemradio`     | When `selectionMode="single"`. Selected item has `aria-checked="true"`.     |
| Each `Item` (multi select)  | `menuitemcheckbox"` | When `selectionMode="multiple"`. Selected items have `aria-checked="true"`. |
| `Section` group container   | `group`             | With `aria-label` derived from the `title` prop.                            |
| Trigger element             | `button`            | With `aria-haspopup="menu"` and `aria-expanded` reflecting open state.      |

**Keyboard interactions**

| Key                                   | Behaviour                                                                |
| ------------------------------------- | ------------------------------------------------------------------------ |
| `Enter` / `Space` (trigger focused)   | Opens the menu and moves focus to the first item.                        |
| `Arrow Down` (trigger focused)        | Opens the menu and moves focus to the first item.                        |
| `Arrow Up` (trigger focused)          | Opens the menu and moves focus to the last item.                         |
| `Arrow Down` / `Arrow Up` (menu open) | Moves focus between items; wraps at list boundaries.                     |
| `Home`                                | Moves focus to the first item.                                           |
| `End`                                 | Moves focus to the last item.                                            |
| `Enter` / `Space` (item focused)      | Activates the focused item; closes the menu (command mode).              |
| `Escape`                              | Closes the menu and returns focus to the trigger.                        |
| `Tab`                                 | Closes the menu and moves focus to the next page element.                |
| Type-ahead                            | Typing a character moves focus to the next item matching that character. |

**Labelling the trigger**

The `MenuTrigger` component does not render a visible label itself. Ensure the trigger element has an accessible name via one of:

- Its text content (e.g., `<ActionButton>Edit</ActionButton>`).
- An explicit `aria-label` (e.g., `<ActionButton aria-label="More options">â€¦</ActionButton>`).

**Disabled items**

Always prefer `disabledKeys` over conditionally omitting items. Removing items from the DOM can confuse screen reader users who expect certain actions to exist (even if unavailable).

## Integration

- **Dependencies**: `@adobe/react-spectrum` â€” `Menu`, `MenuTrigger`, and their Spectrum prop types are imported from this package. `Item` and `Section` are re-exported from `@adobe/react-spectrum` via co-located shim files.
- **ThemeProvider**: Must be present in the React tree. Overlay background, item hover state, focus ring colour, and typography all derive from Spectrum tokens resolved by `ThemeProvider`.

### `onAction` vs `onSelectionChange`

These two callbacks serve fundamentally different purposes and should not be combined on the same `Menu`:

| Callback                  | Mode                                     | Use whenâ€¦                                                                                                                 |
| ------------------------- | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------- |
| `onAction(key)`           | `selectionMode="none"`                   | Each item is an **imperative command** â€” "Cut", "Delete", "Export". The menu has no persistent state.                     |
| `onSelectionChange(keys)` | `selectionMode="single"` or `"multiple"` | Items represent **persistent state** â€” "Sort by Name", "Show hidden files". The caller stores and reflects the selection. |

```tsx
// Command pattern â€” onAction
<Menu
    onAction={(key) => {
        if (key === 'delete') deleteItem(id);
        if (key === 'export') exportItem(id);
    }}
>
    <Item key="delete">Delete</Item>
    <Item key="export">Export</Item>
</Menu>;

// Stateful selection â€” onSelectionChange (controlled)
const [sort, setSort] = useState<Selection>(new Set(['name']));

<Menu selectionMode="single" selectedKeys={sort} onSelectionChange={setSort}>
    <Item key="name">Name</Item>
    <Item key="date">Date Modified</Item>
    <Item key="size">File Size</Item>
</Menu>;
```

**Pairing with `ActionButton`**

The most common trigger is `ActionButton`, which provides an accessible button with optional icon and label:

```tsx
import { ActionButton } from '@adobe/react-spectrum';
import { Menu, MenuTrigger, Item, Section } from '@geti/ui';

<MenuTrigger>
    <ActionButton aria-label="View options">
        {/* icon can be passed as a child */}
        More
    </ActionButton>
    <Menu onAction={(key) => console.log(key)}>
        <Section title="Layout">
            <Item key="grid">Grid view</Item>
            <Item key="list">List view</Item>
        </Section>
        <Section title="Sort">
            <Item key="name">By name</Item>
            <Item key="date">By date</Item>
        </Section>
    </Menu>
</MenuTrigger>;
```

**Pairing with `DialogTrigger`**

When a menu item should open a confirmation dialog, keep `MenuTrigger`/`Menu` and `DialogTrigger`/`Dialog` as separate sibling triggers. Handle the `onAction` callback to imperatively open the dialog via a controlled `isOpen` prop â€” do not nest `DialogTrigger` inside `Menu`.

```tsx
const [confirmOpen, setConfirmOpen] = useState(false);

<>
    <MenuTrigger>
        <ActionButton>Actions</ActionButton>
        <Menu
            onAction={(key) => {
                if (key === 'delete') setConfirmOpen(true);
            }}
        >
            <Item key="duplicate">Duplicate</Item>
            <Item key="delete">Deleteâ€¦</Item>
        </Menu>
    </MenuTrigger>

    <DialogTrigger isOpen={confirmOpen} onOpenChange={setConfirmOpen}>
        {/* trigger placeholder required by Spectrum */}
        <span />
        <AlertDialog
            title="Delete item?"
            variant="destructive"
            primaryActionLabel="Delete"
            onPrimaryAction={() => deleteItem(id)}
        >
            This action cannot be undone.
        </AlertDialog>
    </DialogTrigger>
</>;
```

## Related components

- **`ActionMenu`** â€” use when you want the standard "three dots" icon trigger bundled with the menu automatically; `Menu` + `MenuTrigger` gives you full control over the trigger element.
- **`Tabs`** â€” use when options represent co-equal sibling content sections that should be permanently visible rather than hidden behind a trigger.
- **`Picker`** â€” use when the selected value must remain visible in the trigger after the menu closes and the choice should look like a form field.
- **`ActionGroup`** â€” use when all actions or options should be permanently visible as a row of buttons without requiring a trigger click.
