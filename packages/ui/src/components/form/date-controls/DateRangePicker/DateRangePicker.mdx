# DateRangePicker

<Subtitle>
    A date range input that combines two segmented keyboard fields with a range calendar popover, letting users select a
    start and end date.
</Subtitle>

## Responsibility

`DateRangePicker` is a thin wrapper around Adobe React Spectrum's `DateRangePicker`. It applies the same Safari mobile width fix used by `DatePicker` before forwarding all props. It is the correct component whenever a user needs to specify both a start **and** an end date.

Key responsibilities:

- **Range Selection**: Renders two adjacent `DateField` segments (start and end) in a single composite field, plus a calendar button that opens a `RangeCalendar` popover. The calendar highlights the selected range across days.
- **Safari Mobile Fix**: Applies `width="size-2400"` before the props spread, mirroring the `DatePicker` fix. Callers can override with their own `width` prop.
- **Time Support**: When `granularity="minute"` (or finer), both the start and end fields extend to include time segments.
- **Validation**: Supports `minValue`, `maxValue`, `isDateUnavailable`, and the standard Spectrum form validation props. The end date is automatically constrained to be ≥ the start date.

### When to use

- Filter panels requiring a date range — "from / to" reporting windows, dataset time-range selectors.
- Booking or scheduling workflows where both bounds must be captured in a single action.
- Any form that requires a contiguous date span as a single field with a clear visual relationship between start and end.

### When to avoid

- When only a single date is needed — use `DatePicker` instead.
- When start and end dates are conceptually independent and should be labelled separately — use two `DatePicker` instances with explicit relationship logic.
- When the range can be unbounded on one side — consider two independent `DatePicker` fields for cleaner UX.

## Design

Identical implementation pattern to `DatePicker` — one hardcoded default before the spread:

```tsx
export const DateRangePicker = <T extends DateValue>(props: SpectrumDateRangePickerProps<T>) => (
    <SpectrumDateRangePicker width="size-2400" {...props} />
);

export type { SpectrumDateRangePickerProps as DateRangePickerProps };
```

No CSS modules, no local state, no custom logic. The `width` default can be overridden by the caller because `{...props}` comes after it.

## Integration

### Usage examples

```tsx
// Uncontrolled
<DateRangePicker label="Event range" />

// Controlled with RangeValue<CalendarDate>
import { today, getLocalTimeZone } from '@internationalized/date';

const [range, setRange] = useState({
  start: today(getLocalTimeZone()),
  end: today(getLocalTimeZone()).add({ weeks: 1 }),
});

<DateRangePicker
  label="Report period"
  value={range}
  onChange={setRange}
  minValue={today(getLocalTimeZone())}
/>
// → onChange fires: { start: CalendarDate, end: CalendarDate }

// With time granularity
<DateRangePicker
  label="Event range and time"
  granularity="minute"
/>

// Validation
<DateRangePicker
  label="Scheduled window"
  isRequired
  validationState="invalid"
  errorMessage="Both start and end dates are required."
/>
```

- **Depends on**: `@adobe/react-spectrum` — `DateRangePicker`, `SpectrumDateRangePickerProps`; `@react-types/datepicker` — `DateValue`
- **Peer imports**: `@internationalized/date` — `CalendarDate`, `today`, `getLocalTimeZone`; `@react-types/shared` — `RangeValue` for typing the `value`/`onChange` pair
- **Companion**: `DatePicker` (single date, same Safari fix); `RangeCalendar` (standalone calendar without the text fields)
- **Output shape**: `onChange` fires with `RangeValue<T>` — an object `{ start: T, end: T }` where `T extends DateValue`
- **Generic**: `<T extends DateValue>` inferred from `value` or `defaultValue` — `CalendarDate`, `CalendarDateTime`, or `ZonedDateTime`

### Props

| Prop                | Type                                      | Default       | Description                                                  |
| ------------------- | ----------------------------------------- | ------------- | ------------------------------------------------------------ |
| `value`             | `RangeValue<T>`                           | —             | The current date range (controlled). `{ start: T, end: T }`. |
| `defaultValue`      | `RangeValue<T>`                           | —             | The initial date range (uncontrolled).                       |
| `onChange`          | `(value: RangeValue<T>) => void`          | —             | Called when the user commits a range.                        |
| `label`             | `string`                                  | —             | The visible label for the field.                             |
| `granularity`       | `'day' \| 'hour' \| 'minute' \| 'second'` | `'day'`       | The finest time unit to display in each field.               |
| `minValue`          | `DateValue`                               | —             | The earliest selectable date.                                |
| `maxValue`          | `DateValue`                               | —             | The latest selectable date.                                  |
| `isDateUnavailable` | `(date: DateValue) => boolean`            | —             | Function to mark specific dates as unavailable.              |
| `isDisabled`        | `boolean`                                 | `false`       | Prevents interaction.                                        |
| `isReadOnly`        | `boolean`                                 | `false`       | Displays values but prevents changes.                        |
| `isRequired`        | `boolean`                                 | `false`       | Marks the field as required.                                 |
| `validationState`   | `'valid' \| 'invalid'`                    | —             | Controls inline validation styling.                          |
| `errorMessage`      | `ReactNode`                               | —             | Error message shown when `validationState="invalid"`.        |
| `width`             | `DimensionValue`                          | `'size-2400'` | Override the default width.                                  |

## Accessibility

- **ARIA role**: The composite is a `group` with `aria-label` or `aria-labelledby` from the `label` prop. Inside, start and end `DateField` inputs are each labelled "Start Date" and "End Date" respectively. Each segment within a field is a `spinbutton`.
- **Keyboard navigation for segments**: **Tab** moves between all segments of both start and end fields and the calendar toggle button. **Arrow Up/Down** adjusts the focused segment. Typing digits auto-advances to the next segment.
- **Calendar popover**: The calendar button is labelled `"Calendar"`. Focus enters the `RangeCalendar` grid on open. **Arrow keys** navigate days. **Enter/Space** sets the start date on first press, then the end date on second press. The range is highlighted as the end date is being selected. **Escape** closes the popover.
- **Range constraint**: If the user enters an end date earlier than the start date via keyboard, Spectrum surfaces a `validationState="invalid"` indicator automatically.
- **Error announcements**: `errorMessage` is linked via `aria-describedby` and announced when validation fails.
- **Labels**: Always provide `label`. If suppressed for visual design reasons, supply `aria-label` on the component to keep the `group` accessible.

## Related components

- `DatePicker` — use instead when only a single date (not a range) needs to be selected.
- `RangeCalendar` — use instead when the range calendar must be permanently visible rather than in a popover.
- `RangeSlider` — use instead when the range represents a continuous numeric value rather than calendar dates.
