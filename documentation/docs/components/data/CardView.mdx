# CardView

`CardView` renders a responsive grid of `Card` components from a data array. Use it when you need to display a collection of summary items — projects, models, datasets — in a fixed-column grid layout.

```tsx
import { CardView } from '@geti/ui';
```

- [Geti UI Theming](/components/theming)

## Responsibility

`CardView` is a Geti-authored layout component with no upstream Spectrum equivalent. It wraps the `Card` component and arranges items into a CSS grid. The `renderCard` function gives you full control over each card's content and interactive state.

`CardView` itself handles:
- grid layout and gap via CSS `grid-template-columns`,
- `role="list"` / `role="listitem"` semantics for the collection,
- stable keying via `getItemKey` (falls back to index).

It does **not** manage selection state — pass `isSelected` through `renderCard` from your own state.

## Variants and states

| Prop | Default | Notes |
|---|---|---|
| `columns` | `3` | Number of equal-width grid columns |
| `gap` | `'16px'` | Gap between cards, any CSS value |

Individual `Card` states (`isSelected`, `isDisabled`, `onPress`) are configured per-card through `renderCard`.

## Props (deep dive)

### Content

- `items`: the array of data items to render.
- `renderCard(item, index)`: returns a `CardProps` object for each item — controls `children`, `aria-label`, `isSelected`, `isDisabled`, and `onPress`.
- `getItemKey(item, index)`: optional stable key extractor; defaults to the item's array index.

### Layout

- `columns`: integer number of grid columns.
- `gap`: any valid CSS gap value.

### Accessibility

- `aria-label`: labels the collection as a whole (`role="list"`).
- Set `aria-label` on each card via `renderCard` so screen readers identify each item.

## Related components

- `Card` — the individual card unit; use directly when you are not rendering from a data array.
- `ListView` — use when items need keyboard selection with built-in focus management.
- `TableView` — use when items are tabular with sortable columns.
