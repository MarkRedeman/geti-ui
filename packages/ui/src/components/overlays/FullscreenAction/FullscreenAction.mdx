# FullscreenAction

A composite trigger that promotes any content into a fullscreen takeover dialog — combining a quiet icon button, a tooltip, and a structured fullscreen dialog with a close control into a single, reusable unit.

```tsx
import { FullscreenAction } from '@geti/ui';
```

<Stories />

## Responsibility

`FullscreenAction` is a **Geti-native composite pattern** for "expand to fullscreen" interactions. It encapsulates the entire expand/collapse workflow — trigger button, tooltip, dialog lifecycle, title header, divider, action button slot, and close button — so consuming components never need to assemble this pattern from scratch.

Key responsibilities:
- **Trigger composition**: Combines `TooltipTrigger`, `ActionButton` (with the `FullScreen` workflow icon), and a `Tooltip` into a single quiet, icon-only trigger button.
- **Dialog structure**: Renders a `fullscreenTakeover` `Dialog` with a `Heading`, `Divider`, `Content`, and `ButtonGroup` — matching Spectrum's expected fullscreen dialog anatomy.
- **Container ref forwarding**: Exposes a `containerRef` to the `actionButton` factory prop, enabling canvas-based or DOM-measurement scenarios where the fullscreen content needs a reference to its containing DOM node before rendering.
- **Action button slot**: Accepts either a static `ReactNode` or a factory function `(ref) => ReactNode` in the `actionButton` prop, supporting both simple and ref-dependent action placements.

### When to use
- To promote a panel, editor, image viewer, chart, or canvas into a dedicated fullscreen experience.
- When the fullscreen content requires additional toolbar actions alongside the close button (e.g., "Export", "Reset", "Compare") — use the `actionButton` slot.
- When a DOM reference to the fullscreen container is needed (e.g., a canvas renderer must append to the container node) — use the `actionButton` factory form.

### When to avoid
- For standard modal dialogs that are not tied to a "fullscreen" context — use `Dialog` with `DialogTrigger` directly.
- When the fullscreen trigger is not a `FullScreen` icon button — compose `DialogTrigger type="fullscreenTakeover"` manually with your own trigger.
- For lightweight contextual information overlays — use `Popover` or `ContextualHelp`.

## Design

Fully custom composite — not a Spectrum pass-through. Composes Spectrum primitives from `@adobe/react-spectrum` and workflow icons from `@spectrum-icons/workflow`:

- **`DialogTrigger type="fullscreenTakeover"`** — manages open state and applies Spectrum's fullscreen overlay animation.
- **`TooltipTrigger` + `ActionButton`** — quiet icon button (no border at rest) with `FullScreen` icon and "Fullscreen" tooltip.
- **`Dialog`** — receives `aria-label` from the `title` prop for screen reader announcement.
- **`Heading`** — renders the `title` prop; wired to the dialog via `aria-labelledby` by Spectrum automatically.
- **`Content ref={containerRef}`** — the `containerRef` (`useRef<DOMRefValue<HTMLElement>>`) is wired to the `Content` component and optionally passed to the `actionButton` factory.
- **Close button** — `ActionButton isQuiet` with `FullScreenExit` icon inside a `TooltipTrigger`; calls Spectrum's render-prop `close()` function.

```tsx
// Simple fullscreen viewer
<FullscreenAction title="Image Viewer">
  <img src={imageUrl} alt="Dataset item" />
</FullscreenAction>

// With a static action button
<FullscreenAction
  title="Dataset Preview"
  actionButton={
    <Button variant="secondary" onPress={handleExport}>Export</Button>
  }
>
  <DataPreviewTable items={items} />
</FullscreenAction>

// With a factory action button (needs access to the container DOM ref)
<FullscreenAction
  title="Canvas Editor"
  actionButton={(containerRef) => (
    <Button
      variant="secondary"
      onPress={() => canvasRenderer.mountTo(containerRef?.current)}
    >
      Render
    </Button>
  )}
>
  <div id="canvas-mount-point" />
</FullscreenAction>
```

## Integration

- Depends on `@adobe/react-spectrum`: `ActionButton`, `ButtonGroup`, `Content`, `Dialog`, `DialogTrigger`, `Divider`, `Heading`, `Tooltip`, `TooltipTrigger`.
- Depends on `@spectrum-icons/workflow`: `FullScreen`, `FullScreenExit`.
- The `actionButton` factory receives `containerRef.current` at the time the dialog's `ButtonGroup` renders — which is **after** `Content` mounts. The ref value will be the `DOMRefValue<HTMLElement>` wrapping the Spectrum `Content` DOM node, or `null` on the first render pass.
- `id` is forwarded to the trigger `ActionButton` as `${id}-open-fullscreen`. This supports automated test selectors and deep-linking to the fullscreen open action.
- Used in canvas editors, image viewers, and detail panels throughout the Geti application to support fullscreen review mode.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | **Required.** The content to display inside the fullscreen dialog's `Content` region. |
| `title` | `string \| ReactNode` | — | Dialog heading text. When a `string`, also used to generate the trigger button's `aria-label` (`"Open in fullscreen {title}"`). |
| `actionButton` | `ReactNode \| ((ref: DOMRefValue<HTMLElement> \| null) => ReactNode)` | — | Optional action(s) placed in the `ButtonGroup` before the close button. Pass a factory function to receive the `Content` container ref. |
| `id` | `string` | — | ID prefix for the trigger button (`${id}-open-fullscreen`). |

## Accessibility

- **Trigger `aria-label`**: The trigger `ActionButton` is icon-only and receives `aria-label` dynamically: `"Open in fullscreen {title}"` when `title` is a string, or `"Open in fullscreen"` as a fallback. Always provide a string `title` if possible.
- **Tooltip**: The trigger is wrapped in `TooltipTrigger`, which wires `aria-describedby` from the button to the "Fullscreen" tooltip. The tooltip supplements the `aria-label` with a human-readable label visible on hover.
- **Dialog `aria-label`**: The `Dialog` element receives `aria-label="{title} fullscreen"` (or `"Fullscreen content"` as a fallback), independently of the `Heading` element. This ensures screen readers announce the dialog name even when the `Heading` has not yet been associated via `aria-labelledby`.
- **Focus trap**: Spectrum's `fullscreenTakeover` dialog traps focus inside the overlay. Users cannot Tab outside the dialog while it is open.
- **Focus restore**: When the dialog closes, Spectrum returns focus to the trigger `ActionButton` automatically.
- **`Escape` key**: Closes the fullscreen dialog and calls Spectrum's internal `close()` render-prop function. No additional handler is required.
- **Close button**: The close `ActionButton` carries `aria-label="Close fullscreen"` and a `TooltipTrigger` with "Close fullscreen" tooltip — providing both a keyboard-accessible label and a discoverable hover tooltip.
- **Screen reader flow**: On open, the screen reader announces the dialog name and moves to the first focusable element inside (`Heading`). On close, it returns to the trigger button and announces nothing (standard modal dismiss behaviour).

## Related components

- **`Dialog`** / **`DialogTrigger`** — use directly when the fullscreen trigger is not a `FullScreen` icon button or when you need a custom trigger shape.
- **`Tooltip`** / **`TooltipTrigger`** — `FullscreenAction` composes these internally; use them standalone when the fullscreen pattern is not needed.
- **`ActionButton`** — the underlying trigger primitive; `FullscreenAction` wraps it with icon, tooltip, and dialog lifecycle automatically.
- **`DialogContainer`** — use when the fullscreen dialog must be opened programmatically without a persistent trigger button.

## Example use cases

<details>
  <summary>Typical use cases</summary>

- **Image viewer**: An `<img>` or annotation preview inside a card includes a `FullscreenAction` to expand the image into a full-viewport view for detailed inspection.
- **Canvas editor**: A drawing or annotation canvas is promoted to fullscreen mode via `FullscreenAction`, with the `actionButton` factory used to mount the renderer into the fullscreen container DOM node.
- **Chart / data visualization**: A compact inline chart has a fullscreen expand button so analysts can inspect details without leaving the current page context.
- **Dataset preview table**: A condensed data table inside a panel is expanded to fullscreen for side-by-side comparison, with an "Export CSV" action button provided via the `actionButton` slot.
- **Model output review**: An inference result panel (bounding boxes overlaid on an image) offers a fullscreen mode with a "Download result" action button in the dialog toolbar.

</details>
