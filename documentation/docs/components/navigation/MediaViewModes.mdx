# MediaViewModes

MediaViewModes provides a compact icon button that opens a menu to switch between grid and list view modes. Use it in media galleries, dataset browsers, and any content grid where users should be able to choose how items are displayed.

```tsx
import { MediaViewModes, ViewModes } from '@geti/ui';
```

## Responsibility

`MediaViewModes` is a Geti-specific composite component. It renders a quiet `ActionButton` that shows the icon for the current view mode, wraps it in a `TooltipTrigger`, and opens a `Menu` with all available modes. State management is fully controlled by the caller via `viewMode` and `setViewMode`.

## Variants and states

The displayed icon changes to reflect the current mode:

| ViewMode | Icon | Description |
|---|---|---|
| `ViewModes.LARGE` | `ClassicGridView` | Large thumbnail grid |
| `ViewModes.MEDIUM` | `ModernGridView` | Medium thumbnail grid |
| `ViewModes.SMALL` | `ViewGrid` | Small thumbnail grid |
| `ViewModes.DETAILS` | `ViewList` | Detailed list view |

## Props (deep dive)

### Value / state

- `viewMode` (required): the currently active `ViewModes` value. Controls which icon is displayed.
- `setViewMode` (required): callback called when the user selects a new mode.

### Content

- `items`: subset of `ViewModes` values to show in the menu. Defaults to all four modes.

### Visual

- `isDisabled`: disables the trigger button and prevents the menu from opening.

### Accessibility

- The trigger button has `aria-label="View mode"`.
- A tooltip labelled `"View mode"` appears on hover/focus.
- The menu uses `selectionMode="single"` to communicate the active mode.
