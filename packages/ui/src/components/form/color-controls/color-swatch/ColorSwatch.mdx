## Responsibility

`ColorSwatch` is a thin wrapper around Adobe React Spectrum's `ColorSwatch`. It displays a color as a small, filled visual block â€” it is **not interactive** on its own and carries no selection state. Its purpose is purely presentational: show what color is currently active or available.

Key responsibilities:

- **Color preview**: Renders the given color (`string`, `Color` object, or any Spectrum-parseable format) as a filled visual swatch.
- **Static display**: Has no `onPress`, no selection ring, and no inherent interaction model. It is a display element, not a control.
- **Size and shape variants**: `size` (`S`, `M`, `L`) and `rounding` (`none`, `default`, `full`) control its dimensions and border radius.
- **Trigger decoration**: Used alongside interactive elements (e.g., inside an `ActionButton`) to preview the currently selected color without being the selector itself.

### When to use

- To show the currently selected or active color in a button trigger (e.g., the `ColorPickerDialog` trigger button).
- To display a color label in a legend, tooltip, or data table cell.
- As a non-interactive preview chip in a settings panel or color swatch list where selection is handled by a surrounding element.
- Inside `ColorSwatchPicker` children to visually represent each selectable preset (where the picker manages selection state).

### When to avoid

- When the swatch itself needs to be clickable and selectable â€” use `ColorSwatchPickerItem` inside a `ColorSwatchPicker` instead.
- When you need a full color picker UI â€” use `ColorPickerDialog`, which composes `ColorSwatch` with sliders, a wheel, and a hex field.
- When the color needs a visible label or accessible name beyond the color itself â€” add `aria-label` to communicate the color name to screen readers.

## Variants

### Sizes (`size`)

Three sizes are available: `S` (small â€” inline with text), `M` (medium â€” default), `L` (large â€” standalone display).

### Rounding (`rounding`)

Controls the border radius: `none` (sharp corners), `default` (slightly rounded), `full` (circle). Use `full` for palette-style swatches; use `none` for precise rectangular color blocks.

## Design

`ColorSwatch` is a pure thin wrapper with an explicit interface extension.

```tsx
export interface ColorSwatchProps extends SpectrumColorSwatchProps {}

export const ColorSwatch = (props: ColorSwatchProps) => <SpectrumColorSwatch {...props} />;
```

The `interface extends` (rather than a `type` alias) keeps the type open for future Geti-specific additions without a breaking change.

Typical usage as a color preview in a button trigger:

```tsx
import { ColorSwatch } from '@geti/ui';
import { Flex } from '@geti/ui';

<Flex gap="size-100" alignItems="center">
    <ColorSwatch color={currentColor} size="S" aria-label={`Current color: ${currentColorName}`} />
    <span>Pick color</span>
</Flex>;
```

Standalone display in a legend or tooltip:

```tsx
<ColorSwatch color="#4A90D9" size="M" rounding="full" aria-label="Annotation color: Blue" />
```

## Integration

- **Depends on**: `@adobe/react-spectrum` (`ColorSwatch`, `SpectrumColorSwatchProps`)
- **Color format**: `color` accepts a CSS color string (`'#ff0000'`, `'hsl(0, 100%, 50%)'`), or an Adobe `Color` object from `@adobe/react-spectrum`'s `parseColor()`. Spectrum normalises all formats internally.
- **Composed into**: `ColorPickerDialog` uses `ColorSwatch size="S"` inside the trigger `ActionButton` to preview the active color. `ColorSwatchPicker` children are `ColorSwatch` elements (or `ColorSwatchPickerItem` which wraps `ColorSwatch`).
- **Contrast with ColorSwatchPickerItem**: `ColorSwatchPickerItem` wraps `ColorSwatch` and is used as a child of `ColorSwatchPicker` to participate in its managed selection model. Use `ColorSwatch` alone for display; use `ColorSwatchPickerItem` inside a picker.

## Accessibility

`ColorSwatch` is a static, non-interactive display element with minimal ARIA requirements.

- **Role**: Renders as an `<div>` (or equivalent) with `role="img"` when a `color` is present, communicating that it is a visual representation rather than a control.
- **Labels**: `ColorSwatch` has **no visible text label**. Always provide `aria-label` with a human-readable color name (e.g., `aria-label="Red"`). Without it, screen readers will receive no useful information about what color is being displayed.
- **Checkerboard background**: Spectrum renders a checkerboard pattern behind the color fill to communicate transparency / alpha. This is purely visual and requires no additional annotation.
- **No keyboard interaction**: `ColorSwatch` is not focusable and is not in the tab order. It exists only as a visual cue. If you need a focusable, selectable swatch, use `ColorSwatchPickerItem` inside a `ColorSwatchPicker`.
- **Color contrast**: The swatch itself is the content, so standard contrast requirements do not apply to its fill. However, any border or focus ring applied by the parent element must meet contrast requirements.

## Related components

- `ColorSwatchPicker` â€” use instead when swatches must be selectable in a keyboard-navigable radio group.
- `ColorThumb` â€” use instead for a compact, inline square color preview inside an `ActionButton` trigger.
- `ColorPickerDialog` â€” use `ColorSwatch` inside its trigger button to show the current color at a glance.
