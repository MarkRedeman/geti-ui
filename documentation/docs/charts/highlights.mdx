# Highlights

Interactive highlighting lets users focus on one or more data series by dimming the rest. The chart library supports two highlight modes — **transient** and **pinned** — and several interaction triggers.

## Transient vs pinned highlighting

| Mode | Trigger | Behaviour |
|---|---|---|
| **Transient** | Hover over a geometry (line, bar, dot) or a legend item | Highlight is active only while the pointer is over the target; leaving resets all series to full opacity. |
| **Pinned** | Click a legend item | Highlight is locked until the user clicks the same legend item again (toggle off) or clicks another item. Transient hover still works on top of a pinned state. |

Use transient highlighting for quick exploratory scanning. Use pinned highlighting when the user wants to compare a specific series against the rest while scrolling or interacting with other page content.

## Interaction models

### Geometry hover

Moving the cursor over a line segment, bar, dot, or pie slice highlights the owning series and dims all others. This is the default hover behaviour when `highlight` is enabled.

### Legend hover

Hovering over a legend item highlights the corresponding series without requiring the cursor to land on the chart geometry. Useful for charts where geometry is dense or small (e.g. scatter plots with many points).

### Legend click — pinning

Clicking a legend item _pins_ the highlight on that series. Subsequent clicks on the same item release the pin; clicking a different item transfers the pin. Pinning is opt-in via `highlight.interaction.legendClick`.

### Voronoi-driven highlighting (scatter charts)

`ScatterChart` uses a Voronoi partition of the plot area (powered by [d3-delaunay](https://github.com/d3/d3-delaunay)). Each data point owns a cell; moving the cursor into a cell highlights that point regardless of how far the cursor is from the dot. Set `interaction.voronoi = true` (or use the `useVoronoi` prop on `ScatterChart`) to enable this mode.

## `HighlightConfig` props

| Prop | Type | Default | Description |
|---|---|---|---|
| `enabled` | `boolean` | `false` | Enable series highlighting |
| `mode` | `'single' \| 'multiple'` | `'single'` | `'single'` dims all non-hovered series; `'multiple'` accumulates pinned series |
| `dimmedOpacity` | `number` | `0.2` | Opacity applied to non-highlighted series (0–1) |
| `interaction.legendClick` | `boolean` | `false` | Allow clicking legend items to pin a highlight |
| `interaction.legendHover` | `boolean` | `true` | Highlight on legend item hover |
| `interaction.voronoi` | `boolean` | `false` | Use Voronoi cells for highlight hit-testing (scatter charts) |

> `voronoiMaxDistance` is configured on `ScatterChart` directly (`voronoiMaxDistance` prop), not in `HighlightConfig`.

## Examples

### 1) LineChart

```tsx preview
import { LineChart } from '@geti-ui/charts';

const data = [
  { epoch: 1, train: 1.42, val: 1.50, test: 1.55 },
  { epoch: 2, train: 1.10, val: 1.20, test: 1.28 },
  { epoch: 3, train: 0.87, val: 0.98, test: 1.06 },
  { epoch: 4, train: 0.71, val: 0.82, test: 0.90 },
  { epoch: 5, train: 0.60, val: 0.69, test: 0.78 },
];

export default function App() {
  return (
    <LineChart
      data={data}
      xAxisKey="epoch"
      showLegend
      highlight={{ enabled: true, interaction: { legendClick: true } }}
      series={[
        { dataKey: 'train', name: 'Train loss' },
        { dataKey: 'val', name: 'Val loss' },
        { dataKey: 'test', name: 'Test loss' },
      ]}
      aria-label="Loss curves with pinnable highlight"
      height={280}
    />
  );
}
```

## Optional shared provider

For dashboards where multiple charts should share one highlight state, wrap them in `HighlightProvider`.

```tsx
import { HighlightProvider, LineChart, BarChart } from '@geti-ui/charts';

export default function App() {
  return (
    <HighlightProvider enabled mode="single">
      <LineChart
        data={[{ x: 'A', train: 0.8, val: 0.9 }]}
        xAxisKey="x"
        series={[{ dataKey: 'train' }, { dataKey: 'val' }]}
        showLegend
        highlight={{ enabled: true, interaction: { legendHover: true, legendClick: true } }}
      />
      <BarChart
        data={[{ x: 'A', train: 0.8, val: 0.9 }]}
        xAxisKey="x"
        series={[{ dataKey: 'train' }, { dataKey: 'val' }]}
        showLegend
        highlight={{ enabled: true, interaction: { legendHover: true, legendClick: true } }}
      />
    </HighlightProvider>
  );
}
```

Without a provider, charts keep independent local highlight state (default behavior).

## Using highlighting with primitives

When building charts from primitives you manage highlight state yourself. The recommended pattern is:

1. Track the active series key in component state (`useState`).
2. Apply a reduced opacity to non-active `<Line>`, `<Bar>`, `<Cell>`, or `<Scatter>` elements.
3. Wire `onMouseEnter` / `onMouseLeave` on each geometry element or on the `<Legend>` payload items for hover behaviour.
4. Wire `onClick` on legend payload items for pinning.

```tsx
import { useState } from 'react';
import { Line, LineChart } from 'recharts';
import {
  GetiCartesianGrid,
  GetiLegend,
  GetiResponsiveContainer,
  GetiTooltip,
  GetiXAxis,
  GetiYAxis,
} from '@geti-ui/charts/primitives';

const data = [
  { epoch: 1, train: 1.42, val: 1.50 },
  { epoch: 2, train: 1.10, val: 1.20 },
  { epoch: 3, train: 0.87, val: 0.98 },
  { epoch: 4, train: 0.71, val: 0.82 },
];

const SERIES = [
  { dataKey: 'train', color: 'var(--spectrum-global-color-blue-500)' },
  { dataKey: 'val',   color: 'var(--spectrum-global-color-green-500)' },
];

export default function App() {
  const [pinned, setPinned] = useState<string | null>(null);
  const [hovered, setHovered] = useState<string | null>(null);

  const active = pinned ?? hovered;

  const opacity = (key: string) =>
    active === null || active === key ? 1 : 0.25;

  return (
    <GetiResponsiveContainer height={260}>
      <LineChart data={data}>
        <GetiCartesianGrid />
        <GetiXAxis dataKey="epoch" />
        <GetiYAxis />
        <GetiTooltip />
        <GetiLegend
          onClick={(payload) => {
            const key = payload.dataKey as string;
            setPinned((prev) => (prev === key ? null : key));
          }}
          onMouseEnter={(payload) => setHovered(payload.dataKey as string)}
          onMouseLeave={() => setHovered(null)}
        />
        {SERIES.map(({ dataKey, color }) => (
          <Line
            key={dataKey}
            type="monotone"
            dataKey={dataKey}
            stroke={color}
            strokeOpacity={opacity(dataKey)}
            dot={false}
            isAnimationActive={false}
          />
        ))}
      </LineChart>
    </GetiResponsiveContainer>
  );
}
```

Key points:

- Prefer `strokeOpacity` / `fillOpacity` over toggling `hide` — dimming preserves chart layout and avoids axis rescaling.
- For `<Cell>`-based charts (Pie, RadialBar) apply `opacity` directly on each `<Cell>` element.
- Keep pinned and hovered states separate so releasing a hover does not clear a pin.
