# Overflowable tabs

Use `OverflowableTabs` when tab count can exceed the available width. It keeps the selected tab visible and moves hidden tabs into a compact `N more` picker.

This is useful for dataset/project lists that grow at runtime while keeping a stable header layout.

```tsx
import { OverflowableTabs } from '@geti-ai/blocks';
```

## Responsive tabs in an app header

```tsx preview
import { useState } from 'react';
import { OverflowableTabs } from '@geti-ai/blocks';
import { ActionButton, TabItem, TabPanels, Tabs, Text, ThemeProvider, View } from '@geti-ai/ui';
import { Add } from '@geti-ai/ui/icons';
import { Icon } from '@adobe/react-spectrum';

type Dataset = {
  id: string;
  label: string;
  images: number;
};

const initialItems: Dataset[] = [
  { id: 'train', label: 'Training set', images: 1240 },
  { id: 'val', label: 'Validation set', images: 340 },
  { id: 'test', label: 'Testing set', images: 510 },
  { id: 'edge', label: 'Edge cases', images: 94 },
  { id: 'night', label: 'Night shift', images: 188 },
  { id: 'drift', label: 'Drift watch', images: 72 },
  { id: 'archive', label: 'Archive', images: 3210 },
];

const createItem = (index: number): Dataset => ({
  id: `dataset-${index + 1}`,
  label: `Dataset ${index + 1}`,
  images: 120 + index * 20,
});

export default function App() {
  const [items, setItems] = useState(initialItems);
  const [selectedId, setSelectedId] = useState(initialItems[0].id);

  return (
    <ThemeProvider>
      <Tabs
        aria-label="Datasets"
        selectedKey={selectedId}
        onSelectionChange={(key) => setSelectedId(String(key))}
        UNSAFE_style={{
          '--spectrum-tabs-selection-indicator-color': 'var(--energy-blue)',
          '--spectrum-tabs-emphasized-selection-indicator-color': 'var(--energy-blue)',
          '--spectrum-tabs-quiet-emphasized-selection-indicator-color': 'var(--energy-blue)',
          '--spectrum-tabs-item-gap': 'var(--spectrum-global-dimension-size-200)',
        }}
      >
        <OverflowableTabs
          items={items}
          selectedKey={selectedId}
          onSelectionChange={setSelectedId}
          getItemId={(item) => item.id}
          getItemLabel={(item) => item.label}
          overflowAriaLabel="Collapsed datasets"
          trailingContent={
            <ActionButton
              isQuiet
              aria-label="Add dataset"
              onPress={() => {
                const next = createItem(items.length);
                setItems((current) => [...current, next]);
                setSelectedId(next.id);
              }}
            >
              <Icon>
                <Add />
              </Icon>
            </ActionButton>
          }
        />

        <TabPanels>
          {items.map((item) => (
            <TabItem key={item.id}>
              <View paddingTop="size-150">
                <Text>
                  <strong>{item.label}</strong>
                </Text>
                <Text>{item.images} images</Text>
              </View>
            </TabItem>
          ))}
        </TabPanels>
      </Tabs>
    </ThemeProvider>
  );
}
```

## Props

### `OverflowableTabsProps<T>`

| Prop | Type | Required | Description |
| --- | --- | --- | --- |
| `items` | `T[]` | Yes | Full tab data collection. |
| `selectedKey` | `string` | Yes | Currently selected tab id. |
| `onSelectionChange` | `(id: string) => void` | Yes | Called when user selects a tab from visible tabs or overflow picker. |
| `getItemId` | `(item: T) => string` | Yes | Returns stable id for each tab item. |
| `getItemLabel` | `(item: T) => string` | Yes | Returns text label for each tab item. |
| `renderTab` | `(item: T, isActive: boolean) => ReactNode` | No | Custom tab content renderer. If omitted, label text is used. |
| `overflowAriaLabel` | `string` | No | Accessible label for overflow picker. Default: `'Collapsed items'`. |
| `minVisibleTabs` | `number` | No | Minimum number of tabs to keep visible before overflow. Default: `2`. |
| `trailingContent` | `ReactNode` | No | Optional right-side header content (for example Add button). |
| `trailingContentContainerStyle` | `CSSProperties` | No | Style overrides for trailing content container. |

## Integration guidance

- Keep `selectedKey` controlled from the same state that drives your parent `Tabs`.
- Use `renderTab` to combine with custom tab labels (for example `ManagedTab`).
- Prefer stable ids from `getItemId` (do not use array index).
