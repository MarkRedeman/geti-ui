# ColorSlider

`ColorSlider` lets users adjust a single channel of a color value along a gradient track. Use it as a building block inside a custom color editor or inline to expose a specific channel (e.g. opacity/alpha).

```tsx
import { ColorSlider } from '@geti-ai/ui';
```

- [React Spectrum ColorSlider ‚Üó](https://react-spectrum.adobe.com/react-spectrum/ColorSlider.html)
- [Geti UI Theming](/components/theming)

## Examples

### Red channel

```tsx preview
import { ColorSlider } from '@geti-ai/ui';

export default function App() {
  return <ColorSlider defaultValue="#7f0000" channel="red" />;
}
```

## Responsibility

`ColorSlider` is a direct pass-through of React Spectrum's `ColorSlider`. No defaults are changed.

## Variants and states

| Feature | Prop | Notes |
|---|---|---|
| Channel | `channel` | Required. Determines which color channel is controlled |
| Orientation | `orientation` | `"horizontal"` (default) or `"vertical"` |
| Disabled | `isDisabled` | Non-interactive |
| Hide label | `showValueLabel={false}` | Removes the numeric value label |

## Props (deep dive)

### Content

- `label`: optional override for the channel label shown above the slider.
- `showValueLabel`: whether to show the numeric value (default `true`).

### Value / state

- `value` / `defaultValue`: any CSS color string or `Color` object.
- `channel` (**required**): `"red" | "green" | "blue" | "alpha" | "hue" | "saturation" | "brightness" | "lightness"`.
- `orientation`: `"horizontal" | "vertical"`.
- `isDisabled`.

### Events

- `onChange(color)`: fires during drag.
- `onChangeEnd(color)`: fires when the user releases the thumb.

### Accessibility

- The slider uses `role="slider"` with `aria-valuemin`, `aria-valuemax`, and `aria-valuenow`.
- Always provide a `label` or ensure the surrounding context labels the control.
