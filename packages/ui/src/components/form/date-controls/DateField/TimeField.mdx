# TimeField

<Subtitle>A segment-based time input that lets users type hours, minutes, and seconds independently using the keyboard, scoped purely to time values.</Subtitle>

## Responsibility

`TimeField` is a thin wrapper around Adobe React Spectrum's `TimeField`. It renders a time value as individually editable segments (e.g., `HH : MM : SS AM/PM`) directly in the form layout. Unlike `DateField`, it operates exclusively on time — there are no date segments. Use it when you need to capture or display a time of day, a scheduled time, or a recurring time without a date component.

Key responsibilities:
- **Time-only editing**: Segments cover hours, minutes, seconds, and optionally milliseconds and AM/PM — no date portions.
- **Type safety**: Generic over `T extends TimeValue`, accepting `Time` or `ZonedDateTime` from `@internationalized/date`.
- **Granularity control**: `granularity` limits visible segments (e.g., show only `hour:minute` for appointment scheduling, show `hour:minute:second` for process durations).
- **Timezone awareness**: When a `ZonedDateTime` is passed, an optional timezone segment can be displayed or hidden via `hideTimeZone`.

### When to use
- When collecting a time of day independently of a date — meeting start times, daily alarm triggers, opening hours, scheduled job times.
- When the date is already established elsewhere and only the time needs to be set (e.g., a form with a separate `DateField` and a `TimeField`).
- When you need fine-grained segment control — showing only `HH:MM` for everyday scheduling, or `HH:MM:SS` for technical process configurations.
- When keyboard-only time entry is required — `TimeField` avoids the overhead of a clock picker overlay.

### When to avoid
- When users need to select both a date and a time together — use `DateField` with a `CalendarDateTime` value, which includes time segments.
- When a fuzzy or relative time expression is more useful than an exact value (e.g., "in 30 minutes") — `TimeField` is for precise, absolute times.
- When the UX calls for a visual clock face or a scrollable time picker — `TimeField` is keyboard-segment-only.

## Variants

### Default
Renders `HH : MM AM/PM` segments with a visible label. Locale-aware — renders 24-hour format in locales that use it.

### With description
Add `description` for helper text (e.g., "All times are in UTC+0", "Use 24-hour format").

### Invalid
Set `validationState="invalid"` and `errorMessage` to surface a constraint violation (e.g., "Departure time must be in the future"). The border turns red and the message is announced.

### Disabled
`isDisabled` grays out all segments and removes them from the tab order. Use when the time is system-controlled or locked.

## Design

`TimeField` is a pure pass-through wrapper, generic over Spectrum's `TimeValue` union.

```tsx
export const TimeField = <T extends TimeValue>(
  props: SpectrumTimeFieldProps<T>
) => <SpectrumTimeField {...props} />;

export type { SpectrumTimeFieldProps as TimeFieldProps };
```

Note: `TimeField` is exported from `DateField.tsx` — both live in the same source file. Import from `@geti/ui` regardless.

Typical controlled usage:

```tsx
import { TimeField } from '@geti/ui';
import { Time } from '@internationalized/date';

const [time, setTime] = useState<Time | null>(null);

<TimeField
  label="Meeting time"
  value={time}
  onChange={setTime}
  granularity="minute"
  isRequired
/>
```

For timezone-aware time (e.g., scheduling across regions):

```tsx
import { ZonedDateTime, now, getLocalTimeZone } from '@internationalized/date';

<TimeField<ZonedDateTime>
  label="Scheduled at"
  value={scheduledTime}
  onChange={setScheduledTime}
  hideTimeZone={false}
/>
```

## Integration

- **Depends on**: `@adobe/react-spectrum` (`TimeField`, `SpectrumTimeFieldProps`), `@react-types/datepicker` (`TimeValue`)
- **Time types**: `Time` for time-only without timezone, `ZonedDateTime` for time with timezone. Both from `@internationalized/date`. Avoid raw `Date` — its timezone behaviour is unpredictable.
- **Source location**: `TimeField` is exported from `DateField.tsx` alongside `DateField`. Both wrap Spectrum components from the same import.
- **Contrast with DateField**: `DateField` covers date-only (`CalendarDate`) or date+time (`CalendarDateTime`). `TimeField` is scoped to time-only values. Do not use `TimeField` when you need year/month/day segments.
- **Complement with DateField**: A common pattern is a side-by-side `DateField` (for date) and `TimeField` (for time) feeding a combined `CalendarDateTime` value to the parent form.

```tsx
// Side-by-side date + time entry
<Flex gap="size-200">
  <DateField label="Date" value={date} onChange={setDate} />
  <TimeField label="Time" value={time} onChange={setTime} granularity="minute" />
</Flex>
```

## Accessibility

`TimeField` uses the same [ARIA spinbutton pattern](https://www.w3.org/WAI/ARIA/apg/patterns/spinbutton/) as `DateField`, applied to each time segment.

- **Group role**: The field container has `role="group"` with the label as its accessible name via `aria-labelledby`.
- **Segment roles**: Each time segment (`HH`, `MM`, `SS`, `AM/PM`) has `role="spinbutton"` with `aria-valuemin`, `aria-valuemax`, and `aria-valuenow`.
- **AM/PM segment**: Announced as a spinbutton with two discrete values — screen readers announce "AM" or "PM" as the user toggles.
- **Timezone segment**: When visible, announced with the timezone abbreviation (e.g., "PST").
- **Required/invalid**: `aria-required` and `aria-invalid` are set on the group. `errorMessage` is linked via `aria-describedby`.
- **Keyboard navigation**:

| Key | Behaviour |
|-----|-----------|
| `Tab` | Move focus into the field (first segment) |
| `Tab` / `Shift + Tab` | Move focus between segments |
| `0`–`9` | Type digits into the focused segment; auto-advances when complete |
| `↑` / `↓` | Increment or decrement the focused segment by 1 unit |
| `A` / `P` | Toggle AM/PM when the meridiem segment is focused |
| `Backspace` | Clear the focused segment |

- **Segment bounds**: Typing is range-clamped per segment — e.g., typing `9` in the hour segment (12-hour mode) immediately completes it and advances; typing `1` waits for the next digit to determine if `10`, `11`, or `12` is intended.
- **Locale adaptation**: Spectrum automatically renders 12-hour or 24-hour format based on the active locale, eliminating the need for manual format strings.

## Related components
- `DateField` — use alongside `TimeField` when both a calendar date and a clock time must be captured separately.
- `DatePicker` — use instead when date + time selection together in a single popover-backed input is preferred.
- `NumberField` — use instead if only a raw numeric duration (e.g., seconds) rather than a clock time is needed.
