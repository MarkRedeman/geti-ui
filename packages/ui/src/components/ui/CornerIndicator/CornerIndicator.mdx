# CornerIndicator

A decorative wrapper that overlays a small indicator dot in the corner of a UI element.

```tsx
import { CornerIndicator } from '@geti-ai/ui';
```

<Stories />

## Responsibility

`CornerIndicator` is a Geti-specific layout wrapper that overlays a small blue dot in the top-right corner of its children when `isActive={true}`. It is a **decorator**—it wraps arbitrary content and conditionally renders the dot without modifying the child.

There is no direct equivalent to a positioned notification dot in Adobe React Spectrum. While Spectrum provides `Badge` and `StatusLight`, they differ from `CornerIndicator` in key ways:

1. **Overlay Behavior**: `CornerIndicator` wraps children and overlays them, whereas Spectrum indicators render inline beside content.
2. **Visual-Only Dot**: It renders a purely visual dot without a text label, whereas Spectrum components typically require or recommend visible text.
3. **Presence Signal**: Its purpose is a compact activity signal (e.g., indicating a tool is active or a change is pending) rather than a semantic status or category label.

Use `CornerIndicator` for non-intrusive activity signals on icons, thumbnails, or controls. For status labels visible alongside content, prefer `StatusLight`. For categorization metadata, prefer `Badge`.

The component uses a Spectrum `View` with `position="relative"` to establish a coordinate system, and a `div` with `position="absolute"` for the dot. The dot uses `role="status"` and `aria-label="Pending change"` so assistive technologies announce its presence. The dot size, position, and color are driven by Spectrum global dimension and color tokens.

## Variants

### Active State

When `isActive` is `true`, a small blue dot is rendered in the top-right corner. When `false`, the indicator is completely removed from the DOM.

## Props

<Controls of={DocStories.Default} />

| Prop       | Type        | Default | Description                                     |
| ---------- | ----------- | ------- | ----------------------------------------------- |
| `children` | `ReactNode` | —       | The element to which the indicator is attached. |
| `isActive` | `boolean`   | —       | Whether the indicator dot is visible.           |

## Related components

- `Avatar` / `AvatarGroup` — common parent element to wrap with `CornerIndicator` to indicate user activity.
- `ActionButton` — common parent to signal that a tool or mode is currently active via the indicator dot.
- `Image` / `PhotoPlaceholder` — wrap with `CornerIndicator` to signal unsaved changes or pending updates on a thumbnail.

## Example use cases

<details>
  <summary>Typical use cases</summary>

- Showing a blue dot on an annotation tool icon to indicate the tool is currently selected/active.
- Marking a model card thumbnail to indicate a pending training job is in progress.
- Indicating unsaved configuration changes on a settings panel icon in a collapsed sidebar.
- Overlaying a presence dot on a team member avatar to indicate they are currently online.
