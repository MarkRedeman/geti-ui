## Responsibility

`Calendar` and `RangeCalendar` are direct pass-through wrappers of their React Spectrum equivalents with no changed defaults. They are useful as building blocks inside custom dialogs or panels.

## Variants and states

| Feature | Prop | Notes |
|---|---|---|
| Multiple months | `visibleMonths` | Number of months rendered side-by-side |
| Disabled | `isDisabled` | Non-interactive |
| Read-only | `isReadOnly` | Shows value, no editing |
| Min/max dates | `minValue` / `maxValue` | Dates outside range are greyed out |
| Focus management | automatic | Keyboard navigation via arrow keys |

## Props (deep dive)

### Content

- `aria-label`: **required** ÔÇö identifies the calendar to screen readers.

### Value / state (Calendar)

- `value` / `defaultValue`: `CalendarDate` controlled/uncontrolled value.
- `minValue` / `maxValue`: constrains selectable dates.
- `visibleMonths`: number of months to show (default `1`).
- `isDisabled`, `isReadOnly`.

### Value / state (RangeCalendar)

- `value` / `defaultValue`: `{ start: CalendarDate, end: CalendarDate }`.
- `allowsNonContiguousRanges`: permits ranges spanning disabled dates.
- Same `visibleMonths`, `minValue`, `maxValue`, `isDisabled`, `isReadOnly`.

### Events

- `onChange(value)`: fires when the user selects a date or completes a range.

### Accessibility

- Always provide `aria-label` since these components have no built-in visible label.
- Keyboard: arrow keys navigate days; Page Up/Down move months; Home/End jump to start/end of week.

## Related components

- `DatePicker` ÔÇö calendar + segmented input in a single popover-triggered field.
- `DateRangePicker` ÔÇö range calendar + two segmented inputs with a popover.
- `DateField` ÔÇö bare segmented keyboard input without a calendar.
