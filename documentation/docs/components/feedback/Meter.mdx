# Meter

`Meter` shows a visual representation of a quantity as a filled bar with a threshold-based colour. Use it to communicate levels — storage used, model accuracy, training progress — where the absolute value and threshold state both matter.

```tsx
import { Meter } from '@geti/ui';
```

- [React Spectrum Meter ↗](https://react-spectrum.adobe.com/react-spectrum/Meter.html)
- [Geti UI Theming](/components/theming)

## Responsibility

`Meter` is a thin wrapper around React Spectrum's `Meter`. It passes all props through with no Geti-specific defaults. The wrapper keeps the import path consistent with the rest of `@geti/ui`.

## Variants and states

| `variant` | When to use |
|---|---|
| `informative` (default) | Neutral progress with no threshold meaning |
| `positive` | Value is at a healthy level |
| `warning` | Value is approaching a cautionary threshold |
| `critical` | Value has exceeded a dangerous threshold |

## Props (deep dive)

### Content

- `label`: the text label shown above the meter bar.
- `valueLabel`: custom text shown at the trailing end of the bar (defaults to a percentage).

### Value

- `value`: current value (required).
- `minValue`: minimum of the range. Defaults to `0`.
- `maxValue`: maximum of the range. Defaults to `100`.

### Appearance

- `variant`: `informative | positive | warning | critical`.
- `size`: `S | L` — controls label and bar thickness.

### Accessibility

- `label` is required for screen reader announcements; it is also the visible label.
- The meter renders with `role="meter"` and `aria-valuenow` / `aria-valuemin` / `aria-valuemax` automatically.

## Related components

- `ProgressBar` — use when showing task completion progress rather than a level.
- `ProgressCircle` — use for circular indeterminate or determinate progress.
- `StatusLight` — use for a simple coloured dot status without a quantity.
