## Responsibility

Tabs are used to group related content that is at the same level of hierarchy. They provide a clear visual structure for switching between different views or categories within a single page or container.

Key responsibilities:
- **Sectional Organization**: Breaks down large amounts of content into manageable, related pieces.
- **Context Preservation**: Allows users to switch between sections without losing their place in the application.
- **Visual Feedback**: Clearly indicates which section is currently active and provides a preview of other available sections.

### When to use
- To organize **same-level sibling content** (e.g., General Settings, Security, Notifications).
- When users need to switch frequently between related data sets.
- When vertical space is limited and content can be effectively layered.

### When to avoid
- For primary site-wide navigation (use a sidebar or top navigation bar).
- When items represent actions rather than content sections (use `ActionGroup`).
- For more than 7 sections (consider a vertical navigation list or a nested hierarchy).
- For sequential steps in a process (use a `Stepper` or "Next/Back" flow).

## Variants

### Orientation
- **Horizontal (Default)**: The most common variant, best for 2–5 tabs in a standard page layout.
- **Vertical**: Useful for sidebar-style navigation or when there are many tabs that wouldn't fit horizontally.

### Density
Supports `regular` (default) and `compact` density. `compact` is ideal for information-dense dashboards or property panels where space is at a premium.

### Keyboard Activation
- **Automatic (Default)**: The tab is selected as soon as it receives focus via arrow keys.
- **Manual**: Requires the user to press `Enter` or `Space` to select the tab. Recommended for tabs that trigger expensive data fetching or complex re-renders.

## Props

### Tabs Props
| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `selectedKey` | `Key` | - | The currently selected tab key (controlled). |
| `defaultSelectedKey` | `Key` | - | The initially selected tab key (uncontrolled). |
| `onSelectionChange` | `(key: Key) => void` | - | Handler called when the selection changes. |
| `orientation` | `'horizontal' \| 'vertical'` | `'horizontal'` | The orientation of the tabs. |
| `density` | `'regular' \| 'compact'` | `'regular'` | The visual density of the tabs. |
| `isDisabled` | `boolean` | `false` | Whether the entire tab group is disabled. |

### TabList Props
| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `aria-label` | `string` | - | **Required.** An accessible label for the tab list. |
| `children` | `ItemElement \| ItemRenderer` | - | The tab labels (using the `Item` component). |

## Accessibility

- **Roles**: Implements the WAI-ARIA Tabs pattern (`role="tablist"`, `role="tab"`, `role="tabpanel"`).
- **Keyboard Support**:
  - `Tab` moves focus into the tab list (to the active tab).
  - `Arrow Left/Right` (or `Up/Down` for vertical) moves focus between tabs.
  - `Home/End` jumps to the first or last tab.
  - Subsequent `Tab` press moves focus **into the active tab panel**.
- **Labeling**: The `TabList` must always have an `aria-label` or `aria-labelledby` to provide context.
