# Packaging and releases

Please write down a proposal for how to deal with packaging the component library as an npm package, managing releases, versioning and CI + CD.
Recommend modern tools and practices.
Ideally our setup should follow semver, force conventional commits, automatically publish a new version on patch changes, and automatically generate changelogs.
Consider using and setting up https://git-cliff.org/

Figure out a way how we can automatically detect changes to specific components when making a PR, which we can then use to auto comment to a PR with screenshots of the changes.
