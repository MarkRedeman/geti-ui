## Responsibility

The `DropZone` component provides a designated region where users can drop content (files, text, or objects). It manages the drag-and-drop state, provides visual feedback during interaction, and is fully accessible to keyboard and screen reader users.

### When to use
- For file upload regions, image inlays, or data import workflows.
- Typically paired with [FileTrigger](/docs/components-form-filetrigger--docs) to provide both drag-and-drop and click-to-browse capabilities.

### When to use something else
- Use [FileTrigger](/docs/components-form-filetrigger--docs) alone if only a button-based file selection is needed.

## Variants

### States
- **Drag Over**: The zone highlights when a valid drag enters the area.
- **Filled**: Use the `isFilled` prop when the zone already contains content. This changes the interaction to show a "Drop file to replace" message.
- **Disabled**: The zone is non-interactive. Use `isDisabled`.

### Layout
- **Custom Content**: `DropZone` is a container that can hold labels, icons (e.g., `IllustratedMessage`), or buttons.

## Props

### Events
| Prop | Type | Description |
|---|---|---|
| `onDrop` | `(e: DropEvent) => void` | Called when content is dropped into the zone. |
| `onDropEnter` | `(e: DropEnterEvent) => void` | Called when a valid drag enters the zone. |
| `onDropExit` | `(e: DropExitEvent) => void` | Called when a valid drag leaves the zone. |
| `getDropOperation` | `(types: DragTypes, allowed: DropOperation[]) => DropOperation` | Determines if a drop is allowed and which operation (copy, move, link) to perform. |

### Value / State
| Prop | Type | Default | Description |
|---|---|---|---|
| `isFilled` | `boolean` | `false` | Whether the zone already contains content. |
| `replaceMessage` | `string` | `'Drop file to replace'` | The message shown when dragging over a filled DropZone. |
| `isDisabled` | `boolean` | `false` | Whether the drop zone is disabled. |

### Content
| Prop | Type | Description |
|---|---|---|
| `children` | `ReactNode` | The content to display inside the drop zone. |
| `aria-label` | `string` | An accessible label for the drop zone if no visible text is provided. |

## Accessibility

- Renders a visually hidden `<button>` that is keyboard-focusable, allowing screen reader users to activate the drop zone (e.g., to trigger a file picker).
- Supports the ARIA drag-and-drop pattern with appropriate role and state announcements.
- **Keyboard support**: Focus the zone via **Tab**, and activate its inner trigger via **Enter** or **Space**.
- Automatically handles the `isDropTarget` state to provide clear visual and semantic feedback.
