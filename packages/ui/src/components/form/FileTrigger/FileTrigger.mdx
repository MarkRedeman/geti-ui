## Responsibility

The `FileTrigger` component is a behavioral wrapper that enables any interactive element (such as a Button or Link) to trigger the native OS file selection dialog. It manages the hidden `<input type="file">` and translates the system file selection into an `onSelect` callback.

### When to use
- When you want to trigger a file selection from a styled [Button](/docs/components-ui-primitives-button--docs) or other UI element.
- Typically nested within a [DropZone](/docs/components-form-dropzone--docs) to provide both "browse" and "drag-and-drop" capabilities.

### When to use something else
- Use [DropZone](/docs/components-form-dropzone--docs) alone if you only want to support drag-and-drop.
- Use a native `<input type="file">` if you need direct form-submission integration with `FormData`.

## Variants

### File Selection Options
- **Accepted Types**: Use `acceptedFileTypes` (e.g., `['image/png', 'image/jpeg']`) to limit the picker to specific MIME types.
- **Multiple Files**: Use `allowsMultiple` to let users select more than one file at a time.
- **Directory Selection**: Use `acceptDirectory` to open a folder/directory picker instead of a file picker.

### Standard States
- **Disabled**: To disable the trigger, set `isDisabled` on the **child** component (e.g., the Button), not the `FileTrigger` itself.

## Accessibility

- `FileTrigger` renders a visually hidden `<input type="file">` with `aria-hidden="true"`.
- All keyboard and screen reader interactions are handled by the **child** element (e.g., a Button).
- **Keyboard support**: Focus the child element via **Tab**, and trigger it via **Enter** or **Space**.
- Screen readers announce the child element according to its role (e.g., "Button, Select file").
