## Responsibility

The `Grid` component provides a powerful system for two-dimensional layout composition. It allows for the definition of rows, columns, and areas, making it ideal for page structures, dashboard layouts, and complex forms.

Key responsibilities:
- **Two-Dimensional Layout**: Control both rows and columns simultaneously.
- **Named Areas**: Use the `areas` prop to define named sections of a layout (e.g., `'header header', 'sidebar content'`).
- **Flexible Sizing**: Support for `fr` units, percentage-based widths, and Spectrum dimension tokens.
- **Responsive Control**: Define different grid structures for different viewport sizes using responsive prop objects.

### When to use
- To build the overall structure of a page or complex UI panel.
- To create a layout with a fixed-width sidebar and a fluid-width content area.
- When rows and columns both need to be carefully aligned.
- To handle form layouts where labels and inputs are aligned across multiple rows.

### When to avoid
- For simple horizontal or vertical stacks (use `Flex` instead).
- When you only need to align items along a single axis.

## Variants

### Columns & Rows
Use the `columns` and `rows` props to define the number and size of tracks in the grid. Both props accept arrays of values (e.g., `['200px', '1fr']`).

### Named Areas
The `areas` prop allows for a more descriptive way to manage complex layouts. Children can be assigned to specific areas using the `gridArea` prop on the child component (usually a `View`).

### Gap
The `gap` prop (and `columnGap`/`rowGap`) provides a standardized way to add space between grid items using Spectrum's size tokens.

## Props

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `areas` | `Responsive<string[]>` | - | Named grid areas. |
| `columns` | `Responsive<DimensionValue[] \| string>` | - | The track sizes for columns. |
| `rows` | `Responsive<DimensionValue[] \| string>` | - | The track sizes for rows. |
| `gap` | `Responsive<DimensionValue>` | - | The space between grid items. |
| `justifyContent` | `Responsive<JustifyContent>` | - | How the grid is distributed within the container. |
| `alignItems` | `Responsive<AlignItems>` | - | How items are aligned within their grid areas. |

## Accessibility

- **Semantic HTML**: By default, `Grid` renders as a `<div>`. Use the `elementType` prop to change it to a more semantic element if appropriate for the content (e.g., `main`, `section`).
- **Logical Flow**: Ensure that the grid layout maintains a logical reading and tab order that matches the visual presentation.
