# DropZone

A target area for dragging and dropping files or custom data into the application.

```tsx
import { DropZone } from '@geti/ui';
```

- [React Spectrum V3 DropZone](https://react-spectrum.adobe.com/react-spectrum/DropZone.html)

<Stories />

## Responsibility

`DropZone` is a transparent wrapper around Adobe React Spectrum's `DropZone`. It declares a valid drop target area, handles the full drag-and-drop lifecycle (enter, hover, validate, receive), and renders visual feedback automatically. It also exposes an accessible hidden `<button>` so keyboard and screen reader users can activate the zone without dragging.

- **Drop target**: Accepts `copy`, `move`, or `link` operations as configured.
- **Visual feedback**: Highlights when a valid drag enters; switches to "replace" appearance when `isFilled`.
- **Drag validation**: `getDropOperation` lets you accept or reject a drop based on dragged data types.
- **Keyboard accessibility**: Hidden focusable button inside the zone for non-mouse users.
- **`aria-label` is required**: Names both the drop region and the hidden button for screen readers.

### When to use
- For file upload regions, image inlays, or data import workflows where dragging is primary or supplementary.
- When you need both drag-and-drop **and** click-to-browse — nest a `FileTrigger`-wrapped `Button` inside.
- When the zone needs a "replace" state (`isFilled`) after content has already been uploaded.
- When drop-type validation is required via `getDropOperation`.

### When to avoid
- **Click-to-browse only** — use [`FileTrigger`](/docs/components-form-filetrigger--docs) alone.
- **Compact file attachment flows** — a `FileTrigger`-wrapped `Button` without a `DropZone` is lighter for avatar uploaders or inline attachment pickers.
- **Native `FormData` submission** — manage `FileList` manually from `onDrop` or use a native `<input type="file">`.

## Variants

**Default (empty zone):** Dashed border and placeholder content.

```tsx
<DropZone onDrop={handleDrop} aria-label="Upload dataset images">
    <IllustratedMessage>
        <Heading>Drag and drop files here</Heading>
    </IllustratedMessage>
    <FileTrigger onSelect={handleSelect}>
        <Button>Browse files</Button>
    </FileTrigger>
</DropZone>
```

**Filled state:** Use `isFilled` after content has been uploaded. A "Drop to replace" overlay appears on the next drag.

```tsx
<DropZone isFilled replaceMessage="Drop image to replace" onDrop={handleReplace} aria-label="Replace cover image">
    <img src={uploadedUrl} alt="Cover image" />
</DropZone>
```

**Drag-type validation:** Use `getDropOperation` to accept only specific MIME types.

```tsx
<DropZone
    getDropOperation={(types) =>
        types.has('image/png') || types.has('image/jpeg') ? 'copy' : 'cancel'
    }
    onDrop={handleDrop}
    aria-label="Upload image"
>
    {/* ... */}
</DropZone>
```

## Props

<Controls of={DocStories.Default} />

### Events

| Prop | Type | Description |
|------|------|-------------|
| `onDrop` | `(e: DropEvent) => void` | Called when content is dropped into the zone. `DropEvent.items` contains files and text items. |
| `onDropEnter` | `(e: DropEnterEvent) => void` | Called when a valid drag enters the zone boundary. |
| `onDropExit` | `(e: DropExitEvent) => void` | Called when a valid drag leaves the zone boundary. |
| `getDropOperation` | `(types: DragTypes, allowed: DropOperation[]) => DropOperation` | Validates the incoming drop. Return `'copy'`, `'move'`, `'link'`, or `'cancel'`. |

### State

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `isFilled` | `boolean` | `false` | Whether the zone already contains content. Switches to "replace" state on next drag. |
| `replaceMessage` | `string` | `'Drop file to replace'` | Overlay message shown when dragging over a filled zone. |
| `isDisabled` | `boolean` | `false` | Disables all drop interactions and visually dims the zone. |

### Content

| Prop | Type | Description |
|------|------|-------------|
| `children` | `ReactNode` | Content inside the zone: labels, illustrations, `FileTrigger` buttons. |
| `aria-label` | `string` | **Required for accessibility.** Names the visually hidden button inside the zone. |

## Related components
- `FileTrigger` — the primary pairing; nests inside `DropZone` for both drag-and-drop and click-to-browse.
- `Button` — use as the `FileTrigger` child for a clear visible affordance.
- `Image` — use inside `DropZone` to preview a dropped image before confirming the upload.

## Example use cases
<details>
  <summary>Typical use cases</summary>

- A full-page drop target for uploading images to a dataset in a creation wizard.
- A compact thumbnail upload zone on a project settings page for replacing a cover image.
- A drag-and-drop area for importing a pre-labelled annotation export file (JSON, CSV).
- A media drop zone in a video annotation tool for uploading a video clip to process.
