# TagGroup

A focusable, keyboard-navigable list of labels, categories, or keywords with support for selection and removal.

```tsx
import { TagGroup, Item } from '@geti-ai/ui';
```

- [React Spectrum V3 TagGroup](https://react-spectrum.adobe.com/react-spectrum/TagGroup.html)
- [React Aria TagGroup (Headless)](https://react-spectrum.adobe.com/react-aria/TagGroup.html)

<Stories />

## Responsibility

`TagGroup` is a transparent wrapper around Adobe React Spectrum's `TagGroup`. It renders an interactive collection of tags used for categorizing, filtering, or labeling items. The wrapper exists as an integration boundary — all Spectrum behaviour (keyboard navigation, selection state, ARIA roles, and removal callbacks) is inherited directly from Spectrum. Geti-specific defaults or prop transformations can be introduced here in future without changing the consumer import path.

Key responsibilities:

- **Keyboard navigation**: Provides roving `tabindex` management so users can move between tags with `Arrow` keys without tabbing through every item.
- **Selection**: Supports `none` (default), `single`, and `multiple` selection modes via `selectionMode`.
- **Removal**: Exposes an `onRemove` callback that fires when the user activates the remove button on an individual tag. The remove button only renders when `onRemove` is provided.
- **Overflow management**: Supports `maxRows` to cap the number of visible tag rows, automatically rendering a "Show more" affordance for the rest.
- **Theme integration**: Inherits all Spectrum colour, spacing, and focus-ring tokens from `ThemeProvider` in the Geti dark theme.

### When to use

- **Interactive labels**: Displaying keywords, categories, or metadata that users can interact with (select or remove).
- **Applied filters**: A list of currently active filters that users can dismiss individually.
- **Dynamic collections**: Managing a set of tags that grows or shrinks based on user action (e.g., adding classification labels).
- **Multi-select chips**: When users need to pick one or more tags from a fixed set (use `selectionMode="multiple"`).
- When accessibility requirements demand keyboard navigation and ARIA-compliant semantics for a tag collection.

### When to avoid

- **Static, read-only display labels** — use the Geti `Tag` primitive (`data/Tag`), which has no Spectrum dependency and is lighter weight.
- **Status indicators** (e.g., "Beta", "Active") — use `Badge` or `StatusLight`, which carry semantic colour meaning.
- **Single read-only label paired with a field** — use `Label`.
- **Inline status annotation** (error/warning) — use `StatusLight`.

## Design

`TagGroup` follows the _transparent wrapper_ pattern: it extends `SpectrumTagGroupProps` without modification and delegates all rendering to Spectrum. The `Item` re-export is provided so consumers can import both from `@geti-ai/ui` without importing directly from Spectrum.

```tsx
// packages/ui/src/components/data/TagGroup/TagGroup.tsx

export interface TagGroupProps<T extends object> extends SpectrumTagGroupProps<T> {}

// Re-export of Spectrum Item for use with TagGroup
export const Item = SpectrumItem;

export const TagGroup = <T extends object>(props: TagGroupProps<T>) => <SpectrumTagGroup {...props} />;
```

This means `TagGroup` accepts the complete Spectrum `TagGroup` API. Any future Geti-specific defaults (e.g., a default `label`, a maximum tag count, or a custom remove icon) can be introduced in the wrapper without changing call sites.

**Prop mapping**

All props pass through to Spectrum unchanged:

| Geti `TagGroupProps`  | Spectrum `SpectrumTagGroupProps` | Notes                                                          |
| --------------------- | -------------------------------- | -------------------------------------------------------------- |
| `label`               | `label`                          | Associates with the grid landmark via `aria-labelledby`        |
| `onRemove`            | `onRemove`                       | `(keys: Set<Key>) => void` — triggers when remove is activated |
| `selectionMode`       | `selectionMode`                  | `'none' \| 'single' \| 'multiple'`                             |
| `selectedKeys`        | `selectedKeys`                   | Controlled selection state                                     |
| `defaultSelectedKeys` | `defaultSelectedKeys`            | Uncontrolled initial selection                                 |
| `onSelectionChange`   | `onSelectionChange`              | Fired when selection changes                                   |
| `maxRows`             | `maxRows`                        | Caps visible rows; adds a "Show more" button                   |
| `items`               | `items`                          | Dynamic collection data                                        |
| `children`            | `children`                       | Static `<Item>` nodes or render function                       |
| `...rest`             | all other props                  | Passed through without transformation                          |

**The `Item` export**

`Item` is re-exported from Spectrum so that consumers can import everything they need from `@geti-ai/ui`:

```tsx
import { TagGroup, Item } from '@geti-ai/ui';

<TagGroup label="Labels">
    <Item key="person">Person</Item>
    <Item key="vehicle">Vehicle</Item>
</TagGroup>;
```

**Theme integration**

`TagGroup` inherits Spectrum's tag tokens (background, border, focus ring, remove button colour) from `ThemeProvider`. In Geti's dark theme, these tokens are overridden in `src/theme/tokens.css` — no per-component CSS is required.

## Variants

### Selection modes

`selectionMode` controls whether tags respond to click/keyboard selection:

| Mode               | Behaviour                                              |
| ------------------ | ------------------------------------------------------ |
| `'none'` (default) | Tags are not selectable; they display as static chips. |
| `'single'`         | One tag may be selected at a time.                     |
| `'multiple'`       | Any number of tags may be selected simultaneously.     |

```tsx
// Static display
<TagGroup label="Applied labels" selectionMode="none">
    <Item key="person">Person</Item>
</TagGroup>

// Single selection (e.g., active filter)
<TagGroup label="Category" selectionMode="single" onSelectionChange={setCategory}>
    <Item key="images">Images</Item>
    <Item key="videos">Videos</Item>
</TagGroup>

// Multiple selection
<TagGroup label="Labels" selectionMode="multiple" onSelectionChange={setLabels}>
    <Item key="cat">Cat</Item>
    <Item key="dog">Dog</Item>
</TagGroup>
```

### Removable tags

Tags become removable when `onRemove` is passed. Spectrum renders an `×` remove button inside each tag. The callback receives a `Set<Key>` of the keys that should be removed (typically a single-element set).

```tsx
const [labels, setLabels] = useState(['Cat', 'Dog', 'Bird']);

<TagGroup
    label="Annotation labels"
    items={labels.map((label) => ({ id: label, name: label }))}
    onRemove={(keys) => setLabels((prev) => prev.filter((l) => !keys.has(l)))}
>
    {(item) => <Item key={item.id}>{item.name}</Item>}
</TagGroup>;
```

### Overflow with maxRows

```tsx
<TagGroup label="All labels" maxRows={2}>
    {/* Tags beyond row 2 are hidden behind a "Show more" button */}
</TagGroup>
```

## Props

### Collection

| Prop       | Type                          | Default | Description                                                                                      |
| ---------- | ----------------------------- | ------- | ------------------------------------------------------------------------------------------------ |
| `children` | `ItemElement \| ItemRenderer` | —       | Static `<Item>` nodes or a render function for dynamic collections.                              |
| `items`    | `Iterable<T>`                 | —       | The data source for dynamic collections. Passed alongside a children render function.            |
| `label`    | `ReactNode`                   | —       | The visible label for the tag group. Required for accessibility unless `aria-label` is provided. |

### Selection

| Prop                  | Type                               | Default  | Description                                  |
| --------------------- | ---------------------------------- | -------- | -------------------------------------------- |
| `selectionMode`       | `'none' \| 'single' \| 'multiple'` | `'none'` | Whether and how tags can be selected.        |
| `selectedKeys`        | `Selection`                        | —        | The currently selected keys (controlled).    |
| `defaultSelectedKeys` | `Selection`                        | —        | The initially selected keys (uncontrolled).  |
| `disabledKeys`        | `Iterable<Key>`                    | —        | Keys of tags that should be non-interactive. |
| `onSelectionChange`   | `(keys: Selection) => void`        | —        | Called when the selection changes.           |

### Removal

| Prop       | Type                       | Default | Description                                                                                         |
| ---------- | -------------------------- | ------- | --------------------------------------------------------------------------------------------------- |
| `onRemove` | `(keys: Set<Key>) => void` | —       | Called when a tag's remove button is activated. When provided, a remove button renders on each tag. |

### Layout

| Prop      | Type     | Default | Description                                                                               |
| --------- | -------- | ------- | ----------------------------------------------------------------------------------------- |
| `maxRows` | `number` | —       | Caps the number of visible rows. Overflowing tags are hidden behind a "Show more" button. |

## Accessibility

- **ARIA role**: `TagGroup` renders as `role="grid"`. Each `<Item>` renders as `role="row"` containing a `role="gridcell"`, following the ARIA grid pattern for keyboard navigation.
- **Accessible name**: A `label` prop or `aria-label` is **mandatory**. Without an accessible name, screen readers cannot identify the purpose of the grid landmark.
- **`textValue` on items**: If an `<Item>`'s children are not plain text (e.g., they include icons or custom components), you must provide a `textValue` prop on the `<Item>` so Spectrum can use it for type-ahead and announcements.

```tsx
// Good — plain text children, no textValue needed
<Item key="dog">Dog</Item>

// Required — children contain non-text elements
<Item key="dog" textValue="Dog">
    <DogIcon aria-hidden /> Dog
</Item>
```

- **Keyboard interactions**:

| Key                    | Behaviour                                                                     |
| ---------------------- | ----------------------------------------------------------------------------- |
| `Tab`                  | Move focus into or out of the tag group (lands on the first/last focused tag) |
| `Arrow Left / Right`   | Move focus between tags within the group                                      |
| `Space` / `Enter`      | Select or deselect the focused tag (when `selectionMode` is not `'none'`)     |
| `Backspace` / `Delete` | Remove the focused tag (only when `onRemove` is provided)                     |
| `Home` / `End`         | Jump focus to the first or last tag in the group                              |

- **Remove button**: When `onRemove` is provided, each tag renders an `×` button with an accessible label of `"Remove {textValue}"`. This button is keyboard-reachable independently from the tag itself, so screen reader users can tab to it and activate it with `Space` or `Enter`.
- **Focus management on removal**: After a tag is removed, Spectrum automatically moves focus to the adjacent tag or to the group label, preventing focus loss.
- **Selection announcement**: When a tag is selected, Spectrum updates `aria-selected` on the row and announces the change to screen readers.

## Integration

- **Dependency**: `@adobe/react-spectrum` — `TagGroup` (as `SpectrumTagGroup`) and `Item` (as `SpectrumItem`) are imported from this package.
- **`ThemeProvider`**: Must be present in the React tree for Spectrum tokens (background, focus ring, remove button colour) to resolve correctly.
- **`Item` import**: Always import `Item` from `@geti-ai/ui`, not directly from `@adobe/react-spectrum`, to stay on the correct migration path.
- **Geti `Tag` vs `TagGroup`**: These are **different** components. Geti's `Tag` (`data/Tag`) is a bespoke, non-Spectrum primitive for static display labels. `TagGroup` is a Spectrum wrapper for interactive, keyboard-navigable tag collections. Do not substitute one for the other.
- **Dynamic collections**: For large or server-fetched tag lists, use the `items` prop with a render function child. For small, static sets, static `<Item>` children are simpler.
- **Form libraries**: To integrate with React Hook Form or Formik, use the controlled pattern — read `selectedKeys` from `onSelectionChange` and write back with `setValue` or `setFieldValue`. Remember to convert `Set<Key>` to an array for serialization.

## Related components

- **`Tag`** — use instead for a single static display chip with no selection or removal; `Tag` has no Spectrum dependency and is lighter weight.
- **`Badge`** — use for lifecycle or status metadata labels (`"Beta"`, `"Archived"`) that carry semantic colour and are never interactive.
- **`ListBox`** — use for a permanently-visible selection list when tag-style chip presentation is not required.
- **`Menu`** — use to present a list of selectable options inside an overlay rather than inline on the page.
