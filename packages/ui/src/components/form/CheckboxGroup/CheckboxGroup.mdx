## Responsibility

The `CheckboxGroup` component is a wrapper around Adobe's React Spectrum `CheckboxGroup`. It groups multiple `Checkbox` instances under a shared label and manages their collective selection state, validation, and accessibility relationships.

### Key Features
- **Managed Selection**: Handles a collection of selected values as an array of strings.
- **Orientation**: Supports both `vertical` (default) and `horizontal` layouts.
- **Form Integration**: Provides built-in support for labels, help text, and error messages within Spectrum forms.
- **Semantic Grouping**: Automatically handles ARIA roles (`role="group"`) and labeling for the entire set of checkboxes.

---

## Variants

### Orientation
Use the `orientation` prop to switch between `vertical` and `horizontal` layouts. Vertical is recommended for long lists, while horizontal is best for small sets of short labels.

### Layout Options
- **Label Position**: Labels can be placed on the `top` (default) or the `side`.
- **Necessity Indicator**: Displays whether the field is `required` or `optional` via an icon or text label.

## Props

### Content

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | — | The `Checkbox` elements that make up the group. |
| `label` | `ReactNode` | — | The label for the entire group. |
| `description` | `ReactNode` | — | Help text displayed below the group. |
| `errorMessage` | `ReactNode` | — | Error message displayed when validation fails. |

### Value / State

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string[]` | — | Controlled array of selected values. |
| `defaultValue` | `string[]` | — | Initial selection state (uncontrolled). |
| `orientation` | `'vertical' \| 'horizontal'` | `'vertical'` | The layout axis of the group. |
| `isDisabled` | `boolean` | `false` | Disables all checkboxes within the group. |
| `isReadOnly` | `boolean` | `false` | Prevents the user from changing the selection. |
| `isRequired` | `boolean` | `false` | Whether at least one checkbox must be selected. |

### Events

| Prop | Type | Description |
|------|------|-------------|
| `onChange` | `(value: string[]) => void` | Handler called when the selection changes. Returns the new array of selected values. |

### Styles

The `CheckboxGroup` component accepts all standard [Spectrum layout and spacing props](https://react-spectrum.adobe.com/react-spectrum/styling.html).

## Accessibility

- **Group Labeling**: The `label` prop is used to provide an accessible name for the group. If no visible label is provided, an `aria-label` must be used.
- **Individual Labels**: Every `Checkbox` within the group must have its own visible label.
- **Focus Management**: Focus can be moved through the group using the `Tab` key.

---

## Spectrum 2 Migration

When migrating to Spectrum 2 (`@react-spectrum/s2`), note the following changes:

- **Import Update**: Change from `@adobe/react-spectrum` to `@react-spectrum/s2`.
- **Breaking Change**: The `showErrorIcon` prop has been removed in S2 due to accessibility considerations.
- **New `size` Prop**: S2 adds a `size` prop (`'S' \| 'M' \| 'L' \| 'XL'`).
- **Styles Macro**: Layout and spacing props migrate to the new `styles` macro system.
- **Validation Behavior**: The default `validationBehavior` changes from `'aria'` to `'native'`.
