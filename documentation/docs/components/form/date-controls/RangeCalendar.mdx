# RangeCalendar

`RangeCalendar` is a standalone calendar for selecting a start and end date.
Use it when range selection should be visible inline, rather than hidden in a popover.

```tsx
import { RangeCalendar } from '@geti-ui/ui';
```

- [React Spectrum RangeCalendar ↗](https://react-spectrum.adobe.com/react-spectrum/RangeCalendar.html)
- [Geti UI Theming](/components/theming)

## Examples

### Default

```tsx preview
import { RangeCalendar } from '@geti-ui/ui';

export default function App() {
  return <RangeCalendar aria-label="Date range" />;
}
```

### Two months

```tsx preview
import { RangeCalendar } from '@geti-ui/ui';

export default function App() {
  return <RangeCalendar aria-label="Date range" visibleMonths={2} />;
}
```

### Disabled

```tsx preview
import { RangeCalendar } from '@geti-ui/ui';

export default function App() {
  return <RangeCalendar aria-label="Date range" isDisabled />;
}
```

## Responsibility

`RangeCalendar` is a direct pass-through wrapper for React Spectrum `RangeCalendar`.
It keeps behavior and keyboard support unchanged while standardizing imports.

## Variants and states

| State | Prop | Notes |
|---|---|---|
| Active selection | `value` / `defaultValue` | Date interval with `start` and `end` |
| Disabled | `isDisabled` | Non-interactive |
| Read-only | `isReadOnly` | View-only range |
| Multi-month | `visibleMonths` | Show months side-by-side |

## Props (deep dive)

### Value / state

- `value` / `defaultValue`: `{ start, end }` range value.
- `minValue` / `maxValue`: restrict selectable dates.
- `visibleMonths`: number of rendered months.
- `isDisabled`, `isReadOnly`.

### Events

- `onChange(range)`: called when range selection changes.

### Accessibility

- Provide `aria-label` since the control has no built-in visible label.
- Keyboard navigation is inherited from Spectrum calendar semantics.
