# Card

Card is a bordered, padded container for displaying a unit of summary content. Use it for project tiles, model list items, dataset summaries, or any item that should stand out visually as a discrete entity.

```tsx
import { Card } from '@geti/ui';
```

## Responsibility

`Card` is a Geti-specific layout component built on Spectrum's `View`. When `onPress` is provided, it renders the content inside a `<button>` with `aria-pressed` for selection semantics. Without `onPress`, it renders as an `<article>` for read-only content.

## Variants and states

| State | Props | Notes |
|---|---|---|
| Static | (no `onPress`) | Read-only; renders as `<article>` |
| Interactive | `onPress` | Renders as `<button>`; `aria-pressed` reflects `isSelected` |
| Selected | `isSelected` + `onPress` | Blue border and informative background |
| Disabled | `isDisabled` + `onPress` | Non-interactive; not-allowed cursor |

## Props (deep dive)

### Content

- `children`: any React node rendered inside the card body.
- `aria-label`: accessible label for the card (required when content is not plain text).

### Value / state

- `isSelected`: highlights the card with a focus-colour border.
- `isDisabled`: prevents interaction when `onPress` is set.

### Events

- `onPress`: callback fired when the card is clicked or activated via keyboard. Presence of this prop determines whether the card is interactive.

### Visual

- `UNSAFE_className`: additional CSS class for custom styling overrides.

### Accessibility

- always provide `aria-label` — the card container has no implicit label.
- `aria-pressed` is set automatically based on `isSelected` when interactive.
- disabled cards receive `aria-disabled` and the `disabled` attribute.

## Related components

- `Disclosure` — use for content that should expand/collapse rather than be selected.
- `View` — use for generic layout containers without selection semantics.
