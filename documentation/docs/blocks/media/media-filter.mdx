## Immediate apply model

The filter dialog supports immediate apply behavior:

1. users edit draft rules in the dialog,
2. valid rules are emitted immediately via `onApply`,
3. your app keeps canonical filter state and performs querying.

The blocks do not fetch or persist filters.
