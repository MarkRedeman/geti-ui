# DateRangePicker

`DateRangePicker` lets users select a contiguous date range using two segmented inputs and a shared RangeCalendar popover. Use it whenever a start and end date are both required.

```tsx
import { DateRangePicker } from '@geti-ui/ui';
```

## Examples

### Default

```tsx preview
import { DateRangePicker } from '@geti-ui/ui';

export default function App() {
  return <DateRangePicker label="Event range" />;
}
```

### With time (minute granularity)

```tsx preview
import { DateRangePicker } from '@geti-ui/ui';

export default function App() {
  return <DateRangePicker label="Event range and time" granularity="minute" />;
}
```

### Disabled

```tsx preview
import { DateRangePicker } from '@geti-ui/ui';

export default function App() {
  return <DateRangePicker label="Event range" isDisabled />;
}
```

## Responsibility

`DateRangePicker` wraps React Spectrum's `DateRangePicker` and applies `width="size-2400"` to fix a Safari mobile layout-flicker bug. All other props pass through unchanged.

## Variants and states

| State | Prop | Notes |
|---|---|---|
| Date only | `granularity="day"` (default) | Day/month/year segments |
| Date + time | `granularity="minute"` or `"second"` | Adds time segments to both fields |
| Disabled | `isDisabled` | Non-interactive |
| Read-only | `isReadOnly` | Shows value, no editing |
| Required | `isRequired` | Adds required indicator |
| Invalid | `validationState="invalid"` + `errorMessage` | Error styling |

## Props (deep dive)

### Content

- `label`: visible field label.
- `description`: helper text below the field.
- `errorMessage`: shown when `validationState="invalid"`.
- `startName` / `endName`: form field names for start/end date.

### Value / state

- `value` / `defaultValue`: controlled/uncontrolled as `{ start, end }`.
- `minValue` / `maxValue`: constrains the selectable range.
- `granularity`: smallest time unit (`"day"`, `"hour"`, `"minute"`, `"second"`).
- `allowsNonContiguousRanges`: allows selecting ranges that include disabled dates.
- `isDisabled`, `isReadOnly`, `isRequired`.

### Events

- `onChange({ start, end })`: fires when the user commits a new range.

### Accessibility

- Provides two separate accessible date inputs for screen readers.
- The shared calendar highlights the full selected range.
