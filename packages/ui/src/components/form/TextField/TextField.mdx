# TextField

A single-line text input field for names, emails, short descriptions, and other bounded text values.

```tsx
import { TextField } from '@geti/ui';
```

- [React Spectrum V3 TextField](https://react-spectrum.adobe.com/react-spectrum/TextField.html)

<Stories />

## Responsibility

`TextField` is a transparent wrapper around Adobe React Spectrum's `TextField`. It inherits all Spectrum behaviour: automatic label association via `aria-labelledby`, validation state styling, `description` and `errorMessage` rendering, and keyboard handling — with no additional configuration required.

`onChange` receives the **string value** directly — no `.target.value` unwrapping needed.

### When to use
- For short, bounded text inputs: names, emails, usernames, URLs, short descriptions.
- When the input is a single line and does not require multi-line entry.
- When you need to communicate validation state clearly with `validationState="invalid"` and `errorMessage`.

### When to avoid
- **Multi-line input** — use `TextArea`.
- **Search inputs** — use `SearchField` (built-in clear button and search icon).
- **Numeric inputs** — use `NumberField` (stepper buttons, `role="spinbutton"`).
- **Password inputs with toggle** — use `PasswordField`.
- **When no label is feasible** — always provide `label` or `aria-label`.

## States

| State | Prop | Description |
|-------|------|-------------|
| Default | — | Standard interactive field. |
| Invalid | `validationState="invalid"` | Red border, error icon, `errorMessage` below the field. |
| Valid | `validationState="valid"` | Green border, checkmark. Use sparingly. |
| Disabled | `isDisabled={true}` | Non-focusable, visually dimmed. |
| Read-only | `isReadOnly={true}` | Focusable and copy-able, but not editable. |
| Required | `isRequired={true}` | Visual asterisk and `aria-required="true"`. |
| Quiet | `isQuiet={true}` | Underline-only border for dense or settings-panel layouts. |

Use `labelPosition="side"` to place the label to the left for aligned form layouts.

## Props

<Controls of={DocStories.Default} />

### Content

| Prop | Type | Description |
|------|------|-------------|
| `label` | `ReactNode` | The label above or beside the field. Required for accessibility unless `aria-label` is provided. |
| `placeholder` | `string` | Hint text when empty. Supplement labels — do not substitute for them. |
| `description` | `ReactNode` | Persistent helper text below the field. |
| `errorMessage` | `ReactNode` | Error message shown when `validationState="invalid"`. |
| `contextualHelp` | `ReactNode` | A `<ContextualHelp>` popover anchored to the label. |

### Value / State

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | — | The current value (controlled). |
| `defaultValue` | `string` | — | The initial value (uncontrolled). |
| `type` | `string` | `'text'` | The `<input>` type (`'text'`, `'email'`, `'url'`, `'password'`, etc.). |
| `validationState` | `'valid' \| 'invalid'` | — | Visual and ARIA validation state. |
| `isRequired` | `boolean` | `false` | Adds a visual asterisk and `aria-required`. |
| `isReadOnly` | `boolean` | `false` | Value visible but not editable. |
| `isDisabled` | `boolean` | `false` | Non-interactive and visually dimmed. |
| `maxLength` | `number` | — | Maximum number of characters allowed. |
| `isQuiet` | `boolean` | `false` | Underline-only border style. |

### Events

| Prop | Type | Description |
|------|------|-------------|
| `onChange` | `(value: string) => void` | Called on every value change. Receives the new string directly. |
| `onBlur` | `(e: FocusEvent) => void` | Called when the field loses focus. Commonly used to trigger validation. |

### Layout

| Prop | Type | Description |
|------|------|-------------|
| `labelPosition` | `'top' \| 'side'` | Label position relative to the input. |
| `width` | `DimensionValue` | The width of the entire component. |

## Related components
- `TextArea` — use instead for multi-line free-text entry.
- `SearchField` — use instead when the input's semantic purpose is filtering or querying.
- `NumberField` — use instead for numeric-only inputs with stepper buttons.
- `PasswordField` — use instead when the input must mask its value and include a reveal toggle.
