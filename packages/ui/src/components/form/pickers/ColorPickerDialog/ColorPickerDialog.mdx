## Responsibility

`ColorPickerDialog` is the most complex component in the pickers folder — a stateful composite that assembles `ColorArea`, `ColorWheel`, `ColorSlider` (×2), `ColorField`, and `ColorSwatchPicker` into a single dialog triggered by an `ActionButton`. It is the primary full-featured color picker entry point in the library.

## Design

**Stateful composite** — manages a `useState<Color>` draft internally:

```tsx
const [color, setColor] = useState<Color>(() => colorProp ? parseColor(colorProp) : parseColor('#ff0000'));
useEffect(() => { if (colorProp) setColor(parseColor(colorProp)); }, [colorProp]);
```

Draft changes are made locally while the dialog is open. `onColorChange` is only called on Confirm (`color.toString('hex')`). Cancel discards the draft.

Imports all Spectrum components directly from `@adobe/react-spectrum` — it does **not** use the library's own thin wrappers for the color sub-components. This is intentional to avoid circular imports and keep the composite self-contained.

Trigger button: `ActionButton` showing a small `ColorSwatch` preview + label text.

Dialog layout: `ColorArea` (saturation/brightness 2D) + hue `ColorSlider` + alpha `ColorSlider` on the left; `ColorWheel` + `ColorField` (hex) on the right; `ColorSwatchPicker` with 8 preset swatches below; Cancel + Confirm buttons.

## Flow

1. `color` prop (string) → `parseColor()` → `useState<Color>` draft
2. `useEffect` syncs external `color` prop changes into draft
3. User interacts with any sub-component → `setColor(newColor)` updates draft
4. **Confirm**: `onColorChange(color.toString('hex'))` + `close()`
5. **Cancel**: `close()` — draft discarded, `onColorChange` never called

## Integration

- **Depends on**: `@adobe/react-spectrum` (all color + layout + dialog components imported directly), `react` (`useState`, `useEffect`)
- **Props**: `color?: string`, `onColorChange?: (color: string) => void`, `label?`, `isOpen?`, `onOpenChange?`
- **Used by**: any surface where the user needs to pick or change a color value
