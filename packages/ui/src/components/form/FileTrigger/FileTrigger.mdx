# FileTrigger

## Responsibility

`FileTrigger` is a transparent wrapper around `react-aria-components`' `FileTrigger`. It attaches native file-selection behaviour to any interactive child (typically a `Button` or `Link`) without rendering any visible UI of its own. Internally it manages a visually hidden `<input type="file">` and translates the OS file-selection result into an `onSelect` callback.

Key responsibilities:
- **Hidden file input**: Manages the lifecycle of a `<input type="file">` element, including creating it, wiring change events, and clearing it after selection so the same file can be re-selected if needed.
- **Trigger delegation**: Programmatically focuses and clicks the hidden input when the child element is activated (click, `Space`, or `Enter`), so the child element retains full keyboard and screen reader semantics.
- **Type filtering**: The `acceptedFileTypes` prop translates to the `accept` attribute on the hidden input, restricting the OS picker to specific MIME types or extensions.
- **Pairing with DropZone**: `FileTrigger` is the canonical click-to-browse companion for [`DropZone`](/docs/components-form-dropzone--docs). Nest it inside `DropZone` to provide both drag-and-drop and click-to-browse in a single upload area.

### When to use
- When you want to trigger a file selection from a styled `Button`, `Link`, or other interactive element.
- As the click-to-browse companion inside a [`DropZone`](/docs/components-form-dropzone--docs).
- When you need control over accepted file types (`acceptedFileTypes`), multi-file selection (`allowsMultiple`), or directory selection (`acceptDirectory`).
- On mobile when you want to directly open a camera via the `defaultCamera` prop.

### When to avoid
- **Drag-and-drop only** ‚Äî use [`DropZone`](/docs/components-form-dropzone--docs) alone if no click-to-browse button is needed.
- **Direct form submission** ‚Äî use a native `<input type="file" name="‚Ä¶">` if you need the browser to include the file in native `FormData` on form submit without JavaScript handling.
- When directory-level operations are unsupported in the target browser environment ‚Äî check `acceptDirectory` browser compatibility before relying on it.

## Design

`FileTrigger` follows the *transparent wrapper* pattern: it extends `AriaFileTriggerProps` without modification and delegates all behaviour to `react-aria-components`.

```tsx
// packages/ui/src/components/form/FileTrigger/FileTrigger.tsx

import { FileTrigger as AriaFileTrigger } from 'react-aria-components';
import type { FileTriggerProps as AriaFileTriggerProps } from 'react-aria-components';

export interface FileTriggerProps extends AriaFileTriggerProps {}

export const FileTrigger = (props: FileTriggerProps) => <AriaFileTrigger {...props} />;
```

Note that `FileTrigger` is the **only** component in this library sourced from `react-aria-components` rather than `@adobe/react-spectrum`. This is intentional: `FileTrigger` is a headless primitive that does not require Spectrum's styled component layer ‚Äî only React Aria's behavioural hook.

**Prop mapping**

All props pass through to React Aria's `FileTrigger` unchanged:

| Geti `FileTriggerProps` | React Aria `FileTriggerProps` | Notes |
|-------------------------|-------------------------------|-------|
| `onSelect` | `onSelect` | `(files: FileList \| null) => void` |
| `acceptedFileTypes` | `acceptedFileTypes` | Sets `accept` on the hidden `<input>` |
| `allowsMultiple` | `allowsMultiple` | Sets `multiple` on the hidden `<input>` |
| `acceptDirectory` | `acceptDirectory` | Sets `webkitdirectory` on the hidden `<input>` |
| `defaultCamera` | `defaultCamera` | Sets `capture` attribute for mobile cameras |
| `children` | `children` | The pressable trigger element |
| `...rest` | all other props | Passed through without transformation |

**No visible output**

`FileTrigger` renders only what its `children` render, plus a visually hidden `<input type="file" aria-hidden="true">` outside the visual flow. It has no container element, no border, and no layout impact.

**Theme integration**

`FileTrigger` itself has no Spectrum tokens or CSS. Its visual appearance is entirely determined by the child component (e.g., a `Button`). `ThemeProvider` is only needed if the child element is a Spectrum component.

## Variants

### Single file selection (default)

```tsx
<FileTrigger onSelect={(files) => console.log(files)}>
    <Button variant="primary">Select file</Button>
</FileTrigger>
```

### Multiple file selection

```tsx
<FileTrigger onSelect={handleSelect} allowsMultiple>
    <Button variant="secondary">Select images</Button>
</FileTrigger>
```

### Restricted MIME types

```tsx
<FileTrigger
    onSelect={handleSelect}
    acceptedFileTypes={['image/png', 'image/jpeg', 'image/webp']}
>
    <Button>Upload image</Button>
</FileTrigger>
```

### Directory selection

```tsx
<FileTrigger onSelect={handleSelect} acceptDirectory>
    <Button>Select folder</Button>
</FileTrigger>
```

### Mobile camera capture

```tsx
// Open the front (selfie) camera
<FileTrigger onSelect={handleSelect} acceptedFileTypes={['image/*']} defaultCamera="user">
    <Button>Take selfie</Button>
</FileTrigger>

// Open the rear camera
<FileTrigger onSelect={handleSelect} acceptedFileTypes={['image/*']} defaultCamera="environment">
    <Button>Capture photo</Button>
</FileTrigger>
```

### Disabled trigger

`FileTrigger` itself has no `isDisabled` prop. To disable the trigger, set `isDisabled` on the **child** component:

```tsx
<FileTrigger onSelect={handleSelect}>
    <Button isDisabled>Upload (unavailable)</Button>
</FileTrigger>
```

```
                              ‚îÇ
                              ‚ñº

                              ‚îÇ
                              ‚ñº

                        (same file can be re-selected immediately)
```

### DropZone + FileTrigger composition

The canonical upload pattern in Geti UI nests `FileTrigger` inside `DropZone`, giving users two paths to the same action:

```tsx
const [files, setFiles] = useState<File[]>([]);

const handleDrop = async (e: DropEvent) => {
    const dropped = await Promise.all(
        e.items
            .filter((item): item is FileDropItem => item.kind === 'file')
            .map((item) => item.getFile())
    );
    setFiles((prev) => [...prev, ...dropped]);
};

const handleSelect = (fileList: FileList | null) => {
    if (!fileList) return;
    setFiles((prev) => [...prev, ...Array.from(fileList)]);
};

<DropZone onDrop={handleDrop} aria-label="Upload dataset images">
    <IllustratedMessage>
        <Upload aria-hidden />
        <Heading>Drag and drop files</Heading>
        <Content>or use the button below</Content>
    </IllustratedMessage>
    <FileTrigger
        onSelect={handleSelect}
        acceptedFileTypes={['image/png', 'image/jpeg']}
        allowsMultiple
    >
        <Button variant="secondary">Browse files</Button>
    </FileTrigger>
</DropZone>
```

Both `handleDrop` and `handleSelect` converge on the same `files` state, so downstream processing (validation, upload, preview) is unified.

### Form integration

`FileTrigger` does not render a named `<input>` and does not participate in native form submission. To use selected files in a form:

```tsx
const [selectedFiles, setSelectedFiles] = useState<File[]>([]);

const handleSubmit = async (e: FormEvent) => {
    e.preventDefault();
    const formData = new FormData();
    selectedFiles.forEach((file) => formData.append('files', file));
    await uploadApi.post('/datasets/import', formData);
};

<form onSubmit={handleSubmit}>
    <FileTrigger onSelect={(fl) => setSelectedFiles(fl ? Array.from(fl) : [])}>
        <Button type="button">Select files</Button>
    </FileTrigger>
    <Button type="submit" isDisabled={selectedFiles.length === 0}>Upload</Button>
</form>
```

## Props

<Controls />

### Events

| Prop | Type | Description |
|------|------|-------------|
| `onSelect` | `(files: FileList \| null) => void` | Called when the user selects files from the OS picker. `null` is never returned in practice ‚Äî if the user cancels, the callback is not called. |

### File selection

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `acceptedFileTypes` | `readonly string[]` | ‚Äî | MIME types or file extensions to allow (e.g., `['image/*']`, `['.csv']`). Maps to the `accept` attribute on the hidden input. |
| `allowsMultiple` | `boolean` | `false` | Whether multiple files can be selected in a single picker session. |
| `acceptDirectory` | `boolean` | `false` | Whether to open a directory/folder picker instead of a file picker. |
| `defaultCamera` | `'user' \| 'environment'` | ‚Äî | Mobile only: which camera to open for capture. `'user'` = front, `'environment'` = rear. |

### Content

| Prop | Type | Description |
|------|------|-------------|
| `children` | `ReactNode` | The interactive element that triggers the file picker (e.g., a `Button`). Must be a single pressable element. |

## Accessibility

- **Hidden input**: `FileTrigger` renders a `<input type="file">` with `aria-hidden="true"` positioned off-screen. It is invisible to all assistive technologies ‚Äî the child element carries all the accessible semantics.
- **No extra ARIA**: `FileTrigger` does not add any ARIA attributes to the child element. The child's existing role, label, and state are unaffected. A `<Button>` child will still be announced as a button with its label.
- **Keyboard support**: Focus management and keyboard activation (`Space`, `Enter`) are fully handled by the child element. When the child is activated, React Aria programmatically triggers the hidden input ‚Äî no additional key handling is required.
- **Screen reader announcement**: Screen readers announce the child element per its role. For example, a `<Button>Browse files</Button>` is announced as *"Browse files, button"*. After selecting files, no automatic announcement is made ‚Äî if you need post-selection feedback, render a status message in an `aria-live` region.
- **Labeling**: There is no label on `FileTrigger` itself. Always ensure the child element has a clear, descriptive accessible name (e.g., *"Select images"* rather than *"Upload"*) so screen reader users understand what will happen when they activate it.
- **Disabled state**: Set `isDisabled` on the **child** element. Do not attempt to disable the `FileTrigger` wrapper ‚Äî it has no visual presence to disable.

## Integration

- **Dependency**: `react-aria-components` ‚Äî `FileTrigger` is imported from this package, not from `@adobe/react-spectrum`. This is the only component in `@geti/ui` sourced from `react-aria-components`.
- **`ThemeProvider`**: Required only if the child element is a Spectrum component (e.g., `Button` from `@geti/ui`). `FileTrigger` itself does not use Spectrum tokens.
- **`DropZone`**: The primary composition partner. See the [DropZone docs](/docs/components-form-dropzone--docs) for the full drag-and-drop API and the composed example.
- **`Button`**: The most common child. Import `Button` from `@geti/ui` to ensure consistent styling and press-event handling.
- **`FileList` to `File[]`**: `onSelect` receives a `FileList` (not an array). Convert with `Array.from(fileList)` or the spread operator `[...fileList]` before iterating, filtering, or passing to state.
