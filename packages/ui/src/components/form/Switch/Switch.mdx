# Switch

A binary on/off toggle for settings and features that take effect immediately.

```tsx
import { Switch } from '@geti-ai/ui';
```

- [React Spectrum V3 Switch](https://react-spectrum.adobe.com/react-spectrum/Switch.html)

<Stories />

## Responsibility

`Switch` wraps Adobe React Spectrum's `Switch` with a single Geti-specific enhancement: when `isEmphasized={true}`, the `data-emphasized` CSS attribute is set on the root element, and a CSS Module overrides Spectrum's selected-state tokens with Geti's `--energy-blue` custom properties, giving the switch higher visual prominence.

All other props pass through to Spectrum unchanged. `UNSAFE_className` is merged with the Geti module class via `clsx`.

### When to use

- For **immediate activation or deactivation** of a feature — when the user expects the effect now, with no Submit button.
- In **settings panels**: "Enable dark mode", "Allow notifications", "Auto-save", "Show grid".
- Use `isEmphasized` when the switch represents a **primary or high-impact setting** that warrants stronger visual weight.

### When to avoid

- **Deferred form submission** — use [`Checkbox`](/docs/form-checkbox--docs) when the change is applied only after the user submits a form.
- **Selecting items from a list** — use [`CheckboxGroup`](/docs/form-checkboxgroup--docs).
- **Choosing one from several options** — use [`RadioGroup`](/docs/form-radiogroup--docs).
- **When validation is needed** — `Switch` does not support `validationState`. Use a `Checkbox` with `isRequired` inside a `Form` instead.

## States

| State         | Prop                                            | Description                                                   |
| ------------- | ----------------------------------------------- | ------------------------------------------------------------- |
| Default (off) | —                                               | Standard unselected switch.                                   |
| Selected (on) | `defaultSelected={true}` or `isSelected={true}` | Track filled with the accent colour.                          |
| Emphasized    | `isEmphasized={true}`                           | Track and handle use Geti's energy-blue tokens when selected. |
| Disabled      | `isDisabled={true}`                             | Non-focusable, visually dimmed.                               |
| Read-only     | `isReadOnly={true}`                             | Focusable, state visible, but value cannot be changed.        |

## Props

<Controls of={DocStories.Default} />

| Prop               | Type                            | Default | Description                                                                        |
| ------------------ | ------------------------------- | ------- | ---------------------------------------------------------------------------------- |
| `children`         | `ReactNode`                     | —       | The visible label. Required for accessibility unless `aria-label` is provided.     |
| `isSelected`       | `boolean`                       | —       | Whether the switch is "on" (controlled).                                           |
| `defaultSelected`  | `boolean`                       | `false` | Initial "on" state (uncontrolled).                                                 |
| `isDisabled`       | `boolean`                       | `false` | Non-interactive and visually dimmed.                                               |
| `isReadOnly`       | `boolean`                       | `false` | State visible but not changeable.                                                  |
| `isEmphasized`     | `boolean`                       | `false` | Applies Geti's energy-blue tokens when selected. Use for high-prominence settings. |
| `onChange`         | `(isSelected: boolean) => void` | —       | Called when toggled. Receives `true` when on, `false` when off.                    |
| `UNSAFE_className` | `string`                        | —       | A custom CSS class merged with the Geti switch class.                              |

## Related components

- `Checkbox` — use when the setting is collected on form submission rather than applied immediately.
- `RadioGroup` — use when the user must choose between two or more explicitly named options.
- `ToggleButton` — use when the toggle is part of a toolbar or canvas control.

## Example use cases

<details>
  <summary>Typical use cases</summary>

- Enabling or disabling real-time AI inference on a live annotation canvas.
- Toggling dark mode or notification alerts in a user preferences panel.
- Turning a model's auto-labeling feature on or off in a project settings page.
- Activating or deactivating a specific data augmentation option in a training configuration.
