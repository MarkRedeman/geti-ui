# VirtualizedHorizontalGrid

A virtualised horizontal grid for large item collections — items flow left-to-right with DOM recycling, built on `react-aria-components` and a custom `HorizontalLayout`.

```tsx
import { VirtualizedHorizontalGrid } from '@geti/ui';
```

<Stories />

## Responsibility

Provides a virtualised horizontal grid (items flow left-to-right with DOM recycling), built on `react-aria-components` (RAC) and a custom `HorizontalLayout` class. Used for large horizontally-scrolling image/card galleries.

## Design

Built on RAC, not Spectrum. Key building blocks:

- **`Virtualizer`** (from `react-aria-components`) — manages DOM recycling.
- **`HorizontalLayout`** (co-located `HorizontalLayout.ts`) — a custom class extending RAC's `Layout`. Implements the horizontal flow algorithm: items are laid out left-to-right in rows, with `itemWidth`, `itemHeight`, and `gap` controlling the grid geometry.
- **`orientation="horizontal"`** — passed to the inner `ListBox` to set scroll direction.
- **`listBoxItemStyles`** — a style object passed to every `ListBoxItem`, enabling caller control of item dimensions without changing the layout algorithm.
- **No infinite scroll** — unlike `VirtualizedListLayout`, this component has no `isLoading`/`onLoadMore` mechanism. It renders a fixed `items` array.
- **Render props** — same `renderItem`, `idFormatter`, `textValueFormatter` pattern as `VirtualizedListLayout`.

## Integration

- Used for horizontal image strip / filmstrip views (e.g. video frame selectors, horizontally-scrolling thumbnail rows).
- `HorizontalLayout.ts` is a co-located private implementation — not exported from the component's public API.
- Contrast with `VirtualizedListLayout` (same data/ category) — that is vertical + infinite scroll; this is horizontal + fixed collection.
- Contrast with `CardView` — `CardView` uses CSS grid without virtualisation (best for small/medium counts); `VirtualizedHorizontalGrid` virtualises the DOM for large horizontal collections.

## Props

| Prop                 | Type                     | Default | Description                                                                     |
| -------------------- | ------------------------ | ------- | ------------------------------------------------------------------------------- |
| `items`              | `T[]`                    | —       | **Required.** The array of items to render.                                     |
| `renderItem`         | `(item: T) => ReactNode` | —       | **Required.** Renders the content for each list item.                           |
| `idFormatter`        | `(item: T) => Key`       | —       | **Required.** Returns the unique key for each item.                             |
| `textValueFormatter` | `(item: T) => string`    | —       | **Required.** Returns the accessible text for each item.                        |
| `listBoxItemStyles`  | `CSSProperties`          | —       | Inline styles applied to every `ListBoxItem` wrapper. Controls item dimensions. |
| `ariaLabel`          | `string`                 | —       | **Required.** Accessible label for the list box.                                |

## Accessibility

- Built on `react-aria-components` `ListBox` with `orientation="horizontal"`, which follows the [WAI-ARIA Listbox pattern](https://www.w3.org/WAI/ARIA/apg/patterns/listbox/).
- Always provide `ariaLabel` — without it the list has no accessible name.
- `textValueFormatter` provides the accessible text for each item; ensure it returns a meaningful, non-empty string for every item.
- Keyboard navigation follows standard horizontal list navigation: `Arrow Left/Right` to move between items, `Home/End` for first/last.

## Related components

- **`VirtualizedListLayout`** — use instead for vertical infinite-scroll lists; shares the same render-prop API but uses standard `ListLayout`.
- **`CardView`** — use when item count is small-to-medium and CSS grid layout (without DOM virtualisation) is sufficient.
- **`ListView`** — use when a dense, single-column list with Spectrum styling is preferred over a horizontal strip.
- **`ListBox`** — use for simple, permanently-visible horizontally-arranged option groups with no virtualisation needed.
