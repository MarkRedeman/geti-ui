# ToggleButtons

ToggleButtons renders a group of mutually exclusive buttons — only one option is active at a time. Use it for view mode switching, filter selection, or any compact segmented control where a single choice must be made.

```tsx
import { ToggleButtons } from '@geti/ui';
```

## Responsibility

`ToggleButtons` is a Geti-native component. It:

- renders one Geti `Button` per option with `variant="accent"` for the selected option and `variant="secondary"` for all others,
- sets `aria-pressed` on each button for accessible state communication,
- wraps the group in a `role="group"` div with `aria-label="Selection options"`,
- accepts a `getLabel` mapper for custom rendering (icons, rich content) per option.

## Variants and states

`ToggleButtons` has no visual variants. Selection is shown via the `accent`/`secondary` button variant difference.

| Prop | Type | Notes |
|---|---|---|
| `isDisabled` | `boolean` | Disables every button in the group |

## Props (deep dive)

### Content

- `options`: `T[]` — array of string or number values to display as buttons.
- `getLabel`: `(option: T) => ReactNode` — optional mapper for custom labels or icons per option.

### Value

- `selectedOption`: currently selected value (controlled — must be kept in state by the caller).

### Events

- `onOptionChange`: `(option: T) => void` — called with the newly selected option.

### Accessibility

- `isDisabled`: disables all buttons in the group,
- each button has `aria-pressed` reflecting its selection state,
- the container uses `role="group"` for grouped semantics.

## Related components

- `ToggleButton` — use for a single persistent toggle (not a mutually exclusive group).
- `RadioGroup` — use when options are part of a form submission or need a visible label.
- `Tabs` — use when toggling between full content panels, not just a state value.
