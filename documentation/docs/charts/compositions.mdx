# Chart Compositions

Compositions are fully assembled chart components. Import and render them with data — no Recharts wiring required.

## Primitives vs compositions

- **Compositions** = ready-made components with common chart wiring built in (axes, grid, tooltip, legend, responsive container behavior).
- **Primitives** = lower-level themed parts you compose manually with Recharts when you need custom behavior.

Use compositions by default. Switch to primitives for non-standard layouts.

```ts
import { LineChart, AreaChart, BarChart, ScatterChart, PieChart, DonutChart, RadarChart, RadialBarChart, MeterChart, TreemapChart, Sparkline } from '@geti-ai/charts';
```

## `LineChart`

Renders one or more continuous data series as lines.

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `data` | `object[]` | required | Array of data points |
| `xAxisKey` | `string` | `"name"` | Key for the x-axis values |
| `series` | `SeriesConfig[]` | required | Each series: `{ dataKey, name?, color? }` |
| `height` | `number` | `300` | Chart height in px |
| `aria-label` | `string` | — | Accessible label for the chart |
| `animate` | `boolean` | `false` | Opt-in per-chart animations |
| `xScale` | `AxisScaleConfig` | — | Scale config for the X axis (type, domain, overflow) |
| `yScale` | `AxisScaleConfig` | — | Scale config for the Y axis (type, domain, overflow) |

### Example — single series

```tsx preview
import { LineChart } from '@geti-ai/charts';

const data = [
  { epoch: 1, loss: 1.42 },
  { epoch: 2, loss: 1.10 },
  { epoch: 3, loss: 0.87 },
  { epoch: 4, loss: 0.71 },
  { epoch: 5, loss: 0.60 },
  { epoch: 6, loss: 0.52 },
];

export default function App() {
  return (
    <LineChart
      data={data}
      xAxisKey="epoch"
      series={[{ dataKey: 'loss', name: 'Training loss' }]}
      aria-label="Training loss per epoch"
      height={260}
    />
  );
}
```

### Example — multiple series

```tsx preview
import { LineChart } from '@geti-ai/charts';

const data = [
  { epoch: 1, train: 1.42, val: 1.50 },
  { epoch: 2, train: 1.10, val: 1.20 },
  { epoch: 3, train: 0.87, val: 0.98 },
  { epoch: 4, train: 0.71, val: 0.82 },
  { epoch: 5, train: 0.60, val: 0.69 },
  { epoch: 6, train: 0.52, val: 0.61 },
];

export default function App() {
  return (
    <LineChart
      data={data}
      xAxisKey="epoch"
      series={[
         { dataKey: 'train', name: 'Train loss' },
         { dataKey: 'val', name: 'Val loss' },
       ]}
      aria-label="Training vs validation loss"
      height={260}
    />
  );
}
```

### Example — log scale (loss curve)

Use `yScale` to switch the Y axis to a logarithmic scale. This is ideal for loss curves that drop rapidly at first and then slow down.

> ⚠ **Log scale requires all values to be > 0.** Pass `domain: [minPositive, 'auto']` to prevent Recharts from including zero in the range.

```tsx preview
import { LineChart } from '@geti-ai/charts';

const data = [
  { epoch: 1, loss: 2.31 },
  { epoch: 2, loss: 1.10 },
  { epoch: 3, loss: 0.52 },
  { epoch: 4, loss: 0.24 },
  { epoch: 5, loss: 0.11 },
  { epoch: 6, loss: 0.05 },
  { epoch: 7, loss: 0.023 },
  { epoch: 8, loss: 0.011 },
];

export default function App() {
  return (
    <LineChart
      data={data}
      xAxisKey="epoch"
      series={[{ dataKey: 'loss', name: 'Training loss' }]}
      yScale={{ scale: 'log', domain: [0.005, 'auto'] }}
      aria-label="Log-scale training loss"
      height={260}
    />
  );
}
```

## `BarChart`

Renders categorical data as vertical bars. Supports grouped and stacked layouts.

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `data` | `object[]` | required | Array of data points |
| `xAxisKey` | `string` | `"name"` | Key for the category axis |
| `series` | `SeriesConfig[]` | required | Each series: `{ dataKey, name?, color? }` |
| `stacked` | `boolean` | `false` | Stack bars when true |
| `height` | `number` | `300` | Chart height in px |
| `animate` | `boolean` | `false` | Opt-in per-chart animations |
| `xScale` | `AxisScaleConfig` | — | Scale config for X axis (value axis when `layout="vertical"`) |
| `yScale` | `AxisScaleConfig` | — | Scale config for Y axis (value axis when `layout="horizontal"`) |

> **Stacking API note:** `BarChart` uses a top-level `stacked` prop that applies to all series simultaneously. `AreaChart` uses a per-series `stacked` field on each `SeriesConfig` entry, so individual series can be mixed between stacked and non-stacked in the same chart.

### Example — grouped

```tsx preview
import { BarChart } from '@geti-ai/charts';

const data = [
  { model: 'MobileNet', precision: 84, recall: 80 },
  { model: 'ResNet-50', precision: 89, recall: 87 },
  { model: 'EfficientDet', precision: 91, recall: 90 },
];

export default function App() {
  return (
    <BarChart
      data={data}
      xAxisKey="model"
      series={[
         { dataKey: 'precision', name: 'Precision (%)' },
         { dataKey: 'recall', name: 'Recall (%)' },
       ]}
      aria-label="Model comparison — precision vs recall"
      height={260}
    />
  );
}
```

### Example — stacked

```tsx preview
import { BarChart } from '@geti-ai/charts';

const data = [
  { label: 'Annotated', ok: 900, review: 60, rejected: 40 },
  { label: 'Unannotated', ok: 0, review: 0, rejected: 0 },
  { label: 'Total', ok: 900, review: 60, rejected: 40 },
];

export default function App() {
  return (
    <BarChart
      data={data}
      xAxisKey="label"
      stacked
      series={[
         { dataKey: 'ok', name: 'Approved' },
         { dataKey: 'review', name: 'In review' },
         { dataKey: 'rejected', name: 'Rejected' },
       ]}
       aria-label="Dataset annotation breakdown"
      height={260}
    />
  );
}
```

### Example — log scale on the value axis

Log scale is useful for count data that spans orders of magnitude (e.g. class imbalance).

> ⚠ All bar values must be > 0 when using log scale.

```tsx preview
import { BarChart } from '@geti-ai/charts';

const data = [
  { label: 'Cat',     count: 1200 },
  { label: 'Dog',     count: 340  },
  { label: 'Car',     count: 85   },
  { label: 'Bicycle', count: 12   },
  { label: 'Sign',    count: 3    },
];

export default function App() {
  return (
    <BarChart
      data={data}
      xAxisKey="label"
      series={[{ dataKey: 'count', name: 'Instances' }]}
      yScale={{ scale: 'log', domain: [1, 'auto'] }}
      aria-label="Class imbalance — log scale"
      height={260}
    />
  );
}
```

## `ScatterChart`


Visualises the relationship between two continuous variables.

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `series` | `SeriesConfig[]` | required | Series array with `name` and `data` |
| `xAxisLabel` | `string` | — | X-axis label |
| `yAxisLabel` | `string` | — | Y-axis label |
| `height` | `number` | `300` | Chart height in px |
| `animate` | `boolean` | `false` | Opt-in per-chart animations |
| `useVoronoi` | `boolean` | `true` | Enable Voronoi nearest-point hover (see below) |
| `voronoiMaxDistance` | `number` | `80` | Max pixel distance for Voronoi tooltip activation |
| `xScale` | `AxisScaleConfig` | — | Scale config for the X axis |
| `yScale` | `AxisScaleConfig` | — | Scale config for the Y axis |

### Voronoi hover

By default `ScatterChart` uses **Voronoi-based hover** (powered by [d3-delaunay](https://github.com/d3/d3-delaunay)). Instead of requiring the cursor to land exactly on a small dot, the chart divides the plot area into regions — one per data point — using a Delaunay triangulation. Moving the cursor anywhere inside a region highlights the owning point and shows a tooltip.

Key behaviours:

- **`useVoronoi={true}` (default)** — nearest-point lookup is active. Recharts' built-in cursor line is hidden; a custom active-dot ring and an absolutely-positioned HTML tooltip are rendered instead.
- **`useVoronoi={false}`** — falls back to standard Recharts hover, where the cursor must intersect a dot to show the tooltip.
- **`voronoiMaxDistance`** — clamps the lookup radius. Points farther than this many pixels from the cursor are ignored, so the tooltip disappears in truly empty areas of the chart. Raise this value for sparse data; lower it for very dense scatter plots.

### Example

```tsx preview
import { ScatterChart } from '@geti-ai/charts';

const points = [
  { x: 0.1, y: 0.85, name: 'cat' },
  { x: 0.3, y: 0.72, name: 'dog' },
  { x: 0.5, y: 0.91, name: 'car' },
  { x: 0.2, y: 0.68, name: 'person' },
  { x: 0.7, y: 0.77, name: 'bicycle' },
  { x: 0.4, y: 0.83, name: 'truck' },
  { x: 0.6, y: 0.95, name: 'bus' },
];

export default function App() {
  return (
    <ScatterChart
      series={[{ name: 'Per class', data: points }]}
      xAxisLabel="False positive rate"
      yAxisLabel="True positive rate"
      aria-label="ROC scatter — per class"
      height={280}
    />
  );
}
```

### Example — Voronoi disabled (standard hover)

```tsx preview
import { ScatterChart } from '@geti-ai/charts';

const points = [
  { x: 0.1, y: 0.85, name: 'cat' },
  { x: 0.3, y: 0.72, name: 'dog' },

  { x: 0.7, y: 0.77, name: 'bicycle' },
  { x: 0.4, y: 0.83, name: 'truck' },
  { x: 0.6, y: 0.95, name: 'bus' },
];

  return (
    <ScatterChart
      series={[{ name: 'Per class', data: points }]}
```

### Example — log-log scatter (model size vs accuracy)

Log scales on both axes are useful when comparing data that spans many orders of magnitude.

> ⚠ Both axes require all values to be > 0.

```tsx preview
import { ScatterChart } from '@geti-ai/charts';

const models = [
  { x: 3.4e6,  y: 72 },
  { x: 11.7e6, y: 80 },
  { x: 25.6e6, y: 85 },
  { x: 44.5e6, y: 88 },
  { x: 138e6,  y: 91 },
];

export default function App() {
  return (
    <ScatterChart
      series={[{ name: 'Models', data: models }]}
      xScale={{ scale: 'log', domain: [1e6, 'auto'] }}
      xAxisLabel="Parameters"
      yAxisLabel="Top-1 accuracy (%)"
      aria-label="Model size vs accuracy"
      height={280}
    />
  );
}
```

## `Sparkline`

A minimal inline chart with no axes, ticks, or legend. Designed for use inside tables, cards, and list items.

### Props

| Prop | Type | Default | Description |
|---|---|---|---|
| `data` | `object[]` | required | Ordered data values |
| `width` | `number` | `120` | Width in px |
| `height` | `number` | `40` | Height in px |
| `color` | `string` | theme primary | Line colour (CSS colour or token) |
| `area` | `boolean` | `false` | Fill the area under the sparkline |
| `fillOpacity` | `number` | `0.15` | Area fill opacity when `area` is enabled |
| `animate` | `boolean` | `false` | Opt-in per-chart animations |

### Example

```tsx preview
import { Sparkline } from '@geti-ai/charts';

export default function App() {
  return (
    <div style={{ display: 'flex', flexDirection: 'column', gap: 16 }}>
      <div style={{ display: 'flex', alignItems: 'center', gap: 12 }}>
        <span style={{ color: 'var(--spectrum-global-color-gray-300)', fontSize: 13, width: 120 }}>
          MobileNet v3
        </span>
        <Sparkline data={[72, 75, 79, 81, 83, 85, 88].map((value) => ({ value }))} />
      </div>
      <div style={{ display: 'flex', alignItems: 'center', gap: 12 }}>
        <span style={{ color: 'var(--spectrum-global-color-gray-300)', fontSize: 13, width: 120 }}>
          ResNet-50
        </span>
        <Sparkline data={[80, 82, 84, 86, 87, 89, 91].map((value) => ({ value }))} />
      </div>
      <div style={{ display: 'flex', alignItems: 'center', gap: 12 }}>
        <span style={{ color: 'var(--spectrum-global-color-gray-300)', fontSize: 13, width: 120 }}>
          EfficientDet
        </span>
        <Sparkline data={[60, 65, 70, 74, 80, 84, 90].map((value) => ({ value }))} />
      </div>
    </div>
  );
}
```
