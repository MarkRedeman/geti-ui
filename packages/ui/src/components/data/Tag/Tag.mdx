# Tag

<Subtitle>A compact, chip-style label for displaying categorized metadata such as annotation classes, classification results, and taxonomy entries.</Subtitle>

```tsx
import { Tag } from '@geti/ui';
```

<Stories />

## Responsibility

`Tag` is a fully custom Geti primitive ‚Äî it has **zero dependency on Adobe React Spectrum** (except for the optional tooltip, which borrows Spectrum's `TooltipTrigger`). Its sole job is to render a styled chip that visually communicates a categorical label to the user.

Key responsibilities:
- **Visual identity**: Applies Geti's accent dot (in `--energy-blue`) as the default prefix, giving annotation labels an immediately recognizable appearance throughout the product.
- **Flexible affixes**: Accepts optional `prefix` and `suffix` slots for custom icons, counts, or action elements without breaking the chip's internal layout.
- **Theme awareness**: Switches background between a dark-mode and light-mode gray token via the `darkMode` prop, allowing the chip to sit correctly on both dark panels and light surfaces.
- **Tooltip integration**: Wraps the chip in a Spectrum `TooltipTrigger` when a `tooltip` string is provided ‚Äî the only point where Spectrum enters the component.

### When to use
- To display annotation class labels alongside a model prediction or a manual annotation.
- To render items in a label palette, taxonomy tree, or classification result list.
- When you need a compact, non-interactive label that carries optional supplemental information via a hover tooltip.
- When the chip needs a custom icon or action (e.g., a remove button) via `prefix` or `suffix`.

### When to avoid
- For **interactive, selectable, or removable** tag collections ‚Äî use `TagGroup` instead, which wraps Spectrum's `TagGroup` and manages keyboard navigation and removal callbacks.
- For status indicators (e.g., "Active", "Pending") ‚Äî use `StatusLight` or `Badge`, which carry semantic color meaning.
- When you need a full Spectrum-styled chip with built-in selection and form integration.

## Variants

### Default (with accent dot)
The standard appearance: a rounded chip with a small blue dot (`--energy-blue`) before the text. The dot is rendered as `aria-hidden="true"` since it is decorative.

### No dot (`withDot={false}`)
Removes the dot and adjusts left padding. Use when the `prefix` slot provides its own icon, or when the dot would be visually redundant.

### With prefix / suffix
`prefix` replaces the dot slot entirely. `suffix` appends a node after the text with consistent spacing. Both accept any `ReactNode`.

### Dark / Light mode
`darkMode={true}` uses `var(--spectrum-global-color-gray-100)` (darker background).
`darkMode={false}` (default) uses `var(--spectrum-global-color-gray-200)` (lighter background).

## Design

Styling is 100% inline `CSSProperties` using global CSS custom properties. The component does not import a CSS module and does not require Spectrum's `Provider` to be active ‚Äî as long as the Spectrum global token stylesheet is present in the document.

```tsx
// Default ‚Äî accent dot, dark background
<Tag text="Person" />

// No dot, custom prefix icon
<Tag text="Vehicle" withDot={false} prefix={<VehicleIcon />} />

// With suffix (e.g., a confidence score)
<Tag text="Dog" suffix={<span>92%</span>} />

// Light surface variant with tooltip
<Tag text="Bicycle" darkMode tooltip="Annotation class: Bicycle" />
```

## Flow

```
props { text, prefix, suffix, withDot, darkMode, tooltip, id, className, style }
  ‚Üí hasPrefix = !!(withDot || prefix)
  ‚Üí containerStyle = { ...baseStyle, backgroundColor (dark/light), ...style }
  ‚Üí chipElement = <div style={containerStyle} aria-label={text} title={tooltip}>
      {hasPrefix &&
        <span style={prefixWrapper}>
          {prefix ?? (withDot ? <span style={dot} aria-hidden /> : null)}
        </span>}
      <span>{text}</span>
      {suffix && <span style={suffixWrapper}>{suffix}</span>}
    </div>
  ‚Üí if tooltip:
      <TooltipTrigger>
        {chipElement}
        <Tooltip>{tooltip}</Tooltip>
      </TooltipTrigger>
    else:
      {chipElement}
```

## Integration

- **`--energy-blue`** (`#0168b5` fallback) must be present in the global stylesheet for the dot to render correctly. It is defined in `geti-global.module.css` and the Geti theme tokens.
- **`TagGroup`** (in the same `data/` category) wraps Spectrum's `TagGroup` for interactive, keyboard-navigable collections where tags can be removed. `Tag` here is a distinct, non-Spectrum primitive ‚Äî do not confuse the two.
- The component renders a `<div>` and accepts all standard HTML div attributes via `...rest`, making it composable as a list item, table cell content, or overlay label.
- Can be used **outside `ThemeProvider`** as long as the Spectrum global color tokens and `--energy-blue` are loaded.

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `text` | `string` | ‚Äî | **Required.** The label text displayed inside the chip. Also used as the accessible `aria-label`. |
| `withDot` | `boolean` | `true` | Whether to render the blue accent dot as the default prefix. Ignored when `prefix` is provided. |
| `prefix` | `ReactNode` | ‚Äî | Custom element rendered before the text, replacing the dot. |
| `suffix` | `ReactNode` | ‚Äî | Custom element rendered after the text. |
| `darkMode` | `boolean` | `false` | Switches the background to the darker gray token. |
| `tooltip` | `string` | ‚Äî | Text shown in a Spectrum `Tooltip` on hover. |
| `id` | `string` | ‚Äî | Sets both `id` and `data-testid` on the container div. |
| `className` | `string` | ‚Äî | Additional CSS class names. |
| `style` | `CSSProperties` | ‚Äî | Inline style overrides applied after computed styles. |

## Accessibility

- **`aria-label`**: The container `<div>` receives `aria-label={text}`, giving screen readers a programmatic name for the chip even when the text is accompanied by decorative elements.
- **`title` attribute**: When `tooltip` is provided, it is also set as the `title` attribute on the container, enabling browser-native tooltip behavior for pointer users who may not trigger the Spectrum tooltip.
- **Decorative dot**: The accent dot `<span>` is rendered with `aria-hidden="true"` to prevent it from being announced as content.
- **Tooltip ARIA**: When `tooltip` is provided, Spectrum's `TooltipTrigger` wires `aria-describedby` between the chip element and the tooltip automatically.
- **Keyboard**: The base chip is not focusable. If you need a keyboard-accessible chip (e.g., for removal), use `TagGroup` which provides roving `tabindex` and keyboard removal support.
