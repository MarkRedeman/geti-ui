# RadioGroup

RadioGroup presents a named set of mutually exclusive options. Use it in forms and settings panels when a user must pick exactly one option from a visible list.

```tsx
import { Radio, RadioGroup } from '@geti-ai/ui';
```

- [React Spectrum RadioGroup ‚Üó](https://react-spectrum.adobe.com/react-spectrum/RadioGroup.html)
- [Geti UI Theming](/components/theming)

## Examples

### Default (vertical)

```tsx preview
import { Radio, RadioGroup } from '@geti-ai/ui';

export default function App() {
  return (
    <RadioGroup label="Annotation type">
      <Radio value="detection">Object detection</Radio>
      <Radio value="segmentation">Segmentation</Radio>
      <Radio value="classification">Classification</Radio>
    </RadioGroup>
  );
}
```

### Pre-selected value

```tsx preview
import { Radio, RadioGroup } from '@geti-ai/ui';

export default function App() {
  return (
    <RadioGroup label="Export format" defaultValue="coco">
      <Radio value="coco">COCO JSON</Radio>
      <Radio value="pascal">Pascal VOC</Radio>
      <Radio value="yolo">YOLO</Radio>
    </RadioGroup>
  );
}
```

### Horizontal layout

```tsx preview
import { Radio, RadioGroup } from '@geti-ai/ui';

export default function App() {
  return (
    <RadioGroup label="View mode" orientation="horizontal">
      <Radio value="grid">Grid</Radio>
      <Radio value="list">List</Radio>
    </RadioGroup>
  );
}
```

### Disabled

```tsx preview
import { Radio, RadioGroup } from '@geti-ai/ui';

export default function App() {
  return (
    <RadioGroup label="Locked option" isDisabled defaultValue="a">
      <Radio value="a">Option A</Radio>
      <Radio value="b">Option B</Radio>
    </RadioGroup>
  );
}
```

## Responsibility

`RadioGroup` and `Radio` are thin wrappers around React Spectrum's equivalents. All props pass through unchanged. The wrappers provide `@geti-ai/ui` import paths and Geti theming.

## Variants and states

| Prop | Options | Default |
|---|---|---|
| `orientation` | `'vertical' \| 'horizontal'` | `'vertical'` |
| `isDisabled` | boolean | `false` |
| `isReadOnly` | boolean | `false` |

## Props (deep dive)

### Content

- `children`: `Radio` components. Each must have a `value` string.
- `label`: visible group label.
- `description`: optional helper text.
- `errorMessage`: error text shown when `validationState="invalid"`.

### Value

- `value`: controlled selected value.
- `defaultValue`: uncontrolled initial value.

### Events

- `onChange`: `(value: string) => void`.

### Validation

- `validationState`: `'valid' | 'invalid'`.
- `isRequired`: marks as required in form validation.

### Accessibility

- renders as `<fieldset>` with `<legend>` for the `label` ‚Äî never skip the `label` prop,
- individual `Radio` items render as standard radio inputs with correct labelling.
